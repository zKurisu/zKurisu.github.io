相关概念

Path loss

Path loss 指信号在从发射端 (如基站) 传播到接收端 (如无线终端) 的过程中, 由于距离增加和传播环境的影响, 信号强度逐渐减弱的现象.

RSSI

RSSI, Received Signal Strength Indicator, 是用来衡量接收信号强度的指标, 单位通常是 dBm (分贝毫瓦). 如: -70 dBm 表示信号强度比 1 毫瓦低 70 dB.

  • RSSI 值越小 (越负), 表示信号强度越弱
  • RSSI 值越大 (越接近 0), 表示信号强度越强

Propagation Models

Ebook 中提到的三种:

  • Free-Space, 理想的无障碍环境, 如真空或开阔空间.
    • 没有障碍物、反射、散射或衍射
    • 信号能量仅因距离增加而扩散
  • Log-Distance, 实际环境, 如城市, 郊区或室内.
    • 考虑障碍物, 反射, 散射或衍射
    • 信号强度随距离增加而衰减时, 衰减速率由路径损耗指数 (Path Loss Exponent, n) 决定
  • ITU, International Telecommunication Union, 用于特定环境 (如城市, 郊区, 室内) 或特定频段
    • 提供了一系列标准化的传播模型, 适用于不同的通信场景和频段
    • 这些模型基于大量实测数据和统计分析, 具有较高的准确性

BER

Bit Error Rate, 指在数字通信系统中, 接收端错误接收的比特数与总传输比特数的比率. 其反映信号在传输过程中受到噪声, 干扰和失真影响的程度.

$$
\displaylines
{
\begin{aligned}
BER = \frac{错误接收的比特数}{总传输比特数}
\end{aligned}
}
$$

Transmisster power 越大, BER 越小.

SNR

Signal-to-Noise Ratio, 指信号功率与噪声功率的比值, 用于衡量信号相对于噪声的强度. 其反映信号在传输过程中受到噪声影响的程度.

$$
\displaylines
{
\begin{aligned}
SNR = \frac{P_{signal}}{P_{noise}}
\end{aligned}
}
$$

Transmisster power 越大, SNR 越大.

Internal Signal Range

Internal Signal Range 指信号在设备或系统内部传播的范围, 如: 电路板, 芯片, 模块或其他内部组件之间的信号传输.

External Signal Range

External Signal Range, 指信号在设备或系统外部传播的范围, 如: 无线信号或有线信号在开放环境中的传播.

Simulator, Emulator 和 Testbed

  • Simulator, 模拟器, 通过软件模型模拟系统的行为或环境, 通常不涉及实际硬件 (纯软件)
  • Emulator, 仿真器, 通过软件或硬件模仿特定系统的功能, 能够运行与实际系统相同的软件 (要结合硬件)
  • Testbed, 测试平台, 为实验和测试设计的实际或虚拟环境, 通常包含真实硬件和软件 (实际测试)

Mininet-wifi 属于 Emulator.

SoftMac

SoftMAC 指通过软件实现网络协议栈中的 MAC (媒体访问控制) 层功能, 而不是依赖硬件. MAC 层负责控制数据帧在物理网络介质上的传输, 传统上由硬件实现, 但 SoftMAC 将其移至软件中, 提供更高的灵活性和可编程性

Linux network namespaces

Linux Network Namespaces 是 Linux 内核提供的一种隔离机制, 用于将网络资源 (如网络接口, IP 地址, 路由表等) 隔离到独立的命名空间中, 使每个命名空间拥有独立的网络栈, 仿佛运行在单独的网络环境中.

  • 隔离指每个 Network Namespace 间的网络资源互不干扰

Linux operating system’s network stack

Linux 网络栈是内核中处理网络通信的软件实现, 负责从应用程序到物理网络设备的数据传输. 它遵循 OSI 模型 或 TCP/IP 模型, 将网络通信分为多个层次, 每一层负责特定的功能.

Proactive Routing 和 Reactive Routing

Proactive Routing

Proactive Routing 是指, 每个节点会周期性地更新和维护网络中所有节点的路由信息 (即使没有数据传输需求), 工作流程是:

  1. 节点定期广播路由更新信息
  2. 所有节点都维护有完整的路由表, 记录到网络中其他节点的最短路径
  3. 当需要发送数据时, 直接查找路由表选择路径

其优点在于:

  • 低延迟, 路径以预先计算, 数据传输时无需等待路由发现
  • 可靠性, 在网络拓扑稳定的场景, 能快速响应链路故障

缺点在于:

  • 高开销, 频繁的路由更新会占用宽带和计算资源
  • 扩展性差, 对于大型网络, 路由表的存储和更新所带来的开销更大

典型的协议有 OLSR, DSDV.

Reactive Routing

Reactive Routing 指按需路由, 仅在需要发送数据时, 才动态发现和维护路径, 节点中不会维护全局路由表, 仅在需要时建立路由. 其工作流程为;

  1. 当节点需要发送数据时, 通过广播 Route Request 发起路径发现
  2. 目标节点和中间节点通过 Route Reply 建立临时路径
  3. 路径仅在数据传输期间维护, 在过期后丢弃

其优点在于:

  • 低开销, 不会周期性更新路由, 适合拓扑频繁变化的动态网络
  • 扩展性好, 适合大规模或高移动网络

但缺点在于:

  • 初始延迟, 在路由发现过程中导致首次数据传输延迟
  • 路径维护复杂, 拓扑动态变化时的恢复机制

典型的协议有 AODS, DSR.

Hybrid Routing

结合两者的优点, 平衡开销和延迟, 适合中等规模的动态网络:

  • 在区域内使用主动路由, 在局部范围内维护路由信息
  • 在区域间使用按需路由, 跨区域通信时触发路由发现

如 ZRP, Zone Routing Protocol.

Single transceiver 和 Multitransceiver

Transceiver (收发器) 是一种可以同时发送和接收信号的设备, .

Single transceiver

对于 single transceiver 的设备而言, 其可能无法同时处理多个任务, 如在不同信道或频率上同时发送和接收数据. 且发送和接收模式需要切换.

比如一个基本的无线路由器, 只有一个射频模块, 如要处理 2.4GHz 和 5GHz 频段, 则需要顺序处理, 不能同时处理.

Multitransceiver

Multitransceiver 设备有多个 transceiver 单元, 每一个都能独立发送和接收信号, 即以同时处理多个通信任务, 例如在一个频率上发送数据, 同时在另一个频率上接收数据.

其带来性能提升, 但在设计更复杂和成本更高.

比如现代手机, 有多个射频模块用于处理不同的通信标准 (如 Wi-Fi, 蓝牙, 蜂窝网络).

Background

传统无线网络通常由:

  1. Wireless terminals, 如 laptops, smartphones. 用户与网络间的接口
  2. Radio links, 连接 terminals 和 coverage agent
  3. Base stations, 充当 coverage agent
  4. Switching and control centers, 集中 base stations 与其他通信服务连接

现今的无线协议也有很多, 如 Bluetooth, LTE, Zigbee, WiFi.

Radio waves 的传播可能会经历下面三种状态 (无顺序关系):

  • attenuation, 信号在传播过程中, 随着距离的增加, 能量逐渐减弱的现象 (信号能量在空间中扩散, 导致单位面积上的能量减少)
  • long-term fading, 信号在传播过程中, 由于路径上的障碍物 (如建筑物, 树木等) 导致的信号平均强度的缓慢变化 (障碍物会阻挡或吸收信号, 导致信号强度在较长时间尺度上发生变化)
  • short-term fading, 信号在传播过程中, 由于反射, 散射和衍射等现象导致的信号强度的快速波动

Wireless network emulation

可以用 TC 或 Wmediumd 来模拟 wireless medium.

TC

TC 指 Traffic Control, 用于管理网络接口的队列规则.

  • qdisc, Queueing Discipline, 表示队列规则, 用于控制数据包如何排队和调度
  • tc qdisc 列出当前的队列规则

输出如:

  • dev 后跟设备名
  • qdisc noqueue: 表示这是一个无队列 (noqueue) 的队列规则, 即不需要排队
  • root, 表示这是根队列规则 (即默认的队列规则)
  • refcnt 2: 表示该队列规则被引用了 2 次 (引用计数为 2)
  • qdisc netem: 表示这是一个 netem 队列规则. netem 是 Linux 中用于模拟网络条件的工具, 可以模拟延迟, 丢包, 抖动等
  • limit 1000: 表示队列的最大长度为 1000 个数据包
  • delay 2.5ms: 表示每个数据包会被延迟 2.5 毫秒
  • rate 6688Kbit: 表示带宽限制为 6688 Kbps (约 6.688 Mbps)

在 Infrastructure Network 下, 存在一个中心节点 (如接入点, Access Point, AP) 和多个站点 (Stations, STAs), 此时会根据 AP 和 STA 之间的距离 d, 计算接收信号强度, 然后 TC 根据信号强度动态调整 STA 的带宽, 当 STA 移动时, 距离 d 会变化, 致信号强度和带宽值也随之变化.

在 Non-Infrastructure Network (Ad Hoc 和 Mesh 网络) 下, 没有中心节点 (如 AP), 所有节点都是对等的 (Peer-to-Peer). 这种情况下, TC 和传播模型的应用更加复杂. 例如, STA1 可能与 STA2 和 STA3 同时关联, 但它们之间没有中心节点来计算距离 d. (此时需要借助 Wmediumd)

在 TC 环境下, 其 station 在扫描时不会考虑 wireless medium, 即使 ap 离开其 signal coverage, 也能扫描到.

Wmediumd

Mininet-WiFi 用 mac80211_hwsim 内核模块来虚拟化 WiFi 网卡, 且默认为所有无线节点提供相同的 virtual medium.

这个 virtual medium 指 distance 无效, 因此它们在内部任意位置都是相互可达的, 彼此间可以通过 scanning 来发现 (不论距离).

若想让无线节点可以用距离隔离, 则可以使用 Wmediumd.

Propagation model

Propagation model 用于模拟一些特殊的环境, 比如 indoor, outdoor, interference 等.

当前 Mininet-WiFi 支持的 propagation models 有:

  • Friis Propagation Loss Model, 该模型基于 Friis 方程, 用于计算自由空间中的信号传播损耗. 它假设天线是各向同性的, 没有热损失, 适用于远场区域的传播
  • Log-Distance Propagation Loss Model (default), 考虑了信号强度随距离的对数衰减, 适用于模拟城市环境中的无线信号传播
  • Log-Normal shadowing Propagation Loss Model, 在对数距离模型的基础上增加了对数正态分布的阴影效应, 以模拟由于建筑物, 地形等因素引起的信号强度的随机变化
  • International Telecommunication Union (ITU) Propagation Loss Model, 一套标准化的方法来计算不同环境下的无线信号传播损耗, 包括城市, 郊区和乡村等场景
  • Two-Ray Ground Propagation Loss Model, 考虑了地面反射对信号传播的影响, 适用于模拟地面对信号传播的反射效应. 它使用特定的公式来计算信号的传播损耗, 适用于模拟地面反射对信号传播的影响

选用正确的 Propagation model 比较重要.

有一些模型中, 信号不会随时间变化, 但现实中, 有各种因素会引起信号强度的变化. 可以通过设置 fading_coefficient 来模拟:

相关概念

Managed Mode 和 Ad-Hoc Mode

  • Managed Mode, 指无线网卡连接到一个无线接入点 (AP), 如无线路由器. AP负责管理网络, 包括通过 DHCP 分配 IP 地址, 处理客户端之间的通信等.
  • Ad-Hoc Mode, 指创建了一个没有 AP 的对等 (peer-to-peer) 网络, 每个设备都直接与其他设备通信, 没有中心控制点

Monitor mode

Monitor Mode 指, 允许设备监控经过该接口的数据包, 用于捕获网络流量的详细信息, 包括源地址, 目的地址, 协议类型等.

Mesh 网络

Mesh 网络是一种去中心化的网络架构, 其中每个节点 (设备) 可以连接到多个其他节点, 形成一个网状结构. 这种网络结构允许数据从节点到节点传输, 直到到达目的地, 提供了一个健壮且灵活的通信框架. 与传统网络不同, Mesh 网络中的设备可以直接相互通信, 无需依赖中央集线器或路由器.

Mesh 和 Ad-Hoc 网络的区别

  • 在简单的Ad-Hoc网络中，节点必须彼此直接可见才能通信
  • 在 Mesh 网络中, 节点可以转发其他节点的数据包, 这意味着即使两个节点不在彼此的直接通信范围内, 它们也可以通过中间节点进行通信

Beacons

Beacons (信标) 是一种由无线接入点 (AP) 定期发送的管理帧, 用于宣布无线网络的存在. 它在无线通信中扮演着关键角色, 主要用于网络同步和有效管理无线通信. Beacon 帧包含多个组件, 如时间戳, 网络名称 (SSID), 支持的数据速率, 信标间隔, 能力信息, 流量指示图 (TIM) 以及信息元素 (IEs) 等. 这些信息对于设备发现可用网络, 同步时钟以及获取网络参数等都至关重要.

Channel

“channel” (信道) 是指无线信号用于传输数据的特定频率范围. 无线网络使用电磁频谱的特定部分来发送和接收数据. 由于频谱资源有限, 为了有效利用这些资源并减少干扰, 无线网络将频谱划分为多个离散的频率范围, 每个范围就是一个信道.

比如:

  • channel 1 工作在 2.412MHz
  • chennel 6 工作在 2.437MHz
  • chennel 11 工作在 2.452MHz

每一个 channel 的宽度是 0.005MHz.

Scanning methods

有两种 scanning methods:

  • Active, client 发送一个 probe request, 然后监听来自 AP 的 probe response, probe request 通常包含一个特定的服务集标识符 (SSID), 只有具有该特定 SSID 的 AP 才会响应这个请求 (就像手机输入 SSID 找 WiFi)
  • Passive, client 监听每个信道, 等待 AP 定期发送的 beacons,. 与 Active scan 相比通常需要更长的时间来获取结果, 因为 client 必须等待 beacons 的出现, 然后手动找到接入点

Client 在 channel scanning 时不能接收或发送包, 为了减缓这种情况, 通常使用 background scanning. (由 wpa_supplicant 实现)

Background scanning 会在设备切换到新的 AP 之前进行, 减少客户端在扫描过程中无法传输或接收数据的停机时间，从而加快客户端在不同接入点之间的切换速度. (想象你在房间连一个 WiFi, 此时走到大厅, 要切换到另一个 WiFi, Background scanning 则在与房间 WiFi 断开前进行扫描, 加快连接大厅 WiFi 的速度)

在配置好 bgscan 之后, 其会检测设备与当前 AP 的强度, 一旦低于某个设定值, 就会寻找新 AP.

使用 bgscan 需要保证切换 AP 的 SSID 相同.

过程如下:

  1. h1 为了和 h2 通信, 先发送 ARP/IMCP 给 switch s1
  2. Switch s1 不知道 h2 的位置, 因此 s1 将 packet 转发给 controller (packet-in message, 由 switch 创建, 发送给 controller), controller 会建议 switch 该做什么.
  3. Controller 向 switch 响应 packet-out message, 让 switch 根据 h2 的 IP 地址询问其 MAC 地址
  4. Switch 向其所有 ports 进行 ARP/ICMP 广播 (除 h1 所在的 port)
  5. h2 做 ARP/ICMP 应答
  6. Switch 也不知道如何处理来自 h2 的包, 因此将 packet 转发给 controller, 由此学习到 h2 的位置
  7. Controller 给 switch 发送 flow mod message, 让 switch 把来自 h2 的 packet 转发到 h1
  8. 最后 h1 受到 ICMP 的响应

交换机通常有两种运行模式:

  • Proactive Mode, 主动模式, 此时每个节点会周期性地更新和维护网络中所有节点的路由信息 (即使没有数据传输需求)
  • Reactive Mode, 反应模式, 仅在需要发送数据时, 才动态发现和维护路径

在 Wireless Network 中, AP 只有一个 interface 来与 stations 关联, 且即用作输入, 也用作输出. 因此在应用 OpenFlow 协议时, 需要进行特殊处理.

注意事项

AP 默认不具有路由功能

在 Mininet-WiFi 中, AP 通常是作为 Open vSwitch (OVS) 网桥存在的.

AP 的主要功能是为无线设备提供接入服务, 即接收和转发无线设备的数据帧, 但它们通常不会对数据包进行路由决策, 如 IP 转发.

AP 本质上是一个无线接入点, 其核心功能是桥接无线设备到有线网络. 它类似于一个无线交换机, 主要处理 MAC 地址 (第二层) 的通信, 而不是 IP 地址 (第三层)的路由.

若想让其具有转发功能, 需要手动开启转发并配置路由规则, 如:

或让控制器配置流表.

默认 controller 的功能

Mininet-WiFi 默认使用 ovs-controller, 一个轻量级的 OpenFlow 参考控制器, 仅提供基础的二层转发功能.

AP 的网络命名空间

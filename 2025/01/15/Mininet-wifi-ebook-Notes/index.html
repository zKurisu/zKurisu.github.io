相关概念

Path loss

Path loss 指信号在从发射端 (如基站) 传播到接收端 (如无线终端) 的过程中, 由于距离增加和传播环境的影响, 信号强度逐渐减弱的现象.

RSSI

RSSI, Received Signal Strength Indicator, 是用来衡量接收信号强度的指标, 单位通常是 dBm (分贝毫瓦). 如: -70 dBm 表示信号强度比 1 毫瓦低 70 dB.

  • RSSI 值越小 (越负), 表示信号强度越弱
  • RSSI 值越大 (越接近 0), 表示信号强度越强

Propagation Models

Ebook 中提到的三种:

  • Free-Space, 理想的无障碍环境, 如真空或开阔空间.
    • 没有障碍物、反射、散射或衍射
    • 信号能量仅因距离增加而扩散
  • Log-Distance, 实际环境, 如城市, 郊区或室内.
    • 考虑障碍物, 反射, 散射或衍射
    • 信号强度随距离增加而衰减时, 衰减速率由路径损耗指数 (Path Loss Exponent, n) 决定
  • ITU, International Telecommunication Union, 用于特定环境 (如城市, 郊区, 室内) 或特定频段
    • 提供了一系列标准化的传播模型, 适用于不同的通信场景和频段
    • 这些模型基于大量实测数据和统计分析, 具有较高的准确性

BER

Bit Error Rate, 指在数字通信系统中, 接收端错误接收的比特数与总传输比特数的比率. 其反映信号在传输过程中受到噪声, 干扰和失真影响的程度.

$$
\displaylines
{
\begin{aligned}
BER = \frac{错误接收的比特数}{总传输比特数}
\end{aligned}
}
$$

Transmisster power 越大, BER 越小.

SNR

Signal-to-Noise Ratio, 指信号功率与噪声功率的比值, 用于衡量信号相对于噪声的强度. 其反映信号在传输过程中受到噪声影响的程度.

$$
\displaylines
{
\begin{aligned}
SNR = \frac{P_{signal}}{P_{noise}}
\end{aligned}
}
$$

Transmisster power 越大, SNR 越大.

Internal Signal Range

Internal Signal Range 指信号在设备或系统内部传播的范围, 如: 电路板, 芯片, 模块或其他内部组件之间的信号传输.

External Signal Range

External Signal Range, 指信号在设备或系统外部传播的范围, 如: 无线信号或有线信号在开放环境中的传播.

Simulator, Emulator 和 Testbed

  • Simulator, 模拟器, 通过软件模型模拟系统的行为或环境, 通常不涉及实际硬件 (纯软件)
  • Emulator, 仿真器, 通过软件或硬件模仿特定系统的功能, 能够运行与实际系统相同的软件 (要结合硬件)
  • Testbed, 测试平台, 为实验和测试设计的实际或虚拟环境, 通常包含真实硬件和软件 (实际测试)

Mininet-wifi 属于 Emulator.

SoftMac

SoftMAC 指通过软件实现网络协议栈中的 MAC (媒体访问控制) 层功能, 而不是依赖硬件. MAC 层负责控制数据帧在物理网络介质上的传输, 传统上由硬件实现, 但 SoftMAC 将其移至软件中, 提供更高的灵活性和可编程性

Linux network namespaces

Linux Network Namespaces 是 Linux 内核提供的一种隔离机制, 用于将网络资源 (如网络接口, IP 地址, 路由表等) 隔离到独立的命名空间中, 使每个命名空间拥有独立的网络栈, 仿佛运行在单独的网络环境中.

  • 隔离指每个 Network Namespace 间的网络资源互不干扰

Linux operating system’s network stack

Linux 网络栈是内核中处理网络通信的软件实现, 负责从应用程序到物理网络设备的数据传输. 它遵循 OSI 模型 或 TCP/IP 模型, 将网络通信分为多个层次, 每一层负责特定的功能.

Background

传统无线网络通常由:

  1. Wireless terminals, 如 laptops, smartphones. 用户与网络间的接口
  2. Radio links, 连接 terminals 和 coverage agent
  3. Base stations, 充当 coverage agent
  4. Switching and control centers, 集中 base stations 与其他通信服务连接

现今的无线协议也有很多, 如 Bluetooth, LTE, Zigbee, WiFi.

Radio waves 的传播可能会经历下面三种状态 (无顺序关系):

  • attenuation, 信号在传播过程中, 随着距离的增加, 能量逐渐减弱的现象 (信号能量在空间中扩散, 导致单位面积上的能量减少)
  • long-term fading, 信号在传播过程中, 由于路径上的障碍物 (如建筑物, 树木等) 导致的信号平均强度的缓慢变化 (障碍物会阻挡或吸收信号, 导致信号强度在较长时间尺度上发生变化)
  • short-term fading, 信号在传播过程中, 由于反射, 散射和衍射等现象导致的信号强度的快速波动

SDWN

SDWN, Software-Defined Wireless Networking, 与 SDN 类似, 其在无线网络层面实现控制层面和数据层面分离.

  • 当前最流行的南向接口是 openflow

用 Mininet-wifi 来仿真有两个特点:

  • 能直接接入 third-party tools 使用
  • 其使用 actual network protocol stack

Mininet-wifi 能虚拟 stations, access points, 也能使用 Mininet 中的节点. (Mininet-wifi 结合了 Mininet 以及 SoftMac 无线驱动)

  • Stations 指连接到无线网络的设备, 其工作在 Managed Mode, 通过认证和关联连接到 Access Point
  • Access Point 指提供无线网络连接的设备, 工作在 Master Mode, 负责管理与其关联的站点

Components

  • 内核空间中
    • 内核模块 mac80211_hwsim 虚拟化 WiFi network cards, 其所有特性基于 mac80211 模块
    • 内核模块 cfg80211 提供无线设备的配置和管理接口
    • wlan1, wlan2, wlan3 是 mac80211_hwsim 创建的虚拟无线接口
  • 用户空间中
    • nl80211 进程负责用户空间与内核空间的交互
    • sta1-wlan0, sta2-wlan0, ap1-wlan0, 都是虚拟的无线接口
    • sta1-wlan0 和 sta2-wlan0 代表两个 station
    • ap1-wlan0 代表一个 Access Point
    • ofdatapath -> ofprotocol -> controller 是 OpenFlow 协议栈的一部分
    • wpa_supplicant 负责无线客户端连接的认证和加密
    • iw, 用于配置和管理无线网络接口
    • hostapd 用于配置和管理 AP

Wireless network emulation

可以用 TC 或 Wmediumd 来模拟 wireless medium.

TC

TC 指 Traffic Control.

在 Infrastructure Network 下, 存在一个中心节点 (如接入点, Access Point, AP) 和多个站点 (Stations, STAs), 此时会根据 AP 和 STA 之间的距离 d, 计算接收信号强度, 然后 TC 根据信号强度动态调整 STA 的带宽, 当 STA 移动时, 距离 d 会变化, 致信号强度和带宽值也随之变化.

在 Non-Infrastructure Network (Ad Hoc 和 Mesh 网络) 下, 没有中心节点 (如 AP), 所有节点都是对等的 (Peer-to-Peer). 这种情况下, TC 和传播模型的应用更加复杂. 例如, STA1 可能与 STA2 和 STA3 同时关联, 但它们之间没有中心节点来计算距离 d. (此时需要借助 Wmediumd)

概览

SDN, Software Defined Network, 允许逻辑集中的软件程序控制整个网络的行为.

其将网络的控制逻辑与转发流量的底层物理路由器和交换机分离开来, 网络操作员就可以编写高级控制程序来指定整个网络的行为.

一些概念

Topology-dependent MAC Addressing, 根据网络物理或逻辑拓扑动态分配 MAC 地址. 例如, 在虚拟化环境中, 为不同租户或服务分配唯一的 MAC 前缀, 实现流量隔离和简化路由.

IP Addressing for Application Compatibility, 设计 IP 地址方案时, 考虑遗留系统或特定应用的需求 (如固定 IP 段, 子网划分), 避免地址冲突并支持无缝迁移.

RCP, Routing Control Platform, 指一种集中式路由管理架构, 通过统一平台处理路由决策, 替代传统的分布式 BGP 路由计算. RCP 可视为 SDN 控制器的扩展, 专注于路由优化.

BGP 的基本工作原理

IGP, Internal Gateway Protocol, 如 RIP 或 OSPF, 用于 AS 内部的路由, 不考虑 AS 外的因素.

EGP, External Gateway Protocol, 如 BGP (Border Gateway Protocol), 用于 AS 之间的路由选择, 由于不同 AS 内 “代价” 不同 (有的以时延, 有的以带宽, 有的以距离), 因此需要另外的度量方式. AS 间的路由还要考虑政治, 经济, 安全的策略.

BGP 的目的是寻找一条比较好的路由, 而非寻找一条最佳路由.

4D network structure

配置 network 而非配置 router.

4D network structure 分四层:

  • Dicision, 负责所有的 management 和 control, 如 Path selection, Reachability control
  • Dissemination, 负责与 router 交互, 提供 network-wide view
  • Discovery, 负责 topology and traffic monitoring (收集信息)
  • Data, 负责 processing packets

控制平面

当 flow table 没有匹配 packets 时, 其会被发往 controller 处理, 此时的开销称 “控制器开销”.

若 matched, 可以执行的 actions 如:

  • Forward
  • Drop
  • Modify
  • Enqueue

OpenFlow (v 1.3) 中添加了:

  • Action Set, 对 packet 执行多个 actions
  • Group, a list of action sets, 一个 packet 需要分别由不同 action set 处理并输出
  • pipline 处理

OpenFlow Switch Components 包括:

  • Secure channel
  • Flow table (match and action)
  • (New) Group tables

SDN Controller

选用 SDN Controller 时一般需要考虑:

  • Programming language (与 performance 挂钩)
  • Learning curve
  • User base and community support
  • Focus
    • Southbound API
    • Northbound API
    • Support for OpenStack
    • Education, Research, or Production?

Week 4

Coursera

这一部分主要讲网络虚拟化和 SDN 的关系.

相关概念

  • 网络虚拟化: 允许在单个共享物理平台上托管多个逻辑网络的技术. 该技术通常用于数据中心, 例如允许多个租户共享同一物理数据中心基础设施
  • 网络仿真: 创建模拟真实网络条件的网络环境的过程: 与模拟环境不同, 仿真环境是实时运行的, 还允许实验者运行真实的软件和操作系统. 常用的网络仿真器是 Mininet, 我们将在本课程中使用它
  • 虚拟网络接口: 在虚拟网络节点 (如虚拟机) 中实例化的网络接口. 虚拟网络接口通常必须使用软件/内核交换机 (如 Open vSwitch) “桥接” 到机器的物理网络接口

Network Virtualization

Network 中可以被虚拟化的部分有:

  • Nodes: Virtual machines, 如 Xen, KVM, VMWare, Virtual Box
  • Links: Tunnels, Ethernet GRE
  • Storage

SDN 和 Virtual Network 的对比:

Network Function Virtualization

NFV, Network Function Virtualization, 网络功能虚拟化, 是一种网络架构和运营模型, 旨在将传统的专用硬件网络设备功能 (如路由器, 防火墙, 负载均衡器等, 此时称 middlebox) 转移到通用的服务器, 存储设备和网络设备上.

  • Steering 指的是在网络中引导流量通过一系列虚拟网络功能 (VNFs) 的过程
  • Placement 则涉及到在网络中的物理位置或虚拟位置部署 VNFs 的决策. 在 NFV 中, VNFs 可以根据运行时的需求和网络条件灵活地放置在网络中的任何位置

RouteBricks

硬件速度快但不灵活, 软件更加灵活但速度不佳, 因此要考虑

  • 如何让硬件 programmable
  • 如何让软件更快

RouteBricks 是一种软件路由器架构, 它通过在多个服务器和单个服务器内的多个核心上并行化路由器功能来提高性能. 这种架构旨在解决软件路由器的扩展性问题, 通过利用服务器技术的最新进展, 特别是高速并行处理能力.

Switching fabric 用于描述数据在网络设备之间传输方式, 它本质上是硬件和软件的组合, 共同工作以连接网络中的多个设备, 并允许它们相互通信.

Line card 是网络系统中的一个关键组成部分, 它是一种模块化的电子组件, 对于电信接入网络至关重要. 它提供了用于发送和接收的端口, 连接局域网 (LAN) 和广域网 (WAN) 网络. 在 line card 中, 电路通常内置于每个端口中, 无论是小型还是中型路由器.

传统的 Hardware Router 的处理速率为:

  • 每一个 linecard 处理速率为 $R$
  • 总的 switching 速率为 $N \times R$

RouteBricks 架构如下:

  • Commodity interconnect 指一种通用的互联系统, 用于 servers 间的数据交互
  • 数据由位于 server 上的 linecards 处理
  • 假如一个 server 的处理速率为 $R$
  • 需要满足几个条件 (challenges)
    • internal link rates $< R$
    • Per-server processing rate 为 $c \times R$, $c$ 是一个 small integer
    • Per-server fanout (ports 数量) 为 constant

一个 server 可以视作下面结构:

  • 多核提供并行处理
  • 一些用于加快处理的方法有
    • Large packet buffers to hold multiple packets
    • Batch processing
    • Ethernet GRE

RMT

  • Protocol-independent processing
  • Control and repurpose in the field
  • Implemented by fast, low-power chips

但实际上, 这些功能受当前硬件限制:

  • OpenFlow is protocol dependent

两个 Programmable Hardware Data Planes 的示例:

  • OpenFlow Chip, Generalizable, programmable match-action primitives
  • SwitchBlade, Programmable modularizable FPGA-based data plane

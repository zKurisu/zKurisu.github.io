Architecture

Ryu Controller 的编写逻辑为:

  • 定义继承自 app_manager.RyuApp 的类, 这是入口, 整个文件就是在完善这个类
  • 利用 set_ev_cls 修饰器添加事件处理函数

主要的包

ryu.base

一般导入 app_manager, 用作管理应用程序的生命周期, 所有的 Ryu 应用程序都需要继承 app_manager.RyuApp.

ryu.controller

一般导入 ofp_event 类 (ofp 指 openflow protocol), 其定义了与 OpenFlow 相关的事件类, 表示交换机与控制器之间的各种交互:

常见有:

  • EventOFPPortStatus, 交换机端口状态变化
  • EventOFPFlowRemoved, 交换机流表项被删除
  • EventOFPHello, 交换机与控制器建立连接

将接收的包送往所有 ports

  • @set_ev_cls(event_class, dispatcher_phase), set event class, 该修饰器将一个方法注册为特定事件 (当事件类型为 xxx, 事件处理阶段为 xxx 时) 的事件处理函数
    • event_class 即某一特定事件, 比如 opf_event.EventOFPPacketIn, 指收到一个数据包时
    • dispatcher_phase 指事件处理的阶段, MAIN_DISPATCHER 表示交换机已经完成握手并进入正常工作状态
  • Ryu 中常见的的事件处理阶段有:
    • HANDSHAKE_DISPATCHER, 握手阶段
    • CONFIG_DISPATCHER, 配置阶段
    • MAIN_DISPATCHER, 主调度阶段 (正常工作状态)
    • DEAD_DISPATCHER, 断开连接阶段
  • 所修饰函数的第二个参数是 event (类型就是修饰器中的 event_class), 包含与该事件的所有相关信息

Switching Hub

Switching Hub 所具有的功能有:

  • 学习连接到 port 的 host 的 MAC 地址, 并写入 MAC address table
  • 当接收到的数据包指向一个已知地址的 host 时, 从对应端口发出
  • 当接收到的数据包指向一个未知地址的 host 时, 执行 flooding

OpenFlow 交换机从 Ryu Controller 接收指示后需要完成:

  • 对接收的 packet 进行修改或者将 packet 从指定 port 发出
  • 将 packet 发送到 controller (Packet-In)
  • 根据从 controller 接收到的指示将数据包从指定 port 发出 (Packet-Out)

注意, Packet-In 和 Packet-Out 是 OpenFlow 交换机与控制器之间的两种主要通信类型:

  • Packet-In 消息是由交换机发送给控制器的, 当交换机无法匹配流表规则时, 会将数据包发送给控制器处理
  • Packet-Out 消息是从控制器向 OpenFlow 交换机发送的消息, 是包含数据包发送命令的消息. 控制器可以使用 Packet-Out 消息, 告诉交换机某一个数据包如何处理

每当 Switch 产生 Packet-In packet 时也会学习 MAC 地址与 port 的对应.

当 switch 转发包时:

  • 目的 Host 已知, 则转发到对应 port
  • 目的 Host 未知, 则 flooding

Class Definition and Initialization

构建 Ryu application 的第一步, 就是创建一个继承自 ryu.bash.app_manager.RyuApp 类的类. 且需要用 OFP_VERSIONS = [ xxx ] 来指定 OpenFlow 协议版本.

Event Handler

对于 Ryu 而言, 每接收到一个 OpenFlow message, 就会产生一个对应的事件. 其针对每一个 message 都有对应的 event handler.

这些 event handler 都以 event object 作为参数, 且用 ryu.controller.handler.set_ev_cls 修饰器来修饰.

  • set_ev_cls 指定对哪个 event class message 做处理, 以及此时 switch 需要满足的状态

Event class 的命名规则为: ryu.controller.ofp_event.EventOFP + <OpenFlow message name>, 比如 EventOFPPacketIn.

Switch state 有:

  • ryu.controller.handler.HANDSHAKE_DISPATCHER, 交换 HELLO message 时
  • ryu.controller.handler.CONFIG_DISPATCHER, 等待接收 SwitchFeatures message 时
  • ryu.controller.handler.MAIN_DISPATCHER, normal status
  • ryu.controller.handler.DEAD_DISPATCHER, 与控制器断开连接时

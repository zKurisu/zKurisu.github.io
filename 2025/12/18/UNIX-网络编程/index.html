UNIX-网络编程

socket 的本质

从架构上看, socket 是应用层与传输层之间的抽象层, 封装了网络协议的细节:

在 UNIX/Linux 系统中, socket (套接字) 被抽象为一个特殊的文件类型. 和普通文件一样, 在文件系统中有路径标识, 但是 socket 文件存储的内容并不在磁盘上, 比如: /tmp/tmux-0/default:

  • 该文件是 socket 类型文件, 文件大小是 0, 且没有分配任何数据块

这是源自 UNIX 的 “一切皆文件 (Everything is a file)” 的设计哲学, 不管是普通文件, 硬件设备 (如 /dev/tty), 管道 (pipe), FIFO, 还是 socket, 都可以用统一的文件接口 (如 open(), write(), close() 等系统调用) 进行操作.

当创建一个 Unix domain socket 时, 内核会在文件系统中创建一个特殊的 inode, 其文件类型标记为 socket, 这个 inode 不包含数据块, 但存储了:

  • socket 的元信息 (如协议族, 类型等)
  • 指向内核中对应 socket 数据结构的指针
  • 权限, 所有者等常规文件属性

这个 “空文件” 用作内核对象在文件系统中的句柄或入口点, 而不是用来存储数据的容器.

因此 socket 实际的通信缓冲区, 连接状态, 协议控制块等都完全存在于内核内存中, 与磁盘无关. socket 文件只是方便不同进程能通过路径名来找到这个内核数据结构.

Socket 分类

由于不同应用场景对通信的要求差异很大, 因此 Socket 分为几种类型. 若按通信语义划分, 有:

  1. SOCK_STREAM, 流式套接字. 通常基于 TCP (也可以是其他可靠流协议, 如 SCTP), 用于面向连接, 数据有序, 无重复的可靠传输
  2. SOCK_DGRAM, 数据报套接字. 通常基于 UDP (也可以是其他无连接协议), 用于无连接, 每个数据包独立发送, 不需要保证顺序, 可靠性的场景
  3. SOCK_RAW, 原始套接字, 直接访问底层网络协议 (如 IP, ICMP, TCP 头部等), 可以自定义协议头, 需要 root 权限, 通常用于网络工具开发
  4. 还有 SOCK_SEQPACKET 等较少使用的类型

若按照通信范围, 可以划分为多个 domain, 在 socket(domain, type, protocol) 的第一个参数处指定 (AF 指 Address Family):

Domain Description
AF_INET IPv4 网络通信
AF_INET6 IPv6 网络通信
AF_UNIX/AF_LOCAL 本地进程间通信
AF_PACKET 链路层原始包访问
AF_NETLINK 内核与用户空间通信

默认阻塞的系统调用

默认情况下, socket 是阻塞模式, 比如下面这些系统调用在特定条件下会阻塞当前线程/进程:

系统调用 阻塞条件 说明
accept() 没有客户端连接到达 TCP 服务器等待新连接
connect() 无法立即完成 TCP 三次握手 TCP 客户端发起连接（UDP 不阻塞）
recv() / recvfrom() 接收缓冲区无数据可读 对 TCP：可能等到有数据或对端关闭；对 UDP：等到一个完整数据报
send() / sendto() 发送缓冲区满（TCP） TCP 流控导致无法立即写入内核缓冲区；UDP 通常不阻塞（除非内核缓冲区满）
read() / write() 同 recv/send 因 socket 是文件描述符，也可用通用 I/O 函数

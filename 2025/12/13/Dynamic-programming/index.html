Dynamic-programming

介绍

Dynamic programming (动态规划) 是一种通过分解问题并存储子问题来避免重复计算的算法思想, 主要用于解决具有重叠子问题和最优子结构的复杂问题.

Dynamic programming 的本质是用 “空间换时间”, 通常包含下面步骤:

  1. 分解问题, 将原问题拆分为多个相互关联的子问题
  2. 存储子问题解, 避免重复计算 (记忆化)
  3. 递归求解, 从最小子问题开始逐步推导到原问题, 或者从原问题出发递归求解子问题并缓存结果

适用条件

  1. 最优子结构 (Optimal Substructure), 原问题的最优解包含子问题的最优解. 如果原问题的解为 $f(n)$, 则 $f(n)$ 可由子问题 $f(k), k < n$ 的最优解的组合而成. 比如斐波那契数列: $F(n) = F(n-1) + F(n-2)$, 其中 $F(n)$ 的最优解由 $F(n-1)$ 和 $F(n-2)$ 的最优解组合而成
  2. 重叠子问题 (Overlapping Subproblems), 在递归求解过程中, 相同的子问题会被反复计算多次. 通过存储子问题的解, 避免重复计算. 比如斐波那契数列的递归计算: $F(5) = F(4) + F(3), F(4) = F(3) + F(2)$, $F(3)$ 被使用了两次

与 Divide and conquer 的相似和不同

Dynamic programming 和 divide and conquer 都利用 “分解问题” 的思想, 都可以用递归实现.

核心的区别有:

  • 子问题的 “重叠性” 不同. divide and conquer 要求子问题独立, 无重叠. 而 DP 要求子问题高度重叠, 同一个子问题会被计算多次
  • 解决问题的核心策略不同. divide and conquer 的核心是 “分解后独立解决, 最后合并结果”, 重点是独立性和合并的高效. 而 DP 的核心是 “存储子问题的解, 避免重复计算”

DP 的两种实现

Dynamic programming 有两种经典的实现思路:

  • 自顶向下 (记忆化搜索)
  • 自底向上 (迭代填表)

自顶向下

自顶向下 (Too-Down with memorization) 的核心思路是: 从原问题出发, 递归地分解为子问题, 并用一个哈希表 (或数组) 缓存已计算的子问题解. 若子问题已计算过, 则直接返回缓存结果; 否则递归计算并缓存.

优点是不需要设计复杂的状态转移方程; 缺点是递归深度过大可能导致栈溢出.

自底向上

自底向上 (Bottom-Up Tabulation) 的核心思路是: 从最小的子问题开始, 按从小到大的顺序计算并填充表格 (数组), 逐步推导出原问题的解. 表格的每个位置对应一个子问题的解.

优点是能够避免栈溢出, 空间复杂度可控; 缺点是需要预先设计好状态转移方程和表格结构:

相关概念

在自顶向下中, 下面几个概念都是隐式的:

  • 状态, 隐含在递归函数的参数中. 例如, 斐波那契的记忆化递归函数 fib(n, memo) 中, 参数 n 就是状态 (表示 “第 n 个斐波那契数”)
  • 状态转移方程, 隐含在递归调用的逻辑中. 例如, fib(n) = fib(n-1) + fib(n-2) 就是状态转移方程, 但它没有直接写成 dp[i] = dp[i-1] + dp[i-2] 的形式, 而是通过递归调用实现
  • 边界条件, 就是递归的终止条件 (如 n <= 1 时直接返回结果)

状态

状态 (State) 是用于描述子问题的变量, 通常用 dp[i] 或 dp[i][j] 表示. 状态的定义直接决定了问题的复杂度.

比如:

  • 斐波那契数列问题中, dp[i] 表示 “第 i 个斐波那契数”
  • 0-1 背包问题中, dp[i][j] 表示 “前 i 个物品, 容量为 j 时的最大价值”

这里需要注意 dp 数组维度的选择, 考虑子问题的状态是否需要两个独立维度共同描述, 当状态转移需要 dp[i-1][j], dp[i][j-1], dp[i-1][j-k] 等不同维度状态时才添加维度.

状态转移方程

状态转移方程 (State Transition Equation), 是子问题之间的递推关系, 描述了如何从已知子问题的解推导出当前子问题的解.

比如:

  • 斐波那契数列中: dp[i] = dp[i-1] + dp[i-2]
  • 0-1 背包问题: dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])

边界条件

边界条件是 “最小子问题的解”, 是状态转移的 “起点”, 没有边界条件, 状态转移方程无法启动.

比如:

  • 斐波那契数列: dp[0] = 0, dp[1] = 1
  • 0-1 背包问题: dp[0][j] = 0 (前 0 个物品, 任何容量的价值都是 0), dp[i][0] = 0 (任何物品, 容量 0 时的价值是 0)

常见问题

Weighted Interval Scheduling

Weighted Interval Scheduling (加权区间调度) 是 dynamic programming 的一个经典应用问题, 其核心是在不重叠的区间中选择一组区间, 使得它们的权重之和最大.

问题定义: 给定一组区间, 每个区间 i 有:

  • 开始时间 $s_i$ (start)
  • 结束时间 $e_i$ (end)
  • 权重 $w_i$ (比如重要性)

处理该问题时需要进行预处理: 按结束时间 $e_i$ 对区间进行排序, 排序之后, 区间的索引 $i$ 越大, 结束时间越晚.

State 定义: dp[i] 表示前 i 个区间 (排序后的前 i 个) 时, 能获得的最大权重.

State Transition Equation 定义: 对于第 i 个区间 (排序后第 i 个) 有两种选择:

  • 不选第 i 个区间: 最大权重等于前 i - 1 个区间的最大权重, 即 dp[i] = dp[i - 1]
  • 选第 i 个区间: 需要找到最后一个结束时间 $\le$ 第 i 个区间开始时间 $s_i$ 的区间 p(i), 此时最大权重为 $w_i + dp[p(i)]$ (dp[p(i)] 就是找到前一个最近的相容的区间的最优权重)

这里 p(i) 指最后一个不重叠区间的索引, 即最大的索引 $j < i$, 使得区间 $j$ 的结束时间 $e_j \le s_i$. 计算方法可以是: 在区间已经按结束时间排序后, 通过二分查找高校找到 p(i), 即在区间 [0, i-1] 中查找最大的 $j$, 满足 $e_j \le s_i$.

边界条件: $dp[0] = 0$, 没有区间时, 权重为 0.

总结下算法具体流程:

  1. 排序: 将所有区间按结束时间 $e_i$ 从小到达排序
  2. 预处理 $p(i)$: 对每个区间 i, 用二分查找计算 p(i)
  3. 动态规划填表: 按 i 从 1 到 n 计算 dp[i]
  4. 结果: dp[n] 即最大权重之和

总体的时间复杂度为 $O(n logn)$:

  • 排序耗时 $O(n logn)$
  • 计算每个 p(i) 耗时 $P(log n)$
  • 动态规划填表耗时 $O(n)$

空间复杂度为 O(n), 用于存储排序后的区间, p(i) 数组和 dp 数组.

Sequence Alignment

Sequence alignment (序列对齐) 是生物信息学, 计算机科学中的常见问题, 目标是通过对齐两个或多个序列 (如 DNA, RNA, 蛋白质序列, 字符串序列), 找到它们之间的相似性, 差异或进化关系. 对齐过程中允许插入, 删除或替换字符 (或氨基酸/碱基等), 以最小化 “不匹配” 的代价.

相关概念有:
Alignment (对齐) 指在两个序列中插入空格 (Gap), 使它们的长度相等, 以便于逐位比较. 例如:

  • 序列 1: ACGT
  • 序列 2: AGT

Score (得分) 用于衡量对齐的好坏. 通常由以下三种操作的 cost 决定:

  • Match (匹配): 两个字符相同, 得正分 (如 +1)
  • Mismatch (需替换): 两个字符不相同, 得负分 (如 -1)
  • Gap (需插入/删除): 插入或删除一个字符 (空格), 得负分 (如 -2)

不同的 align 方式会得到不同的分数, 而目标是找到总得分最高的对齐方式 (或者总 cost 最低的对齐). 比如下面 3 中不同的 align:

最经典的 sequence alignment 问题是 “双序列全局对齐”, 即强制对齐两个序列的所有字符 (包括首尾), 可以用 DP 求解.

求解

问题定义: 给定两个序列 $X = x_1 x_2 … x_m$ 和 $Y = y_1 y_2 … y_n$, 以及:

  • 问题得分 match(a, b), 若 a = b 则为 s, 否则为 t
  • 间隙代价 gap, 插入/删除一个字符时的得分, 通常为负数

目标: 找到全局对齐的最大得分.

状态定义: 定义 dp[i][j] 为序列 X 的前 i 个字符 ($x_1…x_i$) 与序列 Y 的前 j 个字符 ($y_1…y_j$) 的最大对齐得分.

状态转移方程: 对于 dp[i][j], 有三种可能的对齐方式:

  1. $x_i$ 与 $y_j$ 对齐, 得 $match(x_i, y_j)$ 分: dp[i-1][j-1] + match(x_i, y_j) (i-1 表示此时 X 消耗一个字符达到 i 状态, 同理 j-1 表示此时需要 Y 消耗一个字符达到 j 状态)
  2. $x_i$ 与空格对齐: dp[i-1][j] + gap (只需要 X 消耗一个字符达到 i 状态)
  3. $y_i$ 与空格对齐: dp[i][j-1] + gap (只需要 Y 消耗一个字符达到 j 状态)

综上, 状态转移方程为:

$$
\displaylines
{
\begin{aligned}
dp[i][j] = max \begin{cases}
dp[i-1][j-1] + match(x_i, y_j) \newline~ \newline
dp[i-1][k] + gap \newline~ \newline
dp[i][j-1] + gap
\end{cases}
\end{aligned}
}
$$

边界条件:

  • 当 i = 0 (空序列 X) 时, 对齐 Y 的前 j 个字符需要插入 j 个间隙: dp[0][j] = j x gap
  • 当 j = 0 (空序列 Y) 时, 对齐 X 的前 i 个字符需要插入 i 个间隙: dp[i][0] = i x gap

算法步骤:

  1. 初始化 DP 表: 创建 (m + 1) x (n + 1) 的二维数组 dp, 填充边界条件 dp[0][j] = j x gap 和 dp[i][0] = i x gap
  2. 填充 DP 表: 按 i 从 1 到 m, j 从 1 到 n 的顺序, 根据状态转移方程计算每个 dp[i][j]
  3. 回溯构造对齐: 从 dp[m][n] 出发, 根据转移路径回溯, 确定对齐方式

时间复杂度是 $O(mn)$.

空间复杂度也是 $O(mn)$.

第二章: 类型和函数

Haskell 中的每个函数和表达式都带有各自的类型. 一个值的类型标识了它和该类型的其他值所共有的一簇属性 (property).

类型不仅会为单纯的字节加上意义, 还会阻止我们混用不同的类型, 避免程序错误. 比如说, 类型系统通常不会允许将一个酒店预约数据当作汽车租赁数据来使用.

Haskell 的类型系统

Haskell 的类型系统有这些特点:

  • 强 (strong) 类型
  • 静态的 (static)
  • 可自动推导 (automatically inferred)

强类型

强类型语言是指:

  • 类型安全, 每个值都有明确的类型, 且操作需要符合类型的语义, Haskell 强类型系统会拒绝执行任何无意义的表达式, 保证程序不会因为这些表达式而引起错误. 比如将整数当函数使用
  • 禁止危险的隐式转换, 值的类型在运行时是严格保持的, 不允许发生未预期的, 隐式的类型转换. 比如将一个整型值传给一个接受浮点数的函数, C 编译器会自动且静默地将参数从整数类型转换为浮点类型, 而 Haskell 编译器会引发编译错误

遵守类型规则的表达式被称为是 “类型正确的 (well typed)”, 而不遵守类型规则, 会引起类型错误的表达式被称为是 “类型不正确的 (ill typed)”.

要在 Haskell 中进行类型转换, 必须显示地使用类型转换函数.

静态类型

静态类型系统指, 编译器可以在编译期 (而不是执行期) 知道每个值和表达式的类型. Haskell 编译器或解释器会发现类型不正确的表达式, 并拒绝这些表达式的执行:

惰性求值

弄清楚 isOdd (1 + 2) 的结果是如何求值的.

在使用严格求值的语言里, 函数的参数总是在应用函数之前被求值. 比如上述例子中: 子表达式 (1 + 2) 会首先被求值, 得到结果 3. 接着将 3 绑定到变量 n, 应用到函数 isOdd. 然后 mod 3 2 返回 1, 而 1 == 1 返回 True.

Haskell 使用了另外一种求值方式 – 非严格求值. 在这种情况下, 求值 isOdd (1 + 2) 时并不会立即将子表达式 1 + 2 求值为 3, 此时编译器会做出一个 “承诺”, 说, “当真正有需要的时候, 我有办法计算出 isOdd (1 + 2) 的值”.

用于追踪未求值表达式的记录被称为 “块 (chunk)”. 编译器通过创建块来延迟表达式的求值, 知道这个表达式的值真正被需要为止. 如果某个表达式的值不被需要, 那么从始至终, 这个表达式都不会被求值. 简单来说, 只要程序还能继续运行且不需要知道这个值的具体内容, 它就不会被计算.

缓冲区 (Buffering)

I/O 子系统是现代计算机中最慢的部分之一. 完成一次磁盘读写的时间是一次内存读写几千倍. 完成一次网络读写也会慢上千倍.

由于这个原因, 现代操作系统和编程语言都提供了工具来帮助程序更快进行 I/O:

  • 操作系统一般采用缓存 (Cache), 把频繁访问的数据片段保存在内存中
  • 编程语言通常采用缓冲区 (Buffering). 它们每次向操作系统请求一大片内存, 通过减少 I/O 请求次数来读取相同数量的数据

缓冲区模式

Haskell 有 3 中不同的缓冲区模式, 其定义为 BufferMode 类型:

  • NoBuffering, 没有缓冲区, 每次向操作系统请求一个字符. 读取和写入都是一个字符一个字符进行, 通常性能很差
  • LineBuffering, 行缓冲, 当换行符输出时会让整个输出缓冲区写入, 或者当缓冲区太大时进行写入. 在输入时, 它会读取块上所有可用的字符, 直到遇到一个换行符
  • BlockBuffering, 块缓冲, 以一个固定的块大小读取或者写入数据. 在批处理大量数据的时候性能最好

默认选用的缓冲区模式依赖于操作系统和 Haskell 的实现:

  • 可以调用 hGetBuffering 来查看系统当前的缓冲区模式
  • 可以用 hSetBuffering 来设置, 如 hSetBuffering stdin (BlockBuffering Nothing)

第十九章: 错误处理

Haskell 中有两类主流的错误处理:

  • “纯” 的错误处理
  • 异常

“纯” 的错误处理, 通常是指算法不依赖任何 IO Monad. 通常会用 Haskell 的类型系统来实现.

Haskell 支持异常, 能在任何地方抛出, 但是只会在 IO monad 中被捕获.

I/O 异常

由于 I/O 异常的类型很多, 因此有单独的 System.IO.Error 来处理.

System.IO.Error 模块中定义了两个函数: catch 和 try, 与 Control.Exception 中的类似, 但这里只处理 I/O 异常.

Haskell 中所有 I/O 错误有一个共同类型 IOError.

需要注意, System.IO.Error 与 Control.Exception 中定义有相同名字的函数, 因此导入时通常要用 qualified 导入以区分.

虽然 Prelude 默认导出 System.IO.Error 中的 catch 函数, 但我们应该尽量用 Control.Exception 中的, 因为后者可以处理任意类型的异常.

抛出异常

Control.Exception 中最通用的抛出异常的函数是 throw, 类型是 Exception -> a, 其可以抛出任何的 Exception, 而且能用在 “纯” 的上下文中.

还有一个类型为 Exception -> IO a 的函数 throwIO, 能在 IO monad 中抛出异常.

还有一个函数是 ioError, 其在 Control.Exception 和 System.IO.Error 中的定义都是相同的, 类型是 IOError -> IO a.

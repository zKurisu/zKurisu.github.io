第二章: 类型和函数

Haskell 中的每个函数和表达式都带有各自的类型. 一个值的类型标识了它和该类型的其他值所共有的一簇属性 (property).

类型不仅会为单纯的字节加上意义, 还会阻止我们混用不同的类型, 避免程序错误. 比如说, 类型系统通常不会允许将一个酒店预约数据当作汽车租赁数据来使用.

Haskell 的类型系统

Haskell 的类型系统有这些特点:

  • 强 (strong) 类型
  • 静态的 (static)
  • 可自动推导 (automatically inferred)

强类型

强类型语言是指:

  • 类型安全, 每个值都有明确的类型, 且操作需要符合类型的语义, Haskell 强类型系统会拒绝执行任何无意义的表达式, 保证程序不会因为这些表达式而引起错误. 比如将整数当函数使用
  • 禁止危险的隐式转换, 值的类型在运行时是严格保持的, 不允许发生未预期的, 隐式的类型转换. 比如将一个整型值传给一个接受浮点数的函数, C 编译器会自动且静默地将参数从整数类型转换为浮点类型, 而 Haskell 编译器会引发编译错误

遵守类型规则的表达式被称为是 “类型正确的 (well typed)”, 而不遵守类型规则, 会引起类型错误的表达式被称为是 “类型不正确的 (ill typed)”.

要在 Haskell 中进行类型转换, 必须显示地使用类型转换函数.

静态类型

静态类型系统指, 编译器可以在编译期 (而不是执行期) 知道每个值和表达式的类型. Haskell 编译器或解释器会发现类型不正确的表达式, 并拒绝这些表达式的执行:

类型推导

Haskell 编译器可以自动推断出程序中几乎所有表达式的类型 (有时要提供一些信息来帮助编译器理解程序代码). 这个过程被成为类型推导 (type inference).

复合数据类型: 列表和元组

纯度

副作用是指: 函数的行为受系统的全局状态所影响.

副作用本质上是函数的一种不可见 (invisible) 输入或输出. Haskell 的函数在默认情况下都是无副作用的: 函数的结果只取决于显式传入的参数.

我们将带副作用的函数称为 “不纯 (impure) 函数”, 而将不带副作用的函数称为 “纯 (pure) 函数”.

惰性求值

弄清楚 isOdd (1 + 2) 的结果是如何求值的.

在使用严格求值的语言里, 函数的参数总是在应用函数之前被求值. 比如上述例子中: 子表达式 (1 + 2) 会首先被求值, 得到结果 3. 接着将 3 绑定到变量 n, 应用到函数 isOdd. 然后 mod 3 2 返回 1, 而 1 == 1 返回 True.

Haskell 使用了另外一种求值方式 – 非严格求值. 在这种情况下, 求值 isOdd (1 + 2) 时并不会立即将子表达式 1 + 2 求值为 3, 此时编译器会做出一个 “承诺”, 说, “当真正有需要的时候, 我有办法计算出 isOdd (1 + 2) 的值”.

用于追踪未求值表达式的记录被称为 “块 (chunk)”. 编译器通过创建块来延迟表达式的求值, 知道这个表达式的值真正被需要为止. 如果某个表达式的值不被需要, 那么从始至终, 这个表达式都不会被求值. 简单来说, 只要程序还能继续运行且不需要知道这个值的具体内容, 它就不会被计算.

会触发求值的典型场景有:

  1. 模式匹配 (pattern matching):

为了选择分支, 必须知道 x 是 True 还是 False.

  1. 输出到外部世界 (I/O)

print 内部会对值调用 show, 而 show 必须知道完整结构.

  1. 数值比较或算术运算:

为了比较 x > 0, 必须知道 x 的数值, 所以 x 会被完全求值.

  1. 使用 seq 或 $! 强制求值

seq 是 Haskell 提供的显式求值的原语.

匿名 (lambda) 函数

Haskell 允许编写完全匿名的函数, 这样就不必为一些简单的辅助函数想名字. 因为匿名函数从 lambda 演算而来, 所以匿名函数通常也被称为 lambda 函数.

在 Haskell 中, 匿名函数以反斜杠符号 \ 为开始, 后跟函数的参数 (可以包含模式), 而函数体定义在 -> 符号之后. 其中, \ 符号读作 lambda.

匿名函数在各个方面的行为都和带名称的函数基本一致, 但是, 匿名函数的定义受到几个严格的限制, 其中最重要的一点是: 普通函数可以通过多条语句来定义, 而 lambda 函数的定义只能有一条语句.

部分函数应用和柯里化

函数签名里的 -> 只有一种作用: 它表示一个函数接受一个参数, 并返回一个值. 其中 -> 符号的左边是参数的类型, 右边是参数返回值的类型.

理解 -> 的含义比较重要: 在 Haskell 中, 所有函数都只接受一个参数. 尽管 dropWhile 看上去像是接受两个参数的函数, 但实际上它是一个接受一个参数的函数, 而这个函数的返回值是另一个函数, 这个被返回的函数也只接受一个参数.

编写可读代码的提示

Haskell 有两个很好的特性: 尾递归和匿名函数. 但这两个特性通常并不被使用.

对列表的处理操作一般可以通过组合库函数如 map, take 和 filter 来进行.

创建包

Cabal 是 Haskell 社区用来构建, 安装和发布软件的一套标准工具. Cabal 将软件组织成包 (package). 一个包有且只能有一个库, 但可以有多个可执行程序.

为包添加描述

Cabal 要求为每个包添加描述信息, 放在 .cabal 文件中. 这个文件需要放在项目的顶层目录里.

每个 Cabal 包都需要有个名字. 通常来说, 包的名字和 .cabal 文件的名字相同. 比如包叫做 mypretty, 那么描述文件就是 mypretty.cabal. 通常, 包含 .cabal 文件的目录名字也和包名相同, 如 mypretty.

GHC 的包管理器

GHC 内置了一个简单的包管理器来记录安装了哪些包以及它们的版本号, 可以用 ghc-pkg 命令来查看包数据库.

GHC 会区分所有用户都能使用的系统包 (system-wide packages, 全局安装的, 主机上所有用户都能用) 和只有当前用户才能使用的用户包 (per-user packages, 用户本地安装的). 用户数据库管理我们没有使用管理员权限安装的包.

常用的 ghc-pkg 子命令有:

  • ghc-pkg list, 列出已安装的包
  • ghc-pkg unregister, 告诉 GHC 我们不再用这个包 (还需要手动删除已安装的文件)

介绍

Functional Programming (FP, 函数式编程), 是一种编程范式 (programming paradigm, 范式这个词可以视作是 “规范/要求” 等意思), 它将所有计算视为函数的求值, 并避免改变状态和可变数据.

相关概念

Functional Programming 强调使用下面这些概念来构建程序.

Purity

在 Functional Programming 中, Purity 通常指 Pure Functions: 对于相同的输入, 总返回相同的输出, 并且在执行过程中没有任何副作用 (side effects).

特点是:

  • 确定性, 函数内部不依赖外部状态 (比如全局变量, 时间, 随机数等)
  • 无副作用, 函数内部不能修改全局变量或外部状态, 不能修改传入的参数, 不执行 I/O, 抛出异常等

  • 函数的返回值也不定, 比如发送同样的 request, 如果远端数据库改变了,其返回的内容也会改变
  • 这些改变都称作 side effects

Functional Programming 并不是完全 pure 的, 还是会产生一些 side effects, 但会尽量进行限制.

Function Composing

Function Composing 指: 将一个复杂任务拆解为一系列简单的子任务 (函数), 每个子任务只专注于完成一件小事, 然后通过组合这些子任务, 想流水线一样逐步处理数据.

在数学上, function composing 记为:

}
$$

Currying

Currying 是指: 将一个多参数函数转换为一系列单参数函数的过程.

在数学上, 对于函数 $f(x,y)$, 其 currying 形式记为 $f_c(x)(y)$, 满足:

$$
\displaylines
{
\begin{aligned}
f(x,y) = f_c(x)(y)
\end{aligned}
}
$$

Curried Function 就是指: 每次只接受一个参数的函数.

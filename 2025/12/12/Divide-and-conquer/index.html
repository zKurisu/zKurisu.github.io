Divide-and-conquer

介绍

Divide and conquer (分治法) 是一种通用的算法思想, 核心是: 将复杂问题分解为简单子问题, 解决子问题后合并结果.

步骤

Divide and conquer 具体可以分 3 个步骤:

  1. Divide (分解), 将原问题分解为若干个规模更小, 结构与原问题相似的子问题. 比如归并排序中, 将数组分解为两个长度减半的子数组; 二分查找中, 将查找区间分解为左半区间和右半区间
  2. Conquer (解决), 递归求解子问题. 若子问题规模足够小 (如长度为 1 或空), 则直接求解 (称为 “基本情况”); 否则继续分解子问题, 直到达到基本情况. 比如归并排序中, 当子数组长度为 1 时, 直接视为有序 (基本情况); 二分查找中, 当查找区间为空时, 返回 “未找到” (基本情况)
  3. Combine (合并), 将子问题的解合并为原问题的解. 比如归并排序而中, 合并两个有序子数组为一个有序数组; 二分查找中, 合并子问题的查找结果 (如确定目标在左半区还是右半区)

适用条件

通常情况下, 一个问题能满足下面条件, 则能用 divide and conquer 求解:

  1. 分解后的子问题相互独立. 一个子问题的解不影响其他子问题的解 (反例: 动态规划斐波那契数列问题, 子问题重叠)
  2. 子问题的结构与原问题一致 (仅规模更小), 因此子问题的解法可直接应用到原问题
  3. 子问题的解能高效合并为原问题的解 (反例: 排序的复杂度为 $O(n log n)$ 但是合并的时间复杂度为 $O(n^2)$, 则整体时间复杂度退化为了 $O(n^2)$)

示例

归并排序

归并排序是 divide and conquer 在排序问题上的一个经典算法.

问题描述: 将一个无序数组 [38, 27, 43, 3, 9, 82, 10] 排序为升序数组.

步骤:

  1. Divide: 用二分法, 每次将数组从中间位置分成左右两个子数组, 直到子数组长度为 1 (单个元素天然有序)
  • 初始数组是: [38, 27, 43, 3, 9, 82, 19]
  • 第一次分解: 从中间位置的 index 是 3, 拆分为左子数组 [38, 27, 43, 3] 和右子数组 [9, 82, 19]
  • 第二次分解: 将 [38, 27, 43, 3] 拆分为 [38, 27] 和 [43, 3]; 将 [9, 82, 19] 拆分为 [9, 82] 和 [19]
  • 第三次分解: 拆为 [38], [27], [43], [3], [9], [82], [19]; 此时所有子数组长度均为 1, 分解结束

  1. Conquer (解决): 递归求解子问题, 这里直接让长度为 1 的子数组参与合并

  2. Combine (合并): 将两个有序子数组合并为一个有序数组. 归并排序中的合并需要一个临时数组辅助:

  • 比较两个子数组的第一个元素, 将较小的元素放入临时数组
  • 重复上述步骤, 直到一个子数组为空
  • 将剩余子数组的元素全部追加到临时数组末尾
  • 将临时数组的结果复制回原数组对应位置

比如合并 [38] 和 [27]:

  • 临时数组初始为空: tmp = []
  • 比较 38 和 27, 将 27 放入临时数组: tmp = [27]
  • 右子数组为空, 左子数组剩下 38, 追加到临时数组中: tmp = [27, 38]
  • 将临时数组复制回原数组, 得到有序子数组: [27, 38]

合并 [27, 38] 和 [3, 43]:

  • 左子数组 L = [27, 38], 右子数组 R = [3, 43]
  • 初始化指针 i=0 (L 的索引), j=0 (R 的索引), 临时数组 temp = []
  • 比较 L[i] = 27 和 R[j] = 3, 取 3 放入 temp: temp = [3], j = 1
  • 比较 L[i] = 27 和 R[j] = 43, 取 27 放入 temp: temp = [3, 27], i = 1
  • 比较 L[i] = 38 和 R[j] = 43, 取 38 放入 temp: temp = [3, 27, 38], i = 2 (此时 L 遍历完毕)
  • 追加 R 的剩余元素 43 到 temp: temp = [3, 27, 38, 43]

实现示例

时间复杂度和空间复杂度计算

归并排序的时间复杂度由分解和合并两部分组成:

  • 分解, 每次二分, 一共分解为 $log_2 n$ 层
  • 合并, 每层合并的总操作次数为 $O(n)$ (每个元素只参与一次)

因此总时间复杂度为 O(n log n).

由于需要额外的临时数组来存储合并结果, 且大小与输入相同, 因此空间复杂度为 $O(n)$.

普通逆序对问题

普通逆序对问题能用归并排序的思路来求解.

普通逆序对 (Normal Inversion Pair) 是指在数组 (或序列) 中, 存在一对元素 (i, j), 满足:

  • i < j (下标 i 在 j 之前)
  • a[i] > a[j] (元素 a[i] 的值大于 a[j] 的值)

简单来说就是数组中 “前面元素大于后面元素” 的数对.

比如数组 [3, 1, 4, 2] 中的普通逆序对有:

  • (0, 1): 3 > 1
  • (0, 3): 3 > 2
  • (2, 3): 4 > 2

总共有三个普通逆序对.

在归并排序的合并阶段, 当 left[i] > right[j] 时, 由于 left 和 right 都是有序的, left[i] 及其后面的所有元素都会大于 right[j], 因此逆序对数量增加 len(left) - i.

显著逆序对问题

显著逆序对 (Significant Inversion Pair) 是普通逆序对的强化版本, 指: 在数组 (或序列) 中, 存在一对元素 (i, j), 满足:

  • i < j (下标 i 在 j 之前)
  • a[i] > 2 * a[j] (元素 a[i] 的值大于 a[j] 值的两倍)

示例, 数组 [4, 3, 2, 1] 中的显著逆序对:

  • (0, 1): 4 > 2 x 3 ?, 否
  • (0, 2): 4 > 2 x 2 ?, 是
  • (0, 3): 4 > 2 x 1 ?, 是
  • (1, 2): 3 > 2 x 2 ?, 否
  • (1, 3): 3 > 2 x 1 ?, 是
  • (2, 3): 2 > 2 x 1 ?, 否

总共有 3 个逆序对.

n 位数相乘

两个 n 位数相乘, 可以将整数拆为 “高位和低位” 两部分, 然后相乘.

先说明 “数的位值表示 (place-value representation)”: 任何一个十进制 (或者任意进制) 的整数, 都可以按 “每一位对应的权值” 展开. 比如一个 n 位十进制数 a, 可以写成:

$$
\displaylines
{
\begin{aligned}
a = a_{n-1} \cdot 10^{n-1} + a_{n-2} \cdot 10^{n-2} + \dots + a_1 \cdot 10^1 + a_0 \cdot 10^0
\end{aligned}
}
$$

  • $a_{n-1}, a_{n-2},…, a_0$ 是每一位上的数字, $10^k$ 是第 k 位的 “权值”.

如果把前 $\frac{n}{2}$ 位当作 “高位”, 后 $\frac{n}{2}$ 位当作 “低位”:

$$
\displaylines
{
\begin{aligned}
a = \underbrace{\bigl(a_{n-1} \cdot 10^{n-1} + \dots + a_{\lfloor n/2 \rfloor} \cdot 10^{\lfloor n/2 \rfloor}\bigr)}_{a_1 \geq 10^{\lfloor n/2 \rfloor}} + \underbrace{\bigl(a_{\lfloor n/2 \rfloor - 1} \cdot 10^{\lfloor n/2 \rfloor - 1} + \dots + a_0 \cdot 10^0\bigr)}_{a_0 < 10^{\lfloor n/2 \rfloor}}
\end{aligned}
}
$$

  • 通常将高位部分提取公因式 $10^{n/2}$, 剩下的写作 $a_1$
  • 将低位直接写为 $a_0$

此时有:
$$
\displaylines
{
\begin{aligned}
a = a_1 \cdot 10^{n/2} + a_0
\end{aligned}
}
$$

比如 n = 4, a = 1234, 那么 $n/2 = 4/2 = 2$:

  • 高位 $a_1 = 12$
  • 低位 $a_0 = 34$

记为:
$$
\displaylines
{
\begin{aligned}
1234 = 12 \cdot 10^2 + 34
\end{aligned}
}
$$

此时整数乘法可以写为:

$$
\displaylines
{
\begin{aligned}
a \times b = (a_1 \cdot 10^{n/2} + a_0) \times (b_1 \cdot 10^{n/2} + b_0)
\end{aligned}
}
$$

展开后变为:

$$
\displaylines
{
\begin{aligned}
a \times b &= a_1 b_1 \cdot 10^{2\lfloor n/2 \rfloor} + (a_1 b_0 + a_0 b_1) \cdot 10^{\lfloor n/2 \rfloor} + a_0 b_0 \newline~ \newline
&= a_1 b_1 \cdot 10^n + (a_1 b_0 + a_0 b_1) \cdot 10^{\lfloor n/2 \rfloor} + a_0 b_0
\end{aligned}
}
$$

此时原问题拆成了:

  • 四个小一点的整数乘法: $a_1 b_1$, $a_1 b_0$, $a_0 b_1$, $a_0 b_0$
  • 几次加减法和移位: 乘以 $10^k$ 相当于移位

之后继续拆解小整数乘法, 直到问题足够小.

上述四个乘法, 可以通过 “Karatsuba” 技巧, 优化为三个乘法:
$$
\displaylines
{
\begin{aligned}
a_1 b_0 + a_0 b_1 = (a_1 + a_0)(b_1 + b_0) - a_1 b_1 - a_0 b_0
\end{aligned}
}
$$

最终可以将时间复杂度降为: 递归做 3 次小乘法 $O(n^{log_2 3})$ 和 $O(n)$ 次加减/移位, 即 $O(n^{log_2 3}) \approx O(n^{1.585})$, 比直接计算的 $O(n^2)$ 快了很多.

实现示例

矩阵乘法

可以用 devide and conquer 思想来优化矩阵乘法, 核心是将大矩阵分解为更小的子矩阵, 递归计算子矩阵的乘积, 再合并结果.

对于两个 n x n 的矩阵 A 和 B, 要计算其乘积 $C = A \times B$.

C 中的元素可以计算为: $C[i][j] = \sum_{k=1}^n A[i][k] \cdot B[k][j]$.

分治的关键在于将 $A$, $B$ 分解为 4 个 $\frac{n}{2} \times \frac{n}{2}$ 的子矩阵 (假设 n 是 2 的幂, 非幂次时可补零), 即:

$$
\displaylines
{
\begin{aligned}
A = \begin{bmatrix}
A_{11} & A_{12} \newline~ \newline
A_{21} & A_{22}
\end{bmatrix}, B = \begin{bmatrix}
B_{11} & B_{12} \newline~ \newline
B_{21} & B_{22}
\end{bmatrix}
\end{aligned}
}
$$

  • 这里的 $A_{11}, A_{12}, A_{21}, A_{22}$ 和 $B_{11}, B_{12}, B_{21}, B_{22}$ 均为 $\frac{n}{2} \times \frac{n}{2}$ 的子矩阵

矩阵 $C$ 也可以分解为 4 个矩阵:

$$
\displaylines
{
\begin{aligned}
C = \begin{bmatrix}
C_{11} & C_{12} \newline~ \newline
C_{21} & C_{22}
\end{bmatrix}
\end{aligned}
}
$$

对于朴素的分治, 这里的每个子矩阵可以计算为:

$$
\displaylines
{
\begin{aligned}
C_{11} = A_{11} B_{11} + A_{12} B_{21} \newline~ \newline
C_{12} = A_{11} B_{12} + A_{12} B_{22} \newline~ \newline
C_{21} = A_{21} B_{11} + A_{22} B_{21} \newline~ \newline
C_{22} = A_{21} B_{12} + A_{22} B_{22} \newline~ \newline
\end{aligned}
}
$$

这种方法需要 8 次 $\frac{n}{2} \times \frac{n}{2}$ 的子矩阵乘法和 4 次子矩阵加法 (加法的时间复杂度是 $O(n^2)$, 毕竟有 $n^2$ 个元素). 设 $T(n)$ 为 $n \times n$ 的矩阵乘法的时间复杂度, 则递归式为:

$$
\displaylines
{
\begin{aligned}
T(n) = 8T(\frac{n}{2}) + O(n^2)
\end{aligned}
}
$$

根据主定理, 递归式的时间复杂度为 $O(n^3)$, 这与优化前一样. 需要考虑其他分解方法.

这里新的分解算法为 Strassen 算法, 能将原来矩阵乘法的时间复杂度从 $O(n^3)$ 降为 $O(n^{log_2 7}) \approx O(n^{2.807})$. 核心思想是: 通过构造 7 个中间矩阵, 将子矩阵乘法次数从 8 次减少到 7 次, 从而突破 $O(n^3)$ 的限制.

定义 7 个 $\frac{n}{2} \times \frac{n}{2}$ 的中间矩阵 $M_1 ~ M_7$:

$$
\displaylines
{
\begin{aligned}
M_1 &= (A_{11} + A_{22})(B_{11} + B_{22}), \newline~ \newline
M_2 &= (A_{21} + A_{22})B_{11}, \newline~ \newline
M_3 &= A_{11}(B_{12} - B_{22}), \newline~ \newline
M_4 &= A_{22}(B_{21} - B_{11}), \newline~ \newline
M_5 &= (A_{11} + A_{12})B_{22}, \newline~ \newline
M_6 &= (A_{21} - A_{11})(B_{11} + B_{12}), \newline~ \newline
M_7 &= (A_{12} - A_{22})(B_{21} + B_{22}).
\end{aligned}
}
$$

通过 $M_1 ~ M_7$ 组合出 C 的 4 个子矩阵:

$$
\displaylines
{
\begin{aligned}
C_{11} &= M_1 + M_4 - M_5 + M_7, \newline~ \newline
C_{12} &= M_3 + M_5, \newline~ \newline
C_{21} &= M_2 + M_4, \newline~ \newline
C_{22} &= M_1 - M_2 + M_3 + M_6
\end{aligned}
}
$$

Strassen 算法仅需 7 次 $\frac{n}{2} \times \frac{n}{2}$ 的子矩阵乘法和 18 次子矩阵加减法 (加减法的时间复杂度还是 $O(n^2)$), 递归式可以写为:

$$
\displaylines
{
\begin{aligned}
T(n) = 7 T(\frac{n}{2}) + O(n^2)
\end{aligned}
}
$$

根据主定理, 递归式的解为 $T(n) = O(n^{log_2 7}) \approx O(n^{2.8707})$.

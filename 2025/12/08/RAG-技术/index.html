RAG-技术

介绍

RAG (Retrival-Augmented Generation) 是一种结合了信息检索和文本生成的 AI 技术架构. 其核心思想是通过在生成回答之前先检索相关信息来增强大语言模型的能力.

RAG 的技术原理分 3 个部分:

  • Retrival (检索), 从知识库中找到相关信息. 将用户的问题转换成向量表示 (embedding), 在向量数据库中计算余弦相似度或其他距离度量, 检索出最相关的 Top-k 个文档片段
  • Augmentation (增强), 将检索到的信息与原始查询组合. 将检索到的文档片段整理成结构化上下文, 设计 prompt 模板, 明确告诉 LLM 如何使用这些信息
  • Generation (生成), 利用增强后的 prompt, 让 LLM 生成最终的答案. 通过答案与检索内容进行一致性检查, 给答案质量评分, 来控制生成质量

RAG 流程架构

RAG 系统通常分两大阶段:

  • 离线索引: 数据准备 (建立知识库)
  • 在线检索: 实时查询和生成答案

离线索引

离线索引 (Offline indexing) 需要在系统运行之前, 提前将知识库文档处理好并存储到向量数据库中.

离线 (offline) 的意思就是:

  • 不在用户查询的实时路径上
  • 可以花费较长时间处理, 通常是批处理人物

文档解析

文档解析指, 将不同格式的文档 (PDF, Word, 图片等) 解析成方便后续程序分析提取的格式.

常见的做法是:

  • 将任意来源的文档统一转换为 PDF 格式 (File-To-PDF)
  • 再从 PDF 中提取结构化文本并生成 Markdown 内容 (PDF-TO-Markdown)

数据清洗

数据清洗 (Data Cleaning), 指去除文档中无用的标签和冗余信息, 进行去重处理, 格式标准化和规范化, 清理无效空白与乱码, 过滤敏感信息等. 目的是提升文档的质量和一致性.

文本分块

文本分块 (Document Chunking/Splitting) 指, 将长文本或结构复杂的文档切分为多个语意连贯, 长度适中, 便于处理和检索的小片段, 以提升后续向量化, 索引构建与检索匹配的效率和准确性.

常见的分片方式有:

  • 基于固定长度的分片 (Fixed-size chunking), 比如每 500 个token 一块
  • 基于语义边界的分片 (Semantic-aware Chunking), 按段落, 章节分
  • 父子分片 (Parent-Child Chunking), 保留层级关系的分片结构

分片能让上下文不至于太大, 也能减少噪声信息, 让检索更准确, 但需要注意, 如果:

  • Chunk 太大, 容易噪声多
  • Chunk 太小, 容易丢失上下文

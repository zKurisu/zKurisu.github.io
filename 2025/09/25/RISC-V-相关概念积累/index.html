RISC-V ISA

RISC 是 “Reduced Instruction Set Computer” (精简指令集计算机) 的缩写. 是加州大学伯克利分校研究团队开源的一个指令集.

V 是罗马数字的 5, 表示这是第五代 RISC 架构.

ISA 是 “Instruction Set Architecture” (指令集架构) 的缩写. 是底层硬件向上层软件提供的一层接口规范:

其定义了, 如 CPU 支持哪些指令, 指令的编码格式, 宽度, 寄存器数量与用途, 内存访问方式, 异常/中断处理等.

指令宽度通常等价于通用寄存器的宽度, 其决定了寻址范围的大小以及数据运算的能力, 比如:

  • 8 位, 寻址范围 $2^8 = 256$
  • 16 位, 寻址范围 $2^{16} = 65536$
  • 32 位, 寻址范围 $2^{32} = 4294967296$
  • 64 位, 寻址范围 $2^{64} = 18446744073709551616$

x86 和 arm 都需要付费授权, 而 RISC-V 是开源免费的, 可以自由使用和修改, 常用于教学和科研. 现在生态也逐渐成熟, 各公司也在推出 RISC-V 芯片.

常见的 ISA 有:

  • x86
  • Power
  • ARM
  • MIPS
  • RISC-V

RISCV ISA 命名格式

格式为:

  • RV 是 RISC-V 的缩写
  • [###] 标识处理器的字宽
  • [abc...xyz] 标识指令集模块集合

比如: RV32IMA:

  • 32 位芯片
  • 支持 I, M, A 三个模块

模块化 ISA

RISC-V ISA 由 1 个基本整数指令集和多个可选的扩展指令集组成, 基础指令集是固定的, 不会改变.

对于 CPU 的实现而言, 基本整数指令集是需要强制实现的.

基本指令集其分为 4 种:

  • RV32I (Integer), 32 位整数指令集
  • RV32E (Embedding), RV32I 的子集, 用于小型的嵌入式场景
  • RV64I, 64 位整数指令集, 兼容 RV32I
  • RV128I, 128 位整数指令集, 兼容 RV64I 和 RV32I

扩展模块指令集有标准化的和非标准化的指令集扩展, 如:

  • M, 整数乘法 (Multiplication) 与除法指令集
  • A, 存储器原子 (Automic) 指令集
  • F, 单精度 (32bit) 浮点 (Float) 指令集
  • D, 双精度 (64bit) 浮点 (Double) 指令集, 兼容 F
  • C, 压缩 (Compressed) 指令集
  • 等等

IMAFD 被称为 General 组合, 用 G 标识.

General Purpose Registers

RISC-V 的 Unprivileged Specification 定义了 32 个通用寄存器以及一个 PC 寄存器, 这些定义对 RV32I, RV64I, RV128I 都一样. 如果实现支持 F/D 扩展则需要额外支持 32 个浮点寄存器.

对于 RV32E, 其将 32 个通用寄存器缩减为 16 个.

每个寄存器在编程时的特定用途和别名定义在 RISC-V Application Binary Interface (ABI) 中.

Privileged level

RISC-V 的 Privileged specification 定义了三个特权级别 (privilege level).

  • User mode
  • Supervisor mode
  • Machine mode

内存管理与保护

分两种:

  • Physical Memory Protection
  • Virtual memory, 需要支持 Supervisor mode, 用于实现高级的操作系统特性

RISC-V 汇编

RISC-V 汇编由多条语句 (statement) 构成, 一条语句有 3 部分组成 (以 GNU 汇编为例):

  • label, 标号, 在 GNU 汇编中, 任何以 : 结尾的标识符都被认为是一个标号
  • operation 分几类:
    • instruction, 对应机器指令的字符串
    • pseudo-instruction (伪指令), 为了提高编写代码的效率, 可以用一条伪指令指示汇编器产生多条实际的 instructions
    • directive (指示/伪操作), 通过类似指令的形式 (但以 . 开头) 通知汇编其如何控制代码的产生, 不对应具体的指令
    • macro, 用 .macro/.endm 自定义的宏
  • comment: 常用 # 开始到当前行结束 (也有以其他开头的)

操作对象

RISC-V 汇编指令的操作对象包括:

  • 寄存器: 比如 32 个通用寄存器 (限于 RV32I 来讲) x0 ~ x31. 在 RISC-V 中, Hart 在执行算术逻辑运算时所操作的数据必须直接来自寄存器


x0 比较特殊, 只能读, 且值为 0.

  • 内存: Hart 可以执行在寄存器和内存之间的数据读写操作, 对于 RV32, 最多可以访问 $2^{32}$ 个字节的内存空间. 注意寻址的基本单位是 1 byte, 即一个地址对应的空间大小是 1 byte

编码格式

RISC-V 汇编 (RV32I) 有 6 种编码格式:

  • funct3/funct7 (“function”, funct3 表示占了 3 个 bits, funct7 表示占了 7 个 bits) 和 opcode (“operate code”) 决定 instruction 类型
  • rs 是 “register source”, 提供数据; rd 是 “register destination” 存储计算结果
  • R-type, “Register”, 每条指令中有三个 fields, 用于指定 3 个寄存器参数
  • I-type, “Immediate”, 每条指令除了带有两个寄存器参数外, 还带有一个立即数参数
  • S-type, “Store” (与内存相关), 每条指令除了带有两个寄存器参数外, 还带有一个立即数参数 (宽度为 12 bits, 但 fields 的组织方式不同于 I-type)
  • B-type, “Branch” (与分支相关), 每条指令除了带有两个寄存器参数外, 还带有一个立即数参数 (宽度为 12 bits, 但取值为 2 的倍数)
  • U-type, “Upper”, 每条指令含有一个寄存器参数再加上一个立即数参数 (宽度为 20 bits, 用于表示一个立即数的高 20 位)
  • J-type, “Jump”, 每条指令含有一个寄存器参数再加上一个立即数参数 (宽度为 20 bits)

opcode 值解析:

  • inst[1:0] 表示 opcode 的前两位 (从右往左看), 都是 11
  • inst[4:2] 表示 opcode 的第 3, 4, 5 位, 横轴
  • inst[6:5] 表示 opcode 的第 6 和 7 位, 纵轴

比如 ADD 指令的就是 0110011. 在上面 table 里属于 OP 这个类型 (“OPeration”).

注意, RISC-V 指令在内存中按照 “小端序” 排列, 即低位放在低地址.

UNIX-环境高级编程-Notes

线程和线程 ID

一个进程内的所有线程共享同一地址空间, 文件描述符, 栈以及与进程相关的属性. 因为它们能访问同一存储区, 所以各线程在访问共享数据时需要采取同步措施以避免不一致性.

与进程相同, 线程也用 ID 标识. 但其只在所属的进程中起作用.

线程是在进程模型建立很久之后引入的, 使用另一套控制函数.

errno 的相关常量定义在 <errno.h> 头文件中. 对于 errno 需要注意两个规则:

  • 如果没有出错, 则 errno 的值不会被例程清除, 因此需要在函数返回后就检验是否出错, 不然 errno 中可能存储的是以前的错误信息
  • 任何函数都不会把 errno 置为 0, 且 <errno.h> 中定义的所有常量都不为 0

用户标识

每个用户都有一个唯一的数值 ID 标识 (uid), 内核通过该 ID 来检验用户是否有执行某些操作的权限.

用户 ID 为 0 的为 root 用户或 superuser, 如果一个进程有 root 权限, 则大多数文件权限检查都不再进行.

每个用户也会包括 组 ID (group id, gid), 允许各个成员之间共享资源 (如文件).

信号

信号, Signal, 用于通知进程发生了某种情况. 当向一个进程发送信号时, 我们必须是那个进程的所有者或者是超级用户.

终端上产生信号的方法一般有:

  • Ctrl+C, Ctrl+\, delete 用于中断当前运行的进程
  • kill 发送指定信号

常见的信号有:

  • SIGINT, 值为 2, 终止进程
  • SIGQUIT, 值为 3, 终止进程 + core dump
  • SIGKILL, 值为 9, 立即终止
  • SIGTERM, 值为 15, 请求终止

可以用 signal 函数捕获指定信号, 并调用一个函数.

  • signum 是要捕获的信号
  • handler 是信号处理函数, 可以是:
    • 自定义函数, 信号发射时调用, 接收信号编号作为参数
    • SIG_IGN, 忽略这个信号
    • SIG_DFL, 使用系统默认行为

返回值:

  • 成功时, 返回信号的处理方式, 可能是一个函数指针, 或 SIG_IGN, SIG_DFL
  • 失败时返回 SIG_ERR, 并设置 errno

时间值

UNIX 使用过两种不同的时间值:

  • 日历时间, 从 1970 年 1 月 1 日 00:00:00 到当前积累的秒数, 一般用于记录文件最近一次修改时间, 通常用 time_t 类型保存
  • 进程时间, 也称 CPU 时间, 用于度量进程使用的 CPU 资源, 以时钟滴答计算, 通常用 clock_t 来保存

UNIX 会为一个进程维护 3 个进程时间值:

  • 时钟时间, 指进程运行的时间总量
  • 用户 CPU 时间, 指执行用户指令所用的时间量
  • 系统 CPU 时间, 指为了执行该进程内核程序所经历的时间

用户 CPU 时间和系统 CPU 时间之和称 “CPU 时间”.

限制

限制分两种:

  • 编译时限制 (比如, short 的最大值是什么)
  • 运行时限制 (比如, 文件名最多有多少字符)

编译时限制可以在头文件中定义, 程序在编译时包含这些头文件.

运行时限制需要进程调用一个函数来获得限制值.

为了解决这些问题, 提供了 3 种限制查询方法:

  1. 编译时限制 (头文件)
  2. 与文件或目录无关的运行时限制 (sysconf 函数)
  3. 与文件或目录有关的运行时限制 (pathconf 和 fpathconf 函数)

运行时查看环境限制

可以用 sysconf, pathconf, fpathconf 三个函数在运行时查询系统运行环境中的各种配置限制或参数, 比如:

  • 文件系统支持的最大路径长度
  • 一个进程最多可以打开多少个文件
  • 当前系统允许的线程数, 进程数, 文件大小限制
  • 等等

这些限制都不是编译时常量, 而是运行时由操作系统决定的配置值.

sysconf

sysconf 用于查询系统级别的配置限制 (与整个操作系统相关), 比如一个进程最多能打开多少文件, 其函数原型为:

  • name 是一个预定义的宏, 即要查询的配置参数
  • 成功时返回对应整数, 出错时返回 -1

常见的 name 参数定义在 <unistd.h> 中, 比如:

  • _SC_ARG_MAX, 命令行参数的最大总长度 (包括环境变量)
  • _SC_CHILD_MAX, 每个用户 ID 最多可拥有的子进程数
  • _SC_OPEN_MAX, 一个进程最多可打开的文件描述符数目
  • _SC_PAGESIZE, 系统内存页大小
  • 等等

可以在 man 3 sysconf 中查看.

pathconf

pathconf 查询与路径相关 (某个文件或目录路径) 的配置限制, 原型为:

  • pathname 文件或目录的路径
  • name 是要查询的限制类型, 也是预定义宏
  • 成功时返回对应数字, 错误时返回 -1

常见 name 参数有:

  • _PC_PATH_MAX, 该路径所在文件系统允许的最大路径长度
  • _PC_NAME_MAX, 该目录下文件名的最大长度
  • _PC_LINK_MAX, 一个文件最多有多少个硬链接

POSIX 标准

POSIX, Portable Operating System Interface, 该标准定义了一系列便于程序在操作系统间移植的接口, 以 UNIX 操作系统为基础但不限于此.

若一个操作系统符合 POSIX 定义, 则必须满足这些头文件内容:

POSIX.1 标准现在由 Austin Group 开放工作组 (http://www.opengroup.org/austin) 维护.

Single UNIX Specification

Single UNIX Specification 是 POSIX.1 标准的一个超集, 定义了一些附加接口.

Open Group 拥有 UNIX 商标, 他们使用 Single UNIX Specification 定义了一系列接口, 一个系统想要称 UNIX 系统, 其实现必须支持这些接口并通过验证测试.

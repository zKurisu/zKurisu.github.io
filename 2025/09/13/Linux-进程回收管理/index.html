Linux-进程回收管理

介绍

这里说明下 <sys/wait.h> 中的 wait 和 waitpid 函数, 一般用于 fork 后, 父进程获取子进程退出的信息.

在一个子进程终止时, 其进程不会立即被完全清理, 而是会保留一部分信息, 直到父进程通过 wait 或类似的函数来获取. 如果父进程没有获取, 子进程终止后就变成了一个 “僵尸进程”:

  • 其不再运行
  • 进程 ID 仍然存在
  • 占用系统进程表项, 可能导致资源泄漏

<sys/wait.h> 标准库

<sys/wait.h> 专门用于进程状态管理, 进程等待, 进程控制等.

其定义了 wait, waitpid 函数, 以及子进程退出状态相关的宏, 比如:

  • WIFEXITED(status), 子进程是否正常退出
  • WEXITSTATUS(status), 获取子进程的 exit code
  • WIFSIGNALED(status), 是否被 signal 杀死

wait 函数

该函数会:

  • 阻塞当前进程, 直到任意一个子进程结束
  • 当有子进程退出时, wait 会消耗其保留信息, 防止其变成僵尸进程
  • 将退出状态信息写入 status 参数中, 待相关宏来解析

返回值:

  • 成功时, 返回结束的子进程 PID
  • 失败时, 返回 -1, 并设置 errno

waitpid

waitpid 相比 wait 更灵活, 可以:

  • 等待某个特定的子进程
  • 可以选择是否阻塞
  • 可以等待一组子进程中的任意一个或所有

这几个参数:

  • pid 的几个取值:
    • pid > 0, 指定特定 PID
    • pid == -1, 等待任意子进程 (同 wait)
    • pid == 0, 等待与父进程同进程组的任意子进程
    • pid < -1, 等待 ID 与 pid 绝对值相等的进程组中的任意子进程
  • status, 用于接收子进程退出状态的整型指针
  • options, 常用的为:
    • 0 表示阻塞
    • WNOHANG 表示非阻塞

Heap-数据结构

medium blog example

介绍

Heap (堆) 是一种 Complete Binary Tree (完全二叉树), 有两种类型:

  • Max Heap, 每个节点的值都大于或等于其子节点的值, 此时根节点 (堆顶) 是整个堆中的最大值
  • Min Heap, 每个节点的值都小于或等于其子节点的值, 此时根节点 (堆顶) 是整个堆中的最小值

Heap 可以用于高效获取最大值或最小值.

相关概念

Complete Binary Tree

Complete Binary Tree 指 Binary tree 中除了最后一层, 其他层都是满的, 而且最后一层的节点都尽量靠左排列 (从左到右填充, 不能有空缺).

Complete Binary Tree 由于排列紧凑, 可以用数组存储, 存储顺序是从上到下, 从左到右:

因此可以无需节点指针, 来进行索引:

  • Left child: 2 * i + 1
  • Right child: 2 * i + 2
  • Parent: (i - 1) / 2

比如这里:

  • node 20, index = 0:
    • left child: index = 2 * 0 + 1 = 1, 正是 node 19
    • right child: index = 2 * 0 + 2 = 2, 正是 node 19
  • node 19, index = 1:
    • left child: index = 2 * 1 + 1 = 3, 正式 node 17
    • right child: index = 2 * 1 + 2 = 4, 正式 node 16

完全二叉树的节点数 n 可以通过高度 h 推断:

  • 最少节点数的情况: h + 1, 每个节点都只有左子树, 每层 1 个节点

  • 最多节点数的情况: $2^{h+1} - 1$, 满二叉树, 所有层都填满

  • 一般情况是: $2^h \le n \le 2^{h+1} - 1$, 也就是最后一层不满

操作系统中的 heap memory

注意操作系统中的 heap 与这里的数据结构完全是不同的概念, 只是名字相同.

操作系统中的 heap 指系统动态内存分配的区域, 用于存储程序运行时动态分配的数据, 只是内存块, 不保证任何顺序. 也不用 heap 数据结构来管理.

Heap 数据结构是一种特殊的完全二叉树, 用于高效获取最大/最小值, 用数组存储且有序.

关键性质

  • Min Heap: arr[parrent] <= arr[child]
  • Max Heap: arr[parrent] >= arr[child]
    (注意与完全二叉树的区别就在此)

Heap 一般用 array 存储, 对于 index = i 的 node 有:

  • Left child: 2 * i + 1
  • Right child: 2 * i + 2
  • Parent: (i - 1) / 2

核心操作

以 min heap 为例, 常见操作有:

  • insert(x)/push(x), 插入一个新元素, 时间复杂度是 O(log n)
  • extract_min()/pop(), 移除并返回最小元素, 时间复杂度是 O(log n)
  • peak()/find_min(), 查看最小元素 (堆顶) 但不移除, 时间复杂度是 O(1)
  • heapify(), 将一个无序数组构建成堆, 时间复杂度是 O(n)

插入元素

向 Heap 插入元素时, 需要维持 heap 的性质.

假设有一个 min heap, 即每个节点的值 $\le$ 其子节点的值, 根节点是最小值.

插入数据的步骤:

  1. 将新元素放在 Heap 的末尾 (即完全二叉树的最后一个位置, 对应数组的最后一个索引)
  2. 向上调整 (Heapify up/Sift up):
    a. 比较新元素与其父节点的值
    b. 如果新元素的值比父节点小, 则交换它们
    c. 重复这个过程, 直到新元素 $\ge$ 父节点或达到根节点

如果是 max heap 思路也一样, 只不过是新元素 $\ge$ 父节点才交换.

弹出堆顶

在 heap 中, 堆顶是优先级最高的元素 (在 Min Heap 中是最小值, 在 Max Heap 中是最大值).

比如 min heap 中索引为 0 的元素:

弹出堆顶的一般逻辑是:

  1. 取出堆顶元素 (通常是 heap[0])
  2. 把堆最后一个元素移动到堆顶 (为了保留完全二叉树的性质)
  3. 通过 heapify down (堆化) 或 sink (下沉) 操作来调整新的堆, 以 min heap 为例子:
    a. 比较当前节点 (i) 与左孩子 (2i + 1) 和右孩子 (2i + 2) 的值
    b. 如果当前节点 > 某个子节点, 则说明其不是最小的, 需要和最小的孩子进行交换
    c. 重复这个过程直到当前节点 $\le$ 所有子节点或者到达叶子节点

Eq 和 PartialEq

在 Rust 中, 并不是所有类型都支持任意两个值都可判定是否相等.

实现 Eq trait 表示, 对于类型中的任意两个值, 都总是可以判定它们是否相等, 且满足:

  • Reflexive (自反性), $a == a$ 为 true
  • Symmetric (对称性), 如果 $a == b$, 则 $b == a$
  • Transitive (传递性), 如果 $a == b$ 且 $b == c$, 则 $a == c$

Eq 是一个标记 trait, 没有任何方法, 继承自 PartialEq, 任何实现了 Eq 的类型, 需要保证对于任意两个值 a 和 b, a == b 的结果是可靠且符合逻辑的.

Ord 和 PartialOrd

Ordering (排序), 指一组元素之间的大小关系或顺序关系.

Total Ordering (完全排序), 指对于集合中的任意两个元素 $a$ 和 $b$, 一定能确定它们之间的关系, 并且只有以下三种情况之一成立:

  1. $a < b$, a 小于 b
  2. $a == b$, a 等于 b
  3. $a > b$, a 大于 b

非空指针

std::ptr::NonNull 智能指针, 包裹一个 *mut T 的可变裸指针, 但能保证指针非空.

NonNull<T> 的常用方法有:

  • .as_ptr(), 获取内部的 *mut T 指针
  • .as_ref(), 获取 &T
  • .as_mut(), 获取 &mut T
  • NonNull::new_unchecked(ptr: *mut T) -> NonNull<T>, 直接构造 NonNull, 需要保证 ptr 非 null

这些方法都需要在 unsafe 块中运行.

FFI 接口

FFI, Foreign Function Interface, 外部函数接口, 允许一种编程语言 (如 Rust) 调用另一种语言 (如 C) 编写的函数.

Rust FFI 可以用来:

  • 调用 C/C++ 等语言的库函数
  • 暴露 Rust 函数供其他语言调用

FFI 的核心是利用 C 语言的 ABI (Application Binary Interface), 因为其最通用和稳定, 几乎所有语言都支持.

裸指针构造和处理

裸指针, Raw pointer, 是一种不提供任何运行时安全检查的指针类型, 类似 C 中的普通指针.

Rust 中有两种裸指针:

  • *const T, 指向类型 T 的不可变裸指针
  • *mut T, 指向类型 T 的可变裸指针

之所以称为 “裸”, 就是因为其能绕过 Rust 的安全规则, 比如借用检查, 生命周期, 可变性规则等. 需要和 unsafe 关键字一起只用.

获取 Iterator 的方法

Rust 标准库实现的 collections (如 Vec<T>, &[T], HashMap 等) 都能转换为 iterator, 用于遍历.

常见的获取迭代器的方法有 3 种:

  • iter(), 调用对象是 &T, 返回的迭代器类型 Iter<'_, T>, 此时元素类型都是 &T (引用), 无法修改
  • iter_mut(), 调用对象是 &mut T, 返回的迭代器类型 IterMut<'_, T>, 此时元素类型都是 &mut T (引用), 可以修改元素
  • into_iter(), 调用对象是 T, 返回的迭代器类型 IntoIter<T>, 此时元素类型都是 T, 会获取元素所有权

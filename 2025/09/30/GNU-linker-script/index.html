语法

Linker scripts 用哦 linker command language 来编写, linker 会把多个 input files (object file) 合并为一个 output file (executable file, 也可以称 object file).

Object file 中的 section 有名称和大小, 有的还会有关联的数据, 称 section contents. 如果一个 section 被标记为 loadable, 表明其 contents 会在运行时加载进内存. 如果一个 section 没有 contents (可能只是占位符, 暂时不包含内容, 但要求分配内存), 通常被标记为 allocatable, 此时不需要加载特定内容到这块内存区域, 但某些情况下需要将这块内存初始化为零.

如果一个 section 既不是 loadable 也不是 allocatable, 通常包含 debugging information.

任何一个 loadable 或 allocatable section 都包含两个地址:

  • VMA, Virtual Memory Address, 程序运行时该 section 所在的地址 (程序看到的地址)
  • LMA, Local Memory Address, 程序被加载到内存中该 section 的地址

通常这两个地址相同, 加载到哪里就从哪里开始运行.

可以用 objdump -h xxx 来查看这两个地址.

Format

Linker scripts 由一系列 commands 组成, 每个 command 可以是一个 keyword 加上 arguments 或者是一个对 symbol 的赋值语句. 多个 commands 之间用 ; 分隔, 忽略 whitespace.

注释采用 C 风格, 用 /* 和 */ 包裹.

最简单的 linker script 只包含 SECTIONS command, 用来描述 output file 的内存布局.

假设程序只包含 .text, .data 和 .bss 段, 假设代码加载到 0x10000 地址, 数据加载到 0x8000000 地址:

  • SECTIONS { ... } 标记对 sections 布局的起始和结束位置
  • . 是 location counter, 表示接下来的内容应该存放在哪里, 在 SECTIONS 的开头部分, 其值为 0
  • . = 0x10000; 是一个 command, 表明接下来的内容存放的位置, 用 ; 与后续内容分隔
  • .text : { *(.text) } 定义一个 output section, 名字叫 .text, 这个 section 的内容来自于所有输入 object files 中的 .text section, 即 *(.text):
    • .text 是 output section 名字
    • : 分隔 section name 和定义的内容
    • { ... } 定义该 section 包含哪些内容
    • *(.text) 是一个通配符, 表示从 * 中提取 .text section 的内容
  • .bss section 紧跟在 .data section 之后

PROVIDE

PROVIDE(symbol = expression), 表示, 当 symbol 没有被定义时, 才会进行定义.

  • 如果程序里定义了 _etext, 这里又有 _text = .; 则会报 multiple definition diagnostic 的错
  • 如果程序里定义了 etext, 那么 PROVIDE(etext = .); 则不会进行定义

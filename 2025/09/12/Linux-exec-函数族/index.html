Linux-exec-函数族

介绍

Unix/Linux 系统编程中 execve() 封装出了多个变体: execl, execlp, execle, execv, execvp, execvpe, 这些函数称 exec 函数族.

这些函数的本质都是用一个新的程序替换当前正在运行的进程, 原进程的代码, 数据, 堆栈都会被新进程取代. 定义这些变体是为了便利不同参数传递方式和环境变量, 路径搜索等.

这些函数名是多个单词的组合缩写:

  • exec, “execute”, 表示执行一个新程序
  • l, “list”, 表示参数以列表形式逐个传入, 用 char *arg0, arg1...
  • v, “vector” 表示参数以数组 argv[] 传入, 用 char *argv[] 数组
  • p, “PATH”, 表示会自动在系统的 PATH 环境变量中查找可执行文件
  • e, “environment”, 允许自定义传递环境变量, 可以传入 char *envp[] 环境变量数组

主要分两大类:

  • 含 l, 参数以分开的列表传递给函数, 如 execl, execlp, execle
  • 含 v, 函数以数组传递给函数, 如 execv, execvp, execve, execvpe

l 类 exec 函数原型

  • 需要注意最后一个参数必须是 NULL, 表示参数列表结束

这类函数的第一个参数和第二个参数是一样的, 比如:

  • 第一个 ls 是要执行的程序名
  • 第二个 ls 是传给程序的 argv[0] , 通常也是程序名, 当然也可以设为其他的, 但这是约定俗称的, 比如 C 的 main 程序的 argv[0] 是默认传入当前文件名

v 类 exec 函数原型

  • 注意数组的最后一个元素需要是 NULL

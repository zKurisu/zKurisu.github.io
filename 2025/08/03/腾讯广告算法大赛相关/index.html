比赛官网

赛题介绍

需要从从海量多模态用户行为中挖掘潜在兴趣, 进行生成式推荐.

赛题为: All-Modality Generative Recommendation, 全模态生成式推荐.

需要基于脱敏处理后用户的协同, 文本, 视觉等全模态历史行为数据, 预测其下一次可能交互的广告. 每个行为包含广告的 ID 类特征, 以及多模态信息. 初赛数据集为 1M, 复赛为 10M, 数据为 JSON 格式.

参考文献:

  1. A Review of Modern Recommender Systems Using Generative Models (Gen-RecSys). SIGKDD 2024.
  2. Recommender Systems with Generative Retrieval. NIPS 2023.
  3. Actions Speak Louder than Words: Trillion-Parameter Sequential Transducers for Generative Recommendations. ICML 2024.

赛方会提供基线模型 (生成式的模型), 提供全流程演示.

提供基于公开大模型的多模态 embedding. 有多套.

评分标准

评价指标: HitRate@K 等指标, 衡量模型从候选库中检索用户真正点击的公告的能力, K 的取值在初赛塞题数据提供时提供. 主办方会提供待检索的广告库, 参赛者需要从该广告库中检索出排名前 K 的广告.

提交和排名: 参赛者提供训练和推理代码, 然后系统会读取测试数据集, 运行推理代码, 得到排名前 K 的广告, 最后计算得到 HitRate@K 等. 每天限提交三次 (一个队伍), 当日 12 点前为当日最佳, 会在 15 点更新排名. 缺席决赛的队伍成绩取消.

可以通过环境变量修改训练中用到的一些路径信息:

  • USER_CACHE_PATH, 用户的缓存路径, 配额 20GB, 可以用来在训练, 评测中共享一些文件
  • TRAIN_DATA_PATH, 训练数据集路径

如果需要在训练的产出页面展示 loss 等可视化指标, Tensorboard Events 文件必须产出到 TRAIN_TF_EVENTS_PATH 路径下.

模型推理与评测

在模型管理页面, 可以指定发布的模型进行评测:

提交后, 评测状态会显示 “待评测”, “评测失败” 或 “评测成功” 等, 评测成功后会在页面展示结果.

注意评测脚本需要统一命名为 infer.py (评测的入口文件), 否则无法执行评测任务.

产出规范, infer.py 需要返回 top10s 和 user_list:

  • top10s, 是 list[list], 推理出由 10 个 item_id 组成的 list
  • user_list, 是 list[str], user_id 组成的 list, 其中 user_id 为 str 格式
  • 注意 top10s 的第一个 list 应为 user_list 中第一个用户的序列推理出的结果, 以此类推.

环境变量设置评测过程中所需的目录路径:

  • USER_CACHE_PATH
  • MODEL_OUTPUT_PATH, 模型路径
  • EVAL_DATA_PATH, 数据集, 推理候选集, 检索索引路径
  • EVAL_INFER_PATH, 推理脚本文件目录
  • EVAL_RESULT_PATH, 产出内容路径, 存放中间结果和 result.json

多头注意力机制维度计算

Multi-head attention 的设计理念是, 将高维特征空间均匀分割为多个子空间 (head), 让每个 head 可以独立关注输入的不同方面, 从而保证并行运算.

线性层计算 QKV

nn.Linear 在这里的作用是将输入投影到适合计算注意力的空间, 让模型能够学习到有意义的 Query, Key, Value 表示

从 QKV 的定义来看, 其实就是线性变换:

$$
\displaylines
{
\begin{aligned}
Q = W_Q X \newline~ \newline
K = W_K X \newline~ \newline
V = W_V X
\end{aligned}
}
$$

dropout_rate 参数的作用

dropout_rate 是深度学习中常用的正则化技术, 在这里用于将部分注意力权重置零, 防止过拟合.

什么是 embedding

在机器学习和自然语言处理中, embedding 是一种将离散数据 (如单词, 类别, 用户 ID 等) 转换为连续向量 (一组实数) 的技术. 其核心是用数学形式捕获数据的语义或关系.

比如将单词 cat 转为 [0.2, 0.5, 0.7].

相似的对象在向量空间中距离相近.

常见的 embedding 类型有:

  • Word Embedding
  • Graph Embedding
  • Image Embedding
  • Category Embedding

CKPT

CKPT 指在训练过程中保存的模型状态快照:

  • 保存训练过程中的权重, 优化器状态等信息, 防止因意外中断导致训练丢失
  • 可以从检查点恢复训练, 无需从头开始
  • 可以比较不同 checkpoint 的性能, 来选择最佳模型

什么是推理

推理, inference 是深度学习模型在实际应用中对新数据进行预测的核型过程, 此时模型参数固定, 仅执行前向计算.

给定训练好的模型参数 $\theta$ 和新输入 $x$, 计算输出:
$$
\displaylines
{
\begin{aligned}
\hat{y} = f_{\theta} (x)
\end{aligned}
}
$$

Sequential Recommendation

Sequential Recommendation, 序列化推荐, 核心思想是利用用户的历史行为序列 (按时间排序) 来预测其下一个可能感兴趣的物品.

传统推荐仅考虑用户静态偏好和物品属性, 而序列化推荐重点关注用户行为的动态变化和顺序模式.

序列化推荐中, 用户的喜好是动态的, 受上下文影响, 比如:

  • 传统推荐: 认为用户偏好是静态的 (“用户A喜欢科幻电影”)
  • 序列化推荐中: 认为用户偏好是动态且受上下文影响的 (“用户 A 在购买了手机后, 接下来很可能会购买手机壳和贴膜”)

数学定义, 给定一个用户 u 的历史行为序列 $S_u = [i_1, i_2, i_3, …, i_t]$ (i 是物品, 下标是时间戳), 序列化推荐的目标是预测用户 u 在 t+1 时刻最可能交互的物品 $i_{t+1}$:

$$
\displaylines
{
\begin{aligned}
P(i_{t+1}|S_u)
\end{aligned}
}
$$

Temporal Recommendation 和 Sequential Recommendation 的区别

Temporal recommendation 更关注精确的时间, 以建立用户和物品之间联系的动态变化. 比如: “用户在不同时间段 (如季节, 时段) 的偏好如何变化?”

  • 可以得到假设如: 时间影响用户偏好 (如 “早晨看新闻, 晚上看电影”)

Sequential recommendation 关注的是行为序列间的顺序, 不需要严格精确的时间戳, 而将用户行为序列按顺序排序, 比如: “用户看了A后，接下来会看什么？”

  • 其中蕴藏一些因果关系

Point-Wise Feed Forward Network

Point-Wise Feed Forward Network, 逐点前馈网络, 其作用是对序列中每个位置的向量独立进行非线性变换, 增强模型的表达能力.

  • 这里的 Point-Wise 就是指对输入张量的每个位置 (如序列中的每个 token) 独立处理, 不跨位置交互

该网络一般由两个 Linear 层组成, 中间加一个激活函数:

$$
\displaylines
{
\begin{aligned}
FFN(x) = W_2 \cdot ReLU(W_1 \cdot x + b_1) + b_2
\end{aligned}
}
$$

  • $W_1,W_2$ 是权重矩阵, $b_1, b_2$ 是偏置

Self-attention 和 causality

对于 sequential model, causality 比较重要, 因为模型在预测 $t+1$ 个元素时, 只能依赖前 $t$ 个元素, 而不能 “偷看” 未来的信息.

标准的 self-attention 机制是 non-causal 的, 因为其允许每个位置的输出依赖整个序列的所有位置 (包括未来位置), 这在训练时问题不大, 但推理时会导致信息泄漏, 此时模型会利用未来的数据导致预测不符合当前逻辑.

需要用 mask 来实现 causal attention, 在计算 $Q_i$ 和 $K_j$ 的注意力权重时, 如果 $j > i$, 则强制设为 $-\infty$, 这样第 $t$ 个位置的输出 $S_t$ 仅依赖于 $x_1,x_2,…,x_t$, 而不会受到 $x_{t+1},…,x_n$ 的影响.

生成一个 causal mask, 一个上三角矩阵, 对角线及以下为 0, 以上为 $-\infty$, 例如对序列长度为 4:

$$
\displaylines
{
\begin{aligned}
mask = \begin{bmatrix}
0 & -\infty & -\infty & -\infty \newline~ \newline
0 & 0 & -\infty & -\infty \newline~ \newline
0 & 0 & 0 & -\infty \newline~ \newline
0 & 0 & 0 & 0 \newline~ \newline
\end{bmatrix}
\end{aligned}
}
$$

在计算注意力权重时应用掩码:

$$
\displaylines
{
\begin{aligned}
A = softmax( \frac{Q K^T}{\sqrt{d}} + Mask) V
\end{aligned}
}
$$

Prediction layer

在 prediction layer 做的计算为:

$$
\displaylines
{
\begin{aligned}
r_{i,t} = F_t^{(b)} N_i^T 或 r_{i,t} = F_t^{(b)} M_i^T
\end{aligned}
}
$$

  • $N_i$ 或 $M_i$ 是物品 $i$ 的 embedding vector, 来自矩阵 $N$ 或 $M$
  • $r_{i,t}$ 是一个标量, 表示给定前 $t$ 个物品, 下一个物品是 $i$ 的可能性

为什么不能直接对 $F_t^{(b)}$ 做 softmax

因为 $F_t^{(b)}$ 本身并不是一个概率分布, 而是一个序列的隐状态表示, 是经过多层注意力计算后的特征向量, 其表示特征空间中的点, 而不是每个类别的概率分数, 对其做 softmax 会导致无意义的概率, 且无法计算 $F_t^{(b)}$ 和候选物品 $i$ 的匹配程度.

预测的过程, 就是拿输出的特征向量与物品的 embedding 相乘, 得到匹配分数, 之后做 softmax 得到概率分数.

$F_t^{(b)}$ 可以看作 “用户当前兴趣” 的编码, 由历史序列 $[s_1, s_2, ,…, s_t]$ 计算而来.

nn.Conv2d

nn.MaxPool1d

nn.MaxPool1d 对序列数据进行 max pool 操作, 以保留重要特征:

  • 输入形状 (batch_size, channels, length)
  • 输出形状 (batch_size, channels, floor((length - kernel_size) / stride) + 1)
  • 一般都取 kernel_size=stride, 此时有 output_length = input_length // kernel_size

nn.MaxPool2d

nn.MaxPool2d 对图像数据进行 max pool 操作:

  • 输入形状: (batch_size, channels, height, width)
  • 输出形状: 当 kernel_size = stride 时有 (batch_size, channels, height // pool_size, width // pool_size)

Computational Graphs

Computational graph 是一个以 Function objects 构成的 directed acyclic graph, 叶子节点是 input tensors, roots 是 output tensors.

在 forward pass 中, autograd 会同时进行两个操作:

  • 执行指定运算以得到 resulting tensor
  • 在 DAG 中维护该操作的 gradient function

在 backward pass 中, 即 DAG root 调用 .backward() 方法, autograd 会:

  • 用 .grad_fn 计算 gradients
  • 将结果累加到各 tensor 的 grad 属性中
  • 用 chain rule 计算到 leaf tensors

torch.nn.Module 类

torch.nn.Module 是一个基类, PyTorch 中所有神经网络模型都继承自这个类.

nn.Module 封装了所有神经网络层, 模型和组件的共同行为, 主要负责:

  • 管理状态 (state), 接管网络的所有 “可学习参数” (如 weight 和 bias)
  • 定义计算, 通过 forward 方法定义数据如何从输入流向输出

参数管理

nn.Module 自动跟踪的所有可学习参数, 可以通过下面方法访问:

  • parameters(), 返回一个迭代器, 包含模型的所有参数
  • named_parameters(), 返回包含参数名称和参数本身的迭代器

模块结构管理

nn.Module 可以嵌套成为一个树, 可以用下面方法访问内部模块:

  • children(), 返回直接子模块的迭代器
  • modules(), 递归返回网络中的所有模块 (包括自身) 的迭代器
  • named_modules(), 递归地返回包含名称和模块本身的迭代器

状态管理

State 以字典形式存储, 可以保存供以后加载:

  • state_dict(), 返回一个字典, 包含模型的所有参数和持久缓冲区
  • load_state_dict(), 从字典中加载参数

设备管理

nn.Module 可以在设备间 (CPU, GPU) 移动:

  • to(device) 方法, 将模型的所有参数和缓冲区移动到指定的设备

注意事项

backward

backward() 方法只能由标量调用, 即 1x1 的张量.

梯度计算本质是 “标量函数对张量的导数”, 即输出每个元素对输入每个元素的偏导数.

假设 $y$ 是标量 (如 loss), 则 y.backward() 会计算 $\frac{\partial y}{\partial x}$, 结果是一个与 x 形状相同的梯度张量.

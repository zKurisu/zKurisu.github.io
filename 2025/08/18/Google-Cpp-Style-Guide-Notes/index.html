Header Files

基本上每个 .cc 文件都会对应一个 .h 文件, 除了单元测试或者只包含 main 函数的 .cc 文件.

Self-contained Headers

.h 的头文件需要是 self-contained 的, 即能够独立编译.

不能独立编译的用 .inc 作为文件后缀.

Header file 应该包含自己需要的所有其他 headers (比如用了一些标准库), 需要包含 header guards, 如:

如果 header file 中声明了 inline function 或 templates, 需要直接包含其定义, 因为它们需要在编译时实例化, 若定义不可见会导致链接错误.

Include What You Use

如果源文件 (.cc) 或头文件 (.h) 使用了其他文件中定义的符号 (如类, 函数, 变量), 必须直接包含该符号的声明头文件, 不能依赖其他啊头文件间接包含的符号.

比如 foo.cc 用了 bar.h 中的符号, 即使 foo.h 中包含了 bar.h, foo.cc 仍需显式包含 bar.h.

Defining Functions in Header Files

只有在函数很短时 (10 行或更短) 才直接在 header file 中定义. 通常用 inline 修饰.

对于必须要放在 header file 中的长函数定义, 需通过注释或命名空间隔离, 而非直接声明在 public 部分:

避免头文件被冗长实现干扰, 突出接口设计.

Names and Order of Includes

包含 headers 的顺序为:

  • Related header, 比如要调试的目标 header
  • C system headers
  • C++ standard library headers
  • other libraries headers
  • your project’s headers

注意不使用 UNIX 目录记号如 . 和 ...

只有下面一些 headers 用 <> 包裹:

  • C 和 C++ standard library headers, 如 <stdlib.h>, <string>
  • POSIX, linux 和 windows system headers, 如 <unistd.h> 和 <windows.h>
  • 部分第三方库, 如 <Python.h>

Static and Global Variables

每一个 object 都有一个 storage duration:

  • 全局变量在程序启动时按源码中的定义顺序初始化, 程序退出时, 全局变量按初始化顺序的逆序析构 (后初始化的先析构)
  • Static storage duration 指 namespace 中定义的变量或 class 中的 data members, 或者 static 修饰的变量, 会在程序启动时初始化
  • Function-local static variables 之函数中定义的变量, 在首次执行到声明时初始化
  • 所有静态变量在程序退出时析构

Trivially Destructible 指一个类型的析构函数 ~T() 满足下面条件:

  1. 析构函数是编译器自动生成的或显示定义为 = default
  2. 析构函数非虚
  3. 析构函数不执行任何额外操作

定义全局变量时, 仅允许 trivially destructible 的类型, 如 int, float, std::array<int, 3> 等. 避免程序退出时的析构顺序问题.

Classes

Doing Work in Constructores

在构造函数内禁止调用 virtual function, 因为子类尚未构造完成, 此时虚函数表 (vtable) 未正确初始化, 导致调用的是当前类的实现, 而非子类重写版本.

使用简单的初始化构造函数, 避免构造失败.

Implicit Conversions

定义 single-argument constructor 时, 加上 explicit 关键字避免隐式类型转换.

Copyable and Movable Types

Copyable 指, 支持通过拷贝构造函数和拷贝赋值运算符从同类型对象复制值, 且源对象不会被修改, 即拷贝后的对象与源对象独立, 修改互不影响. 比如: int, std::string:

Google C++ Style Guide

Header Files

基本上每个 .cc 文件都会对应一个 .h 文件, 除了单元测试或者只包含 main 函数的 .cc 文件.

Self-contained Headers

.h 的头文件需要是 self-contained 的, 即能够独立编译.

不能独立编译的用 .inc 作为文件后缀.

Header file 应该包含自己需要的所有其他 headers (比如用了一些标准库), 需要包含 header guards, 如:

如果 header file 中声明了 inline function 或 templates, 需要直接包含其定义, 因为它们需要在编译时实例化, 若定义不可见会导致链接错误.

Include What You Use

如果源文件 (.cc) 或头文件 (.h) 使用了其他文件中定义的符号 (如类, 函数, 变量), 必须直接包含该符号的声明头文件, 不能依赖其他啊头文件间接包含的符号.

比如 foo.cc 用了 bar.h 中的符号, 即使 foo.h 中包含了 bar.h, foo.cc 仍需显式包含 bar.h.

Defining Functions in Header Files

只有在函数很短时 (10 行或更短) 才直接在 header file 中定义. 通常用 inline 修饰.

对于必须要放在 header file 中的长函数定义, 需通过注释或命名空间隔离, 而非直接声明在 public 部分:

避免头文件被冗长实现干扰, 突出接口设计.

Names and Order of Includes

包含 headers 的顺序为:

  • Related header, 比如要调试的目标 header
  • C system headers
  • C++ standard library headers
  • other libraries headers
  • your project’s headers

注意不使用 UNIX 目录记号如 . 和 ...

只有下面一些 headers 用 <> 包裹:

  • C 和 C++ standard library headers, 如 <stdlib.h>, <string>
  • POSIX, linux 和 windows system headers, 如 <unistd.h> 和 <windows.h>
  • 部分第三方库, 如 <Python.h>

Static and Global Variables

每一个 object 都有一个 storage duration:

  • 全局变量在程序启动时按源码中的定义顺序初始化, 程序退出时, 全局变量按初始化顺序的逆序析构 (后初始化的先析构)
  • Static storage duration 指 namespace 中定义的变量或 class 中的 data members, 或者 static 修饰的变量, 会在程序启动时初始化
  • Function-local static variables 之函数中定义的变量, 在首次执行到声明时初始化
  • 所有静态变量在程序退出时析构

Trivially Destructible 指一个类型的析构函数 ~T() 满足下面条件:

  1. 析构函数是编译器自动生成的或显示定义为 = default
  2. 析构函数非虚
  3. 析构函数不执行任何额外操作

定义全局变量时, 仅允许 trivially destructible 的类型, 如 int, float, std::array<int, 3> 等. 避免程序退出时的析构顺序问题.

Classes

Doing Work in Constructores

在构造函数内禁止调用 virtual function, 因为子类尚未构造完成, 此时虚函数表 (vtable) 未正确初始化, 导致调用的是当前类的实现, 而非子类重写版本.

使用简单的初始化构造函数, 避免构造失败.

Implicit Conversions

定义 single-argument constructor 时, 加上 explicit 关键字避免隐式类型转换.

Structs vs Classes

在 C++ 的语法中, struct 和 class 的唯一语法区别就是默认的访问权限:

  • struct, 默认成员是 public, 默认继承是 public
  • class, 默认成员是 private, 默认继承是 private

但在该规范中, struct 仅用于 passive data objects, 其他时候都用 class, 即:

  • 纯数据集合, 仅存储数据, 不主动管理资源或维护复杂逻辑
  • 可以有简单行为, 比如构造函数, 析构函数或辅助方法, 但不能强制约束数据之间的关系
  • 不需要封装, 所有成员可直接访问

class 用于 active object, 即:

  • 有行为 (方法), 封装操作和逻辑
  • 需要封装, 隐藏实现细节

Structs vs Pairs and Tuples

相比 pair 或 tuple, 优先使用 struct.

当 pair 或 tuple 中的元素没什么实质含义是可以使用这两个数据结构.

Inheritance

Composition (组合) 大多时候都优于 inheritance (继承). 如果必须用继承, 应该用 public 继承.

在 C++ 中继承可以分两大类:

  • Interface inheritance, 接口继承:

    • 继承纯抽象基类, 只有纯虚函数, 无数据成员
    • 目的是定义行为

  • Implementation inheritance, 实现继承:

    • 继承包含具体实现或数据的基类
    • 目的是复用代码或数据

优先使用 interface inheritance, 而 implementation inheritance 需要谨慎.

Composition 指, 一个类包含另一个类的对象作为成员, 而不是继承它:

Operator Overloading

需要谨慎重载运算符 (operator 关键字), 且不使用用户定义的字面量.

就算要重载, 尽量只在自己定义的类别中, 限制在这些 .cc 和头文件以及 namespace 中.

鼓励对 ==, =, << 这些作用直观的运算符进行重载, 而不是定义 Equals(), CopyFrom(), PrintTo() 等函数:

禁止重载 &&, ,, &.

Access Control

classes's data member 应设为 private, 除非它们是常量.

Declaration Order

将相似的声明放在一起, 将 public 部分放在最前面.

一般的顺序是: start with public: section, 之后是 protected:, 之后是 private:.

具体的有:

  1. Types and type aliases (typedef, using, enum, nested structs and classes, and
    friend types)
  2. (Optionally, for structs only) non-static data members
  3. Static constants
  4. Factory functions
  5. Constructors and assignment operators
  6. Destructor
  7. All other functions (static and non-static member functions, and friend
    functions)
  8. All other data members (static and non-static)

Inputs and Outputs

在 C++ 中, 函数输出结果的方式主要有两种:

  1. 返回值, return value, 直接通过 return 返回结果
  2. 输入/输出参数, in/out parameters, 通过修改函数参数传递结果, 而非通过 return, 通常使用指针或引用实现

示例:

优先考虑用 return 显式返回, 之后考虑 reference.

C++ 中提供了 std::optional, 表示一个参数是否可选:

  • Optional 参数, 调用者可以忽略, 此时函数需要处理 “无值” 的情况 (毕竟没传递参数)
  • Non-optional 参数, 调用者必须提供有效值

Non-optional input 应该为直接值或者 const reference.

Non-optional output, 以及 input/output parameters 应该为 reference.

在函数定义的参数顺序上, input-only 的参数放在所有 output parameters 的前面.

Write Short Functions

优先编写 small and focused functions.

当一个函数超过 40 行时, 就该考虑是否能将其拆分.

Function Overloading

函数重载应该语义清晰, 当开发者看到调用处的代码时, 应该能直观理解其意图, 而无需深入研究到底调用了哪个重载版本.

比如参数类型或数量有显著区别:

切要确保所有重载版本应完成逻辑相似的操作, 避免混淆.

Default Arguments

默认参数只能用于 non-virtual function, 避免继承导致的行为不一致. 默认值也要保证始终不变 (编译器常量或字面量).

如果使用默认参数会降低可读性, 则考虑改用函数重载.

Trailing Return Type Syntax

默认情况下使用传统的前置返回类型, 如 int foo();.

仅在以下情况使用尾置返回类型, 如 auto foo() -> int;:

  • 前置语法无法表达
  • 前置语法会降低可读性

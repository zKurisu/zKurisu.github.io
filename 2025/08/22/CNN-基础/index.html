CNN-基础

介绍

CNN (Convolutional Neuron Network), 卷积神经网络, 是一种前馈神经网络, 利用 sparse connectivity 和 parameter sharing 技术来提取空间或时序特征.

相关概念

Sparse connectivity

Sparse connectivity (局部连接), 每个神经元只连接输入数据中的局部区域 (比如图像中的一块像素), 而非全部输入.

Parameter sharing

Parameter sharing (参数共享), 同一个特征在整个图像的不同区域是通用的. CNN 使用卷积核 (滤波器) 在图像上滑动, 共享同一组参数来检测不同位置的相同特征, 能极大减少参数量.

Convolution

参考讲解1

Convolution, “col” 是 “共同” 的意思可以理解为积分, 词根 “vol” 是 “卷/转” 的意思, 因此翻译成卷积.

数学定义:
$$
\displaylines
{
\begin{aligned}
f(t) * g(t) = \int f(\tau)g(t- \tau)d\tau
\end{aligned}
}
$$
* 是卷积运算符, 其本质就是定义了一种运算, 但要比加减乘除高级很多, 卷积的过程包括 (以 2-D 坐标为例):

  • 旋转, 从 $f(t) * g(t)$ 可以看出, 两个函数在同一个因变量 $t$ 下, 比如都在坐标轴右侧:

假设 $f(t)$ 是山洞, $g(t)$ 是火车, 要描述火车进洞的过程, 则需要让 g(t) 旋转, 即 $g(t) \rightarrow g(-t)$, 把所有 $t$ 换成临时变量 $\tau$, 得到 $f(\tau)$, $g(-\tau)$:

再加上 $t$ 得到 $g(t-\tau)$, 则能让 $g$ 沿着 $t$ 轴移动:

  • 相乘, 表示 $f(\tau)$ 和 $\g(t-\tau)$ 某一时刻 (特定 $t$) 的重合 (表示相互作用)
  • 积分, 计算面积

简单来说, 就是固定一个 $t$ 值后, 求两个函数重叠的面积. 若 $t$ 在变化, 就像是一个滑动的过程, 但需要不断求面积.

也可以交换 $f(t)$ 和 $g(t)$ 的身份.

核心组件

Convolutional layer

Convolutional layer (卷积层) 负责特征提取, 关键参数有:

  • Kernel/Filter (卷积核), 小的权重矩阵 (如 3x3)

Pooling layer

Pooling layer (池化层), 负责降维, 缩小特征图的空间尺寸.

Fully connected layer

Fully connected layer (全连接层), 负责将特征映射到输出, 通常位于 CNN 末尾.

卷积层梯度计算

这里计算卷积层输出对于输入, 卷积核权重, 偏置的梯度.

以 conv1d 为例, 忽略填充, 设步长为 1, 假设:

  • 输入 $X$ 的形状为 $(C_{in}, L_{in})$
  • 卷积核权重 $W$ 的形状为 $(C_{out}, C_{in}, K)$
  • 偏置 $b$ 的形状为 $(C_{out}$
  • 输出 $Y$ 的形状为 $(C_{out}, L_{out})$

输出的计算过程:
$$
\displaylines
{
\begin{aligned}
Y[c,i] = \sum_{k=0}^{K-1} \sum_{c_{in} = 0}^{C_{in} - 1} W[c,c_{in},k] \cdot X[c_{in}, i+k] + b[c]
\end{aligned}
}
$$
(事实证明这个计算过程比较复杂, 因为输出的形状较大)

卷积层的反向传播需要计算三个梯度:

  1. 输入梯度 (input_grad), 损失对输入数据的梯度 $\frac{\partial L}{\partial X}$
  2. 权重梯度 (weight_grad), 损失对卷积核权重的梯度 $\frac{\partial L}{\partial W}$
  3. 偏置梯度 (bias_grad), 损失对偏置项的梯度 $\frac{\partial L}{\partial b}$

kernel_size, stride, padding 的数值关系

对于输入尺寸为 $(H_{in}, W_{in})$ 的特征图, 输出尺寸 $(H_{out}, W_{out})$ 的计算公式为:

$$
\displaylines
{
\begin{aligned}
H_{out} = [ \frac{H_{in} + 2 \times padding - kernel_size}{stride} + 1] \newline~ \newline
W_{out} = [ \frac{W_{in} + 2 \times padding - kernel_size}{stride} + 1]
\end{aligned}
}
$$

若要保证尺寸不变, 当 stride=1 时, 需要满足:

$$
\displaylines
{
\begin{aligned}
padding = \frac{kernel_size - 1}{2}
\end{aligned}
}
$$

  • 比如: kernel_size = 3 $\rightarrow$ padding = 1; kernel_size=5 $\rightarrow$ padding=2

stride 用于缩小尺寸, 当 stride > 1 时, 输出尺寸按比例缩小, 比如:

  • stride=2, 尺寸约减半 (配合 padding 调整)

在设计时, 一般要注意让 $(H_{in} + 2 \times padding − kernel_size)$ 能够被 stride 整除, 避免向下取整导致边缘信息丢失.

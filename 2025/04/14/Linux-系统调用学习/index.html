Linux-系统调用学习

介绍

Linux system call 的定义一般包含在 <unistd.h> (UNIX Standard) 头文件中.

学习时常用

  • man 2 xxx 查看手册
  • strace 追踪调用情况

追踪调用:

Pipe

对于 pipe 而言, 而要两个 “管道文件描述符”, 一个用于 child 读取, 一个用于 parent 写入.

  • <sys/types.h> 提供基本的系统类型, 如 pid_t
  • <unistd.h> 提供 POSIX 系统调用, 如 pipe, fork, getpid 等
  • <stdio.h> 提供标准 I/O 函数, 如 printf
  • UNIX 文件描述符是一个 integer
  • pipe(pipefds) 用指定数组创建管道

汇编调用 open 和 sendfile

  • open 用来打开一个文件 (一个文件描述符)
  • sendfile 将一个文件发送到另一个文件 (这里把文件内容发送到 STD_OUT)

汇编调用 getcwd

  • getcwd 系统调用, 用于获取当前目录

Simple distro

学习自 Youtube

需要 3 个组件:

  1. linux kernel, 负责管理硬件资源, 进程调度, 内存管理, 设备驱动等底层功能
  2. busybox, 一个集成了数百个常用 Linux 命令的单一可执行文件, 专为嵌入式系统和救援环境设计, 这里用作 user space
  3. syslinux, 一组轻量级的 BIOS/UEFI bootloader, 以启动 Linux 系统

启动流程为:

  1. syslinux 加载 linux kernel
  2. linux kernel 启动 busybox

这里的构建流程在 docker 中进行:

  • bzip2 提供压缩工具, 用于解压内核源码 (.tar.bz2 格式)
  • git 用于下载内核源码
  • vim 用于编辑配置
  • make 用于构建
  • gcc 用于编译内核
  • libncurses-dev 提供 ncurses 库的开发文件, 支持 make menuconfig 的文本界面配置
  • flex, 词法分析器生成器, 用于构建内核时解析某些配置文件
  • bison, 语法分析生成器, 与 flex 配合使用
  • bc, 内核配置过程的数学运算
  • cpio, 处理 initramfs 的归档工具
  • libelf-dev, 提供 libelf 库, 解析 elf 文件
  • libssl-dev, 提供 OpenSSL 开发库, 内核某些加密功能依赖

介绍

U-Boot, Universal Bootloader, 是一个开源的跨平台 Bootloader, 主要用于嵌入式系统.

其主要任务是在操作系统启动前, 完成硬件初始化, 加载内核和根文件系统.

BIOS, 引导扇区与引导程序的关系

  • BIOS, 是固件程序, 负责开机自检(POST), 硬件初始化, 并加载引导程序
  • 引导扇区, 是磁盘的第一个扇区 (512 bytes), 存储引导程序和分区表
  • 引导程序, 一个程序, 负责加载操作系统的核心文件

由于 u-boot 程序通常大于 512 bytes, 因此, 在引导扇区中先存储微型程序, 称 SPL. Secondary Program Loader, 其负责加载 U-Boot 主程序到内存. u-boot 主程序存储在后续扇区或特定分区.

然后运行:

  • -M, machine, 指定开发板
  • -serial stdio 指定输出到当前终端
  • -device loader,file=u-boot.bin,addr=0x60800000,cpu-num=0,force-raw=on 指定设备加载参数
    • loader 是 QEMU 的通用设备加载器
    • file=u-boot.bin 指定要加载的二进制文件
    • addr=0x60800000 指定要将文件加载到的地址
    • cpu-num=0 指定 CPU 核心
    • force-raw=on, 强制以原始二进制格式加载 (不解析 ELF 头)

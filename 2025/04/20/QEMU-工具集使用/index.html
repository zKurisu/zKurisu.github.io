QEMU-工具集使用

介绍

QEMU, Quick Emulator, 是一个开源的硬件虚拟化平台, 支持跨架构的全系统模拟和用户空间程序模拟.

qemu-img

qemu-img 用于创建虚拟硬盘:

  • -f, 指定 format
  • ubuntu.qcow2 为文件名
  • 20G 为容量

qemu-system-x86_64

qemu-system-x86_64 是 QEMU 项目提供的针对 x86_64 架构系统的模拟器, 其能够模拟完整的计算机系统, 包括 CPU, 内存, 存储设备, 外设等.

不加参数的默认行为

  • QEMU 调用内置的 SeaBIOS 尝试从 disk.qcow2 启动
  • 如果 disk.qcow2 中包含有效的引导扇区, 则继续启动, 否则报错 No bootable device
  • 默认配置为:
    • 1 个 CPU 核心
    • 128MB 内存
    • -vga std 显卡
    • disk.qcow2 为存储设备
    • 无网络

安装 linux 到硬盘

  • -m 指定 memory 大小
  • -smp, symmetric multi-processing, 用于配置多核 CPU, 这里指定要使用的核心数
  • -hda, 指定硬盘 (hda, hdb 这些是 Linux 对硬盘设备的编号)
  • -cdrom, Compact Disc Read-Only Memory, 这里指定 Linux 安装镜像
  • -boot, 指定启动顺序, d 意思从 cdrom 启动, 默认值为 c, 从 -hda 指定的硬盘启动
  • -enable-kvm 启动 KVM 加速
  • -vga, video graphic array, 指定显卡模拟类型, 主流 Linux 发行版都会内置 virtio 显卡驱动
  • -display 指定虚拟机显示方式, gtk 会打开一个 GTK 图形窗口, vnc=:0 能远程访问 (vncviewer localhost:5900), none 则为字符输出

从 qcow 硬盘启动虚拟机

指定 BIOS 固件

  • -bios, 指定 BIOS 固件程序, 其会在虚拟机启动时最先执行, 负责硬件初始化和引导操作系统
  • -serial 指定串口输出方式, stdio 为当前终端, mon:stdio 结合 QEMU monitor 和串口

qemu-system-riscv32

运行裸机程序

裸机程序 (Bare-Metal Program), 指不依赖任何操作系统, 直接在硬件上运行一段程序:

  • -nographic, 不使用图形界面, 将控制到输出重定向到终端
  • -smp 1, 表示只模拟一个 CPU 核心
  • -machine virt, 表示使用 QEMU 内置的一个通用的, 软件定义的虚拟平台
  • -bios none, 表示不加载 BIOS 或固件. 因为硬件在 CPU 启动时会先执行一段固件代码
  • -kernel xx.elf, 指定 QEMU 加载并运行的文件 (通常是内核)

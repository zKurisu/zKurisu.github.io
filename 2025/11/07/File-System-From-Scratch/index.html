High level view of design

有 3 种类型的 block:

  • Superblock: 1 个
  • Inode blocks: 要占大约 10% 的硬盘空间
  • Data blocks: 剩余磁盘空间

Super block 也是 512 bytes, 大致结构如下:

  • 500 bytes reserved for boot sector codes
  • 2 bytes reserved
  • 2 bytes 记录 blocks 的数量
  • 2 bytes 记录 inode blocks 的数量
  • 2 bytes 记录 filesystem 中的 inodes 数量
  • 4 bytes Magic string

一个 Inode 占 32 bytes, 结构如下:

  • 1 bytes, valid/type field, 其中:
    • 0~3 bits
    • 4~6 bits, 表示文件类型
    • 最后 1 bit 表示是否空闲, 0 表示 not valid, 1 表示 valid
  • 2 bytes, size, 表示文件大小
  • 11 bytes, 表示文件名: 前 8 bytes 是文件名, 后 3 bytes 是扩展名
  • 2 bytes, indirect pointer, 指向一个包含指针数组的 data block (用于大文件)
  • 16 bytes, direct pointers (8 个), 直接指向 data block (8 * 512 = 4k)

Magic Number 常用来标识文件系统类型, 合法性, 数据块类型等, 在文件系统设计中非常重要. 其通常就是一个特殊的, 预定义的数值.

Magic Number 通常被写入磁盘的 superblock, 文件头, inode, 数据块标记等位置. 便于操作系统或文件系统工具在读取磁盘数据时通过检查这些 magic number 来判断其是否是合法的文件系统, 这个块是不是 inode/目录/数据块 等.

Superblock

如:

Superblock 是文件系统的核心数据结构, 通常位于文件系统的起始位置 (比如磁盘的某个固定的偏移处), 其存储了关于整个文件系统的重要全局信息, 这些信息可以便于文件系统的挂载, 访问和管理等.

Superblock 的具体内容因文件系统类型而异, 通常会包含:

  • 文件系统类型/Magic number, 用来标识该文件系统的类型, 防止误识别
  • 块大小 (Block size), 文件系统的基本单位 (如 4096 字节)
  • 总块数 (Total blocks), 整个文件系统中有多少个块
  • 空闲块数 (Free blocks), 当前还有多少块未被使用
  • inode 总数 (Total inode), 文件系统支持多少个文件/目录描述符
  • 空闲 inode 数 (Free Inodes), 当前还有多少 inode 可分配
  • 块位图位置 (Block bitmap location), 指向记录哪些块已用的位图
  • inode 位图位置 (Inode bitmap location), 指向记录哪些 inode 已用的位图
  • inode 表位置 (Inode Table Location), inode 数据结构存放的起始位置
  • 文件系统状态 (Filesystem state), 如 “clean” 或 “dirty”, 用于判断是否需要 fsck
  • 挂载次数/最大挂载次数, 用于安全卸载和强制检查
  • 最近一次挂载时间/写入时间, 用于日志, 审计等
  • 版本/特性标志, 比如是否支持日志, 大文件, 扩展属性等

Inode 结构

Inode (Index node, 索引节点), 是文件系统中用于描述一个文件或目录的元数据结构, 其通常不包含文件名本身, 但包含与该文件相关的几乎所有其他信息, 比如:

  • 文件类型, 是普通文件, 目录, 符号链接, 设备文件等
  • 权限信息, 用户/组/其他用户的读, 写, 执行权限
  • 所有者/组, UID 和 GID
  • 文件大小, 文件占用的字节数
  • 时间戳, 包括创建时间 (ctime), 最后修改时间 (mtime), 最后访问时间 (atime)
  • 数据块指针, 指向文件内容实际存储在磁盘上哪些数据块的指针, 包括直接指针 (10~12 个), 一级间接指针, 二级, 三级间接指针等 (用于大文件)
  • 链接计数 (link count), 有多少个目录项 (硬链接) 指向该 inode
  • 状态信息, 比如是否被打开, 是否脏等

文件系统通常用 “目录” 把文件名和 “inode” 编号关联起来:

对于绝大多数类 UNIX 文件系统而言, 在格式化初始化时, 第一个 inode 会被专门用于表示根目录, 是整个文件系统目录树的起点.

Block 结构

union 是一种特殊的数据结构, 其所有成员共享同一块内存空间, 大小等于最大成员的大小. 同一时间只能使用其中一个成员, 但可以根据需要, 将同一个 block 解释为不同的逻辑结构.

这里相当于 block 有多种类型, 可以是:

  • 超级块, 存放整个文件系统的元信息
  • 数据块, 存放文件的实际内容
  • 指针块, 存放指向其他数据块的指针
  • inode 块, 将多个 inode 存放在一个块内

Formatting

Formatting 指, 在存储设备上初始化并创建一套用于管理文件和数据的结构, 比如超级块, inode, 位图, 数据块等.

比如一块空白硬盘, 本质上就是一大块二进制存储空间, 此时操作系统不能直接理解怎么存文件, 找文件等, 格式化的目的, 就是让操作系统知道:

  • 文件系统是什么类型
  • 哪里存放文件的元信息
  • 哪里存放文件的内容
  • 那些块是空闲的
  • 如何管理文件等

具体需要做的通常有:

  1. 创建超级块 (superblock)
  2. 初始化 inode 表
  3. 初始化数据块区域
  4. 初始化 bitmap
  5. 创建根目录
  6. 设置文件系统类型和版本等等

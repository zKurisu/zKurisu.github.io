错误处理

所有的错误信息应该传递给 parser, 不需要在 lexer 中打印出. 通过传递一个特殊的 token ERROR.

常见场景有:

  • 遇到非法字符时, 报告 error string, 然后继续后续字符
  • 遇到 unescaped newline, 报告 Unterminated string constant, 然后从下一行开始
  • 字符串太长, 报告 String constant too long
  • 注释没有闭合, 报告 EOF in comment
  • 字符串没有闭合, 报告 EOF string constant
  • 注释之外遇到 *), 报告 Unmatched *)

scanner 需要正确把转移字符转换为 ASCII 值:

当遇到 \0 时应该报错, 且转换为 0.

需要维护 curr_lineno 变量.

C++ 版本注意

  • 每次调用 scanner 都会返回 next token 和 lexeme
  • cool_yylex 函数返回 integer code, 代表 syntactic category, 定义在 cool-parse.h 文件中
  • lexeme 从 cool_yylval (global union) 中读取, 类型为 YYSTYPE
  • class identifiers, object identifiers, integers, strings 的 lexeme 应该存储在 cool_yylval.symbol 中
  • boolean constants 的 lexeme 应该存储在 cool_yylval.boolean 中
  • 当遇到 lexical error 时, cool_yylex 应该返回 token ERROR, lexeme 为错误信息, 并存储到 cool_yylval.error_msg

A Tour of the Cool Support Code

获取该文档:

下面是该文档的阅读笔记.

Introduction

CS143 课程提供了一系列工具以便于编写 Cool compiler, 包括:

  • a string table package
  • a symbol table package
  • miscellaneous utilities for the parser and lexer
  • a package for managing Cool abstract syntax trees
  • routines for printing abstract syntax trees
  • miscellaneous routines for handling multiple input files and command-line flags
  • the runtime system

Cool system 用 C++ 编写, 主要用到了 classes, single inheritance, virtual functions, overloading, and templates 这些主题.

String table

所有编译器都需要管理大量 strings, 包括 program identifiers, numerical constants, string constants 等. 但很大部分这些字符串都是重复的 (比如一个 identifier 在多处调用), 没有必要为所有这些字符串都创建一个副本. 此时就出现了一个新的数据结构 string table.

String table 是一个查找表, 维护字符串的单个副本.

在 Cool compiler 项目提供的工具代码 stringtab.h 中, 提供了 Cool string table class, 能够向 string table 插入和查询内容.

很多 production compilers 用 hashed data structures 来实现 string tables, 但这里用 lists 来实现 (见 stringtab_functions.h).

Cool string tables 由 Entry 组成, 每个 Entry 保存:

  • a string
  • the length of the string
  • an integer index unique to the string

在 Cool compiler 中, 有三个独立的 string tables:

  • stringtable, 保存 string constants
  • inttable, 保存 integer constants
  • idtable, 保存 identifiers

Code generator 可以根据这几个 table 对这些常量做区分, 让 string constant 和 integer constant 生成不同的代码.

注意这三个 table 中的 Entry 类型不同:

  • StrEntry
  • IntEntry
  • IdEntry

但都继承自 Entry.

有三个方法可用于向 table 中添加元素, 返回值都为 Entry:

  • add_string(char *s, int m), 添加字符串 s, 长度最大为 m
  • add_string(char *s), 直接添加字符串 s
  • add_int(int i), 将 integer i 转为 string 然后添加在 table 中

get_string 方法用于从 entry 中提取出 string. 在使用这些方法前, 先阅读 stringtab.cc 文件.

Symbol Tables

Symbol table (符号表) 是一个用于管理 scope 的数据结构. 本质上也是一个 lookup table, 关键点在于其 key 存储 symbol name, 而 value 存储与该符号相关的信息.

如果同一名称在不同作用域中定义 (如全局变量 x 和局部变量 x), 符号表可以正确关联当前作用域的名称.

Introduction

Cool, the Classroom Object-Oriented Language. 是一门教学用的小型语言, 但具有现代语言的一些特性如 objects, static typing, automatic memory management.

Cool 程序由一组类 (classes) 组成, 类是程序的基本模块化单元, 类似 Java/C++, 所有代码必须封装在类中.

类将变量 (成员属性) 和过程 (成员方法) 捆绑在一起, 形成一个数据类型.

类的实例称对象, 在 Cool 中, 类名直接作为类型名使用. 也可以让子类集成父类的属性和方法.

Cool 也是 expression language, 即几乎所有代码都是表达式, 每一个表达式有一个 value and type.

Cool 是 type safety 语言, 编译器会严格检查类型以确保函数正确调用, 比如不能对 Int 类型调用 String 的方法.

Getting Started

  • 其生成 MIPS assembly code

spim 是一个 MIPS simulator, 用于运行 coolc 生成的汇编.

Classes

所有 Cool 代码都必须组织在 class 之下, 定义语法为:

  • 类名需要以大写字母开头
  • feature 指 attribute 或 method, 必须以小写字母开头, 且不能重复定义, 但 attribute 名可以和 method 一致
  • 每个类定义需要用 ; 分隔

Cool 中:

  • 所有 attribute 都是 local scope to the class
  • 所有 methods 都是 global scope

Cool 中有自动的内存管理, 不需要用户手动释放.

Inheritance

语法:

子类可以覆盖父类的方法, 但不能重复定义 attribute.

在 Cool 中, 如果一个类定义没有指定父类, 则默认继承自 Object 类.

除了 Object 类外, Cool 还定义了 4 个基础类:

  • Int
  • String
  • Bool
  • IO

Types

Cool 中的 class name 也都是一个 type.

类型声明的语法为:

  • x 是变量名
  • T 是类型名

Cool 中的类型具有兼容性, 如果某个方法或变量需要类型 P 的值, 那么任何 P 的子类 C 的值都可以替换它, 这种关系称 C conforms to P, 或 $C \le P$

SELF_TYPE

SELF_TYPE 是 self 变量所在类的类型 (或者 conforms to that type), 其会动态变化为对应类别, 比如:

  • 定义 copy() 时, 其返回 SELF_TYPE
  • 但在 (new Sally).copy() 这里返回的是 Sally 类型

Type checking

Cool 利用对每个标识符进行类型声明, 便于 type checker 推测类型.

表达式的类型最终由 type checker 分配的称 “static type”, 而如果在运行时分辨, 则称 “dynamic type”.

静态类型检查能够给出更加详细的信息.

可以用 isvoid expr 来判断一个表达式是否为 void.

Expressions

Constants

Boolean constants: true 和 false. 属于 Bool 类.

Integer constants 指非字符串的数字, 如 0, 123, 007. 属于 Int 类.

String constants 指由 " 包裹的字符序列, 如 "This is a string", 最长为 1024 字符. 属于 String 类.

Identifiers

Identifiers 指:

  • 局部变量名
  • method 的形式参数
  • self
  • class attributes 等

Assignment

Dispatch

就是指调用方式. 有 3 种.

  • 如果返回值没有显式类别则为 SELF_TYPE

另外两种调用语法:

  • 等价于 self.<id>(<expr>, ..., <expr>)
  • 访问父方法, 如 e@B.f(), B 为父类

Conditionals

  • 其返回值为运行的分支的结果
  • 这里的 predicate 必须为 Bool 类型

Loops

  • predicate 必须为 Bool 类型
  • 当 predicate 为 false 时, 终止循环并返回 void

Blocks

  • 返回值为最后一个 expr 的值

Let

  • 重复用 let 定义时, 后面的覆盖前面的
  • in <expr> 指定其生效的表达式
  • 整个 let 的返回值为 <expr> 的结果
  • 初始化 x 为 10
  • 初始化 y 为 20
  • 计算 x * y
  • 返回 200

可以嵌套:

  • 这里没有 default 或 otherwise 分支, 但可以用 x : Object => ... 来实现

New

Isvoid

算术运算

二元运算支持 +, -, *, /, 语法为:

  • 两个子表达式的 static type 必须为 Int
  • 只有整数除法

三种比较运算符: <, <=, =. 结果为 Bool 类型.

  • 不同类型比较不会报错, 但会返回 false

not <expr> 取 boolean 值的非.

Basic Classes

Object

Object 类是 inheritance graph 的根节点, 有下面几个方法:

  • abort() : Object, 暂停程序运行并报错
  • type_name() : String, 返回类名
  • copy() : SELF_TYPE, 返回对象的复制

IO

IO 类提供下面方法来进行简单的输入输出操作:

  • out_string(x : String) : SELF_TYPE, 输出 string 并返回 self
  • out_int(x : Int) : SELF_TYPE, 输出 int 并返回 self
  • in_string() : String, 从标准输入读取字符串, 直到一个换行符
  • in_int() : Int, 读取读取单个数字, 数字之后直到换行符的部分都会丢弃

Int

Int 类没有特殊的方法, 初始化默认值为 0.

String

String 类方法为:

  • length() : Int, 返回字符串长度
  • concat(s : String) : String 返回与自己连接后的字符串
  • substr(i : Int, l : int) : String, 指定初始索引和截取长度, 返回截取后的子串

初始化的默认值为 "", 空字符串.

Bool

Bool 类只有两个值: true 和 false, 默认初始值为 false.

Main Class

每一个程序都需要包含 Main 类, 且包含 main 方法, 不接受任何参数, 作为程序的入口.

每次程序运行相当于运行 (new Main).main().

Lexical Structure

Cool 中的 lexical units 有:

  • integers, 非空的由 [0-9] 组成的字符串
  • type identifiers 和 object identifiers, 非关键字字符串, 由 [_0-9a-zA-Z] 组成, Type identifiers 由大写字母开头, Object identifiers 由小写字母开头
  • special notation, 如 self 和 SELF_TYPE, 属于特殊的 identifiers, 而非关键字, 还有 (, ), = 等等
  • strings, 由 "" 包裹, 有几个特殊字符:
    • \b , backspace
    • \t, tab
    • \n, newline
    • \f, formfeed
  • keywords, 有 class, else, false, fi, if, in, inherits, isvoid, let, loop, pool, then, while, case, esac, new, of, not, true. 除了 true 和 false 外都是大小写不敏感
  • whitespace, 包括空格, \n, \f, \r, \t, \v

有两种注释形式:

  • -- 开头的行
  • *...* 之间的内容

Cool Syntax

语法描述为:

  • [...] 表示可选
  • [[...]] 包裹 meta-symbol

比如:

  • program ::= [[class;]]+ 表示程序至少要有一个类作为入口 (Main 类的 main 方法)

Game loop

一个游戏, 基本上都是位于一个无限循环中, 类似 while(true) 或 while(1), 在每一个 loop 中更新游戏状态, 一般分三步:

  1. 处理输入. 比如检测用户是否按下一个按键, 移动手柄, 移动鼠标等
  2. 更新游戏状态. 根据输入更新游戏 object 的位置, 得到的分数等
  3. 渲染游戏. 将游戏内容绘制到屏幕

Infinite Scrolling

绘制两层背景, 且滚动速度不一致, 这种称 “滚动视察 (Parallax Scrolling)”. 其核心原理是模拟人眼观察远近物体时的视觉差异. 是一种让不同图层以不同速度移动来营造深度感的2D游戏技术.

比如:

  • 离玩家 “近” 的物体 (如前景草木) 移动速度快
  • 离玩家 “远” 的物体 (如远山, 云朵) 移动速度慢
  • 静止的背景 (如星空) 完全不移动

SpriteSheet 和 Quad

SpriteSheet 指, 将多个小图 (sprite) 合并成一个大图. 避免多次加载图片造成的性能损失.

Quad 是大图中的某个矩形区域, 用于裁剪出特定的子图 (其只是定义位置信息, 不包含图片内容).

Particle Systems

Particle System 通过大量微小粒子的集体行为, 来模拟动态视觉效果, 如火焰, 烟雾, 魔法特效等.

核心术语有:

  • Particle, 粒子, 系统的基本单元, 通常为带属性的 2D 四边形 (含位置, 速度, 颜色, 生命周期等)
  • Emitter, 发射器, 控制粒子的生成规则 (如数量, 方向, 初始属性)
  • Controller, 控制器, 动态调整粒子的行为 (如重力影响, 颜色渐变等)

Love2D 中提供 love.graphics.newParticleSystem 接口来创建一个粒子系统.

Tweening

Tween (从…之间), 补间动画, 是一种平滑过渡的技术, 通过在起始值和目标值之间自动插入中间帧, 让属性 (如位置, 透明度, 缩放等) 随时间变化.

如:

  • endX 相当于目标位置
  • timer / MOVE_DURATION 计算动画的进度 t, 范围 [0, 1]
  • endX * t 实现从 0 到 endX 的线性插帧 (Lerp)
  • math.min 防止浮点数误差导致的越界

也可以使用 knife.timer 库 (或者 flux 库):

  • duration 是动画的持续时间
  • definition 中定义插值属性的目标值

Stencil

Stencil (模板) 是一种基于像素的遮罩技术, 用于控制渲染过程中的绘制区域. 其本质上是任意不可见的形状, 控制其覆盖的部分渲染.

应用 stencil 的过程称 stencil test.

Love2D 中提供的创建 sencil 接口为:

  • func 用于绘制 stencil 的形状
  • action 是使用该 stencil 的效果, 比如不渲染
  • value 是给 stencil 覆盖的像素所赋予的值, 没有覆盖的默认为 0
  • keepvals 默认为 false, 此时每次调用该 stencil 都会将 value 先设置为 0. 若为 true 则不会 reset 为 0

应用 stencil testing:

  • comparemode 指选中 pixels 的方式, 如 greater, less
  • comparevalue 指用于比较的值

StateStacks

State Stack 是一种管理游戏运行时状态的设计模式, 通过栈结构 (先进后出) 组织多个游戏状态, 其核心在于:

  • 能够允许同一时间下激活多个 state (同时渲染)
  • 能够通过出栈快速返回上一状态 (如关闭菜单后继续游戏)
  • 但每次只处理顶层 state 的 input, 比如按键, 鼠标等, 即每次只更新顶层 state 的内容

State Stack 和 State Machine 之间的区别在于:

  • State Stack 能够层级管理, 适合处理全局状态的叠加
  • State machine 适合局部行为的流转

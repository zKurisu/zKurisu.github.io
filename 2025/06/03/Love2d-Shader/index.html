介绍

Shade (阴影), 源自古英语 sceadu, 与光线的遮挡, 明暗变化相关.

-er 后缀, 表示执行动作的工具或程序.

Shader 因此可直接理解为生成阴影/明暗的工具, 或处理光照的程序, 添加这些效果后, 还需计算出最终颜色, 因此称 “着色器”.

Shader 能够用于硬件加速 Vertex 和 Poxel 的特效:

  • Vertex shader 会对每个 vertex 执行一次, 例如一个四边形有 4 个 vertexes, 一个复杂的 3D 模型可能有数千个. 负责 vertex 的位置变换, 法线计算等
  • Pixel shader 对物体覆盖的每个屏幕 pixel 执行一次, 决定该像素的最终颜色, 如光照, 特效处理

Vertex shader 先运行, 处理完所有 vertexes 之后, 光栅化生成像素, 再由 pixel shader 处理.

Shader 是运行在 GPU 上的程序, 因为其并行架构 (数千个核心) 使其能高效处理顶点和像素的批量计算.

Shader Language 介绍

Shader 并不以 Lua 编写, 而是一个特殊的语言 GLSL (OpenGL Shading Language), 其语法与 C 类似.

Love 对 GLSL 做了一些调整和别名以简化使用, 如:

GLSL 术语 LÖVE Shader Language 对应术语 备注
sampler2D Image 表示 2D 纹理
sampler2DArray ArrayImage 表示 2D 纹理数组
samplerCube CubeImage 表示立方体贴图（Cubemap）
sampler3D VolumeImage 表示 3D 纹理
texture2D(tex, uv) (GLSL 1) Texel(tex, uv) 兼容所有 GLSL 版本的纹理采样函数
texture(tex, uv) (GLSL 3) Texel(tex, uv) 同上，LÖVE 统一使用 Texel 简化调用
float number (已弃用) 早期版本别名，建议直接使用 float
uniform extern (已弃用) 早期版本别名，建议直接使用 uniform

需要注意:

  1. Vertex shader code 至少要包含一个函数 position 用于处理顶点坐标变换
  • mat4 transform_projection 受 love.graphics.translate 影响, 是当前的变换矩阵 (transformation matrix)
  • vec4 vertex_position 是变换前的顶点坐标
  • 返回值为 vec4 output_pos, 指 vertex 的最终坐标
  1. Pixel shader code 至少要包含一个函数 effect 计算每个像素的颜色
  • vec4 color 受 love.graphics.setColor, 是原本要绘制的颜色
  • Image tex 指当前要绘制的 texture 或 canvas
  • vec2 texture_coords 是 textrue 的局部坐标系, 用于标记内部像素点的位置, 范围一般为 normalized 为 (0,0) 到 (1,1), 左上角为 (0,0)
  • vec2 screen_coords 是正常的屏幕坐标系
  • 返回值为 vec4 output_color, 表示最终要绘制的像素颜色
  • vec4 是 4 元数组, vec2 是 2 元数组

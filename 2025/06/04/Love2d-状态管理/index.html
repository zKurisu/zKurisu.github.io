Love2d-状态管理

介绍

介绍两种状态管理方式, 都负责控制 State 的切换, 更新和渲染等, 控制 State 的行为:

  • StateMachine, 以图的方式组织和管理 States, 一次处理一个 State, 适合局部行为的流转
  • StateStack, 更高级的方式, 以 Stack 方式组织和管理 States, 能够同时激活多个状态, 处理顶层状态的输入, 返回上一个状态等

这里用 CS50 课程中 Colton Ogden 的实现来学习.

BaseState

  • 定义一个基础类, 以及这些空方法的好处在于, 其继承类不会出现调用一个不存在的方法的情况, 都会有一个统一的接口

这种设计是 Template Method Pattern 的一种变体, 基类负责定义子类需要实现的接口框架, 而子类通过重写这些方法来实现具体行为.

这里一个 State 有 5 中基本的行为:

  • init() 是初始化
  • enter() 对应激活 State 时的逻辑
  • exit() 对应退出 State 的时的逻辑
  • update(dt) 处理帧更新
  • render() 处理绘制

StateMachine

  • StateMachine 负责控制所有 States, 在没有状态时, 运行 empty state 的行为
  • 接受 states 列表作为输入, 默认为 {} 空表
  • processAI 方法用于把一些行为从 update 中分离出

注意, Lua 中允许向函数传入少于定义的参数个数.

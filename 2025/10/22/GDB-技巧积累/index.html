多进程调试

GDB 默认情况下只跟踪父进程, 的可以更改一些选项来调整.

follow-fork-mode

follow-fork-mode 选项有两个值:

  • parent (默认), 在 fork 后继续调试父进程, 子进程不受影响
  • child, 在 fork 后调试子进程, 父进程被 detached (不再调试)

比如:

detach-on-fork

detach-on-fork 控制是否在 fork 之后 detach 掉其中一个进程, 选项有两个:

  • on (默认), 只保留一个进程 (由 follow-fork-mode 决定的那个), 另一个被 detached
  • off, 不 detach, 可以同时调试父子两个进程 (需要配合 info inferiors 和 inferior 来切换)

指定调试目标

用 target 命令, 指定当前调试会话所连接的 “目标” 程序运行环境. 常用于需要调试远程机器上运行的程序, 或者虚拟平台中运行的程序.

比如:

  • target remote :1234, 连接到本地的 1234 端口, 通常是一个 gdbserver 在监听
  • target exec <prog>, 直接指定一个可执行文件作为目标, 类似 file

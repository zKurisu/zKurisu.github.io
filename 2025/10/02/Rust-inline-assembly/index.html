Rust-inline-assembly

Rust 圣经 inline assembly 部分

介绍

Rust 通过 asm!, naked_asm! 和 global_asm! 宏来提供内联汇编支持.

支持的指令集架构有:

  • x86, x86-64
  • ARM
  • AArch64, Arm64EC
  • RISC-V
  • LoongArch
  • s390x

Operand type

  • in(<reg>) <expr>, 将 Rust 表达式传递给汇编代码, 并让 Rust 编译器自动选择一个合适的通用寄存器来存放. 因为不同架构的寄存器名可能不同, 这样写能增加可移植性

也可以手动指定寄存器, 如:

  • out(<reg>) <expr>, 从某个通用寄存器中读取一个值, 并将该值赋给 Rust 变量, 需要注意, 此时该寄存器不能用于输入或其他用途
  • lateout(<reg>) <expr> 和 out 类似, 但允许在汇编中先用作 in 或其他用途, 最后输出到 Rust 左值
  • inout(<reg>) <expr>, 表明这个 Rust 左值既能传递到某个寄存器中, 也能从寄存器中读取
  • inout(<reg>) <in expr> => <out expr>, 用 in expr 传值给寄存器, 用 out expr 从寄存器中读取结果
  • inlateout(<reg>) <expr> / inlateout(<reg>) <in expr> => <out expr>, 允许绑定的寄存器在其他地方提前使用
  • sym <path>, 让汇编代码可以直接引用 Rust 中的某个函数或静态变量
  • const <expr>, 传入一个编译期常量, 不占用寄存器

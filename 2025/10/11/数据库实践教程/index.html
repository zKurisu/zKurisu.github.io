记录数据库开发一些概念和技巧. 教材推荐: “数据库系统实现”, “数据库系统概念”.

概述

数据库的接口是:

  • 数据定义语言 DDL
  • 数据查询语言 SQL
  • 数据操作语言 DML

数据库具体需要实现:

  • DDL, DML, SQL
  • 持久化能力
  • 故障恢复能力
  • 并发控制与事物管理

早期的数据库管理系统通常是层次模型或网状模型, 缺点是没有高级查询语言且操作复杂.

当前主流的是关系数据库系统, 并提供 SQL.

常见的非关系数据库是 NoSQL 数据库:

  • 通过 API 访问
  • 不需要预定义模式
  • 分布式支持
  • 对象存储

数据库简史:

NewSQL

SQL 引擎, 事务引擎, 存储引擎.

数据库架构:

关系数据库 = 存储 + 事务 + SQL:

  • 存储: 关系数据库的数据结构
  • 事务: 关系数据库的算法
  • SQL : 关系数据库操作的描述语言 (一层接口)

工业界实现存储通常有两种数据结构:

  • 哈希表, $O(1)$, 支持 put/get, 不支持范围 scan
  • B+ 树, $O(logn)$, 支持 put/get, 支持范围 scan

事务中最重要的是 ACID:

  • 原子性 (Atomic), 事务操作要么全部成功, 要么全部失败
  • 一致性 (Consistent), 一个事务只能使数据库从一个一致的状态跳转到另一个一致的状态
  • 隔离性 (Isolation), 多个并发事务互相不影响, 就像多个事务串行执行一样
  • 持久性 (Durability), 一旦事务成功提交, 其对数据库的影响是永久的

MiniOB 介绍

线程模型:

  • 如果一个线程处理一个请求: 通常线程在等待 client 发出指令时会有空闲, 因此会导致资源浪费
  • 线程池可以让空闲的线程继续处理下一个请求

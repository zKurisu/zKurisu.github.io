获取文件绝对路径

用 vim.fs.abspath({path}) 接口:

  • 会将 ~ 展开为 HOME 环境变量的值 (家目录)
  • 在 path 的基础上加上 pwd 的值
  • 不处理 . 和 ..
  • 不会判断文件是否存在
  • 不会判断是否是链接文件

标准化一个文件名

用 vim.fs.normalize({path}, {opts}) 接口, 将文件 path 展开为标准格式:

  • Path 开头的 ~ 会替换为 user’s home directory
  • 将环境变量替换为对应值
  • 会解析 . 和 ..

遍历一个目录

用 vim.fs.dir({path}, {opts}) 接口, 返回一个基于 path 的迭代器:

  • {path} 是一个目录的 absolute 或 relative 地址
  • {opts} 可以指定下面参数:
    • {depth?} (integer, default: 1), 要遍历的深度
    • {skip?}, 一个匿名函数, 声明为: fun(dir_name: string): boolean, 返回 false 时停止遍历
    • {follow?} (boolean, default: false), 是否跟随链接文件

迭代器 Iterator 每次 yields 两个值 name 和 type:

  • name 是 item 的 basename
  • type 是文件类型 file, directory, link, fifo, socket, char, block, unknown 中的一种

查找文件/目录

用 vim.fs.find({names}, {opts}) 接口, 从 {path} ({opts} 中指定) 开始查找列举在 {names} 中的文件, 当 {upward} 设置为 true 时会查找父目录; 否则向下查找. 会递归查找, 可以通过 {stop} 来提前结束, 当找到的个数到 {limit} (default 1) 也会结束. 可以设置 {type} 指定查找类型.

示例:

  • {names} 是 string|string[]|fun(name: string, path: string): boolean, 只能是 basename, 不支持绝对和相对路径. 如果 name 是一个 function, 则会在遍历时对每个文件调用, 匹配时返回 true 即可, 这里的 {name} 是 basename, {path} 是 fullpath
  • {opts} 是包含参数的 table:
    • {path}, 指定开始查找的路径, 默认是当前目录
    • …

遍历所有父目录

用 vim.fs.parents({start}) 接口, 比如 {start} 是 /tmp/test/hello/world, 那么其返回的迭代器会包括:

  • /tmp/test/hello/world
  • /tmp/test/hello
  • /tmp/test
  • /tmp
  • /

删除文件/目录

用 vim.fs.rm({path}, {opts}) 接口:

  • {path} 是文件/目录的路径
  • {opts} table 中的参数有:
    • {recursize} (boolean), 是否递归删除
    • {force} (boolean), 是否强制删除

找到包含某文件/目录的 parent directory

用 vim.fs.root({source}, {marker}) 接口, 如:

  • {source} (integer|string), 若为数字, 表示 buffer 编号, 若为字符串, 则表示开始的文件路径
  • {marker}, 表示标记的文件/目录

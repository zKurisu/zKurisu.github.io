Prometheus-基本使用

参考 B 站
Prometheus 官网
Prometheus 官方文档

介绍

Prometheus 是一个开源的系统监控和告警工具.

主要组件

  • Prometheus Server, 负责抓取数据和存储时序数据
  • Client library, 用于检测程序代码的客户端库
  • Push gateway, 一个汇总节点, 处理批量, 短期的监控数据
  • Exporter, 收集信息并向 Prometheus 提供监控数据
  • Alertmanager, 用于告警通知

相关概念

总的与 Prometheus 相关的术语表

Metrics

Metrics, 指标, 指一组用来描述系统状态的量化数据.

在 Prometheus 中, Metrics 以 “时间序列数据” 的格式存储, 即随着时间的推移, 记录一系列数据点.

其由以下几部分组成:

  • Metric Name, 指标名称. 描述该指标的含义, 如 cpu_usage, 表示 CPU 使用率
  • Label, 标签, 键值对, 指标之下的细分, 用于筛选, 如 cpu_usage{host="server1", core="0"}, 表示 server1 的第 0 个核心的 CPU 使用率
  • Value, 浮点数值 (一个 label 如果没有对应的 value, 可以视为不存在)
  • Timestamp, 时间戳, 记录指标值的时间点

Metric Types

Prometheus client library 提供了 4 种 metric types:

  • Counter, 计数器, 表示一个累积值, 只能单调递增或者清零. 通常用于记录事件发生的次数
  • Gauge, 仪表盘, 可以表示任意变化的数值. 通常表示测量值
  • Histogram, 直方图, 可以导出多组 time series, 表示数值的分布情况
  • Summary, 摘要, 与 Histogram 类似, 但直接表示概率分布

但 Prometheus server 并没有利用类型信息, 统一视为 time series.

Push gateway

Push Gateway 是一个中间组件, 用于接收来自短期任务或批处理作业的指标数据, 并临时存储这些数据, 供 Prometheus 拉取.

Exporter

Exporter 是将第三方系统的监控数据转换为 Prometheus 格式的工具. 它通常以独立的进程运行, 暴露一个 HTTP 接口, Prometheus 可以通过该接口拉取 (pull) 指标数据.

Rules

Prometheus 利用 rules 来创建新的 time series 然后 generate alerts.

Jobs 和 instances

在 Prometheus 中, 一个用来 scrape 的 endpoint 则称为一个 instance, 通常也指代一个 single process.

一组有相同 purpose 的 instance, 可以相互作为备份, 以便实现 scalability 和 reliability 的称一个 job.

比如, 一个 API server job 以及四个 replicated instances 的示例为:

  • job: api-server
    • instance 1: 1.2.3.4:5670
    • instance 2: 1.2.3.4:5671
    • instance 3: 5.6.7.8:5672
    • instance 4: 5.6.7.8:5673

分位数和滑动窗口

Quantile (分位数) 是统计学中的概念, 用于描述数据分布的 “位置” (一个百分数). 例如:

  • P50, 表示 50% 的数据小于等于该值
  • P90, 表示 90% 的数据小于等于该值
  • P99, 表示 99% 的数据小于等于该值

而 Sliding window (滑动窗口), 指一种时间窗口, 其随着时间推移而滑动, 只保留一段时间内的数据, 比如:

  • 如果滑动窗口的大小为 5 分钟, 那么它始终只保留过去 5 分钟内的数据
  • 当新的数据进入窗口时, 旧的数据会被移除

因此 “滑动窗口内的分位数” 就是描述一个值在窗口内的位置信息.

Rules

Prometheus 支持两种 rules 的定义:

  • Recording rules, 预先计算复杂的 PromQL 查询结果并存储为新的 time series
  • Alerting rules, 用于定义触发告警的条件, 当 Prometheus 检测到某个 PromQL 表达式的值满足特定条件时, 会触发告警并将其发送到 Alertmanager, 由 Alertmanager 负责处理和分发告警

PromQL

PromQL, Prometheus Query Language, 用于从 TSDB 中筛选 time series 数据.

  • instance query, 指查询某个时间点的数据
  • range query, 指以一个 time step 来查询一个时间段内的数据 (等价于多次执行 instance query)

在 Prometheus UI 中:

  • Table tab 用于 instance queries
  • Graph tab 用于 range queries

对于其他程序而言, 可以用 HTTP API 传递 PromQL 来查询.

Expression evaluation types

PromQL 的 expression 返回的内容可以分为 4 类:

  • Instance vector, 一组共享相同时间戳 (瞬时数据) time series, 每一个 time series 包含一个样本, 比如用 http_requests_total 来查询
  • Range vector, 一组 time series, 每个 time series 包含一段时间范围内的多个样本, 比如用 http_requests_total[5m] 来查询
  • Scalar, 一个浮点数值, 没有时间序列或时间戳
  • String, 一个字符串值, 但目前还没在 PromQL 中使用

Operators

PromQL 支持基本的运算操作符:

  • +, -, *, /, %, ^

需要考虑不同类型操作数之间二元运算的结果:

  • Between two scalars, 结果仍然是 scalar
  • Between an instant vector and a scalar, 输出一个 vector, 但 metric name is dropped
  • Between two instance vectors, 输出一个 vector, 但只有标签匹配的会出现在结果中, 同样 drop the metric name

其支持的比较运算符有:

  • ==, !=, >, <, >=, <=

同样考虑不同类型操作数之间二元运算的结果:

  • Between two scalars, 需要提供 bool 修饰符, 结果为另一个 Scalar, 值为 0 或 1

  • Between an instant vector and a scalar, instant vector 中的每个样本数据都会应用比较:

    • 默认情况下, 如果结果为 true, 则保留该时间序列, 如果结果为 false, 则丢弃该时间序列
    • 如果提供 bool 修饰符, 结果为 true 时时间序列值为 1, 结果为 false 则时间序列值为 0, 此时会丢弃 metric name

  • Between two instance vectors

    • 默认情况下, 比较运算符会应用于匹配的时间序列, 如果比较结果为 true, 则保留该时间序列, 反之
    • 如果提供 bool 修饰符, 如果比较结果为 true, 则时间序列值为 1, 如果比较结果为 false 或没有匹配项, 则值为 0, 此时也会丢弃 metric name

逻辑运算符

支持:

  • and (intersection)
  • or (union)
  • unless (complement)

示例:

  • vector1 and vector2, 结果 vector 的 metric name 和 values 都来自左边的 vector (即 vector1). vector1 中与 vector2 中 element 标签匹配的返回在结果中
  • vector1 or vector2, 返回全部 vector1 的元素以及 vector2 中不与 vector1 中元素匹配标签的元素
  • vector1 unless vector2, 返回 vector1 中, 不与 vector2 匹配标签的元素

聚合操作

有:

  • sum, (calculate sum over dimensions)
  • min, (select minimum over dimensions)
  • max, (select maximum over dimensions)
  • avg, (calculate the average over dimensions)
  • group, (all values in the resulting vector are 1)
  • stddev, (calculate population standard deviation over dimensions)
  • stdvar, (calculate population standard variance over dimensions)
  • count, (count number of elements in the vector)
  • count_values, (count number of elements with the same value)
  • bottomk, (smallest k elements by sample value)
  • topk, (largest k elements by sample value)
  • quantile, (calculate $\phi$-quantile ($0 \le \phi \le 1$) over dimensions)
  • limitk, (sample n elements)
  • limit_ratio, (sample elements with approximately 𝑟 ratio if 𝑟 > 0, and the complement of such samples $r = -(1.0 - r)$)

Histogram

指 Prometheus 中的 Histogram Metric Type, 用于记录值的分布情况.

一个核心概念是 Bucket (桶), 其用于将连续的数值范围划分为多个区间, 统计每个区间内的样本数量.

比如定义一组 buckets 为 [0.1, 0.5, 10., 2.0, 5.0], 其分别统计的范围为:

  • (0, 0.1]
  • (0, 0.5]
  • (0, 1.0]
  • (0, 2.0]
  • (0, 5.0]
  • (0, +inf), 自动统计超过的值

在创建一个 Histogram 后, Prometheus 会暴露以下指标:

  • <metric_name>_bucket{le="<upper_bound>"}, 每个桶的样本数量
  • <metric_name>_count, 总的样本数量
  • <metric_name>_sum, 所有样本值的总和

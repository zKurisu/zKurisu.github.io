Prometheus-基本使用

参考 B 站
Prometheus 官网
Prometheus 官方文档

介绍

Prometheus 是一个开源的系统监控和告警工具.

主要组件

  • Prometheus Server, 负责抓取数据和存储时序数据
  • Client library, 用于检测程序代码的客户端库
  • Push gateway, 一个汇总节点, 处理批量, 短期的监控数据
  • Exporter, 收集信息并向 Prometheus 提供监控数据
  • Alertmanager, 用于告警通知

相关概念

总的与 Prometheus 相关的术语表

Metrics

Metrics, 指标, 指一组用来描述系统状态的量化数据.

在 Prometheus 中, Metrics 以 “时间序列数据” 的格式存储, 即随着时间的推移, 记录一系列数据点.

其由以下几部分组成:

  • Metric Name, 指标名称. 描述该指标的含义, 如 cpu_usage, 表示 CPU 使用率
  • Label, 标签, 键值对, 指标之下的细分, 用于筛选, 如 cpu_usage{host="server1", core="0"}, 表示 server1 的第 0 个核心的 CPU 使用率
  • Value, 浮点数值 (一个 label 如果没有对应的 value, 可以视为不存在)
  • Timestamp, 时间戳, 记录指标值的时间点

Metric Types

Prometheus client library 提供了 4 种 metric types:

  • Counter, 计数器, 表示一个累积值, 只能单调递增或者清零. 通常用于记录事件发生的次数
  • Gauge, 仪表盘, 可以表示任意变化的数值. 通常表示测量值
  • Histogram, 直方图, 可以导出多组 time series, 表示数值的分布情况
  • Summary, 摘要, 与 Histogram 类似, 但直接表示概率分布

但 Prometheus server 并没有利用类型信息, 统一视为 time series.

Push gateway

Push Gateway 是一个中间组件, 用于接收来自短期任务或批处理作业的指标数据, 并临时存储这些数据, 供 Prometheus 拉取.

Exporter

Exporter 是将第三方系统的监控数据转换为 Prometheus 格式的工具. 它通常以独立的进程运行, 暴露一个 HTTP 接口, Prometheus 可以通过该接口拉取 (pull) 指标数据.

Rules

Prometheus 利用 rules 来创建新的 time series 然后 generate alerts.

Jobs 和 instances

在 Prometheus 中, 一个用来 scrape 的 endpoint 则称为一个 instance, 通常也指代一个 single process.

一组有相同 purpose 的 instance, 可以相互作为备份, 以便实现 scalability 和 reliability 的称一个 job.

比如, 一个 API server job 以及四个 replicated instances 的示例为:

  • job: api-server
    • instance 1: 1.2.3.4:5670
    • instance 2: 1.2.3.4:5671
    • instance 3: 5.6.7.8:5672
    • instance 4: 5.6.7.8:5673

配置

官方详细的配置文档

Prometheus 用 YAML 来配置.

精简后的内容如下:

主要包含 3 个 blocks:

  • global, 包含 prometheus server 的全局配置
    • scrape_interval, 指定 prometheus 从 targets 处获取数据的频率
    • evaluation_interval, 指定 prometheus 检查 rules 的频率
  • rule_files, 指定存储 rules 的文件路径
  • scrape_configs, 控制 prometheus 所监控的数据 (包括 prometheus server 自己产生的)
    • prometheus 是默认的任务, 拉取 prometheus server 自己的状态数据
    • 虽然在 targets 中指定 http://localhost:9090, 但其默认从 /metrics 路径拉取, 即 http://localhost:9090/metrics

Prometheus-基本使用

介绍

Prometheus 是一个开源的系统监控和告警工具.

主要组件

  • Prometheus Server, 负责抓取数据和存储时序数据
  • Client library, 用于检测程序代码的客户端库
  • Push gateway, 一个汇总节点, 处理批量, 短期的监控数据
  • Exporter, 收集信息并向 Prometheus 提供监控数据
  • Alertmanager, 用于告警通知

相关概念

Metrics

Metrics, 指标, 指一组用来描述系统状态的量化数据.

在 Prometheus 中, Metrics 以 “时间序列数据” 的格式存储, 即随着时间的推移, 记录一系列数据点.

其由以下几部分组成:

  • Metric Name, 指标名称. 描述该指标的含义, 如 cpu_usage, 表示 CPU 使用率
  • Label, 标签, 键值对, 指标之下的细分, 用于筛选, 如 cpu_usage(host="server1", core="0"), 表示 server1 的第 0 个核心的 CPU 使用率
  • Value, 浮点数值,
  • Timestamp, 时间戳, 记录指标值的时间点

Push gateway

Push Gateway 是一个中间组件, 用于接收来自短期任务或批处理作业的指标数据, 并临时存储这些数据, 供 Prometheus 拉取.

Exporter

Exporter 是将第三方系统的监控数据转换为 Prometheus 格式的工具. 它通常以独立的进程运行, 暴露一个 HTTP 接口, Prometheus 可以通过该接口拉取 (pull) 指标数据.

Rules

Prometheus 利用 rules 来创建新的 time series 然后 generate alerts.

  • global, 包含 prometheus server 的全局配置
    • scrape_interval, 指定 prometheus 从 targets 处获取数据的频率
    • evaluation_interval, 指定 prometheus 检查 rules 的频率
  • rule_files, 指定存储 rules 的文件路径
  • scrape_configs, 控制 prometheus 所监控的数据

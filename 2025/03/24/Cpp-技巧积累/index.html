指针和引用的区别

指针 (pointer) 和引用 (reference) 都能用于访问变量.

对于指针:

  • 一个变量, 存储另一个变量在内存中的地址
  • 可以指向任何类型的变量, 包括 nullptr
  • 可以在任何时候重新赋值, 指向不同变量
  • 是一个独立的变量，占用内存

对于引用:

  • 一个变量的别名, 必须在初始化时绑定到一个变量, 且不能重新绑定到其他变量
  • 引用不能为 nullptr, 必须始终指向一个有效的变量
  • 引用本身不占用额外的内存
  • 引用不能指向动态分配的内存, 必须绑定到已存在的变量

在文件输入或重定向输入中, EOF 在文件末尾会自动触发.

Iterator

Iterator, 迭代器, 是一种对象, 用于遍历容器. 类似于指针, 可以访问和修改容器元素.

vector 的迭代器

对于 std::vector 的迭代器, 支持的操作有:

  • *, 解引用, 访问迭代器指向的元素
  • ++, 自增, 移动到下一个元素
  • --, 移动到上一个元素
  • +, -, 指针风格运算
  • ==, !=, <, > 等比较两个迭代器的位置

获取迭代器的方法有:

  • begin(), 返回指向第一个元素的迭代器
  • end(), 返回指向最后一个元素的迭代器
  • cbegin(), 返回指向第一个元素的常量迭代器 (不能修改元素)
  • cend(), 返回指向最后一个元素的常量迭代器 (不能修改元素)

Lambda 表达式

语法:

  • Capture list 指 Lambda 表达式可以访问的外部变量, 指定方式为:
  • []: 不捕获任何外部变量
  • [=]: 以值的方式捕获所有外部变量
  • [&]: 以引用的方式捕获所有外部变量
  • [var]: 以值的方式捕获指定变量 var

  • [&, var]: 以引用的方式捕获所有外部变量，但以值的方式捕获 var

std::nth_element, 对容器中的元素进行部分排序, 确保排序后的容器中的第 n 个元素 (从 0 开始) 放置在其正确的位置, 同时保证:

  • 第 n 个元素之前的元素都小于或等于它 (在这个范围内)
  • 第 n 个元素之后的元素都大于或等于它

  • first 和 last 表示要处理的范围
  • nth 指第 n 个元素的迭代器
  • comp 是自定义比较函数

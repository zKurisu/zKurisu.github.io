指针和引用的区别

指针 (pointer) 和引用 (reference) 都能用于访问变量.

对于指针:

  • 一个变量, 存储另一个变量在内存中的地址
  • 可以指向任何类型的变量, 包括 nullptr
  • 可以在任何时候重新赋值, 指向不同变量
  • 是一个独立的变量，占用内存

对于引用:

  • 一个变量的别名, 必须在初始化时绑定到一个变量, 且不能重新绑定到其他变量
  • 引用不能为 nullptr, 必须始终指向一个有效的变量
  • 引用本身不占用额外的内存
  • 引用不能指向动态分配的内存, 必须绑定到已存在的变量

Virtual functions

Virtual functions 是实现 runtime polymorphism (运行时多态) 的关键机制, 允许子类重写基类的成员函数, 并通过基类指针或引用调用正确的函数版本.

Virtual function 指在基类中用 virtual 关键字声明的成员函数, 可以确保通过基类指针/引用调用函数时, 实际执行的是对象所属子类的版本 (动态绑定):

每个对象内部有一个隐藏的 pointer, 指向所属类的 vtable, 当调用 virtual function 时, 程序通过 vptr 查找 vtable, 再跳转到正确的函数, 如:

  1. ptr 指向 Derived 对象。

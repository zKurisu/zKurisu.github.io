介绍

Flex, Fast Lexical Analyzer Generator, 是 Lex 的增强版本, 用于生成词法分析器 (Lexer). 可以通过定义正则表达式和动作 (匹配时执行一定操作), 然后编译 flex 程序以生成一个 C 程序, 其可以识别输入流中的 Tokens.

正则

在 Flex 文件中, 可用的正则表达式有:

  • x, 单个字符 “x”
  • "x" an “x”, even if x is an operator.
  • \x an “x”, even if x is an operator.
  • [xy] the character x or y.
  • [x-z] the characters x, y or z.
  • [^x] any character but x.
  • . any character but newline.
  • ^x an x at the beginning of a line.
  • <y>x an x when Flex is in start condition y.
  • x$ an x at the end of a line.
  • x? an optional x.
  • x* 0,1,2, … instances of x.
  • x+ 1,2,3, … instances of x.
  • x|y an x or a y.
  • (x) an x.
  • x/y an x but only if followed by y.
  • {xx} the translation of xx from the definitions section.
  • x{m,n} m through n occurrences of x

需要注意的只有几个:

  • "x", 相当于 \Q...\E, 可以将元字符当作匹配字符
  • <y>x, 相当于 (?<=...), 只不过前面只能是 start state
  • x/y, 相当于 (?=...), 向后看
  • {xx} 相当于变量插值
  • \\ 表示一个反斜线字符, \\. 表示任意转义字符
  • 没有捕获

Flex 变量

常用的 Flex 变量有:

变量 说明
yytext 当前匹配的字符串
yyleng 匹配字符串的长度
yyin 输入文件指针（默认 stdin）
yyout 输出文件指针（默认 stdout）
yylineno 当前行号（需启用 %option yylineno）

Flex 内置函数

常用的 Flex 内置函数有:

函数 说明
yylex() 启动词法分析器
yymore() 保留当前匹配的文本, 下次匹配追加到 yytext
yyless(n) 回退部分匹配的字符, 保留前 n 个字符
unput(c) 将字符 c 放回输入流
input() 读取下一个字符
yyrestart(FILE*) 重置输入源

yylex()

yylex() 函数会读取输入流, 并逐个扫描字符, 尝试匹配用户定义的词法规则, 匹配成功后会执行 action 操作, action 的返回值就是 yylex() 的返回值.

  • 在规则的 action 中, 明确了 return 才会继续 yylex 之后的 main 逻辑; 如果没有返回, 则继续位于 yylex 的捕获状态中
  • yylex() 本身就是一个循环, 内部实现为: “循环读取输入并持续匹配”, 在 action 中遇到一次 return 则会返回

yywrap

yywrap 函数会在 Flex 认为输入结束时自动调用. 其用于控制, 在 Flex 到达输入末尾时, 是否 “换一个输入源继续读取”, 还是 “停止词法分析” (直接 return).

默认情况下, Lex/Flex 从标准输入 (stdin) 读取. 也可以用 yyin 指定一个 FILE * 流.

当 Lex/Flex 读取到当前输入流的末尾 (EOF) 时, 其不会立即退出, 而是会先调用 yywrap 函数. 通常情况下:

  • yywrap 返回 1 (或非 0), 表示没有设置其他输入源了, 词法分析结束
  • yywrap 返回 0, 表示在 yywrap 中设置了新的输入源, 继续读取

状态声明

默认情况下, flex 会让输入字符尝试所有正则, 但有时候, 只想让一些正则在指定情况下再进行匹配, 此时可以通过定义一系列 State 来实现.

每个状态代表一种词法分析上下文, 可以让规则在指定状态下生效.

状态声明语法:

  • %s, 排他状态, 同一时间只能激活一个状态, 比如 INITIAL
  • %x, 包含状态, 可与其他状态叠加, 比如同时处于 STRING 和 ESCAPE 状态

常见状态

在大多数编程语言中, 都需要处理这几个状态:

  • INITIAL, 默认的状态, 不需要手动定义
  • COMMENTS, 多行注释
  • INLINE_COMMENTS, 行内注释
  • STRING, 字符串

一般的处理顺序也是先注释和字符串, 然后是其他关键字和字符.

回到初始状态

可以用: BEGIN 0; 回到 INITIAL 状态.

为什么不统一定义 KEYWORD

因为每一个关键字的语义不同, 用更细的 token class, 可以便于后续的语义分析.

在实际语言的设计中, 如 C 语言, if, while 等等关键字都有独立 token 类型.

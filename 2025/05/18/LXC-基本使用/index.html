介绍

LXC, LinuX Container, 是一种轻量级的操作系统级虚拟化技术.

LXC 与 Docker 有一些区别:

特性 LXC Docker
虚拟化级别 操作系统级虚拟化（共享内核） 应用级虚拟化（镜像封装）
用途 运行完整 Linux 系统 运行单个应用或微服务
镜像管理 基于模板（如 ubuntu） 基于分层镜像（Dockerfile）
依赖 直接依赖内核功能 依赖 Docker 引擎

相关概念

特权容器和非特权容器

特权容器, privileged container, 指:

  • 容器内的 root 用户与宿主机的 root 是同一个用户, 拥有几乎相同的权限
  • 没有 UID/GID 映射, 容器进程在宿主机上以相同的 UID 运行 (容器内以 UID 0 运行的进程, 在宿主机上查看也是 UID 0 运行的)
  • 拥有全部内核能力, 如挂载文件系统, 修改网络配置等
  • 隔离性低, 安全性弱

非特权容器, unprivileged container, 指:

  • 容器内的 root (UID 0) 被映射到宿主机的一个非特权用户 (如 UID 10000+), 使其权限受限
  • 无法直接操作宿主机资源
  • 依靠 subuid/subgid 来配置用户名称空间映射

/etc/subuid 和 /etc/subgid 配置

/etc/subuid (Subordinate User ID, 从属用户 ID) 和 /etc/subgid (Subordinate Group ID, 从属组 ID) 是用户命名空间的核心配置文件. 可以用来实现非特权容器的 UID/GID 映射.

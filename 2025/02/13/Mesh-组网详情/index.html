Mesh-组网详情

参考 B 站

介绍

Mesh 是一种无线拓扑结构, 由多个节点 (如路由器或接入点) 组成, 这些节点相互连接, 形成一个覆盖范围广, 稳定性高的网络. 每个节点都可以与其他节点通信, 数据可以通过多条路径传输, 从而提高网络的可靠性和覆盖范围.

Mesh 网络的帧格式, 路由协议 (HWMP) 和安全机制都定义在 IEEE 802.112 (数据链路层标准).

MAC 层的作用

Mesh 网络中的每个节点 (AP) 通过 MAC 地址唯一标识, 节点间通过 MAC 层协议 (如 IEEE 802.11s 的 HWMP 协议) 动态发现邻居并维护链路状态.

数据帧的转发依赖 MAC 地址. 例如, HWMP 协议通过 MAC 地址构建路径, 选择最优链路进行跳转, 无需依赖IP地址.

802.11 Network Design

802.11s 协议定义了 Mesh 网络的 MAC 层标准, 如:

  • HWMP, Hybrid Wireless Mesh Protocol, 是默认的路由协议, 用于路径选择 (可替换为 B.A.T.M.A.N.Advanced, OLSR 等)
  • Mesh Peering Management, 自动管理节点间的邻居关系

在 802.11 network 中, 最基础的网络称 Basic service set (BSS), 可以仅由两个 station 组成, 例如一个 STA 和另一个 STA 直接通信. 此时也可称 Independent BSS, 两个 STA 工作在 Ad-hoc 模式下.

若 BSS 中存在 AP (Access Point), 提供 integration service (将无线网络与有线网络连接起来), 此时称 Infrastructure BSS.

若多个 BSS 通过 DS (Distribution System) 连接在一起, 形成一个逻辑上的单一网络, 则称 ESS (Extended Service Set). 其目的是扩展无线网络的覆盖范围, 支持 STA 在不同 BSS 之间无缝漫游.

DSM (Distribution System Medium) 是 DS 的物理或逻辑传输介质 (有线或无线都有可能), 负责在不同 AP 之间传输数据帧.

Portal 指 DS 与非 802.11 网络 (如以太网, 互联网) 之间的逻辑接口, 负责协议转换和网络集成. (也是将无线网络的 802.11 数据帧转换为有线网络的 802.3 帧, 实现无线网络与外部网络的互连)

802.11s 同样引出了 mesh BSS (MBSS) 的概念, 指由多个支持 Mesh 功能的无线节点 (Mesh 节点) 组成的自组织网络单元. 节点通过 Mesh Peering Management (MPM) 协议自动发现邻居并建立对等连接. 整个 MBSS 内所有设备都共享同一广播域, 类似传统以太网的广播机制.

802.11s 网络对外表现为一个单一的以太网段, 外部设备 (如路由器, 交换机) 无需感知内部 Mesh 拓扑, 直接将其视为传统以太网络.

Mesh 网络对上层协议 (如 IP, ARP) 也是完全透明的, 无需修改配置就能处理帧的路由和转发.

Mesh STA 之间直接交互, 但不直接与非 Mesh STA (传统 Wi-Fi 客户端) 通信. 通过让部分 Mesh STA 兼任网关, 与非 Mesh 网络跨网通信.

802.11 frame format

  • Frame Control, 2 bits, 包含协议版本, 帧类型 (控制/管理/数据), 标志位 (To DS/From DS, 重传等), 子类型 (如 ACK, Beacon 等)
  • Duration/ID, 2 bits, 在数据帧中表示 NAV, 用于虚拟载波监听, 预留信道时间
  • Sequence Control, 2 bits, 12 位序列号 (用于去重) 和 4 位分片号 (用于分片重组)
  • QoS Control, 2 bits, 含流量优先级 (TID), ACK 策略, 流量调度参数等
  • HT Control, 4 bits, 支持高吞吐量 (HT) 和极高吞吐量 (VHT) 的物理层参数控制
  • Body, 0~7955 bits, 含 payload (如 IP 包) 或管理帧信息
  • FCS, 4 bits, 校验序列

3 种帧类型有一定区别:

  • Data frame, 包含 higher-layer data (封装到下层就成了 payload 吧)
  • Control frame, 用于 acknowledgments 和 reservations
  • Management frame, 用于 maintain a WLAN and the local link

为了实现 multihop, 802.11s 用 additional mesh control field 来扩展 data and management frames, 其中 TTL 和 sequence number fields 用于防止 frame 的 looping forever.

当 Mesh station 间通过 single hop (指不需要经过中间节点) 通信时, 其 frames 不会包含 mesh control field.

Mesh flags 字段指示 Mesh control 字段中是否有 additional MAC address 的存在. 这个 address extension 中包含额外的 3 个地址字段 (此时 802.11 帧中共 6 个地址), 其可以是非 Mesh 网络的地址, 可以用于 Mesh 站点作为代理的情况.

Mesh Formation and Management

Mesh station’s beacon 会携带 mesh 信息, 便于其他节点检测和加入 mesh.

在 Mesh-specific 场景下, 不管是 passive scanning 中的 beacons, 还是 active scanning 中的 probe frame, 这两种载荷中, 都会包含

  • Mesh ID (the name of a mesh)
  • Configuration element (指示Mesh 网络能够提供的服务类型以及支持的参数)
  • Parameters (Mesh station 支持的传输参数, 如频率, 速率等)

当一个 mesh station 找到一个 peer, 则会用 Mesh Peer Link Management protocol 来建立 peer link. 当链路断开时, 也会维护该链路信息, 以便快速重连.

若一个 mesh station 有多个 transceiver, 每一个工作在不同频段, 而其内部通过 802 bridge 连接在一起, 可以传递 frames, 此时可以将多个 mesh 视作一个大 mesh 网络:

在某些情况下, 由于监管部门 (Regulatory bodies) 的要求, mesh network 需要变更所在的 channel (或频率), 为了不让 mesh network splitting (即不同的 station 选择变更到不同的 channel), 选用一种基于 31-bit random channel precedence value 的分布式算法来变更.

比如一个 mesh station 检测到一个 radar station, 此时需要变更 frequency channel, 则会通知 mesh network 中的其他节点, 谁手中的 precedence value 更大, 则变更到谁广播出的新 frequency channel 上.

Path selection in 802.11s

在一个 mesh 网络中, 所有的 mesh station 都使用相同的 path metric 和 path selection protocol, 默认情况下是 Hybrid Wireless Mesh Protocol (HWMP). 也可以用其他协议.

Mesh Peer Link Management Protocol

Mesh Peer Link Management Protocol (PLMP), 是 802.11 中定义的用于管理和维护 wireless mesh network 中 mesh peers 之间连接的协议.

其主要负责:

  • 链路建立和认证, 当两个 mesh 节点进入通信范围时, PLMP 负责建立初始连接, 包括身份认证和协商通信参数
  • 链路状态维护, 持续检测链路质量 (如信号强度, 丢包率), 检测链路可用性 (通过心跳机制), 在链路失效时进行修复或切换
  • 链路断开管理, 当节点离开网络或链路不可恢复时, 终止连接并通知路由协议更新路径
  • 资源管理, 管理贷款分配, 冲突避免等

只要两个 MP (Mesh Point) 之间建立了 ML (Mesh Link), 就称作 peer MP.

NP-complete-Problem-基础

有几个相关概念:

  • P 问题, Polynomial Time Problems
  • NP 问题, Nondeterministic Polynomial Time Problems
  • NP 完全问题, NP-Complete Problems
  • NP 难问题, NP-Hard Problems

P 问题

P 问题指, 那些可以在多项式时间内被确定性图灵机 (即经典计算机) 解决的问题.

其较容易解决, 因为它们的计算时间随着输入规模的增加而增长得相对较慢.

比如排序问题, 最短路径问题.

多项式时间

Polynomial Time, 多项式时间, 表示算法的运行时间与输入规模之间的关系可以用一个多项式函数来表示, 用于描述算法的时间复杂度.

用数学方式表示为:
$$
\displaylines
{
\begin{aligned}
T(n) = O(n^k)
\end{aligned}
}
$$

  • 一个算法 $T(n)$ 可以表示为输入规模 $n$ 的多项式函数, 这里的 $k$ 是一个常数
  • 常见的多项式时间有:
    • $O(n)$, 线性时间
    • $O(n^2)$, 平方时间
    • $O(n^3)$, 立方时间
    • $O(n^k)$, k 次多项式时间

非多项式时间如:

  • $O(2^n)$, 指数时间
  • $O(n!)$, 阶乘时间

确定性图灵计算机

确定性图灵计算机, deterministic Turing Machine, DTM, 是计算理论中的一个基本模型, 其特点是每一步操作都是确定的, 即给定当前状态和输入符号, 下一步操作是唯一确定的.

而非确定性图灵机:

  • 每一步操作可能有多种选择
  • 给定当前状态和输入符号，下一步操作可以有多个可能性

NP 问题

NP 问题是指那些可以在多项式时间内被非确定性图灵机解决的问题, 即那些可以在多项式时间内验证一个给定解是否正确的问题.

但目前不清楚 NP 问题是否能在多项式时间内被解决.

比如旅行商问题, 布尔可满足性问题.

NP-Complete 问题

NP-Complete 问题是 NP 问题中的一个子集, 满足以下两个条件:

  • 本身是 NP 问题
  • 任何 NP 问题都可以在多项式时间内归约 (Reduction) 到该问题

也就是说 NP-Complete 问题是 NP 问题中最 “难” 的问题, 如果有一个 NP 完全问题可以在多项式时间内解决, 那么所有 NP 问题都可以在多项式时间内解决 (即 $P = NP$).

比如旅行商问题, 布尔可满足性问题, 背包问题.

NP-Hard Problems

NP-Hard 问题是指那些至少和 NP 完全问题一样难的问题, 但不一定属于 NP 类. 有下面特点:

  • NP 难问题不一定是 NP 问题 (即它们的解可能无法在多项式时间内验证)
  • 所有 NP-Complete 问题都是 NP-Hard 问题, 但 NP-Hard 问题不一定是 NP-Complete 问题

ICMPv6 协议

ICMPv6, Internet Control Message Protocol version 6, 是 ICMP 协议在 IPv6 环境中的扩展. 其功能有:

  • 错误报告, 如数据包无法到达目标地址时, 发送错误信息 (中间路由器或目标主机发回给源主机)
  • 网络诊断, 如 ping 测试, 利用 ICMPv6 Echo Request 和 Echo Reply 消息测试网络连通性
  • 邻居发现, NDP (Neighbor Discovery Protocol) 和 RDP (Router Discovery Protocol) 协议都依赖 ICMPv6
  • 地址自动配置等.

消息类型

主要分两大类.

错误报告消息

用于报告网络中的错误情况:

  • Type 1, Destination Unreachable, 目标不可达
  • Type 2, Packet Too Big, 数据包过大
  • Type 3, Time Exceeded, 超时
  • Type 4, Parameter Problem, 参数问题

信息性消息

用于网络诊断和管理:

  • Type 128: Echo Request (回显请求)
  • Type 129: Echo Reply (回显回复)
  • Type 133: Router Solicitation (路由器请求)
  • Type 134: Router Advertisement (路由器通告)
  • Type 135: Neighbor Solicitation (邻居请求)
  • Type 136: Neighbor Advertisement (邻居通告)

NDP 协议

NDP, Neighbor Discovery Protocol, 是 ICMPv6 的重要组成部分, 用于 IPv6 网络中的地址解析, 路由发现, 地址自动配置, 其取代了 IPv4 中的 ARP (Address Resolution Protocol) 以及 ICMP 路由发现功能.

其主要功能有:

  • 地址解析: 通过 Neighbor Solicitation 和 Neighbor Advertisement 消息, 将 IPv6 地址解析为链路层地址 (如 MAC 地址)
  • 路由器发现: 通过 Router Solicitation 和 Router Advertisement 消息，主机可以发现网络中的路由器并获取网络前缀
  • 地址自动配置: 主机可以通过路由器通告消息自动配置 IPv6 地址 (无状态地址自动配置)
  • 重复地址检测 (DAD): 在分配 IPv6 地址之前, 主机使用 Neighbor Solicitation 消息检查地址是否已被使用
  • 邻居不可达检测, 通过 Neighbor Unreachable Detect 发现邻居链路状态 (也属于 NS 报文, 但此时的源地址为 fe80::link-local 地址, 目标也是单播), 邻居状态有 5 中, INCOMPLETE, REACHABLE, STALE, DELAY, PROBE

地址解析

主要依靠两种报文:

  • Neighbor Solicitation, NS, 邻居请求, 将目标 IPv6 地址解析为 MAC 地址

报文格式为:

  • Type: 135
  • Code: 0
  • Checksum: ICMPv6 报文的校验和
  • Reserved: 保留字段, 必须设置为 0
  • Target Address: 需要解析的 IPv6 地址
  • Options: 可选字段, 通常用来包含源 MAC 地址

其工作流程为

  1. 当主机需要将目标 IPv6 地址解析为链路层地址时, 它会发送一个 NS 报文
  2. NS 报文的目标地址是目标 IPv6 地址的请求节点组播地址 (Solicited-Node Multicast Address)
  3. 目标主机收到 NS 报文后, 会回复一个 NA 报文

Solicited-Node Multicast Address, 其格式为:

  • FF02::1:FF 是固定前缀, 表示这是一个 Solicited-Node Multicast Address
  • XX:XXXX 是目标 IPv6 地址的最后 24 位

其对应的组播 MAC 地址格式为:

  • 33:33 是固定的前缀, 表示是一个 IPv6 组播 MAC 地址
  • XX:XX:XX:XX 是 IPv6 组播地址的最后 32 位


(此时是组播报文)

  • Neighbor Advertisement, NA, 邻居通告, 用于回复 NS 报文, 提供 MAC 地址

报文格式为:

  • Type: 136
  • Code: 0
  • Checksum: ICMPv6 报文的校验和
  • Flags: 标志字段, 包含的标志位有:
    • R, Router Flag, 表示发送者是路由器
    • S, Solicited Flag, 表示该报文是对 NS 报文的相应
    • O, Override Flag, 表示该报文应覆盖缓存中的现有条目
  • Target Address: 需要通告的目标 IPv6 地址
  • Options: 可选字段, 通常包含目标 MAC 地址

其回复流程为:

  1. 当主机收到 NS 报文时, 它会发送一个 NA 报文作为响应
  2. NA 报文的目标地址是 NS 报文的源地址
  3. 如果主机需要主动通知邻居自己的链路层地址, 它也可以发送一个未请求的 NA 报文 (Unsolicited NA)


(此时是单播报文)

其不依靠广播, 而是用组播地址.

Duplicate Address Detection, DAD, 冲突地址检测, 同样利用 NS 和 NA 报文.

此时, NS 部分报文内容为:

  • source 为 ::, 即未知地址

如果没有回复, 则说明没有冲突.

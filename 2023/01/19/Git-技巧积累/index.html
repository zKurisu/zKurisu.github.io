冲突标注信息

当遇到一个 git merge conflict 时, 会出现一些标注信息, 比如:

  • <<<<<<< HEAD 从这一行开始的部分, 是当前分支的代码
  • ||||||| 49a257a 表示冲突发生前, 共有的版本所包含的内容
  • ======= 分隔行, 分开当前分支的版本和试图合并的版本
  • >>>>>>> main, 表示到这一行之前的内容, 是正在合并的分支上的版本

stash 冲突

比如:

  • <<<<<<< Updated upstream 表示当前分支的版本
  • ======= 是分隔线
  • >>>>>>> Stashed changes 是 stash 恢复的内容, 即 stash 中的代码版本

撤销 commit

用 git revert 来生成反向提交: 新增一个提交, 只不过内容是 “反向变更” (比如之前修改了两个文件, 这次修改回去), 从而抵消原来提交的效果.

常用选项:

  • -n 或 --no-commit, 反转变更但不自动提交, 可以后续修改后在手动 commit
  • -e 或 --edit, 允许编辑撤销提交的说明信息

Git Hook 脚本

Git Hook 是 Git 的一种自动化机制, 其能够在特定 Git 操作 (如 commit, push, merge 等) 前后插入执行自定义脚本.

脚本 (可执行文件) 存储在 .git/hooks/ 目录下, 可以是任何语言编写.

Hook 的生效位置通过文件名识别.

客户端 hook (本地生效)

包括 (下面是文件名):

  • pre-commit, 输入提交信息前触发
  • prepare-commit-msg, 启动提交信息编辑器前触发
  • commit-msg, 提交信息输入完成后触发
  • post-commit, 提交完成后触发
  • pre-push, 执行 git push 前触发

服务端 hook (远程仓库生效)

包括:

  • pre-receive, 推送到达仓库时最先执行
  • update, 对每个分支的推送分别触发
  • post-receive, 推送完成后执行

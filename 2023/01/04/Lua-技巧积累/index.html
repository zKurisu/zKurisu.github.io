Lua 中的正则表达式

Lua 中的字符串模式匹配通过 string.match, string.find, string.gmatch, string.gsub 等函数提供, 是一个轻量级且具有特殊语法的匹配系统, 不是完整的正则表达式.

Lua 中用于字符串匹配的模式称 Lua Patterns, 不叫 Regex, 虽然语法有点像正则, 但是元字符更少, 更简单, 但功能有限.

常用的元字符有:

元字符 含义
. 匹配任意单个字符 (除了换行符)
%a 匹配任意字母 (A-Z, a-z)
%d 匹配任意数字 (0-9)
%l 匹配任意小写字母 (a-z)
%u 匹配任意大写字母 (A-Z)
%W 匹配任意字母或数字 (A-Z,a-z,0-9), 相当于 [A-Za-z0-9]
%s 匹配任意空白字符 (空格, 制表符, 换行符)
%p 匹配任意标点符号
%x 匹配任意十六进制数字 (0-9,a-f,A-F)
%z 匹配 Lua 中的字符零 (NULL 字符, ASCII 0)
% 转义字符, 用来转义匹配特殊字符本身, 比如 %. 能匹配点号, %% 能匹配 %
^ 匹配字符串开头的锚点
$ 匹配字符串结尾的锚点
* 匹配前一个字符/模式 0 次或者多次 (贪婪)
+ 匹配前一个字符/模式 1 次或者多次 (贪婪)
- 匹配前一个字符/模式 0 次或者多次 (非贪婪)
? 匹配前一个字符/模式 0 次或者 1 次
(...) 捕获分组, 可以提取
%1, %2... 在 string.gsub 等中的第 n 个捕获组

Lua chunk

Lua 并不直接被解释器 (lua) 解释执行, 而是先编译 (luac) 为字节码, 然后交给 Lua 虚拟机 (lua) 执行. (lua 命令包含了隐式编译和执行字节码)

Lua 中一段可以被解释器执行的代码称作 chunk. 编译为字节码后的结构称 precompiled chunk (binary chunk).

查看如下:

  • 这里包含了 main 函数的原型
  • 0+ params 表示主块不接受显式参数, 即可变参数未使用
  • 2 slots 表示虚拟机栈需要两个槽位, 用于临时变量和计算
  • 1 upvalue 表示引用了一个外部变量, 这里是 _ENV
  • 0 locals 表示 chunk 中没有局部变量
  • 2 constants 表示常量池中有两个值, 这里是 print 和 Hello Lua!
  • 0 function 表示没有子函数定义
  • VARARGPREP 指令指定可变参数的栈帧起始位置
  • GETTABUP 指令从 upvalue 中获取 print 函数, 操作数 0 0 0 表示:
    • 将结果存入栈槽 0
    • 从 upvalue[0] 中读取, 这里是 _ENV
    • 读取 _ENV 的键 print (constants 中索引 0)
  • LOADK 指令将常量池中的值加载到栈中, 操作数 1 1 表示:
    • 存入栈槽 1
    • 加载 constants 中索引 1, 即 Hello Lua!
  • CALL 指令调用函数, 操作数 0 2 1 表示:
    • 调用栈槽 0 的函数, 这里是 print
    • 函数参数数量是 2, Hello Lua! 和函数本身
    • 期望的返回值数量为 1
  • RETURN 指令结束 chunk 的执行, 操作数 0 1 1 表示:
    • 从栈槽 0 开始返回
    • 返回值数量为 1
    • Lua 5.4+ 的附加标志, 可忽略

版本问题

生成 binary chunk 时, 会嵌入当前 Lua 版本, 当加载 binary chunk 时, 会与使用的 Lua 版本进行对比, 若不匹配, 则拒绝加载.

binary chunk header

  • signature 是一段特殊字节, 用于快速识别文件格式, 十六进制表示为 0x1B4C7561, Go 语言字符串字面量是 \x1bLua

ipairs 和 pairs

  • ipairs 只能遍历索引为数字的表

  • pairs 能遍历所有内容, 包括键值对部分

取 table 长度符号 # 的缺陷

在 Lua 中, 使用 # 操作符可以获取一个表 (table) 的长度. 然而, 在给表赋值时, #my_table 并不会实时更新表的长度. 它只能计算出表中连续的整数键的数量.

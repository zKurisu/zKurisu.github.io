特殊变量

Makefile 中的特殊变量, 是由 Make 工具本身预定义的, 或者是在规则中自动根据规则内容填充的变量, 包括 3 类:

  1. 自动变量 (Automatic Variables), 引用当前规则中的目标, 依赖等
  2. 内置变量 (Built-in/Implicit Variables), Make 预定义的常用变量, 如编译器, 编译选项等
  3. 其他常用特殊变量或环境变量

自动变量

  • $@, 当前规则的目标文件名 (Target), 比如 foo.o: foo.c 中的 foo.o
  • $<, 第一个依赖文件名 (First Prerequisite), 在 foo.o: foo.c 中, 指 foo.c
  • $^, 所有的依赖文件名 (All Prerequisites), 以空格分隔, 且会去重. 比如 foo.o: a.c b.c 中的 a.c b.c
  • $+, 所有的依赖文件名, 以空格分隔, 但不去重
  • $?, 比目标新的依赖文件
  • $*, 目标文件名中通配符匹配的部分

内置变量

通常用于指定编译器, 编译选项, 链接器等, 可以直接使用或者覆盖:

变量名 含义 / 默认值 常见用途
CC C 编译器，默认通常是 cc（通常是 gcc 的软链接） 指定用什么编译 C 代码
CXX C++ 编译器，默认通常是 g++ 或类似的 指定 C++ 编译器
CFLAGS C 编译选项，比如 -Wall -O2 -g，用于编译 C 文件 添加编译选项
CXXFLAGS C++ 编译选项，类似 CFLAGS，用于 C++ C++ 特有的编译选项
CPPFLAGS C/C++ 预处理器选项，比如 -Iinclude/（头文件路径） 用于 #include 的路径等
LDFLAGS 链接器选项，比如 -L/path/to/libs 链接时的选项，非库文件路径
LDLIBS 或 LIBS 链接时要用的库文件，比如 -lm -lpthread 指定链接哪些库（如数学库）
AR 静态库打包工具，默认是 ar 用于生成 .a 静态库
ARFLAGS ar 工具的选项，比如 rv 打包静态库时的选项
MAKEFLAGS 传递给 make 的选项 控制 make 自身的行为
RM 删除命令，默认是 rm -f 通常在 clean 规则里用到

Github Docs

Github 的账号类型

有三种账号:

  • Personal accounts
  • Organization accounts
  • Enterprise accounts

Personal accounts

个人创建, 个人用于登录访问 github.

Organization accounts

共享账户, 能拥有仓库等, 但是不能直接登录到一个 Organization 中. 而是登录自己的账号, 对 Organization 仓库进行修改.

Organization 可以分配不同的 role 给 personal accounts, 来管理访问权限.
(role 是权限的集合)

Organization owners 以及 security managers 的权限最大.

Repository roles for an organization

可用的 roles 有:

  • Read, 可读
  • Triage, 可管理 issue, discussions, and pull requests 的合并请求
  • Write, 可写, 可操作代码以及分支
  • Maintain, 可以管理仓库的设置, 保护分支, 管理合并请求, 管理问题和讨论, 但不能执行敏感操作, 如删除仓库
  • Admin, 所有权限

Organization owners

拥有 organization 的全部权限.

Organization members

默认情况可以创建 repositories 和 projects.

Organization moderators

基本权限和 members 相同, 还可以 block and unblock non-member contributors, set interaction limits, and hide comments in public repositories owned by the organization.

Billing managers

处理 payment information.

Security managers

可分配给一个 team, 然后 team 之下的 members 都具有管理 settings for code security 的权限.

Enterprise accounts

在 Github Enterprise Cloud 或 Github Enterprise Server 上创建的账号.

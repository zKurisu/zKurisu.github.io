PVE 中文文档

介绍

Proxmox VE (Virtual Environment) 是基于 Debian Linux 的开源虚拟化平台, 支持 KVM (Kernal based Virtual Machine) 虚拟机和 LXC (Linux Container) 容器.

PVE 可以单机部署, 也能组建多节点集群.

基本概念

组织单元

即如何组织数据的.

Datacenter

Datacenter 是 PVE 的最高级别组织单元, 它代表了一个完整的虚拟化环境.

一个 Datacenter 可以包含多个 Cluster, 而一个 Cluster 会包含多个 Nodes.

在 PVE 创建时, 会创建一个 Datacenter, 然后创建一个默认的 Cluster 包含当前物理服务器作为唯一的 Node.

需要注意的是, 一个 Node 只能归属于一个 Datacenter (包括默认创建的本地 Datacenter), 但可以加入多个 Cluster.

Nodes

Node 代表一个具体的 PVE 物理服务器节点, 提供计算, 内存和存储资源.

每一个 Node 需要向 Datacenter 注册才能进行管理.

Cluster

Cluster 是多个 Nodes 组成的集群, 提供高可用和负载均衡等.

Storage Pool

Storage Pool 是在 Datacenter 层级下 (就是说使用对象是 Datacenter) 创建的资源组织单位, 一个 Pool 可以供多个 Datacenter 使用, 而一个 Datacenter 也可以使用多个 Pool.

Storage Pool 管理虚拟机和容器使用的存储资源, 可以添加本地存储设备, 也可以用网络存储.

网络管理

SDN

每个 PVE Datacenter 都有自己独立的 SDN 环境, 拥有自己的虚拟交换机, 路由器等网络组件, 但多个 Datacenter 间也能建立虚拟网络连接.

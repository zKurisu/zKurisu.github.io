数据结构与算法基础知识

算法的定义

算法 (Algorithm) 是对特定问题的求解方法和步骤的一种描述, 是指令的有限序列, 其中每个指令表示一个或多个操作.

对于同一个问题, 使用不同的算法, 所消耗的资源也是不同的, 这里的资源指时间和空间:

  • 时间维度, 指执行当前算法所消耗的时间, 通常用 “时间复杂度” 来描述
  • 空间维度, 是指执行当前算法需要占用多少内存空间, 通常用 “空间复杂度” 来描述

大O表示法

大O表示法是一种用于描述算法时间复杂度和空间复杂度的数学符号. 它表示的是算法在最坏情况下的性能.

如:

  • T(n), 表示算法在处理规模为 n (n 随数据结构不同而含义不同, 如图结构, n 表示顶点数或边数) 的输入时所需要的时间 (或步骤数)
  • f(n) 是一个函数, 这里描述 T(n) 的增长速度
  • O(f(n)) 表示函数 f(n) 的上界 (似乎是表示 f(n) 的正比例函数)

具体说, 当 n 趋于无穷大时, T(n) 的增长速度不会超过 f(n) 的常数倍.

为什么说描述的是一个增长速度?

因为始终考虑的是 f(n) 这一函数, 而非 n 这个值.

数学定义

要找到 f(n), 需要知道: T(n) = O(f(n)) 意味着存在常数 c 和 n0, 使得对于所有 $n \ge n_0$, 都有:
$$
\displaylines
{
\begin{aligned}
T(n) \le c \cdot f(n)
\end{aligned}
}
$$

理解

使用大O表示法, 就是找到一个函数 f(n), 用其正比例值来描述算法的复杂度.

例子1: 线性时间复杂度

假设有:
$$
\displaylines
{
\begin{aligned}
T(n) = 3n + 2
\end{aligned}
}
$$

用大O表示其复杂度:
$$
\displaylines
{
\begin{aligned}
\because 能找到常数 c 和 n_0,使得对于所有的n \ge n_0 都有 \newline~ \newline
3n + 2 \le c \cdot n \newline~ \newline
比如取 c=4: \newline~ \newline
3n+2 \le 4n, 当 n \ge 1 时成立
\end{aligned}
}
$$

因此可以表示为 O(n).

例子2: 平方复杂度

假设有:
$$
\displaylines
{
\begin{aligned}
T(n) = 2n^2 + 3n + 1
\end{aligned}
}
$$

用大O表示其复杂度:

$$
\displaylines
{
\begin{aligned}
\because 能找到常数 c 和 n_0,使得对于所有的n \ge n_0 都有 \newline~ \newline
2n^2 + 3n + 1 \le c \cdot n^2 \newline~ \newline
比如取 c=3, n_0=1: \newline~ \newline
2n^2 + 3n + 1 \le 3n^2, 当 n \ge 1 时成立
\end{aligned}
}
$$

因此可以表示为 O(n^2).

数据结构

数据结构是一种组织和存储数据的方式, 以便能够高效地访问和修改数据. 数据结构不仅仅是数据的集合, 还包括数据之间的关系以及可以对数据进行的操作.

很多非数值 (比如, 文字, 图像, 声音) 计算问题的数学模型不能用数学方程表示, 而是诸如表, 树和图之类的具有逻辑的数据, 这些就是数据结构.

线性数据结构

线性关系数据结构是指数据元素之间存在一对一的关系, 即每个元素有且仅有一个前驱和一个后继 (除第一个和最后一个元素外). 这些数据结构中的元素排列成一条直线, 如:

  • 数组, Array
  • 链表, Linked List
  • 栈, Stack
  • 队列, Queue

非线性关系数据结构

非线性关系数据结构是指数据元素之间存在多对多的关系, 即每个元素可能有多个前驱和多个后继. 这些数据结构中的元素排列成一种层次或网络结构. 如:

  • 树, Tree
  • 图, Graph

常见数据结构操作的时间复杂度

O(n) 通常指需要遍历数据集.

  • 访问 List 元素 –> O(1)
  • 查询 List 元素 –> O(n)
  • 向 List 开头或末尾插入或删除元素 –> O(1)
  • 向 List 中间插入元素 –> O(n)
  • 查询/插入/删除 Set/Dict 中的元素
  • 向 Deque (双端队列) 两端插入/删除元素 –> O(1)
  • 向 Deque (双端队列) 中间插入/删除元素 –> O(n)

基础数据结构

链表

链表, link list, 一种线性数据结构, 其中每个节点包含:

  • 数据域, 存储实际数据
  • 指针域, 指向下一个节点的地址 (单链表), 或前驱或后继节点的地址 (双向链表)

链表一般分为:

  • 单链表, 每个节点只有指向下一个节点的指针 (next)
  • 双向链表, 每个节点有指向前驱 (prev) 和后继 (next) 的指针, 支持双向遍历
  • 循环链表, 尾节点指向头节点, 形成环状结构 (可以是单循环或双循环)

链表的性质主要有:

  • 动态内存分配: 无需连续内存空间, 节点可以分散存储
  • 插入/删除高效: 时间复杂度通常为 O(1) (已知位置时)
  • 随机访问低效: 访问第 i 个元素需要从头遍历, 时间复杂度 O(n)
  • 无固定大小: 可以动态扩展或收缩 (与静态数组对比)

基本操作的时间复杂度为:

  • 访问元素 O(n) 需要从头节点开始逐个遍历
  • 头部插入 O(1) 修改头指针和新节点的 next
  • 尾部插入 O(1) 双向链表可直接操作, 单链表需遍历到尾部 (若未维护尾指针则为 O(n))
  • 已知位置插入 O(1) 修改相邻节点的指针
  • 已知位置删除 O(1) 同上
  • 搜索元素 O(n) 需要遍历链表

优先队列

优先队列, Priority Queue, 类似于普通队列, 但每个元素都有一个 priority, 优先级最高的元素总是最先被删除 (或访问).

优先队列支持的操作有:

  • Push, 将元素插入队列, 并根据优先级调整位置, O(logn)
  • Pop, 移除并返回优先级最高的元素, O(logn)
  • Top, 返回优先级最高的元素 (不删除), O(1)
    (因为其底层基于二叉堆, binary heap)

差分数组

差分数组用于应对要 “对区间进行修改” 的场景. 其核心思想是: 将对区间的操作转化为对区间端点的操作.

给定一个原始数组 A[0..n-1], 其差分数组 diff[0..n] 定义为:

  • diff[0] = A[0]
  • diff[i] = A[i] - A[i-1], $i \ge 1$
  • diff[n] 用于防止越界

如果要对原数组 A 的某个区间 [L, R] 统一加 x, 原始的解法为:

  • 修改的时间复杂度为 O(R-L+1)
  • 最坏情况为修改整个数组, L=0, R=n-1, 此时单次修改的时间为 O(n)
  • 若要连续进行 m 次修改, 则总时间复杂度为 O(mn)

若使用差分数组, 则等价于修改差分数组的两个点, 然后利用差分数组来还原:

  • 因为修改 diff[L] 和 diff[R+1] 会连锁影响到 A[L..R] 的值, 示例:
  • 修改 diff 数组的时间复杂度为 O(1), m 次修改总时间为 O(m)
  • 还原数组的时间复杂度为 O(n)
  • 总时间复杂度就是 O(m+n)

提升就在于, 对 A 数组本身的修改只进行了一次, 相当于用 diff 数组记住了每次修改的量和位置, 最后统一施加到 A 数组上. (感觉不用差分数组也行, 只要能记录修改的量和位置)

注意为什么要 diff[R+1] -= x

  • diff[L] += x 会导致从 L 开始的所有元素都增加 x
  • diff[R+1] -= x 会抵消从 R+1 开始的所有元素的增加
  • 这样的话, 只会有 [L,R] 的元素增加了 x

为什么从 0 开始恢复

因为可能操作了多个区间的内容, 从 0 开始能让所有的修改都正确传播.

循环队列

循环队列是一种特殊的线性数据结构, 只是逻辑上看作一个闭环, 还是最寻先进先出原则 (FIFO), 其特点是:

  • 固定大小: 循环队列的容量固定
  • 高效利用空间: 可以重复利用出队后空出的空间
  • 避免数据搬移: 不需要像普通队列那样在出队后移动所有元素

  • front 指针, 指向队首元素 (出队位置)
  • rear 指针, 指向队尾下一个空位 (入队位置)
  • 当指针到达数组末尾, 通过 (rear+1) % capacity 或 (front + 1) % capacity 回到起点. 比如, 容量为 5 的队列, 当 rear=4 时, 下一个位置是 (4+1) % 5 = 0

需要注意, 为了区分队列满和队列空, 循环实际的实际大小是 capacity + 1:

  • 判断空: front == rear
  • 判断满: (rear + 1) % capacity == front

一些矩阵相关的知识

Trace

矩阵 A 的 trace 定义为其主对角线元素的和, 对于方阵而言, 有:

$$
\displaylines
{
\begin{aligned}
tr(A) = \sum_{i=1}^n a_{ii}
\end{aligned}
}
$$

  • 线性性: $tr(A + B) = tr(A) + tr(B)$
  • 相似变换不变性: $tr(P^{-1}AP) = tr(A)$
  • 与特征值的关系: $tr(A) = \sum \lambda_i$

特征值描述了线性变换的缩放因子, 因此 trace 可以看作变换的总缩放量, 类似于 “留下的痕迹”.

Norm

Norm 是矩阵所有元素平方和的平方根, 即:
$$
\displaylines
{
\begin{aligned}
|| A ||F = \sqrt{\sum{i=1}^m \sum_{j=1}^n | a_{ij} |^2}
\end{aligned}
}
$$

  • 其适用于任何矩阵
  • 满足范数的三条公理:
    • 非负性: $|| A || \ge 0$
    • 齐次性: $|| cA || = |c| \cdot ||A||$
    • 三角不等式: $||A+B|| \le ||A|| + ||B||$

取模运算的基本性质

Modular Arithmetic, 取模运算定义为:
$$
\displaylines
{
\begin{aligned}
a\ mod\ m = r \newline~ \newline
0 \le r < m\ and\ a = km + r
\end{aligned}
}
$$

比如: a = 10, m = 3, 有 a mod m = 10 mod 3 = 1.

注意结果 $$ 的范围始终是 $0 \le r < m$.

加法性质

$$
\displaylines
{
\begin{aligned}
(a + b) mod m = ((a mod m) + (b mod m)) mod m
\end{aligned}
}
$$

例如: (17 + 20) mod 5 = (2 + 0) mod 5 = 2 (), 17 mod 5 = 2, 20 mod 5 = 0

减法性质

$$
\displaylines
{
\begin{aligned}
(a - b) mod m = ((a mod m) - (b mod m) + m) mod m
\end{aligned}
}
$$

  • +m 是避免负数结果

例如: (3 - 5) mod 4 = (3 - 1 + 4) mod 4 = 2

乘法性质

$$
\displaylines
{
\begin{aligned}
(a \times b) mod m = ((a mod m) \times (b mod m)) mod m
\end{aligned}
}
$$

例如: (11 x 7) mod 5 = (1 x 2) mod 5 = 2.

幂性质

$$
\displaylines
{
\begin{aligned}
a^k mod m = ((a mod m)^k) mod m
\end{aligned}
}
$$

  • 例如: $2^{10} mod\ 7 = (2 mod 7)^{10}\ mod\ 7 = 1024\ mod\ 7 = 2$
  • 如果指数过大, 可以使用快速幂算法来优化, 时间复杂度为 O(logk)

负数的模运算

(-a) mod m = (m-a mod m) mod m, 例如: -7 mod 5 = (5-2) mod 5 = 3.

除法性质

需要借助乘法逆元:
$$
\displaylines
{
\begin{aligned}
\frac{a}{b}\ mod\ m = (a \times b^{-1}) mod m
\end{aligned}
}
$$

  • 其中 $b^{-1}$ 是 $b$ 的 modular inverse, 满足 $b \times b^{-1} \equiv 1\ mod\ m$
  • 当 $b$ 和 $m$ 互质时, 可以用费马小定理来计算: $b^{-1} \equiv b^{m-2}\ mod\ m$
  • 通用计算为扩展欧几里得算法

例如: $\frac{4}{3}\ mod\ 5$:

  • 逆元 $3^{-1} = 3^{5-2}\ mod\ 5 \equiv 2$
  • 结果: $4 \times 2 \equiv 8 \equiv 3\ mod\ 5$

快速幂算法

快速幂算法是一种高效计算大整数幂 (如 $a^b$) 或模幂 (如 $a^b\ mod\ m$) 的方法, 时间复杂度为 $O(log b)$, 比朴素乘法的 $O(b)$ 快得多.

为什么需要快速幂

  • 如果计算 $a^b$ (如 $3^{100}$) 时, 需要进行 $b-1$ 次乘法, 效率低.
  • 可以利用二进制分解和分治思想, 将指数 $b$ 拆解为 $2$ 的幂次组合, 减少计算次数

原理

将指数 $b$ 表示为二进制形式, 如:
$$
\displaylines
{
\begin{aligned}
b = (b_k b_{k-1}…b_1 b_0)_2 = \sum_{i=0}^k b_i \cdot 2^i
\end{aligned}
}
$$

  • $b_i \in {0,1}$ 是二进制的每一位

利用指数法则 $a^{x+y} = a^x \cdot a^y$ 将 $a^b$ 拆解为:

$$
\displaylines
{
\begin{aligned}
a^b = a^{\sum b_i \cdot 2^i} = \prod_{i=0}^k a^{b_i \cdot 2^i}
\end{aligned}
}
$$

  • 只有当 $b_i = 1$ 的项需要纳入计算

递推计算 $a^{2^i}$:
$$
\displaylines
{
\begin{aligned}
a^1 &= a, \newline~ \newline
a^2 &= a^1 \times a^1, \newline~ \newline
a^4 &= a^2 \times a^2, \newline~ \newline
.
.
.
a^{2^i} = a^{2^{i-1}} \times a^{2^{i-1}}
\end{aligned}
}
$$

算法步骤

以计算 $a^b$ 为例:

  1. 初始化: 结果 res = 1, 当前底数 base = 1, 指数 b
  2. 循环直到 b=0:
    • 如果 b 是奇数 (即当前最右侧二进制是 1, b & 1 = 1), 将 res 乘以 base
    • 将 base 平方 (base x base)
    • 将 b 右移一位 (b = b >> 1)
  3. 返回 res

示例, 计算 $3^{13}$:

$$
\displaylines
{
\begin{aligned}
3^{13} = 3^{(1101)_2} = 3^8 \times 3^4 \times 3^1 = 6561 \times 81 \times 3 = 1594323
\end{aligned}
}
$$

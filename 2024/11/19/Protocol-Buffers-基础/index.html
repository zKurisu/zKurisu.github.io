Protocol-Buffers-基础

protobuf 官方文档

介绍

Protocol Buffers 是一种跨语言 (支持多种语言接口), 跨平台的用于序列化数据的机制 (可以看作一种语言, 也需要编译处理). 它和 JSON 类似, 但是更快更轻量, 常用于存储数据或服务器间通信.

注意已经分配的 Field Number 要避免重复分配, 不然相当于删除数据然后重新添加.

  • repeated, 可以有多个值, 保留顺序

Any 类型

Any 类型允许你在 message 中嵌入任意类型的值, 而无需在当前的 .proto 文件中事先定义这些消息. 这种灵活性使得你可以在运行时传递不同类型的数据, 而不需要修改消息定义.

使用 Any 类型需要导入 google/protobuf/any.proto, 如:

Oneof 类型来管理互斥字段

oneof 类型中的所有字段共享同一块内存. 这意味着在同一时间只能有一个字段被设置, 从而节省内存.

在 oneof 中, 设置任意一个字段会自动清除其他所有字段. 这保证了在任何时候, 只有一个字段处于 “已设置” 状态.

可以通过 case() 或 WhichOneof() 方法来检查当前哪个字段被设置.

如果试图同时设置多个字段 (虽然实际上只能设置一个), 最后设置的字段将覆盖之前的设置. 这是基于字段在 .proto 文件中的定义顺序.

Options

Options 用于提供注解和上下文以便特殊处理. 其可以是:

  • file-level, 对整个 .proto 文件有效
  • message-level, 对整个 message 定义有效
  • field-level, 对一条 field 定义有效

比如 optimize_for (file option), 可选值为:

  • SPEED
  • CODE_SIZE
  • LITE_RUNTIME

Option Retention

Option Retention, 选项保留决定了这些选项是否会在生成的代码中保持可用. 保留的选择影响生成的代码的大小和运行时的行为.

默认是运行时可用, 即会保留在编译后的文件中

可以设置为 RETENTION_SOURCE, 即选项仅存在于源代码中, 在生成的二进制文件中不会占用空间.

命令行工具使用

主要使用编译器 protoc, 语法为:

  • --proto_path, -I, 用来指定 import 的查找路径, 该参数可以使用多次
  • --*_out 指明为特定语言生成代码
  • path/to/file.proto 也可以指定多个

编译输出

Protocol buffer 编译器的输出取决于要用哪个语言来处理, 比如:

  • Python, 生成一个模块, 包含了每种消息类型的静态描述符 (descriptor)
  • Go, 生成一个 .pb.go 文件, 其中定义了一个类型, 包含 .proto 中的 message

BlockChain-基础

介绍

区块链, block chain, 是一种分布式账本技术 (数据不是存储在单一的节点上), 它允许多个参与者在没有中心化控制的情况下共同维护和更新数据. 这种技术以区块 (指数据结构的基本单元) 的形式存储数据, 每个区块包含一组交易信息 (比如交易的发送者和接受者, 以及交易金额等), 并通过加密哈希链接到前一个区块 (确保顺序性), 从而形成一个链条.

为什么需要区块链

参考 B 站讲解视频
区块链中的货币都不是实体, 依靠 “记账” 来进行交易, 当一个区块变动后, 各节点 (记录所有或部分区块信息) 间可以通过 P2P 协议进行更新 (各区块间的联系就形成了一个网络), 每一个区块相当于一个账本, 都是独立存在的.

区块间通过链连接在一起:

(每日的期末结余与第二天的期初余额相等)

记账的人通过为别人记账获取收入.

而谁来记账则需要团体达成一种共识 (即共识机制进行选举, 毕竟记账的人可以拿钱).

综上, 区块链具有几个特征:

  • 各节点通过 P2P 网络路由
  • 分布式账本, 去中心化
  • 通过共识机制选举记账人
  • 账本内容难以篡改 (毕竟分布式)

相关概念

区块

区块指 block chain 中的基本数据结构, 携带一组交易信息, 比如此次交易的发送者和接受者, 以及交易金额等.


(Block 就是把一组交易信息打包, 把这个 block 链接到以前的交易记录上, 就成了 block chain)

当前区块的哈希值是基于区块内所有数据计算得出的, 如果一个区块的内容被更改, 其哈希值也会改变, 从而使后续区块失效. (从而防止篡改吧)

Block 的组成

  • Hash Pointer, 指向前一个 block 的 Hash (当前 block 的 Hash 要用这个来算)
  • Merkle Tree, 一种二叉树 (binary tree), 每个叶子节点代表区块中的一笔交易, 而每个非叶子节点 (non-leaf node) 是其两个子节点哈希值的哈希 (即父节点的哈希是其两个子节点哈希的组合), 最顶层的称根节点 (root), 代表整棵树的 hash, 用于快速验证交易是否在某个 block 中以及确认某个交易是否被更改 (从该交易的叶子节点向上沿着树走, 计算路径上的哈希值. 通过与根节点的哈希值进行比较, 可以确认这笔交易是否有效)

用户

用户, Users, 指使用区块链技术进行交易或交互的个人或实体, 负责:

  • 发起交易: 创建和发送交易, 例如将数字货币转账给其他用户
  • 接收交易: 接收来自其他用户的交易
  • 使用应用: 在一些区块链平台 (如以太坊) 上, 与去中心化应用 (DApps) 交互

节点

节点, Nodes, 指在区块链网络中运行的计算机或设备. 每个节点都有能力存储区块链的副本, 并参与网络的维护和数据处理.

其分为三类:

  • 全节点, Full Nodes (fully validating node), 存储整个区块链的完整副本, 并参与验证交易和区块, 全节点提高了网络的安全性和去中心化程度
  • 轻节点, Light Nodes, 只存储区块链的一部分数据 (如区块头), 依赖全节点来获取交易信息, 通常用于资源有限的设备
  • 矿工节点, Miner Nodes, 专门用于挖矿的节点, 参与区块的创建和验证, 通常会竞争解决复杂的数学问题以获得奖励

用户通过与节点交互进行交易等操作.

由于每个节点都有相同的 “账本” 信息 (区块信息), 因此是去中心化的 (即分布式账本).

网络

在区块链 (blockchain) 中, 网络 (Network) 是指由多个节点 (Nodes) 组成的系统, 这些节点通过点对点 (P2P) 的协议相互连接和通信, 以共同维护和更新区块链的账本.

网络一般也分三种类型:

  • 共有链, Public BlockChain, 任何人都可以参与和访问的开放网络, 如比特币和以太坊
  • 私有链, Private BlockChain, 受限制的网络, 只有特定的参与者可以访问和操作, 通常用于企业内部应用
  • 联盟链, Consortium BlockChain, 由多个机构共同管理的网络, 参与者通常是特定的组织, 如银行联盟

挖矿

挖矿, Mining, 指工作量证明中计算复杂数学问题的过程. (因为有人称 Bitcoin 为数字黄金, 得到它的过程就是挖矿)

比特币

比特币, Bitcoin, 是一种去中心化 (没有中央管理机构, 由网络中的用户共同维护) 的数字货币, 其使用区块链技术 (一种去中心化的记账系统) 来记录所有交易, 确保交易的安全性和透明性.

比特币会在打包区块的过程中产生, 按照最初的构想 (2008 年), 每打一个包, 产生 50 个比特币, 每十分钟打一次包, 但是 4 年之后, 每打一个包则产生 25 个比特币, 以此类推, 每过 4 年, 产生的比特币都为之前的一半, 以次可以计算出能够产生的比特币总量为:
$$
\displaylines
{
\begin{aligned}
50 \times 6 \times 24 \times 365 \times 4 \times (1 + \frac{1}{2} + (\frac{1}{2})^2 + ….) \approx 21 milion
\end{aligned}
}
$$

PoW 工作量证明

PoW, Proof-of-Work, 每一个想打包 (记账) 的人都需要计算一个很难的数学题, 如果做出来了, 那就有权利进行打包, 也就是能获取手续费以及打包奖励. 而这个计算的过程则称为挖矿.

哈希算法

哈希函数, 也可称数字摘要或散列函数, 它可以把一个字符串通过一定的运算, 转化成一个摘要的形式, 而且难以进行逆运算, 比如 SHA256 (美国国家安全局发明的一种算法), 如果输入一个字符串, 则会输出一个 256 位的二进制数 (不管输入的长度如何), 而且, 字符串的细微变化都会导致生成的二进制数大不相同.

其特点可以简单归纳为:

  • 正向计算容易
  • 反向计算困难 (大部分情况只能慢慢试输入的字符串)

挖矿的具体原理

网络中的每一个人都收集了交易信息打包成块, 想要拼接到当前区块链后以获取奖励:

但在拼接之前, 需要计算出一个复杂的数学问题:

  1. 一个字符串包含当前块的头部, 账单信息, 时间戳, 随机数
  2. 对字符串做两次 SHA256 运算: Hash = sha256(sha256(string))
  3. 要求 256 位 2 进制数的前 n 位必须是 0
  4. 如果算出来了 (通过改变随机数慢慢试), 则用这个 Hash 值作为新块的头部, 并把打包信息拼接到 blockchain 后

n 的确定与全部挖矿人员的算力相关.

注意每一个 block 可以拆为两部分:

  • Header, 包含 hash 值
  • Body, 包含账单信息, 时间戳

PoS, Proof-of-Stock

Smart Contracts

Smart Contracts 指写入 blockchain 中的一段代码, 用于自动执行一些操作.

Blockchain 工作流程

  1. 用户在 blockchain network 中注册账户
  2. 用户发起一笔交易 (Tx), 如资产转移或信息更新
  3. 用户用私钥对交易进行数字签名, 以保证交易的真实性和完整性
  4. 已签名的交易被发送到网络中的 validation nodes
  5. Validation nodes 检查交易的有效性, 确保符合网络规则
  6. 有效的交易被集中在一个交易池中, 等待进一步处理
  7. Miner nodes 从交易池中选择多笔交易打包成 block
  8. Miner nodes 计算解决数学问题, 为新 block 找到有效的 hash 值
  9. 新区块再次接受验证 (查看 hash 值是否符合要求之类的), 以确保其合法性和准确性
  10. 验证成功的区块由 miner nodes 添加到 blockchain 上
  11. 新添加的 block 信息被广播到整个 blockchain 网络, 通知其他 nodes 更新

02-BTC 密码学原理

虽说比特币是加密货币, 但实际上比特币网络中的所有信息都是公开的.

主要用到两种密码学技术:

  • 哈希
  • 签名

哈希

哈希, 密码学中称: cryptographic hash function, 主要用到其三个性质

  • collision resistance, 指难以哈希碰撞

    两个不同的输入算出同样的哈希值, 就是哈希碰撞, 一般来说不可避免. 可以利用这个性质来得到 digest (只要一改变, digest 就会改变)

  • hiding, 指 Hash 的计算是单向的, 难以逆向计算, 也就是说, Hash 值没有泄漏出这个输入的任何信息

两个性质结合起来, 可以用来实现 digital commitment (或称 digital equipment of a sealed envelope).

如果输入空间不够大, 一般在输入之后添加几位随机数.

  • puzzle friendly, 指 Hash 值的计算一般不可预测, 只是看到输入, 很难猜到输出. 要想得到结果, 只能一个一个试, 所以才能用作工作量证明 (proof of work)

比特币中用到的 Hash 算法为 SHA-256 (Secure Hash Algorithm, 256 位输出)

签名

比特币中的账户管理是去中心化的, 创建账户的过程就是创建一对公私钥:

其思想来自 asymmetric encryption algorithm. (加密和解密用的不是同一个密钥, 加密用公钥, 解密用私钥), 解决对称加密中密钥分发不方便的问题.

比特币账户中:

  • 公钥相当于银行账户, 只要对方知道你的公钥, 就能向你转钱
  • 私钥相当于账户密码, 只要对方知道了你的私钥, 就能将钱转走

在这里, 公私钥用于签名:

  • 自己交易之后, 用私钥签名并将交易信息散布到网络
  • 别人用你的公钥来验证

产生公私钥对时, 需要确保用一个 good source of randomness, 不然可能与别人产生相同的公私钥对.

03-BTC 数据结构

Hash pointers

Block chains 中的链表与普通链表的区别在于:

  • 用 hash pointer 替代了普通指针

普通指针, 在计算机内存中直接指向一个内存地址, 通常用于引用变量或对象. 而 Hash Pointer: 不仅指向一个内存地址, 还包含该地址所指向的数据的哈希值, 用于确保数据的完整性和防篡改.

假设一个区块链如下:

  • 第一个 block 称 genesis block (创世区块), 最近产生的是 most recent block
  • 一个 block 中的 hash pointer 是根据前一个 block 的所有内容计算出的
  • 如果一个 block 内容被篡改, 则会导致后一个 block 的 hash pointer 无法使用
  • 因此只需要在本地保存最后一个 hash pointer 就能知道区块链是否被篡改

普通链表和区块链的区别:

  • 普通链表, 可以更改其中的元素, 而不影响剩下的元素
  • 区块链, 改动一个元素, 会导致后面所有区块不可用

得益于 Hash pointer, 在本地可以只保存最近的一部分区块, 其他区块在需要时可以向网络中的其他节点询问.

Merkle tree

  • 上层是 hash pointers
  • 最下面一层是数据块 (transaction, 交易数据块)
  • 根节点也能计算一个 root hash

好处在于, 只要记住 root hash, 当树中任意一处修改, 都能检测出.

比特币中, 各个区块用 hash pointer 连在一起, 而每个区块所包含的交易是组织成一个 merkle tree 的形式

一个区块分两个部分:

  • block header, 包含 hash pointer, 比特币协议版本, Merkle tree 的根哈希值, 挖矿难度阈值 (target), 随机数 (nonce) 等信息, 不含任何交易信息
  • block body, 含交易列表 (也就是很多个交易)

Merkle tree 的一个用途在于提供 merkle proof, 验证交易信息是否写入 merkle tree:

  • 最下面颜色填充的是当前交易
  • 从当前交易开始网上计算 hash 值, 得到最终的 hash 值后与 root hash 对比

只要一个数据结构是无环的, 其实都能用哈希指针替换普通指针. (有环的会形成循环依赖)

04-BTC 协议

数字货币防范的主要挑战在于: double spending attack (一个货币多次使用).

一个中心化的解决方法是, 每一次交易都由中心机构验证. 需要维护一个数据结构, 验证货币是否已被花出.

而一个去中心化的货币要解决两个问题:

  • 货币的发行由谁决定 (在 BTC 中由挖矿决定)
  • 如何验证交易的有效性, 如何防止 double spending attack (实现这个的数据结构就是区块链)

验证不合法的区块不会被接收到区块链中.

Distributed consensus

分布式共识, 系统中所有节点都要维护一张全局的哈希表. (但几乎很难实现所有都一致)

CAP Theorem:

  • Consistency
  • Availability
  • Partition tolerance

Paxos 协议能实现 Consistency.

05-BTC 实现

比特币是 transsaction-based ledger, 每一个区块记录的是交易信息.

每一个用户的账户信息需要根据所有的区块来推算 (这是一个坏处, 而以太坊系统中是 accout-based ledger, 则不存在这个问题).

每一个全节点都需要维护一个 UTXO (Unspent Transaction Output) 的表. 用来防范 double spending attack, 验证这个币是否被花掉了.

每个交易可以由多个输入以及多个输出, 但需要保证: total inputs = total outputs, total outputs 可以略微低于 total inputs, 这一部分作为打包的奖励 (transaction fee) 给打包的人. (防止有些节点只打包自己的交易)

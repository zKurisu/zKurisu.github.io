Lecture1 Introduction

介绍下常见的几个指令集

  • MIPS, Microprocessor without Interlocked Pipeline Stages, 一种精简指令集计算 (RISC) 架构, 广泛用于嵌入式系统和网络设备
  • x86, 复杂指令集计算 (CISC) 架构, 主要用于个人电脑和服务器, 支持丰富的指令集和高兼容性
  • Alpha, 由数字设备公司 (DEC) 开发的RISC架构, 以其高性能和64位支持著称, 但已不再更新
  • PIC, Peripheral Interface Controller, 一种微控制器架构, 主要用于嵌入式应用, 具有低功耗和高效率的特点
  • ARM, Advanced RISC Machine, 一种流行的RISC架构, 以其高能效和小尺寸而闻名, 广泛用于移动设备和嵌入式系统

Pipline

在现代处理器中，指令通常被拆解为多个步骤 (如取指, 译码, 执行, 写回), 这些步骤被视为流水线中的不同阶段, 多个指令在不同阶段同时执行, 以提高处理效率. 这类似于工厂的生产线, 每个阶段并行工作, 提高指令吞吐量.

举个例子, 假设对一个指令的处理分 5 个阶段:

  • IF (Instruction Fetch): 取指令
  • ID (Instruction Decode): 译码并读取寄存器
  • EX (Execute): 执行运算或内存地址计算
  • MEM (Memory Access): 访问数据存储器（如加载或存储数据）
  • WB (Write Back): 将结果写回寄存器

周期 ADD SUB LOAD STORE
1 IF(ADD)
2 ID(ADD) IF(SUB)
3 EX(ADD) ID(SUB) IF(LOAD)
4 MEM(ADD) EX(SUB) ID(LOAD) IF(STORE)
5 WB(ADD) MEM(SUB) EX(LOAD) ID(STORE)
6 WB(SUB) MEM(LOAD) EX(STORE)
7 WB(LOAD) MEM(STORE)
8 WB(STORE)

此时总耗时为 8 个周期, 效率高很多.

而无互锁的流水线, 虽然减少了硬件复杂度, 但是会增加程序员和编译器的负担.

NRE costs 和 Recurrent costs

NRE costs 指是在产品开发阶段产生的一次性工程费用. 这些费用在产品的整个生命周期内只发生一次, 主要包括设计, 研发, 原型制作, 测试和验证等活动的成本. 这些成本不会随着每个产品的生产重复发生, 因此称为 “非重复性工程成本”.

Recurrent costs 指在产品生命周期内会定期或持续发生的费用. 这些成本通常与产品的生产, 运营, 维护和支持相关, 且会随着生产数量或时间周期的增加而增加.

Die cost

Die Cost (芯片单元成本) 是指从一片**硅晶圆 (wafer) 中制造出的单个裸片 (die)**的平均成本. 这是芯片制造中的一个关键指标, 直接影响半导体产品的最终价格.

Yield

Yield, 指良率, 表示在整个制造过程中生产出合格产品的比例. 计算为:
$$
\displaylines
{
\begin{aligned}
Y = \frac{No.\ of\ good\ chips\ per\ wafer}{Total\ number\ of\ chips\ per\ wafer} \times 100%
\end{aligned}
}
$$

Die cost 可以计算为:
$$
\displaylines
{
\begin{aligned}
Die\ cost = \frac{Wafer\ cost}{Dies\ per\ wafer \times Die\ yield}
\end{aligned}
}
$$

Tristates

Tristates buffer 是一种特殊的逻辑电路, 除了具有普通缓冲器的功能外, 还能在其输出端产生三种状态:

  • 高电平, 1
  • 低电平, 0
  • 高阻态, Z

  • 当 Enable 使能时, 此时根据输入信号来输出相应的高电平或低电平
  • 当使能信号被禁用时, 三态缓冲器不再对输入信号做出响应, 此时称 “高阻态” (相当于电路断开)

三态缓冲器广泛应用于总线系统中, 多个设备可以共享同一数据线. 通过控制使能信号, 只有一个设备能够向总线发送数据, 而其他设备则处于高阻态.

Nonrestoring Tristate

Nonrestoring Tristate 指一种特殊的 tristate bufferm 其输出在特定情况下不恢复到逻辑高或逻辑低的状态, 而是保持在高阻态或某种其他状态.

Delay Definitions

有这些类型的 delay 需要考虑:

Propagation Delay

Propagation delay 指, 有效输入到开始有有效输出 (输出稳定时) 之间的时间

若要减小 propagation delay, 需要:

  • keep capacitances low
  • keep transistors fast

Contamination Delay

Contamination delay 指, 当输入已经改变, 但输出仍保持 old value 的时间.

Delay Estimation

用 RC delay models, 假设:

  • C 大小为 total capacitance on output node
  • R 为 effective resistance, 把 transistor 视为 resistor
  • $t_{pd} = RC$

R 的计算为:
$$
\displaylines
{
\begin{aligned}
R \approx \frac{1}{\beta (V_{gs} - V_t)} = \frac{1}{\mu C_{ox}} \frac{L}{W} \frac{1}{(V_{gs} - V_t)}
\end{aligned}
}
$$

Elmore Delay

不把 transistor 视为 resistor, 把 Pullup 或 Pulldown network 视为 RC ladder, 此时计算 delay:
$$
\displaylines
{
\begin{aligned}
t_{pd} = \sum_{nodes\ i} R_{i-to-source}C_i
\end{aligned}
}
$$

Size CMOS transistor

Size CMOS transistor 是为了让 $R_p = R_n$, 即 P 和 N channel 的阻值相等.

需要用到的公式如下:
$$
\displaylines
{
\begin{aligned}
R \approx \frac{1}{\beta (V_{gs} - V_t)} = \frac{1}{\mu C_{ox}} \frac{L}{W} \frac{1}{V_{gs} V_t}
\end{aligned}
}
$$

可以看出, CMOS transistor 的宽度 W 和长度 L 会影响 R 的取值.

如果有 $\mu_n = r \cdot \mu_p$, r 取 2 或 3. 即相同 size 下, PMOS 的阻值是 NMOS 的 r 倍. (从上面的公式看出来的)

Schemetic 中标记 transistor dimension 示例如下:

  • $8/2$ 是 $Width/Length$, 没写单位就表示 “1 unit”.
  • 题中求最小, 就找最小整数
  • 单位宽度时, 阻值为 $R_n$, 那么宽度为 2 时, 阻值就该是 $R_n/2$
  • 也要注意串联和并联
  • 调整 size 似乎一般只改变 width
  • $\gamma = 2$, 则有 $C_p = 2C_n$
  • 要让每一个 channel 的相等, 一个边就是一个 channel
  • $W_{ni} = 1, W_{pi} = 2$ 意味着 $\mu_n = 2\mu_p$

有 $W_p = 2W_n$ 等价于 $C_p = 2C_n$.

\begin{aligned}
P(t) = V(t) \times I(t)
\end{aligned}
}
$$

计算能量 (Energy):
$$
\displaylines
{
\begin{aligned}
E = \int_{t_1}^{t_2} P(t) dt
\end{aligned}
}
$$

计算平均功率 (Average Power):
$$
\displaylines
{
\begin{aligned}
P_{avg} = \frac{1}{T} \int_0^T P(t) dt
\end{aligned}
}
$$

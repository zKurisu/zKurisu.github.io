XML-基础语法

W3School XML 教程

介绍

XML, eXtensible Markup Language, 可扩展标记语言, 是一种用于存储和传输数据的标记语言, 其设计易于人类阅读以及及其解析.

特点

可扩展

XML 允许用户自定义标签.

平台无关

ML 文件是纯文本格式, 可以在不同的操作系统和编程语言之间共享和传输.

Do Nothing

字面意思, XML 不具有任何功能, 只是用 tag 把数据包裹起来了而已. 如:

与 HTML 的区别

  • XML 主要设计用来携带数据
  • HTML 主要设计来展示数据
  • XML 的 tags 不像 HTML 那样是预定义好的

比如上述例子中的 <to>, <from>, 这些都不是由一个组织规定好的, 而是由文档的编写者自己定义的.

而 HTML 标签如 <p>, <h1> 都是预定义好的.

XPath

XPath, 是一种路径表达式, 用于定位 XML 文档中的某个节点或 node-sets, 以查找信息.

比如:

下面是一些 XPath 表达式以及其对应的数据:

  • /bookstore/book[1], 对应第一个 book element
  • /bookstore/book[last()], 对应最后一个 book element
  • /bookstore/book[position()]<3, 对应前两个 book element (也就是序号小于 3)
  • //title[@lang], 对应包含 lang 属性的 title element (// 表示不管第一层级是什么)
  • /bookstore/book[price>35.00], 对应 price 子元素值大于 35.00 的 book element

XSLT

XSLT, eXtensible Stylesheet Language Transformations, 是将 XML 进行转换和处理的语言, 它允许将 XML 数据转换为其他格式.

XSLT 使用样式表来定义如何将输入的 XML 数据转换为所需的输出格式.

XML Validator

用于检测 XML 语法是否正确. 在 Linux 中, 可以用工具如 xmllint

  • --noout, 告诉 xmllint 在验证完成后不输出任何内容. 通常情况下, 如果 XML 是有效的, xmllint 会输出该 XML 文件的内容
  • --schema 指定一个 XSD (XML Schema Definition) 文件, 用于验证 XML 文件的结构和内容

XML DTD

DTD, Document Type Definition, 用于定义 XML 文档的结构以及有那些 legal elements and attributes. (也就是如何自定义 tag)

实际上 XML 可以不需要 DTD, 用 DTD 只是方便你确认文档是否 valid.

比如:

  • <!DOCTYPE note SYSTEM "Note.dtd"> 告诉浏览器或解析器该文档使用的 DTD 文件信息
  • note 指定文档 root element
  • SYSTEM "Note.dtd" 指定了外部 DTD 文件的名称

这里:

  • <note>
  • <to>
  • <from>
  • <heading>
  • <body>

这些 element 由 DTD 文档定义如下:

  • !DOCTYPE note 定义 note 为 root element
  • !ELEMENT note (to,from,heading,body) 定义 note element 必须包含四个子元素 to, from, heading, body
  • !ELEMENT to (#PCDATA) 定义 to element 的类型是 #PCDATA (parseable character data)
  • ….

Entity References

  • &, 用 &amp 表示 (ampersand)
  • <, 用 &lt 表示 (less than)
  • >, 用 &gt 表示 (greater than)
  • ", 用 &quot 表示 (quotation mark)
  • ', 用 &apos 表示 (apostrophe)

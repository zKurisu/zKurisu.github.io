Types of Attacks

大致分为三类:

  • Virus, capable of replication program, 即能在某种情况下创建自身的副本, 会直接修改被感染主机文件, 依赖用户触发某种操作来传播
  • Worm, self-replicating, 不依赖其他程序或文件, 通过某种机制创建自身的副本, 通常通过网络, 电子邮件传播
  • Trojan, 一种伪装成合法程序的恶意软件 (posing as legitimate programs), 不自我复制, 依赖用户行为来传播

CIA triad

  • Confidentiality, Data confidentiality, Privacy, 数据是否授权给他人查看
  • Integrity, Data integrity, System integrity, 数据是否被人篡改, 系统行为是否被人改变
  • Availability, 系统对于授权人员是否可用

OSI security architecture

分三个部分:

  • Security attack, 威胁 information 的行为
  • Security mechanism, 防范, 检测 security attack 的 process
  • Security service, 加强 security of data transfering 的 service

Active and Passive Attack

  • Passive attack, 攻击者的目标只是 obtain information
  • Active attack, 攻击者会 change the data or harm the system

Active attack 的例子

  • Masquerade, 伪装攻击, 攻击者假冒成合法用户或系统, 获取未授权访问, 示例: 一个攻击者获取了某个用户的凭证 (如用户名和密码), 然后使用这些凭证登录到系统, 进行未授权的操作, 比如查看敏感信息或修改数据
  • Replay, 重放攻击, 攻击者截获合法用户的通信数据, 并在稍后重新发送, 以欺骗接收方. 示例: 假设用户 A 发送了一条包括交易指令的消息给服务器, 攻击者截获了这条消息. 攻击者随后在某个时刻再次发送相同的消息, 导致服务器重复执行该交易
  • Modification of messages, 消息修改攻击, 攻击者在传输过程中篡改消息内容, 影响通信的完整性. 示例: 用户 A 向用户 B 发送一条消息, 内容为 “我同意支付100元”.攻击者可以截获这条消息, 并将其修改为 “我同意支付1000元”, 从而影响交易结果
  • Denial of service, 拒绝服务攻击, 攻击者通过各种手段使目标系统无法提供正常服务, 攻击者向一个网站发送大量请求, 超出其处理能力, 导致正常用户无法访问该网站. 这种攻击通常通过分布式拒绝服务 (DDoS) 来实现

Fundamentals of Secure design

包括:

  • Economy of mechanism, 设计简洁, 以减少复杂性和潜在漏洞
  • Fail-safe defaults, 系统在出现故障时应默认拒绝访问, 而不是允许访问
  • Complete mediation, 每次访问资源时, 都必须进行权限检查, 而不是依赖于先前的检查结果
  • Open design, 系统的设计和实现应公开, 以便于审查和发现潜在漏洞
  • Separation of privilege, 系统应要求多个条件才能授予特定权限, 降低单一故障的风险
  • Least privilege, 用户和程序应仅被赋予完成任务所需的最小权限
  • Least common mechanism, 不同用户和程序应尽量避免共享相同的机制, 以减少信息泄露风险
  • Psychological acceptability, 安全措施应易于使用,以免用户绕过安全机制
  • Isolation, 将不同的系统或功能隔离, 以限制潜在攻击的影响
  • Encapsulation, 将数据和操作封装在一个模块中, 以限制对内部实现的访问
  • Modularity, 将系统分解成独立的模块, 以便于管理和维护
  • Layering, 采用多层安全措施, 以提供额外的保护
  • Least astonishment, 系统的行为应符合用户的期望, 减少意外操作

Attach Surfaces

Attack Surface, 指系统中所有可能被攻击者利用的入口点或漏洞的集合. 它包括所有与外部交互的部分, 如网络接口, 用户输入, API, 软件组件等.

Attack Trees

Attack Trees, 是一种图形化工具, 用于分析和可视化系统可能受到的攻击方式. 每个树的节点代表一个攻击目标, 分支表示实现该目标的不同攻击路径. 叶子节点表明 attack 是如何初始化的. root node 表示攻击的目标.

Three Components involved in authentication:

  • User terminal and user (UT/U), 指用户用来访问系统的设备, 如个人电脑, 智能手机
  • Communications channel (CC), 指用户终端与服务器之间的数据传输路径, 如互联网
  • Internet banking server, 指提供银行服务和进行身份验证的后端系统

Five Attack Strategies:

  • User credential compromise, 用户凭证泄露, 攻击者通过各种手段获取合法用户的凭证 (如用户名和密码), 以便进行未授权访问
  • Injection of commands, 命令注入, 攻击者向应用程序输入恶意命令, 利用应用程序的漏洞执行未授权操作
  • User credential guessing, 用户凭证猜测, 攻击者尝试通过猜测用户的用户名和密码组合来获得访问权限
  • Security policy violation, 安全政策违规, 用户或系统未遵循既定的安全政策, 可能导致安全漏洞
  • Use of known authenticated session, 使用已知的已认证会话, 攻击者利用合法用户的已认证会话来进行未授权操作

Symmetric Cipher Model

基本术语:

  • Plaintext, 指原始消息数据
  • Ciphertext, 指加密后的数据
  • Enciphering 或 Encryption, 指把 plaintext 变为 ciphertext
  • Deciphering 或 Decryption, 指把 ciphertext 变为 plaintext

Symmetric Cipher Model 指对称加密, 收发双方用同一把密钥:

Cryptanalysis and Brute-Force Attack

Cryptanalysis Attack, 密码分析攻击, 攻击者分析加密算法的结构和实现，以找到弱点.

Brute-Force Attack, 暴力破解攻击, 指攻击者通过尝试所有可能的密钥组合, 直到找到正确的密钥.

Unconditionally Secure Scheme

Unconditionally Secure Scheme 是一种加密方法, 其安全性不依赖于任何计算假设或攻击者的计算能力. 即使攻击者拥有无限的计算资源, 也无法破解这种加密方案.

当前没有任何算法能实现这点, 但可以向两个方向努力靠近:

  • 攻击者破解加密所需的成本高于被加密信息的价值
  • 破解加密所需的时间超过信息的有效使用寿命

Computationally Secure Scheme

Computationally Secure Scheme 是指一种加密方法, 其安全性依赖于攻击者在合理的时间内无法有效破解加密.

Substitution Techniques

Encryption techniques 中最重要的两点是:

  • Substitution, plaintext 中的 letters 被替换为另外的 letters
  • Transposition, plaintext 被重新排序

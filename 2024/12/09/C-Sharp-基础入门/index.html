什么是 .NET

.NET 是微软提供的一个开发平台, 为多种编程语言 (C#, VB.NET 等) 提供应用程序开发的库和工具.

.NET 主要由四个部分组成:

  • CLR (Common Language Runtime): 运行时环境, 负责执行 .NET 应用程序, 并提供垃圾回收, 类型安全等服务
  • 类库: 提供丰富的预构建类和函数, 简化开发过程
  • ASP.NET: 用于创建 Web 应用和服务的框架
  • Entity Framework: 用于数据访问的对象关系映射 (ORM) 框架

其实 .NET 平台也分几个:

  • .NET Framework, 由微软开发, 但只能在 Windows 上运行
  • MONO, 由社区开发, 可以跨平台运行
  • .NET CORE, 由微软开发, 可以跨平台运行

C# 和 .NET 的关系

C# 是 .NET 平台上的一种主要编程语言. 开发者使用 C# 编写应用程序, 而这些应用程序在 .NET 环境中运行.

值类型和引用类型

C# 中的数据类型分两大类:

  • Value Types, 值类型
  • Reference Types, 引用类型

两者在内存存储上有较大区别:

  • Value Types, 直接包含其数据, 存储在栈 (stack) 中. 每个值类型变量都有自己的数据副本
    • 当 Value Types 变量被赋值给另一个变量时, 会复制其值
    • 默认值为其零值
    • 常见的 Value Types 有: int, float, double, char, bool, struct, enum

输出:

  • Reference Types, 在栈中存储的是对数据的引用, 实际数据存储在堆 (heap) 中. 多个引用类型变量可以指向同一个对象.
    • 当 Reference Types 变量被赋值给另一个变量时, 复制的是引用, 而不是数据本身
    • Reference Types 的默认值为 null, 表示不指向任何对象
    • 常见的 Reference Types 有: class, string, array, interface, delegate

继承

继承, 用于通过已有类来创建新类, 以便重用, 扩展类的行为.

其成员被继承的类称为 “基类”, 继承这些成员的类称为 “派生类”. 派生类只能有一个直接基类. 但是, 继承是可传递的. 如果 ClassC 派生自 ClassB, 并且 ClassB 派生自 ClassA, 则 ClassC 将继承在 ClassB 和 ClassA 中声明的成员.

可以把派生类理解为基类的专门化, 比如, 有一个基类 Tool, 字可以有一个名为 Hand (手动工具) 的派生类, 以及另一个名为 Automatic (自动工具) 的派生类, 他们都是 Tool, 但有不同的专门化用处.

派生类会隐式获取基类的所有成员 (除了构造函数和终结器).

Archiso 构建 Arch 镜像

参考 Archwiki archiso 部分
Archiso 是一个高度可定制的, 用于构建 Arch Linux live CD/USB ISO 镜像的工具.

配置

在安装 archiso 之后, 有两个配置文件目录:

  • /usr/share/archiso/configs/releng, Release Engineering, 用于创建正式的每月安装 ISO
  • /usr/share/archiso/configs/baseline, 是一种最低限度的配置, 它只包括从介质启动 Live 环境所需的最低限度的软件包

archiso 通过指定配置目录来构建 ISO 镜像, 因此, 想要自己定义, 可以复制上述目录后修改:

配置目录结构

具体内容可以查看 /usr/share/doc/archiso/README.profile.rst 文件 (用的是 reStructuredText, reST 格式, 这是一种轻量级的标记语言常用于撰写文档), 可以转为其他格式查看:

先以 baseline 目录为例:

  • profiledef.sh 中定义了一些变量, 比如 <iso_name>-<iso_version>-<arch>.iso, 会改变其他配置文件的行为
  • bootstrap_packages.arch, 这里的 arch 要替换为具体的架构, 如 x86_64. 用于 buildmode 为 bootstrap 时, 其列出生成 bootstrap image 时需要安装的软件包 (一行一个, # 开头的行忽略)
  • packages.arch, 同样将 arch 替换为具体的架构. 用于 buildmode 为 iso 或 netboot 时, 其列出需要安装到 ISO 镜像中的包 (mkinitcpio 和 mkinitcpio-archiso 两个包必须包含)
  • pacman.conf, pacman 的配置文件
  • airootfs/ 目录, 在包被安装之前就复制其下的文件或目录到根目录, 不保留权限, 复制之后都归属于 root, 具体可以通过 profiledef.sh 设置, 可以借助这个来设置 airootfs/etc/shadow 以及 airootfs/etc/gshadow 目录来设置用户
  • efiboot/ 目录, 包含 systemd-boot 的配置, 当 bootmodes 中含 uefi-x64.systemd-boot.esp 或 uefi-x64.systemd-boot.eltorito 时该目录必须
  • syslinux/ 目录, 包含 syslinux, isolinux, pxelinux 的配置文件, 当 bootmodes 中含 bios.syslinux.mbr 或 bios.syslinux.eltorito 时必须
  • grub/ 目录, 包含 GRUB 的配置, 当 bootmodes 中含 uefi-ia32.grub.esp 或 uefi-ia32.grub.eltorito 或 uefi-x64.grub.esp 或 uefi-x64.grub.eltorito 时必须

选择包

修改 packages.x86_64 文件, 逐行列出想要在 Live 镜像上安装的软件包即可.

向镜像根目录添加文件

放进 airootfs/ 目录即可, Airootfs 目录用作映像上 Live 系统的根目录 (/) 的起点. 在安装包之前, 它的所有内容都将被复制到工作目录中.

添加 pacman 仓库

内核

可以把想要安装的内核版本都写入 packages.x86_64 文件中.

所有 work_dir/airootfs/boot/vmlinuz-* (自己准备好放这的) 以及 work_dir/boot/initramfs-*.img (通过指定软件包安装的) 内核文件都会被包含到 ISO 中.

Boot Loader

Archiso 支持:

  • syslinux 用于 BIOS 引导, 可修改 syslinux/ 目录来设置
  • systemd-boot (修改 efiboot/ 目录) 或 GRUB (修改 grub/ 目录) 用于 UEFI 引导

用户和密码

若要创建 Live 环境中可用的用户, 可以手动编辑下面文件:

  • myarch/airootfs/etc/passwd
  • myarch/airootfs/etc/shadow
  • myarch/airootfs/etc/group
  • myarch/airootfs/etc/gshadow

注意语法.

更改 os-release 名称

将 /etc/os-release 文件复制到 airootfs/etc/ 下即可.

构建 ISO 镜像

  • -w, 指定 working directory, 存放临时文件, 不指定则选用当前目录
  • -o, 指定生成的 ISO 镜像放在哪个目录, 不指定则默认为当前目录的 out/ 目录下
  • /path/to/profile/ 这里是 /tmp/test-archiso/baseline, 指定配置文件目录

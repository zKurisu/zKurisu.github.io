Haskell-State-Monad-使用

介绍

Haskell 中的 Monad 都比较难理解, 但很多地方都需要用到, 因此单独为每个 Monad 准备一篇记录.

主要记住一点, State Monad 是为了简化值和状态传递而设计的. >>= 会负责取值以及自动传递状态.

State Monad 一般用在需要管理多个不相关计算的共享状态时, 只需要简单的递归参数传递时没必要使用.

定义

State Monad 描述 “带状态的计算的组合”, State 类型本质上是一个计算, 因此其定义为:

  • State 类型包裹一个函数 (也就是一个计算)
  • runState 函数类型是 State s a -> s -> (a, s), 输入计算和一个状态返回值和新的状态
  • s, state, 表示计算过程的环境或上下文信息, 计算结果可以选择依赖这个值
  • a, result, 表示这个 State 计算最终要返回给调用者的有用信息

Monad instance 定义如下:

  • (State h) 这个模式匹配用于取出 State 内包裹的函数

>>= 的实现也可以写为:

  • 可以理解为:
    • h 是 State 类型, 表示一个计算, 向该计算输入一个环境, 会对应一个值和新的环境 (输入一个状态, 返回值和新的状态)
    • f 表示将一个值包裹进一个计算 (当然也可以舍弃这个值直接构建计算)

>>= 干的事就是自动取值进行计算以及自动传递状态, 是一种简化设计.

介绍

Hydra 是一个开源 Python 框架, 用来构建 hierarchical configuration, 并且能组合多个配置, 在命令行动态修改配置, 同时运行多个配置.

其底层用 OmegaConf 读写 YAML, 后面一并说明.

  • 在 client.url 中 ${server.host}, 引用的就是前面的 server.host = localhost; 同理后面的用 ${} 包裹的内容
  • . 表明指定的是当前层级的 node; .. 表明指定 parent 层级的 node

控制 Hydra 的行为

yaml 文件中的 hydra: 键之下的内容, 用来控制 Hydra 框架本身的行为 (如输出路径, 日志格式等), 示例:

  • hydra.run.dir 是单次运行 (python my_app.py) 的输出路径
  • hydra.sweep.dir 是多次运行 (python my_app.py --multirun) 的根目录
  • hydra.job_logging.handlers.file.filename 是日志文件名
  • hydra.job.chdir 指定是否在 Python 运行时将进程的目录切换到 hydra.run.dir 指定的目录, 启用的一个坏处是: 如果数据集用相对路径读取, 可能会出错

介绍

OpenSpiel 包含了很多应用在 games 上的 general reinforcement learning 的环境和算法, 是一个较为全面的 collection.

OpenSpiel 的主要接口是 C++20 和 Python3. 大多 games 和 utility function 由 C++ 编写, 然后用 pybind11 绑定到 Python 上.

基本概念

open_spiel/spiel.h 中定义有两个关键概念:

  • Game, 该 object 包含一个 game 的 high level description, 比如是 simultaneous 还是 sequential, 有多少 players, maximum 和 minimus scores 是多少等
  • State, 描述一个 trajectory 中某一点的信息

游戏中所有可能的 trajectory 会构成一棵树, 树中的每个节点代表一个 State, 而 node 间的连接关系构成用户 moves 的 history.

Chance 指代由 “环境” 决定的随机事件. 当路径进入 Chance node 时, 接下来的走向根据一个概率分布随机产生. 在 RL 的数学框架 (MDP) 中, Chance 对应的是状态转移概率 $P(s’|s,a)$. 比如机器人走迷宫的例子:

  • Agent 决定 “向北走” (Action)
  • Chance 介入: 由于地板上有油, 机器人有 80% 的概率真的向北走, 有 10% 的概率滑向左边, 有 10% 的概率滑向右边
  • 这里的 “打滑” 就是一个 Chance 事件

在大多数 games 中, 通常用一个 explicit player (the “chance” player, id 通常是 kChancePlayerId) 来处理 chance.

Chapter 1: Basic Concepts

State, action, policy, …

用到的例子是 grid-world, 一个机器人在网格世界中移动, 每个 cell 有 Accessible/forbidden/target 其中一个属性, 且 world 存在 boundary:

强化学习中的 State 定义为: agent 在当前环境中的 status. 对应到 grid-world 中, 就是 agent 的 location, 对于复杂的问题, 可能还包括速度, 加速度, 温度等等信息:

将所有 state 集合到一起 (形成一个集合), 就是 State space: $S = \{s_i\}_{i = 1}^9$.

Action 就是在一个 state 下, 可以采取的行为, 比如:

  • $a_1$, 向上移动
  • $a_2$, 向下移动
  • …

一个 state 的所有可能 actions 构成 action space: $A(s_i) = \{a_i\}_{i=1}^5$.

需要注意 action space 是依赖于 state 的, 不同 state 下的 action space 不同.

采取一个 action 后, agent 从一个 state 移动到了另一个 state 的过程称为 State transition:

$$
\displaylines
{
\begin{aligned}
s1 \overset{a2}{\rightarrow} s2
\end{aligned}
}
$$

State transition 本质上定义了 agent 与环境的交互. 其可以用 tabular representation 来表现:

但只适用于 deterministic cases (一个 action 只可能进入一个 state).

对于 State transition 的更一般的表达是用 state transition probability, 用概率来描述 state transition. 比如下面例子, 描述在 $s_1$ 采用 action $a_2$ 回到达 $s_2$:

$$
\displaylines
{
\begin{aligned}
p(s_2 | s_1, a_2) = 1 \newline \newline
p(s_1 | s_1, a_2) = 0, \forall i \ne 2
\end{aligned}
}
$$

该方式既能描述 deterministic case, 也能用于 stochastic cases.

Policy 会告诉 agent 在一个 state 下采取哪一个 action. 如果从示例上直观理解的话, 一个 policy 就表示一个箭头:

基于 policy 运行到底, 就能得到一个 path 或 trajectory:

Policy 用 conditional probability 作为更一般的数学表示, 比如对 $s_1$ 有:

$$
\displaylines
{
\begin{aligned}
\pi (a_1 | s_1) = 0 \newline
\pi (a_2 | s_1) = 1 \newline
\pi (a_3 | s_1) = 0 \newline
\pi (a_4 | s_1) = 0 \newline
\pi (a_5 | s_1) = 0
\end{aligned}
}
$$

针对一个状态的所有概率之和应该等于 1. 上述描述的是一种 deterministic case; 如果是 stochastic case, 则会有多个概率不为 0.

也可以用 tabular representation 来表示 Policy, 该表示方法也是 general 的:

Reward, return, Markov decision process

Reward 是 RL 中一个独特的概念, 其本质是一个 real number, 每次采取一个 action 都会得到一个 reward:

  • 如果 reward 是 positive, 表明鼓励采取这种 action
  • 如果 reward 是 negative, 表明不希望采取这种 action, 相当于 punishment

Reward 通常被视为一种 human-machine interface, 即我们通过设计 reward 来与机器交互, 引导 agent 做出我们期望的行为.

比如在 grid-world 例子中可以设计:

  • 进入 forbidden area 时 $r_{forbid} = -1$
  • 进入 boundary 时 $r_{bound} = -1$
  • 到达 target 时 $r_{target} = 1$
  • 其余时候 $r = 0$

这样能引导 agent 走向目标区域. 同样可以用 tabular representation 来表示 reward transition, 但注意其只能表示 deterministic 的情况:

更 general 的方式也是 conditional probability, 比如:

  • $p(r = -1 | s_1,a_1) = 1$
  • $p(r \ne -1 | s_1,a_1) = 0$

对于 stochastic 的场景下, reward 本身是不确定的, 可能随环境改变.

Tranjectory 表示一个 state-action-reward chain (path), 比如:

对应的数学表示可以写为:

$$
\displaylines
{
\begin{aligned}
s_1 \overset{a_2}{\underset{r=0}{\rightarrow}} s_2 \overset{a_3}{\underset{r=0}{\rightarrow}} s_5 \overset{a_3}{\underset{r=0}{\rightarrow}} s_8 \overset{a_2}{\underset{r=1}{\rightarrow}} s_9
\end{aligned}
}
$$

可以看到其包含了 3 个部分: state, action 和 reward.

一个 trajectory 的 return 表示沿着该 trajectory 的所有 rewards 之和 (也可以说是 cumulative future reward), 比如对于上面例子有:
$$
\displaylines
{
\begin{aligned}
return = 0 + 0 + 0 + 1 = 1
\end{aligned}
}
$$

不同的 policy 会得到不同的 trajectory, 也会有不同的 return.

Return 可以用来衡量 policy 的好坏.

在上述例子中, 到达 target location 之后, 策略其实还在继续运行:

$$
\displaylines
{
\begin{aligned}
s_1 \overset{a_2}{\underset{r=0}{\rightarrow}} s_2 \overset{a_3}{\underset{r=0}{\rightarrow}} s_5 \overset{a_3}{\underset{r=0}{\rightarrow}} s_8 \overset{a_2}{\underset{r=1}{\rightarrow}} s_9 \overset{a_5}{\underset{r=1}{\rightarrow}} s_9 \overset{a_5}{\underset{r=1}{\rightarrow}} s_9 …
\end{aligned}
}
$$

此时的 return 就变为:

$$
\displaylines
{
\begin{aligned}
return = 0 + 0 + 0 + 1 + 1 + 1 + … = \infty
\end{aligned}
}
$$

解决的办法就是引入 discount rate $\gamma \in [0, 1)$, 得到 discounted return:

$$
\displaylines
{
\begin{aligned}
discounted\ return &= 0 + \gamma 0 + \gamma^2 0 + \gamma^3 1 + \gamma^4 1 + \gamma^5 1 + … = \newline \newline
&= \gamma (1 + \gamma + \gamma^2 + …) = \gamma^3 \frac{1}{1-\gamma}
\end{aligned}
}
$$

此时能得到一个有限解. 如果 $\gamma$ 接近 0, 那么 discounted return 就取决于更近未来 (near future, 越往后, $\gamma^n$ 越小, 后面项的影响就越小) 的 rewards; 而 $\gamma$ 接近 1 时, 其取决于更远未来 (far future, 衰减得慢, 积累起来的 reward 就会大一些) 得到的 return.

当 agent 沿着 policy 到达 terminal states, 得到的 trajectory 称为 episode (or a trial). 这种有限步的也被称为 episodic tasks.

有些任务没有 terminal states, 意味着其会永远与环境交互下去. 这类任务称为 continuing tasks.

通常会统一 episodic 和 continuing tasks, 有两种方法, 都是将 episodic task 转换为 continuing task:

  • 将 target state 视为 absorbing state, agent 进入该状态后, 任何 action 都是原地踏步留在该状态, 且后续的 reward 为 $r = 0$
  • 将 target state 视为普通的带 policy 的 state, 策略好的时候留在该状态; 不好时跳出

这里用的第二个方法.

Markov decision process (MDP)

MDP 是一个框架, 这里把上述提到过的概念都套在这里.

MDP 涉及几个 Sets:

  • State: 状态的集合, 就是 state space $S$
  • Action: action 的集合, 就是 action space $A(s), s \in S$
  • Reward: reward 的集合, $R(s, a)$

另一个要素是几个 probability distribution:

  • State transition probability: 在状态 $s$ 下, 采取 action $a$, 状态转移到 $s’$ 的概率 $p(s’|s,a)$
  • Reward probability: 在状态 $s$, 采取 action $a$, 得到的 reward $r$ 的概率 $p(r|s,a)$

第三个要素是 Policy: 在状态 $s$, 选择 action a 的概率是 $\pi (a|s)$.

最后 MDP 需要满足 Markov property, 也称 memoryless property, 即当前的概率与历史无关, 比如下面式子表明, 当前求 $p(s_{t+1}|a_{t+1}, s_t)$, 与 $a_1, s_0$, $a_2, s_1$ 这些历史状态无关:

$$
\displaylines
{
\begin{aligned}
p(s_{t+1} | a_{t+1}, s_t, …, a_1, s_0) = p(s_{t+1} | a_{t+1}, s_t) \newline \newline
p(r_{t+1} | a_{t+1}, s_t, …, a_1, s_0) = p(r_{t+1} | a_{t+1}, s_t)
\end{aligned}
}
$$

可以用 Markov decision process 这个名词来记忆上述组成:

  • Markov 对应 Markov property
  • decision 对应 policy
  • process 对应 sets 和 probability distribution

如果 policy 确定, 那么 markov decision process 就变成了 markov process.

Chapter 2: Bellman Equation

Why return is important

从之前的定义可以知道, return 是求一条 trajectory 上所有 reward 的累加, 其可以用来衡量一个 policy 的好坏. 对于下图而言, return 的计算为:

$$
\displaylines
{
\begin{aligned}
return = 0.5(-1 + \frac{\gamma}{1 - \gamma}) + 0.5(\frac{\gamma}{1 - \gamma})
\end{aligned}
}
$$

本质是求从 1 开始向下和向右两个 policy 的期望值. 严格来说其不是 return 了, 而是后面要提到的 state value.

由于从每个 state 出发的 trajectory 都能得到一个 return, 比如下面例子:

假设从 s1 开始 policy 得到的 return 为 v1; 从 s2 开始得到的是 v2; s3 是 v3; s4 是 v4, 那么就有:

$$
\displaylines
{
\begin{aligned}
v1 = r_1 + \gamma v_2 \newline \newline
v2 = r_2 + \gamma v_3 \newline \newline
v3 = r_3 + \gamma v_4 \newline \newline
v4 = r_4 + \gamma v_1 \newline \newline
\end{aligned}
}
$$
这比较直观, s1 经一个 action 到 s2, 那么其 return 就是该 action 的 reward 加上 s2 的 return; 对于其他情况也是这个原理. 这种一个 state 的 return 依赖于其他 state 的 return 的 idea 在强化学习中被称为 Bootstrapping (Bootstrapping 原意是将两侧鞋带往上提的动作, 后引申出让自己原地腾空的讽刺意义, 而这里指代迭代自己得到结果的行为).

上述式子写成矩阵的形式就是:

简写为:
$$
\displaylines
{
\begin{aligned}
v = r + \gamma P v
\end{aligned}
}
$$

这个式子就是该特定 deterministic 问题的 Bellman equation.

这个例子中最重要的点就是 bootstrapping 的 idea, 一个 state 的 value 会依赖于其他 state 的 value. 还有就是用 matrix-vector form 来求解会比较容易.

State value

考虑一个 single-step process:
$$
\displaylines
{
\begin{aligned}
S_t \overset{A_t}{\rightarrow} R_{t + 1}, S_{t + 1}
\end{aligned}
}
$$

  • $t, t+1$ 都是离散的 time instances
  • $S_t$ 表示 t 时刻的 state
  • $A_t$ 表示 $S_t$ 采取的 action
  • $R_{t+1}$ 表示采取 $A_t$ 后得到的 reward
  • $S_{t+1}$ 表示采取 $A_t$ 后转移得到的 state

注意 $S_t, A_t, R_t$ 这些都是随机变量, 这里的每个操作都是用 probability distribution 来决定的:

  • $S_t \rightarrow A_t$, 一个 state 会采取什么样的 action, 由 $\pi(A_t = a | S_t = s)$ 决定
  • $S_t, A_t \rightarrow R_{t+1}$, 在一个 state 下采取一个 action, 会得到什么 reward, 由 $p(R_{t+1} = r | S_t = s, A_t = a)$ 决定
  • $S_t, A_t \rightarrow S_{t+1}$, 在一个 state 下采取一个 action, 会跳转到什么 state, 由 $p(S_{t+1} = s’ | S_t = s, A_t = a)$ 决定

当然可以把 single-step 扩展到 multi-step 的场景:

$$
\displaylines
{
\begin{aligned}
S_t \overset{A_t}{\rightarrow} R_{t+1},S_{t+1} \overset{A_{t+1}}{\rightarrow} R_{t+2},S_{t+2} \overset{A_{t+2}}{\rightarrow} R_{t+3},S_{t+3}
\end{aligned}
}
$$

将该 trajectory 上所有 reward 累加, 在附上 discounted rate, 就能得到 discounted return:

$$
\displaylines
{
\begin{aligned}
G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + …
\end{aligned}
}
$$

有了上述这些概念, 就能够定义 state value (state value function) 了, 其就是 $G_t$ 的期望值:

$$
\displaylines
{
\begin{aligned}
v_\pi (s) = E[G_t | S_t = s]
\end{aligned}
}
$$

  • 不同的 $s$ 能得到不同的 state value
  • 不同的 policy 会得到不同的 trajectory, 也会得到不同的 state value
  • $v_\pi (s)$ 也可以写为 $v(s, \pi)$

Bellman equation

Bellman equation 描述了不同 state 间 state value 的关系. 主要依赖的推导思想是: 当前 state 的 return 等于 immediate reward 加上下一个时刻的 return.

考虑一个 random trajectory:
$$
\displaylines
{
\begin{aligned}
S_t \overset{A_t}{\rightarrow} R_{t+1},S_{t+1} \overset{A_{t+1}}{\rightarrow} R_{t+2},S_{t+2} \overset{A_{t+2}}{\rightarrow} R_{t+3},S_{t+3}
\end{aligned}
}
$$

其 return $G_t$ 可以写为:
$$
\displaylines
{
\begin{aligned}
G_t &= R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + … \newline\newline
&= R_{t+1} + \gamma(R_{t+2} + \gamma R_{t+3} + …) \newline\newline
&= R_{t+1} + \gamma G_{t+1}
\end{aligned}
}
$$

写成 state value 的定义就有:
$$
\displaylines
{
\begin{aligned}
v_\pi (s) &= E[ G_t | S_t = s] \newline\newline
&= E[ R_{t+1} + \gamma G_{t+1} | S_t = s ] \newline\newline
&= E[ R_{t+1} | S_t = s ] + \gamma E[G_{t+1} | S_t = s]
\end{aligned}
}
$$

之后把两项都解出来就能得到 Bellman equation.

首先是 $E[R_{t+1} | S_t = s]$, 其等于在 $s$ 处采取所有可能的 action 后, 得到的 reward 的期望, 也就是 immediate reward 的 mean 值:

$$
\displaylines
{
\begin{aligned}
E[R_{t+1} | S_t = s] &= \sum_a \pi (a|s) E [R_{t+1} | S_t = s, A_t = a] \newline \newline
&= \sum_a \pi (a|s) \sum_r p(r | s, a) r
\end{aligned}
}
$$

  • $\pi (a|s)$ 是在 $s$ 处采取 action $a$ 的概率
  • 由于在 $s$ 处采取 $a$ 得到的 reward 也是一个概率分布, 因此需要求期望 $E [R_{t+1} | S_t = s, A_t = a]$
  • $p(r|s, a)$ 表示在 $s$ 处采取 $a$ 得到 reward $r$ 的概率

接下来展开第二项 $E[G_{t+1} | S_t = s]$, $G_{t+1}$ 是下一个 state 的 return, 因此会涉及到从 $s$ 到 $s’$ 的概率分布; 而从 $s$ 需要采取某个 action 才能到 $s’$, 因此涉及到从 $s$ 如何选取 action 的概率分布. 该项的本质是求 mean of future rewards:

$$
\displaylines
{
\begin{aligned}
E[G_{t+1} | S_t = s] &= \sum_s’ E[G_{t+1} | S_{t+1} = s’] p(s’|s) \newline \newline
&= \sum_s’ v_\pi (s’) p(s’|s) \newline \newline
&= \sum_s’ v_\pi (s’) \sum_a p(s’ | s,a) \pi (a|s)
\end{aligned}
}
$$

完整的 Bellman equation 如下:

对状态空间中所有 state 列出上述公式, 然后进行求解, 这个过程也称作 policy evaluation, 因为其与 policy 相关.

这里的 $p(r | s,a)$ 和 $p(s’ | s,a)$ 代表 dynamic model.

Bellman equation: Matrix-vector form

先对上述 Bellman equation 做一个变换, 让整个式子看起来简化一些:

$$
\displaylines
{
\begin{aligned}
v_\pi (s) &= \sum_a \pi (a|s) [\sum_r p(r|s,a)r + \gamma \sum_s’ p(s’|s,a) v_\pi (s’)] \newline \newline
v_\pi (s) &= r_\pi (s) + \gamma \sum_s’ p_\pi (s’|s) v_\pi (s’) \newline \newline
其中: r_\pi (s) &\overset{\triangle}{=} \sum_a \pi (a|s) \sum_r p(r|s,a)r \newline \newline
p_\pi (s’|s) &\overset{\triangle}{=} \sum_a \pi (a|s) p(s’|s,a)
\end{aligned}
}
$$

为了写成 matrix-vector 的形式, 还需要为 state 标上号, 写为:

$$
\displaylines
{
\begin{aligned}
v_\pi (s_i) = r_\pi (s_i) + \gamma \sum_{s_j} p_\pi (s_j|s_i) v_\pi (s_j) \newline \newline
\end{aligned}
}
$$

写出所有 $s_i$ 的 bellman equation, 就得到矩阵形式:
$$
\displaylines
{
\begin{aligned}
v_\pi = r_\pi + \gamma P_\pi v_\pi
\end{aligned}
}
$$

  • $v_\pi = [v_\pi (s_1),…,v_\pi (s_n)]^T \in R^n$
  • $r_\pi = [r_\pi (s_1),…,r_\pi (s_n)]^T \in R^n$
  • $P_\pi \in R^{n \times n}$, $[P_{\pi}]_{i j} = p_\pi (s_j | s_i)$, 该矩阵也称 state transition matrix

下面是一个例子:

解上述矩阵的过程就叫 “policy evaluation”.

其 closed-form solution 可以写为:

$$
\displaylines
{
\begin{aligned}
v_\pi = (I - \gamma P_\pi)^{-1} r_\pi
\end{aligned}
}
$$

但实际中一般不用这个求解, 因为维度高时, 求逆的计算量会比较大. 通常用 iterative solution:

$$
\displaylines
{
\begin{aligned}
v_{k+1} = r_\pi + \gamma P_\pi v_k
\end{aligned}
}
$$

  • 注意 $v$ 的下标是 $k$ 和 $k+1$, 与之前的 $\pi$ 并不冲突

这样一直得到一个序列 ${v_0, v_1, v_2,…}$, 我们可以证明, 当 $k \rightarrow \infty$ 时, $v_k$ 就收敛到了 $v_\pi$:

$$
\displaylines
{
\begin{aligned}
v_{k} \rightarrow v_\pi = (I - \gamma P_pi)^{-1} r_\pi, k \rightarrow \infty
\end{aligned}
}
$$

核心的证明思路是证明 $\delta_k = v_k - v_\pi$, 在 $k \rightarrow \infty$ 时, $\delta_k \rightarrow 0$.

Action value

这里说明下 state value 和 action value 的区别:

  • State value 是从一个 state 出发得到的 average return
  • Action value 是从一个 state 出发采取某个 action 后得到的 average return

Action 的选择通常就是根据 action value 来判断, 越大越容易选择. 其定义写为:

$$
\displaylines
{
\begin{aligned}
q_\pi (s,a) = E[G_t | S_t = s, A_t = a]
\end{aligned}
}
$$

  • 在 policy $\pi$ 下, 固定 state 和 action 得到的 return $G_t$ 的期望值

当然可以用 action value 来写 state value 的表达式:

同样, 如果知道了 state value, 也能用来求 action value:

$$
\displaylines
{
\begin{aligned}
q_\pi (s,a) = \sum_r p(r|s,a) r + \gamma \sum_s’ p(s’|s,a) v_\pi (s’)
\end{aligned}
}
$$

Summary

几个关键概念和公式这里汇总下:

  • State value: $v_\pi (s) = E[G_t | S_t = s]$
  • Action value: $q_\pi (s, a) = E[G_t | S_t = s, A_t = a]$
  • The Bellman equation (elementwise form): $v_\pi (s) = \sum_a \pi (a|s) q_\pi (s,a)$
  • The Bellman equation (matrix-vector form): $v_\pi = r_\pi + \gamma P_\pi v_\pi$

Chapter 3: Bellman Optimality Equation

当当前策略不是最优时, 可以用 action value 来判断哪个 action 最优, 进而调整 policy:

$$
\displaylines
{
\begin{aligned}
\pi_{new} (a | s_1) =
\begin{cases}
1, a = a^* \newline \newline
0, a \ne a^*
\end{cases} \newline \newline
where\ a^* = argmax_a\ q_\pi (s_1, a) = a_3
\end{aligned}
}
$$

  • $a^*$ 表示最优的 action

用严格的数学工具说明上述过程的可靠性就是 Bellman Optimality Equation.

Optimal policy

State value 可以用来衡量一个 policy 的好坏, 如果有:

$$
\displaylines
{
\begin{aligned}
v_{\pi_1} (s) \ge v_{\pi_2} (s) \ \ for\ all\ s in S
\end{aligned}
}
$$

即, 对任意状态都有 $\pi_1$ 带来的 state value 更大, 则说明 $\pi_1$ 比 $\pi_2$ 好.

因此正式的定义就是: 如果 policy $\pi^*$ 是 optimal, 那么就有 $v_{\pi^*} (s) \ge v_{\pi}(s)$, 对于任何 $s$ 或任何 policy $\pi$ 都成立.

Bellman optimality equation 定义如下 (elementwise form), 与 Bellman equation 的区别就是加上了 $\max_{\pi}$:

$$
\displaylines
{
\begin{aligned}
v(s) &= \max_{\pi} \sum_a \pi(a|s) (\sum_r p(r|s, a)r + \gamma \sum_{s’} p(s’|s,a)v(s’)),\ \forall s \in S \newline \newline
&= \max_{\pi} \sum_a \pi(a|s) q(s,a)\ \ s \in S
\end{aligned}
}
$$

Matrix-vector form 如下:

$$
\displaylines
{
\begin{aligned}
v = \max_{\pi} (r_\pi + \gamma P_\pi v) \newline \newline
[r_\pi]_s \overset{\Delta}{=} \sum_a \pi (a|s) \sum_r p(r|s,a) r \newline \newline
[P_\pi]_{s,s’} = p(s’|s) \overset{\Delta}{=} \sum_a \pi (a|s) \sum_s’ p(s’ | s,a)
\end{aligned}
}
$$

通常 $p(r|s,a)$ 和 $p(s’|s,a)$ 是已知的, 在给 $v(s’)$ 赋初始值之后, 整个 $q(s,a)$ 都是已知的.

求解 elementwise form, 其实就等价于:

$$
\displaylines
{
\begin{aligned}
\max_{\pi} \sum_a \pi(a|s) q(s,a) = \max_{a \in A(s)} q(s,a)
\end{aligned}
}
$$

也就是直接让最大的 $q(s,a)$ 的概率为 1:

$$
\displaylines
{
\begin{aligned}
\pi(a|s) =
\begin{cases}
1\ a = a^* \newline \newline
0\ a \ne a^* \newline \newline
where\ a^* = arg\ max_a q(s,a)
\end{cases}
\end{aligned}
}
$$

Contraction mapping theorem

这里引入 fixed point 的概念, 对于函数 $f: X \rightarrow X$, 如果有 $x \in X$:

$$
\displaylines
{
\begin{aligned}
f(x) = x
\end{aligned}
}
$$

就称 $x$ 是一个 fixed point.

如果一个 $f$ 是 contraction mapping (contraction function), 需要满足:

$$
\displaylines
{
\begin{aligned}
\vert \vert f(x_1) - f(x_2) \vert \vert \le \gamma \vert \vert x_1 - x_2 \vert \vert
\end{aligned}
}
$$

  • $\gamma \in (0,1)$, 也就是说映射后距离变短了

这里引出 Contraction Mapping Theorem: 对于任何 $x = f(x)$ ($x$ 通过一个以它为参数的函数求得) 这种形式的 equation, 如果 $f$ 是一个 contraction mapping, 那么就会有下面性质:

  • Existence: 存在一个 fixed point $x^*$, 满足 $f(x^*) = x^*$
  • Uniqueness: Fixed point $x^*$ 是 unique 的
  • Algorithm: 考虑序列 ${x_k}$, 且 $x_{k+1} = f(x_k)$, 那么当 $k \rightarrow \infty$ 时, 有 $x_k \rightarrow x^*$. 且 convergence rate 是指数级别的. 也就是说可以用迭代的方法找到 fixed point

对于 Bellman optimality equation, 可以写为:

$$
\displaylines
{
\begin{aligned}
v = f(v) = max_\pi (r_\pi + \gamma P_\pi v)
\end{aligned}
}
$$

其中可以证明 $f(v)$ 是 contraction mapping, $\gamma$ 是 discount rate. 那么我们就能通过迭代找到解.

假设 $v^*$ 是 Bellman Optimality equation 的解, 那么就满足:

$$
\displaylines
{
\begin{aligned}
v^* = \max_\pi (r_\pi + \gamma P_\pi v^*) \newline \newline
\pi^* = arg\ \max_\pi (r_\pi + \gamma P_\pi v^*) \newline \newline
v^* = r_{\pi^*} + \gamma P_{\pi^*} v^*
\end{aligned}
}
$$

可以看出, Bellman Optimality equation 其实就是一种特殊的 Bellman equation, 只不过采取 $\pi^*$ 最优策略, 得到的也是最优 state value $v^*$. 其实就是采取让 $a^*$ 概率最大的策略的 Bellman equation.

Analyzing optimal policies

影响 optimal policy ($\pi(a|s)$) 决策的主要有 3 个 factors:

  • Reward design: $r$
  • System model: $p(r|s,a), p(s’|s,a)$
  • Discount rate: $\gamma$

通常调整 $r$ 和 $\gamma$ 是比较容易的.

Chapter 4 Value Iteration & Policy Iteration

Value Iteration algorithm

$$
\displaylines
{
\begin{aligned}
v_{k+1} = f(v_k) = \max_\pi (r_\pi + \gamma P_\pi v_k), k= 1,2,3…
\end{aligned}
}
$$

这个算法其实叫做 “value iteration”.

需要注意这里的下标, $k$ 表示的不是 “时间步”, 而是 “迭代次数”:

  • $v_k$ 表示当前手里的 “旧的, 不太准确的” 价值表
  • $v_{k+1}$ 表示经过一次 Bellman 算子作用后, “更新的, 更准一点的” 价值表

该式子分两步计算, 会给定一个初始的 $v_0$:

  1. Policy update, 先找到最优的 $\pi$

$$
\displaylines
{
\begin{aligned}
\pi_{k+1} = arg\ max_\pi (r_\pi + \gamma P_\pi v_k)
\end{aligned}
}
$$

  1. Value update, 用找到的 $\pi$ 更新 value

$$
\displaylines
{
\begin{aligned}
v_{k+1} = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_k
\end{aligned}
}
$$

Policy iteration algorithm

给定一个 random initial policy $\pi_0$, 然后分两步迭代:

  1. Policy evaluation (PE), 计算在 $\pi_k$ 下的 state value:

$$
\displaylines
{
\begin{aligned}
v_{\pi_k} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}
\end{aligned}
}
$$

  1. Policy improvement (PI), 更新成新的 policy $\pi_{k+1}$:

$$
\displaylines
{
\begin{aligned}
\pi_{k+1} = arg\ \max_\pi (r_\pi + \gamma P_\pi v_{\pi_k})
\end{aligned}
}
$$

整个过程如下:

$$
\displaylines
{
\begin{aligned}
\pi_0 \overset{PE}{\rightarrow} v_{\pi_0} \overset{PI}{\rightarrow} \pi_1 \overset{PE}{\rightarrow} v_{\pi_1} \overset{PI}{\rightarrow} \pi_2 \overset{PE}{\rightarrow} v_{\pi_2} …
\end{aligned}
}
$$

每次迭代一定有:

$$
\displaylines
{
\begin{aligned}
v_{\pi_0} \le v_{\pi_1} \le v_{\pi_2} \le … \le v^*
\end{aligned}
}
$$

可以通过 Convergence of Policy Iteration theorem 来证明.

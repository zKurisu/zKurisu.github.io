Except Monad

Except Monad 描述可能会抛出异常的计算. 其定义了一套标准的方法, 可以在 “抛出” 错误并在后续流程中 “捕获” 错误, 而不需要手动解构 Either 或其他数据类型.

注意区分 ExceptT 和 throwError with Exception/SomeException and catch. 后者是 GHC 内置的, 且需要包裹在 IO Monad 中使用.

该 library 中的 catching 机制没法捕获 Control.Exception 中抛出的异常, 反之亦然.

  • e 和 m 是两个类型参数. e 表示错误信息, m 代表容器类型
  • m :: Type -> Type 表明 m 是一个类型构造器, 接受一个具体的类型, 返回一个类型
  • | m -> e 是函数依赖, 其告诉编译器, 如果知道了 Monad m, 那么错误类型 e 就唯一确定了

MonadError 最重要的特性是, 当抛出错误时, 如调用 throwError, 计算就会停止; 然后链条中所有后续的 >>= (bind) 操作都会被跳过; 计算会直接跳转到最近的一个处理函数 (通常是 catchError).

如果 m 是 Either e, 那么大部分情况其已经被实现为 MonadError 的 instance 了; 如果想自定义错误类型或用其他的 monad type constructor, 则需要手动定义.

主要要实现下面两个方法:

其接收一个错误值, 并产生一个代表 “失败” 的计算.

其接收一个可能失败的计算, 以及一个错误处理的函数. 如果第一个计算成功, 则返回结果; 如果失败, 则允许处理函数.

Monad Transformer

Monad Transformer 是一种组合多个 Monad 的工具, 因为在现实代码实现中, 单一 Monad 的能力通常是有限的, 比如:

Monad Capability
Maybe 失败/空值
[] 非确定性
IO 副作用
Reader r 只读环境
State s 可变状态

而 Monad Transformer 的核心是把一个 Monad “套进” 另一个 Monad:

这样来看: IO (Maybe a) 通常我们会用 r <- xx 来取出里面的 Maybe a, 我们最终看到的是 Maybe 这一层. 因此 MaybeT IO a 这里的 MaybeT 设计在最外层, 其将 IO 升级为 “带失败处理的 IO”.

常见的 transformer 有:

Transformer Monad Meaning
MaybeT m a m (Maybe a) 可能失败的计算
EitherT e m a m (Either e a) 带错误信息的计算
ReaderT r m a r -> m a 读取环境
StateT s m a s -> m (a, s) 状态
WriterT w m a m (a, w) 日志

mtl

Hackage mtl 文档

MTL 是一个 collection of monad classes, 扩展了 transformer package.

MTL 和 transformers 共享同一套 module 导入, data type 和 function 的名称定义方式, 比如我们有 Foo monad 的 transformer, 那么在 Control.Monad.Foo 中会定义:

  • 一个 typeclass MonadFoo, 其描述了该 transformer 的操作
  • 一个 data type FooT, 实现为所有 monad transformer 的 instance
  • 用 runFooT 来运行 transformed computation

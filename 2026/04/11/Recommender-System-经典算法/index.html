Recommender-System-经典算法

微软开源库 Recommenders

简介

该文档主要记录 recommenders 这个仓库的学习过程, 设计算法原理和代码技巧.

相关概念

这里积累一些推荐算法中的术语. 涉及几个解释方向:

  • 什么是 xxx
  • xxx 对推荐会有什么影响
  • 如何减缓 xxx 的影响

Cold item

Cold item 是指系统中那些几乎没有或完全没有历史交互数据 (如点击, 购买, 评分, 收藏等) 的物品.

这通常被称为 “冷启动问题” (Cold Start Problem). 因为一些推荐系统 (尤其是 collaborative filtering algorithm) 高度依赖历史行为来捕捉物品的特征, 这些缺乏数据的 cold item 就很难被准确地推送给潜在感兴趣的用户.

在下面场景中通常会出现 cold item:

  • 新货上架: 电商平台刚上架的新产品
  • 突发新闻: 新闻资讯类应用刚发布的文章
  • 长尾物品: 一些及其冷门, 长时间无人问津的存量物品

在 Collaborative filter 中, 核心思想通常是 “购买了 A 的人也买了 B”, 说明 A 和 B 是相似的. 但如果 A 是个冷门物品, 没有人买过, 那么算法也就没法找出它的相似物品.

又因为系统倾向于推荐热门物品, 这会导致 cold item 越来越难以得到曝光, 形成恶性循环.

处理 Cold item 的常见思路有:

  • 配合 Content-Based filtering, 既然 item 没有 “行为数据”, 那就利用其本身携带的 “元数据 (metadata)” , 比如一个新电影 (coid item) 的标签是 “科幻, 克里斯托弗诺兰, 星际”, 那么系统可以通过标签进行推荐
  • Exploration & Exploitation. Exploitation 会让推荐系统曝光热物品, 而 Exploitation 时会给冷物品一定的曝光机会
  • Embedding, 利用多模态信息 (比如物品的图片, 文本描述), 通过网络生成出 Embedding, 这样即使没有交互, 也可以用来计算余弦相似性, 找到与冷物品最接近的已有物品

Semi-cold user

Semi-cold User (半冷启动用户), 是介于 “完全陌生” 和 “非常活跃” 之间的一个中间状态. 如果说 Cold User 是系统中完全没有交互的新面孔, 那么 Semi-cold User 就是指那些有极少量交互记录, 但数据量不足以构建准确兴趣模型的用户.

这类用户可能只点击过 1-2 个视频, 或者只买过一件商品, 由于样本量太小, 推荐算法很难判断这些行为是用户的 “真实长期偏好”, 还是偶然的 “误操作” 或 “一次性需求”.

MAP

MAP (Mean Average Precision), 是一个衡量推荐系统性能的常用指标 (系统整体的准确率), 理解这个之前还需要先看 Precision 和 Average Precision 这两个概念 (但用户的准确率).

在推荐场景下, 我们通常会给用户一个包含 K 个项的列表:

$$
\displaylines
{
\begin{aligned}
Precision@K = \frac{前 K 个结果中相关物品的数量}{K}
\end{aligned}
}
$$

它只关心 “有多少是相关的”, 而不关心这些相关的物品排在第几位.

而 AP (Average Precision) 是针对某一个 user 或 item 而言, 其将 Ranking (排名) 的先后顺序考虑了进去, 它会计算不同位置的 precision 并取平均:
$$
\displaylines
{
\begin{aligned}
AP = \frac{\sum_{k=1}^n ( P(k) \times rel(k) )}{相关物品总数}
\end{aligned}
}
$$

  • $P(k)$ 是前 k 个结果的 precision
  • $rel(k)$ 是一个指示函数, 如果第 k 个结果是相关的, 则为 1, 否则为 0

比如这个示例: 返回的 top-k 有 5 个物品 (K=5), 用户点击了排名第 1, 3, 5 的物品 (也就是有 3 个相关项):

  • 在 k=1 处, 相关, 有 $P(1) = 1/1 = 1.0$
  • 在 k=2 处, 不相关, 跳过 (因为 rel(2) = 0)
  • 在 k=3 处, 相关, 有 $P(3) = 2/3 \approx 0.67$
  • 在 k=4 处, 不相关, 跳过
  • 在 k=5 处, 相关, 有 $P(5) = 3/5 = 0.60$

因此总的是 $AP = (1.0 + 0.67 + 0.60)/3 = 0.756$, 同时也可以看出, 如果相关物品排在更靠前的位置, AP 值会显著提高.

而 MAP 则是对所有用户进行 AP 计算, 然后取平均值. 假如系统中有 Q 个用户, 每个用户的平均精度为 $AP_q$, 则有:

$$
\displaylines
{
\begin{aligned}
MAP = \frac{1}{Q} \sum_{q=1}^Q AP_q
\end{aligned}
}
$$

SAR

SAR (Simple Algorithm for Recommendation) 是一个简单且扩展性强的算法, 主要基于用户的 transaction history, 能够处理 “cold item” 和 “semi-cold user” 场景.

SAR 是 neighborhood based algorithm, 会排序检索出 top items for each user.

SAR 的核心思路是: 如果一个用户与一个 item 有交互, 则称 user has an affinity to an item. 假如两个 items 相似, 用户对其中一个有 affinity, 那么就很可能对另一个 item 也有 affinity. 也就是一种 collaborative filtering 思路, 如果一个用户喜欢一个物品, 那么他很可能会喜欢与该物品相似的另一个物品.

SAR 算法中会计算 co-occurence matrix. Co-occurence 的意思是, 两个 items 同时出现在一个用户的交互序列中, 而同时出现意味着它们可能是相似的. 而 Co-occurence matrix 则会统计物品间两两同时出现的次数. 之后利用这个矩阵来计算 item similarity matrix (通过缩放 co-occurence, 总体感觉就是一起出现的次数越多, 就越相似).

还会计算 affinity matrix, 即每个 user 与每个 item 之间的 strength of relationship (比如通过 user 对 item 的评分之类的来计算).

用 affinity matrix $A$ 乘以 similarity matrix $S$ 就能得到推荐结果, 一个 recommendation score matrix $R$, 然后会算出 top-k 的结果.

根据 co-occurence matrix 算 similarity matrix

前面讲了从 co-occurence matrix 通过放缩就能变为 similarity matrix, 有 3 种放缩策略:

  • Jaccard: $s_{ij} = \frac{c_{ij}}{c_{ii} + c_{jj} - c_{oj}}$. $s_{ij}$ 表示 similarity matrix 中的一个位置, $c_{ij}$ 表示 co-occurence matrix 中对应的一个位置, $c_{ii}$ 和 $c_{jj}$ 都是 co-occurence 中的对角线位置上的值, 比较符合直觉的是, 自己和自己同时出现的次数肯定是最大的 (只要出现了就算)
  • lift: $s_{ij} = \frac{c_{i j}}{c_{ii} \times c_{jj}}$
  • counts: $s_{ij} = c_{ij}$

计算 user affinity scores

Affinity 受下面两个 factors 的影响:

  • 不同类型的 interaction events 有不同的 weight
  • Event 发生的时间, 比如很久很久以前的点击 event 的分数会比刚刚的点击事件低

具体的计算公式如下:
$$
\displaylines
{
\begin{aligned}
a_{i j} = \sum_k \omega_k (\frac{1}{2})^{\frac{t_0 - t_k}{T}}
\end{aligned}
}
$$

  • $a_{ij}$ 表示 user i 对 item j 的 affinity score
  • $k$ 是事件的总类数, $\omega_k$ 是该类型时间的 weight
  • $\frac{1}{2}^{\frac{t_0 - t_k}{T}}$ 用来根据时间进行缩放, $t_0$ 是当前时间, $t_k$ 是时间发生时间. $T$ 是一个常数, 用来控制衰减的速度, 如果 $t_0 - t_k = T$, 则称为半衰期, 如果 $T$ 设置得很大, 则分数衰减慢, 此时系统更看重用户的长期兴趣; 而 $T$ 设置很小, 此时分数会迅速减小, 更看重实时兴趣

假如有 n 个用户, m 个 items, 则会得到一个 n x m 的矩阵.

ALS

Matrix factorization by ALS (Alternating Least Squares) 是一种 collaborative filtering algorithm.

这里要先了解一些数学基础.

秩 (Rank) 用来衡量矩阵中包含的 “非冗余信息” 的量. 一个包含信息的矩阵, 如果数据不是随机生成的, 那么它内部通常存在某种关联性, 比如一个 3x3 的矩阵:

  • 如果 rank 为 3, 说明 3 个向量 (一行/一列是一个向量) 都指向不同的方向, 此时它们可以 “撑起” 整个三维空间, 可以用它们的线性组合到达空间中的任何一个点
  • 如果 rank 为 2, 说明 3 个向量中, 有一个能通过另外两个相加或缩放得到 (共面), 相当于这个向量的信息是无效的, 已经包含在其他向量中了, 此时它们就只能撑起一个 “二维空间”
  • 如果 rank 为 1, 说明 3 个向量都指向一个方向 (共线), 此时只能撑起一条 “直线”

从线性代数的术语来讲, rank 是矩阵中线性无关的行 (或列) 的最大数量. 如果一行可以通过其他行的加减乘除组合出来, 那这一行就是 “多余” 的:

  • 满秩 (Full Rank), 所有的行都是独立的, 没有任何信息是重复的
  • 秩亏 (Rank Deficient): 矩阵中存在冗余

比如:
$$
\displaylines
{
\begin{aligned}
A = \begin{bmatrix}
1 & 2 \newline \newline
2 & 4
\end{bmatrix}
\end{aligned}
}
$$

第二行恰好是第一行的 2 倍, 这意味着第二行没有提供任何新信息, 因此这个矩阵的 rank 就是 1, 尽管它是一个 2 x 2 矩阵.

去除一个高维矩阵中冗余信息的一个方法就是矩阵分解 (低秩分解), 如果 mxn 矩阵的秩 r 远小于其维度, 即 $r << m,n$, 意味着这个矩阵可以用两个更小的矩阵相乘来精确表达:

$$
\displaylines
{
\begin{aligned}
A_{m \times n} \approx U_{m \times k} \times V_{k \times n}
\end{aligned}
}
$$

  • 这里的 k 就是我们选定的 rank
  • 原矩阵需要存储 m x n 个元素, 而分解后只需要存储 (m + n) x k 个元素, 当 k 很小时, 能实现极大的数据压缩
  • $A$ 矩阵的每一行维度是 n, $U$ 就表明将每一行投影到了 k 维上 (高维投影到低维); $V$ 矩阵相当于对 $U$ 的每一行做线性变换, 近似还原出 $A$
  • $U$ 的每一行也可以称为原始向量的 “隐向量”

此时回到推荐系统的场景, 假设有一个巨大的表格 (评分矩阵 $R$), 行代表用户 (Users), 列代表物品 (Items). 表格中的每个单元格是用户对物品的评分. 用户肯定是没办法看完全部电影的, 因此这张表的绝大多数地方都是空白的. 此时就能够通过矩阵分解来压缩 $R$ 的维度, 寻找两个维度更小的矩阵 $P$ (用户矩阵) 和 $Q$ (物品矩阵), 使得它们的乘积尽可能地还原原始矩阵 $R$:

$$
\displaylines
{
\begin{aligned}
R \approx P \times Q^T
\end{aligned}
}
$$

  • $P$ 的每一行代表一个用户的 “特征向量”
  • $Q$ 的每一行代表一个物品的 “特征向量”
  • 这里的特征都不是显性的 (比如导演, 题材等自己加上的标签), 而是算法自动学习出来的 “隐向量”, 表示某种特征

将大矩阵拆为两个小矩阵的好处在于, 原本的稀疏矩阵中, 用户与某个电影很可能没有交互, 因此无法计算相关分数. 而拆分后的 “隐向量” 中包含某种关联, 可以计算出一个 “预测分”.

通常损失函数的定义是让 $P \times Q^T$ 与已知评分的差值最小 (预测分与实际分数的差值越小, 说明预测越准). 由于 $P$ 和 $Q$ 都是未知的需要求解, 如果要同时优化它们, 方程很难直接求解, 因此有了 ALS, 其核心思路是: 固定一个矩阵, 求解另一个矩阵, 然后交换后继续求解, 直到两个矩阵的数值收敛:

  1. 固定 $Q$, 优化 $P$. 假设物品的特征向量是已知的, 此时问题退化为一个简单的 “最小二乘法” 线性回归问题, 此时能通过解析解算出每个用户的最佳特征向量
  2. 固定 $P$, 优化 $Q$. 拿到更新后的用户向量后, 反过来假设用户是已知的, 再优化物品向量
  3. 重复迭代, 直到两个矩阵的数值趋于稳定

这里回顾下最小二乘法, “最小” 指的是误差最小, “二乘” 指计算式子里的平方项. 其核心目标是通过让误差的平方和最小, 来找到一个最合适的直线 (或平面).

假如你在纸上画了很多个点, 这些点大致排成一条线, 但并不完全笔直, 此时想用一条直线来代表这些点的趋势. 由于无论怎么画, 都无法让一条直线经过所有的点, 因此只能找到一条近似的直线. 衡量这条直线的好坏, 就是算各个点与直线的距离, 然后取平方 (消除正负号的影响, 并且放大较大的误差) 后相加. 通过最小化平方和, 来调整直线的位置.

用数学语言表达就是, 假设有一组观测数据 $(x_i, y_i)$, 找一个线性函数 $f(x) = wx + b$ 来拟合, 用最小二乘法的损失函数就是:

$$
\displaylines
{
\begin{aligned}
S = \sum_{i=1}^n (y_i - f(x_i)^2) = \sum_{i=1}^n (y_i - (wx_i + b))^2
\end{aligned}
}
$$
之后通过反向梯度传播, 来更新 $w$ 和 $b$.

解析解 (Analytical solution) 这个概念也经常用到, 其也叫做封闭解 (Closed-form solution), 是指通过数学公式, 代数运算和已知的数学函数 (如 sin, cos, log, exp 等), 直接推导出的解. 特点是只要把变量代入公式, 就能立刻算出精确结果.

比如求解一个一元二次方程 $ax^2 + bx + c = 0$, 其解析解就是 $x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}$, 不管输入多复杂, 总能得到精确答案.

与之对应的是数值解 (Numerical solution), 是一组近似解, 存在计算误差, 通过迭代来拟合, 好处就是普适性很强, 任何问题都能找出一个近似值.

ALS 中的每一步都能算出当前时候的最优解, 由于固定了一个矩阵, 可以算出解析解: $\hat{x} = (A^T A)^{-1} A^T b$.

RBM

Joint probability distribution

先介绍一个数学概念: joint probability distribution, 其核心在于, 不再只关注一个随机变量, 而是关注两个或多个随机变量在同一实验或同一时刻同时发生的概率.

比如有两个随机变量 $X$ 和 $Y$:

  • 对于单一分布, 回答 “X 等于某个值的概率是多少”
  • 对于联合分布, 回答 “X 等于 a 且 Y 等于 b 的概率是多少”

如果是离散型变量, 通常用 Joint PMF 来表示:

$$
\displaylines
{
\begin{aligned}
P(X = x, Y = y)
\end{aligned}
}
$$

对于连续型变量, 用 Joint PDF 表示:

$$
\displaylines
{
\begin{aligned}
f(x,y)
\end{aligned}
}
$$

Chain Rule

在概率论中有一个 Chain Rule, 对于任何两个随机变量, $P(A,B)$ 都可以拆解为 “先发生 A 的概率” 乘以 “在 A 发生条件下 B 发生的概率”, 有:

$$
\displaylines
{
\begin{aligned}
P(v,h) = P(v|h) P(h) = P(h|v) P(v)
\end{aligned}
}
$$
(条件概率的计算式子移个项就是上面式子)

Partition Function

Partition Function (配分函数), 通常用符号 $Z$ 表示, 是一个源自物理学 (统计力学) 的核心概念. 在机器学习中, 可以把它视为 “归一化系数”, 直观理解其就是概率的分母.

比如扔一个骰子, 其点数 1~6 分别对应不同的分数: 10, 20, 30, 40, 50, 60. 这些分数的总和并不等于 1 (和是 210), 所以它们不是概率值. 为了把这些分数转换成真正的概率, 需要用分数除以总和. 而这个总和, 就是这个系统的 partition function $Z$:

$$
\displaylines
{
\begin{aligned}
P(i) = \frac{score_i}{Z}
\end{aligned}
}
$$

在 RBM 这种基于能量的系统中, 每个状态 (v,h) 对应一个能量 $E(v,h)$, 而在计算概率时, 需要将能量写作 $e^{-E}$ 的形式. 因此概率计算为:

$$
\displaylines
{
\begin{aligned}
P(v, h) = \frac{1}{Z} e^{-E(v,h)} \newline \newline
Z = \sum_{v,h} e^{-E(v,h)}
\end{aligned}
}
$$

在物理和信息论中, 通常希望系统是可组合的:

  • 能量能够对称相加: 如果你有两个独立的系统 A 和 B, 总能量应该是 $E_{total} = E_A + E_B$
  • 概率时独立相乘的: 如果两个系统相互独立, 它们同时发生的概率应该是 $P_{total} = P_A \times P_B$

而数学上能够把 “加法” 映射到 “乘法” 的, 只有指数函数:

$$
\displaylines
{
\begin{aligned}
e^{-(E_A + E_B)} = e^{-E_A} \times e^{-E_B}
\end{aligned}
}
$$

这里添加负号的意义在于, 能量越低 (越稳定), $e^{-E}$ 越大. 这符合物理常识, 即系统倾向于停留在低能量状态. 比如山谷里的石头, 大部分时间会停留在地势最低的地方 (地能量), 偶尔会因为扰动出现在高处.

也能通过最大熵原理 (Principle of Maximum Entropy) 来解释为什么这样计算. 假设想为一个系统建立概率模型, 但你只知道一些观察到的特征 (比如平均能量), 而对其他一无所知. 此时为了不引入主观偏见, 应该选择一个 “最随机 (熵最大)” 的分布. 而可以通过数学证明 (通过拉格朗日乘子法) 得到, 在给定平均能量限制下, 使熵最大化的概率分布形式一定是指数形式: $P \propto e^{-\lambda E}$.

Partition function 的一个重要特征时, 对其对数求导能得到概率的期望值. 假设这里 $Z = \sum_s e^{b \cdot s}$, $s$ 是状态, $b$ 是参数:

  1. 直接求导有: $\frac{\partial Z}{\partial b} = \sum_s s \cdot e^{b \cdot s}$
  2. 如果是对其对数求导:

$$
\displaylines
{
\begin{aligned}
\frac{\partial log Z}{\partial b} = \frac{1}{Z} \cdot \frac{\partial Z}{\partial b} = \frac{\sum_s s \cdot e^{b \cdot s}}{Z} = \sum_s s \cdot \frac{e^{b \cdot s}}{Z}
\end{aligned}
}
$$

这里的 $\frac{e^{b \cdot s}}{Z}$ 恰好是一个 $s$ 对应的概率, 而结果整体表示 $\sum_s s \cdot P(s)$ 是期望值的定义.

Boltzmann Distribution

Boltzmann distribution (玻尔兹曼分布), 在数学上也常被称为吉布斯分布 (Gibbs Distribution), 是统计力学和机器学习中描述系统状态分布的最基本公式. 其回答了一个核心问题: 在一个处于平衡状态的系统中, 某个特定状态出现的概率是多少.

比如在一个处于特定温度 $T$ 下的系统, 状态 $i$ 出现的概率 $P_i$ 取决于该状态的能量 $E_i$:

$$
\displaylines
{
\begin{aligned}
P_i = \frac{1}{Z}e^{-\frac{E_i}{k}}
\end{aligned}
}
$$

  • $E_i$ 是该状态下的能量. 能量越低, 概率越高
  • $k$ 是玻尔兹曼常数, 是一个用来连接微观能量与宏观温度的常数
  • $T$ 指系统所处的环境温度
  • $Z$ partition function, 是归一化常数, 确保所有概率之和为 1. 即 $Z = \sum_j e^{e-\frac{E_j}{k}}$

在机器学习中, 通常简化 $k$ 为 1.

Hamiltonian

在 RBM 语境下, Hamiltonian (哈密顿量) 指代系统的总能量函数 $E(v,h)$. 在统计物理中, Hamiltonian 描述系统的总能量; 而在 RBM 中, 它描述了 visible layer 和 hidden layer 特定状态组合的 “不和谐度”.

对于一组状态 $(v,h)$, 其 Hamiltonian 通常写为:

$$
\displaylines
{
\begin{aligned}
H(v,h) = -\sum_i a_i v_i - \sum_j b_j h_j - \sum_{i,j} v_i w_{i j} h_j
\end{aligned}
}
$$

式子由三部分组成:

  • $-\sum a_i v_i$, 表示 visible unit $v_i$ 自带的能量倾向 ($a_i$ 是一个偏置项). 其反应数据本身的先验分布, 比如推荐系统中某个电影非常热门, 那么其 $a_i$ 就会很大. 加上符号也是希望让能量低的概率高
  • $-\sum b_j h_j$. 表示 hidden 特征的自己激活倾向. 反应隐藏特征出现的频率
  • $-\sum v_i w_{i j} h_j$, 描述 visible layer 和 hidden layer 之间的耦合能量, 用来衡量 $v_i$ 和 $h_j$ 的一致性. 如果 $w_{i j}$ 是正数, 当 $v_i = 1$ 且 $h_j = 1$ 同时发生时, 能量会大幅下降 (因为加上了负号), 这意味着模型 “喜欢” 它们同时出现; 反之

Gibbs Sampling

Gibbs Sampling 是一种特殊的马尔可夫链蒙特卡洛算法, 其核心作用是: 当你很难直接从一个复杂的联合概率分布 $P(x_1, x_2, …, x_n)$ 中进行采样时, 它可以让你通过不断地从简单的条件概率分布中采样, 最终得到符合联合分布的样本.

想象你要找到从一个洒满了点的二维平面中选择一个点的概率, 这可能比较复杂, 不知道从哪里下手. 但如果你站在平面的某条线上, 找到沿着该线的某点的概率, 就会清楚一些.

Gibbs sampling 的做法是:

  1. 随机给所有变量赋一个初始值
  2. 固定住其他所有变量, 只根据条件概率更新其中一个变量的值
  3. 轮流对所有变量执行这个操作
  4. 重复多次后, 变量的状态组合就会趋近于真实的联合分布

Idea

RBM (Restricted Boltzmann Machine) 是一个 generative neural network model, 通常用 unsupervised learning 方式训练. 从结构上来看, 它是一个二分图, 包含两层神经元:

  • Visible layer, $v$ (可视层): 对应我们能观测到的数据 (即喂给模型的真实数据, 在推荐系统中, 通常是用户的评分或点击行为)
  • Hidden layer, $h$ (隐藏层): 对应数据背后的 “隐含特征” (即模型能捕捉到的隐藏模式, 比如电影的类型, 导演风格, 演员组合等)

Restricted 的意思是, 上述两层的神经元之间没有连接. 这意味着给定可视层时, 隐藏层的每个单元是条件独立的; 反之亦然, 这种结构能极大简化计算.

RBM 的目标是学习到一个 joint probability distribution $P(v, h)$. 此时模型能够理解什么样的隐藏特征 ($h$) 组合会产生什么样的观测数据 ($v$), 比如给定一个电影的隐藏特征, 其能够通过 $P(v|h)$ 在可见层 $v$ 给出用户评分.

RMB 的核心思想是用物理学中的能量模型来描述变量之间的相关性. 其为每一种 $(v,h)$ 的状态组合赋予一个能量值, 公式如下:

$$
\displaylines
{
\begin{aligned}
E(v,h) = -\sum_i a_i v_i - \sum_j b_j h_j - \sum_{i, j}v_i w_{i j} h_j
\end{aligned}
}
$$
能量越低, 发生的概率越大. 模型训练的目的就是让观察到的样本 (用户的真实行为) 处于能量最低点, 即发生的概率最大.

在推荐场景下, RBM 通过隐藏层 h 来解释用户的评分行为:

  • 前向过程 ($v \rightarrow h$): 将用户的评分映射为一组隐藏特征
  • 反向过程 ($h \rightarrow v$): 利用隐藏特征重新 “重建” 评分. 如果重建的评分与原始评分一致, 说明模型抓住了数据的本质规律

将 RBM 用于 collaborative filtering 时, 思路通常为:

  1. 输入: 每一个用户对应一个 RBM, 可视层单元的数量等于电影数量, 输入是该用户对电影的评分
  2. 处理缺失值: 由于用户只看过极少数电影, RBM 只对用户评价过的电影节点进行参数更新, 忽略未评价的节点
  3. 预测评分: 通过已有的评分激活隐藏层, 再由隐藏层反向预测用户每看过 (缺失) 的电影评分

具体涉及的公式下面说明.

这里先说明一个技巧:
$$
\displaylines
{
\begin{aligned}
(1 + f(x))(1 + f(y)) = 1 + f(x) + f(y) + f(x)f(y)
\end{aligned}
}
$$
假设 x 和 y 是系统所有可能的状态, f(x) 是 x 的贡献; f(y) 是 y 的贡献. $(1 + f(x))(1 + f(y))$ 这个表达式写出了所有 x 和 y 的组合:

  • 1: x 和 y 都不发生时对系统的贡献
  • f(x): 只有 x 发生时对系统的贡献
  • f(y): 只有 y 发生时对系统的贡献
  • f(x)f(y): 当 x 和 y 同时发生时对系统的贡献

由上面技巧, 计算给定 $v$ 和偏执 $b$ 时 (Positive phase) 所有 $h$ 的组合的贡献可以写为:

$$
\displaylines
{
\begin{aligned}
Z[v,b] = \product_j \sum_{h_j = 0,1} e^{(\sum_i w_{i j}v_i + b_j) h_j} = \product_j (1 + e^{\sum_i w_{i j} v_i + b_j})
\end{aligned}
}
$$

  • $Z[v,b]$ 既然表示所有组合的贡献, 因此就是 partition function
  • 先说内部的 $\sum$, 每个 $h_j$ 有 0 (不激活) 和 1 (激活) 两种状态, $w_{i j} v_i + b_j$ 是一个 v_i 经过线性变换后的结果, $\sum_i w_{i j} v_i + b_j$ 是累加整个 visible layer 的线性变换结果. 累加 0 和 1 两个状态的结果就能得到单个 $h_j$ 的贡献
  • 对于变换后的式子, $e^{\sum_i w_{i j} v_i + b_j}$ 表示 $h_j$ 激活对系统的贡献, 累乘能得到所有 $h_j$ 激活/不激活的组合

如果对 $Z$ 的 log 求某一个激活神经元 $h_j$ 的偏置的导数, 就能够得到其平均激活程度 (前面可以看到, 求 $Z$ 的 log 导数恰好是求概率的期望):

$$
\displaylines
{
\begin{aligned}
\frac{\partial}{\partial b_j} log Z[v,b] = \frac{1}{1 + e^{-(\sum_i w_{i j} v_i + b_j)}} = \sigma(\phi_j (v,b))
\end{aligned}
}
$$

其中 $\phi_j (v,b) = \sum_i w_{i j} v_i + b_j$. 也就是说神经元的 “平均激活概率” 的计算实际上可以直接用 sigmoid 函数来算.

对于 Negative phase, 固定刚才采样得到的 $h$, 来求 $P(v_i = q | h)$, 比如 $q = 1,…,5$:

$$
\displaylines
{
\begin{aligned}
P(v_i = q | h,a) = \product_{v_i = 1}^q e^{v_i (\sum_j w_{i j} h_j + a_i)} / Z_q
\end{aligned}
}
$$

  • 对于一个 visible layer 的神经元 $v_i$, 要计算 5 个可能分数的概率
  • 这里其实是 one-hot 的写法, $v_i$ 是一个 5 维的向量, 每次只有一个位置为 1, 比如 $v_i = 1$ 其实是 [1, 0, 0, 0, 0], 即第一个位置置 1, 其他位置置 0
  • $Z_q$ 是 partition function, 作为归一化分母
  • 整体相当于 softmax 函数

由上述式子能够得到新的 $v_i$, 之后又固定这个 $v_i$ 来更新 h, 这样交替进行.

在经过 k 次迭代后, 我们通过初始 free energy 和此时 free energy 的差来求梯度并反向更新参数:

$$
\displaylines
{
\begin{aligned}
F = - log Z = -log \sum_{v_i,h_i} P(v,h) \newline \newline
\Delta F = F_0 - F_k \newline \newline
\frac{\partial}{\partial b_j} \Delta F = \frac{\partial}{\partial b_j} (log Z_0 [v,b] - log Z_k[v,b]) = P_0 (h_j = 1 | v,b) - P_k(h_j = 1 | v,b) \newline \newline
\frac{\partial}{\partial w_{i j}} \Delta F = v_i P_0 (v_i = q|h,a) - v_i P_k (v_i | h,a) = \langle v_i \rangle_0 - \langle v_i \rangle_k
\end{aligned}
}
$$

  • $\langle v_i \rangle_0$ 表示初始分布采样出的 $v_i$
  • $\langle v_i \rangle_k$ 表示经过 k 次更新迭代后的分布采样出的 $v_i$

当 $\Delta F = 0$ 时, 说明学习到的 distribution 已经完全符合输入了.

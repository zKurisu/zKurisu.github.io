Recommender-System-经典算法

微软开源库 Recommenders

简介

该文档主要记录 recommenders 这个仓库的学习过程, 设计算法原理和代码技巧.

相关概念

这里积累一些推荐算法中的术语. 涉及几个解释方向:

  • 什么是 xxx
  • xxx 对推荐会有什么影响
  • 如何减缓 xxx 的影响

Cold item

Cold item 是指系统中那些几乎没有或完全没有历史交互数据 (如点击, 购买, 评分, 收藏等) 的物品.

这通常被称为 “冷启动问题” (Cold Start Problem). 因为一些推荐系统 (尤其是 collaborative filtering algorithm) 高度依赖历史行为来捕捉物品的特征, 这些缺乏数据的 cold item 就很难被准确地推送给潜在感兴趣的用户.

在下面场景中通常会出现 cold item:

  • 新货上架: 电商平台刚上架的新产品
  • 突发新闻: 新闻资讯类应用刚发布的文章
  • 长尾物品: 一些及其冷门, 长时间无人问津的存量物品

在 Collaborative filter 中, 核心思想通常是 “购买了 A 的人也买了 B”, 说明 A 和 B 是相似的. 但如果 A 是个冷门物品, 没有人买过, 那么算法也就没法找出它的相似物品.

又因为系统倾向于推荐热门物品, 这会导致 cold item 越来越难以得到曝光, 形成恶性循环.

处理 Cold item 的常见思路有:

  • 配合 Content-Based filtering, 既然 item 没有 “行为数据”, 那就利用其本身携带的 “元数据 (metadata)” , 比如一个新电影 (coid item) 的标签是 “科幻, 克里斯托弗诺兰, 星际”, 那么系统可以通过标签进行推荐
  • Exploration & Exploitation. Exploitation 会让推荐系统曝光热物品, 而 Exploitation 时会给冷物品一定的曝光机会
  • Embedding, 利用多模态信息 (比如物品的图片, 文本描述), 通过网络生成出 Embedding, 这样即使没有交互, 也可以用来计算余弦相似性, 找到与冷物品最接近的已有物品

Semi-cold user

Semi-cold User (半冷启动用户), 是介于 “完全陌生” 和 “非常活跃” 之间的一个中间状态. 如果说 Cold User 是系统中完全没有交互的新面孔, 那么 Semi-cold User 就是指那些有极少量交互记录, 但数据量不足以构建准确兴趣模型的用户.

这类用户可能只点击过 1-2 个视频, 或者只买过一件商品, 由于样本量太小, 推荐算法很难判断这些行为是用户的 “真实长期偏好”, 还是偶然的 “误操作” 或 “一次性需求”.

MAP

MAP (Mean Average Precision), 是一个衡量推荐系统性能的常用指标 (系统整体的准确率), 理解这个之前还需要先看 Precision 和 Average Precision 这两个概念 (但用户的准确率).

在推荐场景下, 我们通常会给用户一个包含 K 个项的列表:

$$
\displaylines
{
\begin{aligned}
Precision@K = \frac{前 K 个结果中相关物品的数量}{K}
\end{aligned}
}
$$

它只关心 “有多少是相关的”, 而不关心这些相关的物品排在第几位.

而 AP (Average Precision) 是针对某一个 user 或 item 而言, 其将 Ranking (排名) 的先后顺序考虑了进去, 它会计算不同位置的 precision 并取平均:
$$
\displaylines
{
\begin{aligned}
AP = \frac{\sum_{k=1}^n ( P(k) \times rel(k) )}{相关物品总数}
\end{aligned}
}
$$

  • $P(k)$ 是前 k 个结果的 precision
  • $rel(k)$ 是一个指示函数, 如果第 k 个结果是相关的, 则为 1, 否则为 0

比如这个示例: 返回的 top-k 有 5 个物品 (K=5), 用户点击了排名第 1, 3, 5 的物品 (也就是有 3 个相关项):

  • 在 k=1 处, 相关, 有 $P(1) = 1/1 = 1.0$
  • 在 k=2 处, 不相关, 跳过 (因为 rel(2) = 0)
  • 在 k=3 处, 相关, 有 $P(3) = 2/3 \approx 0.67$
  • 在 k=4 处, 不相关, 跳过
  • 在 k=5 处, 相关, 有 $P(5) = 3/5 = 0.60$

因此总的是 $AP = (1.0 + 0.67 + 0.60)/3 = 0.756$, 同时也可以看出, 如果相关物品排在更靠前的位置, AP 值会显著提高.

而 MAP 则是对所有用户进行 AP 计算, 然后取平均值. 假如系统中有 Q 个用户, 每个用户的平均精度为 $AP_q$, 则有:

$$
\displaylines
{
\begin{aligned}
MAP = \frac{1}{Q} \sum_{q=1}^Q AP_q
\end{aligned}
}
$$

SAR

SAR (Simple Algorithm for Recommendation) 是一个简单且扩展性强的算法, 主要基于用户的 transaction history, 能够处理 “cold item” 和 “semi-cold user” 场景.

SAR 是 neighborhood based algorithm, 会排序检索出 top items for each user.

SAR 的核心思路是: 如果一个用户与一个 item 有交互, 则称 user has an affinity to an item. 假如两个 items 相似, 用户对其中一个有 affinity, 那么就很可能对另一个 item 也有 affinity. 也就是一种 collaborative filtering 思路, 如果一个用户喜欢一个物品, 那么他很可能会喜欢与该物品相似的另一个物品.

SAR 算法中会计算 co-occurence matrix. Co-occurence 的意思是, 两个 items 同时出现在一个用户的交互序列中, 而同时出现意味着它们可能是相似的. 而 Co-occurence matrix 则会统计物品间两两同时出现的次数. 之后利用这个矩阵来计算 item similarity matrix (通过缩放 co-occurence, 总体感觉就是一起出现的次数越多, 就越相似).

还会计算 affinity matrix, 即每个 user 与每个 item 之间的 strength of relationship (比如通过 user 对 item 的评分之类的来计算).

用 affinity matrix $A$ 乘以 similarity matrix $S$ 就能得到推荐结果, 一个 recommendation score matrix $R$, 然后会算出 top-k 的结果.

根据 co-occurence matrix 算 similarity matrix

前面讲了从 co-occurence matrix 通过放缩就能变为 similarity matrix, 有 3 种放缩策略:

  • Jaccard: $s_{ij} = \frac{c_{ij}}{c_{ii} + c_{jj} - c_{oj}}$. $s_{ij}$ 表示 similarity matrix 中的一个位置, $c_{ij}$ 表示 co-occurence matrix 中对应的一个位置, $c_{ii}$ 和 $c_{jj}$ 都是 co-occurence 中的对角线位置上的值, 比较符合直觉的是, 自己和自己同时出现的次数肯定是最大的 (只要出现了就算)
  • lift: $s_{ij} = \frac{c_{i j}}{c_{ii} \times c_{jj}}$
  • counts: $s_{ij} = c_{ij}$

计算 user affinity scores

Affinity 受下面两个 factors 的影响:

  • 不同类型的 interaction events 有不同的 weight
  • Event 发生的时间, 比如很久很久以前的点击 event 的分数会比刚刚的点击事件低

具体的计算公式如下:
$$
\displaylines
{
\begin{aligned}
a_{i j} = \sum_k \omega_k (\frac{1}{2})^{\frac{t_0 - t_k}{T}}
\end{aligned}
}
$$

  • $a_{ij}$ 表示 user i 对 item j 的 affinity score
  • $k$ 是事件的总类数, $\omega_k$ 是该类型时间的 weight
  • $\frac{1}{2}^{\frac{t_0 - t_k}{T}}$ 用来根据时间进行缩放, $t_0$ 是当前时间, $t_k$ 是时间发生时间. $T$ 是一个常数, 用来控制衰减的速度, 如果 $t_0 - t_k = T$, 则称为半衰期, 如果 $T$ 设置得很大, 则分数衰减慢, 此时系统更看重用户的长期兴趣; 而 $T$ 设置很小, 此时分数会迅速减小, 更看重实时兴趣

假如有 n 个用户, m 个 items, 则会得到一个 n x m 的矩阵.

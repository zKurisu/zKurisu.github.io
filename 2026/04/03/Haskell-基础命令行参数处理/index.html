Haskell-基础命令行参数处理

Hackage System.Console.GetOpt

简介

System.Console.GetOpt 模块绑定了 GNU getopt lib, 用于提供简单的命令行参数处理, 注意这里只是做 parse, 捕获命令行参数用 System.Environment 中的 getArgs 函数.

数据结构

data OptDescr a 用来描述一个 option 的信息, 便于 usageInfo 打印或者 getOpt 进行解析, 其提供一个 constructor:

  • [Char], short option 列表. 比如 ['h', 'H'], 解析时对应 -h 和 -H
  • [String], long option 列表. 比如 ["help", "HELP"], 解析时对应 --help 和 --HELP
  • ArgDescr a, 描述该 Option 对应的参数信息, 有 3 个 constructors:
    • NoArg a, 表示该 Option 不需要接收参数
    • ReqArg (String -> a) String, 表示该 Option 必须接受一个参数, String -> a 表示如何将参数包裹到 a 中, String 是提示信息
    • OptArg (Maybe String -> a) String, 类似 ReqArg, 但可以没有参数

usageInfo 函数能根据 [OptDescr a] 来打印命令行常见的 usage 信息 (参数加上描述信息).

getOpt 函数能根据 [OptDescr a] 和实际输入的 [String], 利用 ArgDescr a 来取出 Flag 类型的数据.

data ArgOrder a 是传给 getOpt 的一个类型, 用来指示如何处理 options 和 non-options, 有 3 个 constructors:

  • RequireOrder, 第一个 non-options 前面的作为 options 处理; 后面的都作为 non-options
  • Permute, options 和 non-options 随意放置, 根据 ArgDescr a 的处理来进行解析
  • ReturnInOrder (String -> a), 用 String -> a 将 non-options 也包裹为 options

  • 第一个 String 是自定义的一个顶部的描述信息
  • 最后的 String 表示返回值

解析 options

  • ArgOrder a 表示如何处理 options 和 non-options
  • [OptDescr a] 是 options 的描述信息, 指示如何进行解析
  • [String] 是具体的命令行参数
  • 返回值是一个三元组, 包含:
    • [a] 解析出的 options 列表
    • [String], 未处理的 non-options 列表
    • [String], 错误信息列表

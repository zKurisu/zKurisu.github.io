Linux-udev-以及设备相关知识

介绍

udev (userspace device manager) 是 Linux 内核的设备管理器, 运行在用户态. 它会监听内核发出的设备事件 (比如插入/拔出), 然后按照预设的规则执行动作, 最常见的就是创建设备文件, 设置权限, 创建符号链接等.

为什么需要 udev

Linux 遵循 “一切皆文件的哲学”. 当插入一个 U 盘后, 内核检测到硬件, 之后会在 /dev 下创建设备节点, 比如 /dev/sdb. 但是有一些问题:

  1. 每次插拔 U 盘后创建的文件名不同, 这次是 /dev/sdb, 下次可能是 sdc, 这取决于插入顺序
  2. 权限不固定, 默认可能只允许 root 读写
  3. 无法区分同型号设备, 插入两个同样的 CH340, 一个叫 ttyUSB0, 另一个叫 ttyUSB1, 无法根据名称来区分

而 udev 可以根据具体的设备属性 (比如 VID/PID, 序列号, 物理端口等) 来识别出 “你是谁”, 然后给出一个固定的名字和正确的权限. 具体的工作流程如下:

这里涉及的几个路径比较关键:

  • /sys/, 内核导出的设备属性树 (是 udev 的数据来源)
  • /lib/udev/rules.d/, 系统自带的规则 (通常不修改)
  • /etc/udev/rules.d/, 自定义规则, 优先级更高 (用户自己编写的就放在这里)
  • /dev/, 设备文件放置的位置

先是一系列匹配条件, 都满足后执行后续一系列的动作, 注意都使用 , 来分隔.

常用的匹配条件有:

条件 含义 来源
SUBSYSTEM==”tty” 设备类型 设备所属子系统
KERNEL==”ttyUSB*” 内核设备名 /dev/ 下的原始名称
ATTRS{idVendor}==”1a86” USB 厂商 ID lsusb 可查
ATTRS{idProduct}==”7523” USB 产品 ID lsusb 可查
ATTRS{serial}==”xxx” 设备序列号 部分设备有
KERNELS==”1-4.2:1.0” USB 物理路径 取决于插在哪个口

常用的动作有:

动作 含义 示例
SYMLINK+=”name” 创建符号链接 /dev/gps_serial
MODE:=”0666” 设权限 所有人可读写
GROUP:=”dialout” 设所属组 避免只能 root 访问
RUN+=”/path/script” 触发后执行脚本 自动挂载等
  • == 用来比较/匹配
  • := 用来赋值并锁定 (后续规则不可更改)
  • += 用来追加 (允许多个值叠加)

udevadm 使用

udevadm 有下面几个常用的子命令.

udevadm info, 查看设备属性

udevadm control, 控制服务

udevadm trigger, 重新触发设备事件

规则更新后, 已插入的设备不会自动重新匹配, 需要手动触发:

udevadm monitor, 实时监听设备事件

udevadm test, 模拟规则的执行 (不实际应用)

改完规则后, 想测试它会匹配哪些设备但不真正执行:

udevadm settle, 等待事件处理完毕

常用于脚本中: trigger 后, 需要用 settle 等所有符链接就绪再继续. 否则可能出现 “设备还没 ready 就读取” 的竞态.

udevadm hwdb, 硬件数据库管理

sysfs

sysfs (根据包含的内容可以称其为 “设备属性树”) 是 Linux 内核导出的内存文件系统, 挂载在 /sys 路径, 用于把内核中所有设备, 驱动, 总线层级关系以目录树的形式暴露给用户空间. udev 的所有设备属性 (如 ATTRS{...}, KERNELS== 等) 都来自这里.

需要注意它不是存在于硬盘上的文件, 而是内核数据结构的实时映射. 比如 cat /sys/class/tty/ttyUSB0/dev 读取到的不是磁盘文件, 而是内核在那一刻返回的值.

核心的子目录是下面四个:

目录 内容 组织方式
/sys/class/ 按功能分类 “所有 tty 设备”、”所有网卡”、”所有块设备”
/sys/bus/ 按总线类型分类 “所有 USB 设备”、”所有 PCI 设备”、”所有I2C 设备”
/sys/devices/ 按物理拓扑 真实硬件树的唯一路径
/sys/block/ 按设备名 sda, mmcblk0 等块设备

比如一个设备的硬件树路径:

1-1.4.2:1.0 这种命名规则是:

  • 1-1: 总线 1 上的第一个设备
  • .4: 该设备第 4 个端口
  • .2: 该端口下的第 2 个设备
  • :1.0: 该设备的第 0 个接口 (一个 USB 设备可以有多个功能接口)

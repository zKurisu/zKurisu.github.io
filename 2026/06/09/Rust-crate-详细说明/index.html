Rust-crate-详细说明

介绍

Crate 是 Rust 中编译的最小单元. 一个 .rs 文件不等于一个 Crate. rustc 一次编译会产生一个 crate.

Crate 最常用的类型有两种:

类型 入口文件 产物
二进制 crate (binary) main.rs (当然也可以是别的名字) 可执行文件
库 crate (library) lib.rs .rlib 库文件（给别人用的）

Crate 可以提供命名空间的隔离:

  • 当前 crate 下的模块, 可以用 use crate::mod_name::func_name; 来访问
  • 其他 crate 下的模块, 比如 use core::arch::asm, 需要指定具体的 crate 名称

纯 rustc 环境

所有可用的 crate 类型有:

值 产物 说明
“bin” 可执行文件 必须有 fn main()
“lib” .rlib Rust 库，给其他 Rust 代码使用
“dylib” .so / .dll 动态链接库，C/其他语言也能用
“staticlib” .a / .lib 静态库，给 C/其他语言链接用
“cdylib” .so / .dll 专给 C 调用的动态库（剥离了 Rust 元数据）
“proc-macro” .so / .dll 过程宏专用

Gymnasium-基本使用

Gymnasium 官方文档
Gymnasium Github

Intro

Reinforcement learning 本质是通过 trial and error 来学习, agent 通过尝试 actions, 获取 feedback (reward), 之后调整自己的行为. 能够让 agent 执行 action 并得到 feedback 的地方就是一个 environment.

可以用下图来描述这个 agent-environment loop:

  1. Agent 先观察到当前 state (就像玩家看游戏屏幕)
  2. Agent 基于 state 选择一个合适的 action (比如按下一个手柄按钮)
  3. Environment 响应新的 state 和一个 reward (游戏状态更新, 记分更新. 一次 action-observation 称 a timestep)
  4. 重复上述过程直到 episode 结束

Gymnasium 主要面向 single-agent 场景, 它提供了一套标准接口来定义环境, 以及如何让 agent 与其交互.

在 Gymnasium 中, 如果 environment 因 task 正常结束或失败而终止, 那么 step() 会返回 terminated=True. 如果想要 environment 在固定 timestep 后结束, 则等待 truncated=True 这个信号. 通常这两个有一个为 True, 则考虑终止该 episode.

Action 和 observation spaces

每个 environment 都会特别指出 action_space 和 observation_space (state space) 这两个 attributes 的格式. 便于了解该 environment 的输入和输出.

Action space 说明了 agent 能干什么. 而 observation space 说明了 agent 能看到什么 (比如图像, structured data).

Env.action_space 和 Env.observation_space 都是 Space 类的 instance (其实基本都是 Space 的子类的 instance), 有两个重要的 functions: Space.contains() 和 Space.sample().

Gymnasium 提供了下面 Space 的子类:

  • Box, 用 upper 和 lower limits 来描述 space 的边界
  • Discrete, 用一组离散值来描述 space, 比如 {0, 1, ..., n-1}
  • Dict, 用一个 dictionary 描述 space
  • Tuple, 用一个 tuple 来描述 space
  • 其余的可以在官方文档上看

修改已有环境

可以用 Wrapper 来修改已有的环境, 并且 Wrappers 可以 chained 来联合使用. 比如大部分用 gymnasium.make 创建的环境, 其实都已经被 TimeLimit (在运行到一个指定 steps 后停止 episode) wrapped 处理了一层.

只需要将一个现有的 environment 作为参数传给一个 wrapper 即可, 比如:

常见的 wrapper 有:

  • TimeLimit, 当 timestep 达到一个数值时传递一个 truncated signal (即返回 truncated=True)
  • ClipAction, 将传递给 step 的一个 action clip 到有效的 action space 中
  • RescaleAction, 将 action 缩放到另一个范围中
  • TimeAwareObservation, 将 current timestep 信息添加到 observation 中
  • 其余的见官方文档

    • downsample, 降采样到 84x84, 方便 CNN
    • pixel, 像素级表示, 稀疏观测
    • rectangle, 矩阵特征, 非视觉 RL

简述训练原理

先从环境拿到一帧游戏画面, 然后将图像交给 agent (卷积神经网络), agent 会输出一个 action 概率分布, 从分布中采样一个 action 再返回给环境, 之后环境会返回新的游戏图像和 reward.

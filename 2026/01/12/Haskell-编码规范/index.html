命名约定

Haskell 中:

  • 所有类型都是大写字母开头, 如 Int, Double
  • 所有函数都是小写字母开头, 如 map, filter

对于缩写也不写为全大写的形式, 比如 HTTPServer 应该写为 HttpServer. 两个字符的缩写是例外, 比如 IO.

名称较长时采用 lowerCamelCase, 名字要具有描述意义. 当然对于 type, typeclass, contructor 这些用 UpperCamelCase.

对于模块的命名都用单数, 比如 Data.Map, 而不是 Data.Maps.

Good Programming Practice

  • 函数要尽可能短, 而且只负责一个任务
  • 避免使用 {} 和 ; 进行分隔的语法
  • 尽量添加 type signatures
  • 减少 unused variables

Let or where expressions

避免混用或者嵌套 let 和 where 来声明 local name.

Application notation

用 $ 来减少括号, 比如:

  • f (g x) 写为 f $ g x
  • f (g (h x)) 写为 f $ g $ h x 或者 f . g $ h x

Imports

import 的库应该分为三类, 每一类之间用空行分隔:

  1. standard library imports
  2. related third party imports
  3. local application/library specific imports

每类 imports 之间按照 alphabetic 的顺序排序.

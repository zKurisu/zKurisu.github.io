2.2 迭代和递归

尾递归

递归函数每次调用自身时, 系统都会为新开启的函数分配内存, 以存储局部变量, 调用地址和其它信息等, 这会导致:

  • 函数的上下文数据都存储在称为 “栈帧空间” 的内存区域中, 直到函数返回后才会被释放. 因此递归通常比迭代更耗费内存空间
  • 递归调用函数会产生额外的开销, 因此递归通常比循环的时间效率更低

但如果函数在返回前的最后一步才进行递归调用, 则该函数可以被编译器或解释器优化, 使其在空间效率上与迭代相当, 这种情况称为 “尾递归 (tail recursion)” (具体如何优化的呢??).

尾递归具体来说, 就是递归调用是函数执行的最后一步, 函数返回上一层后, 无需继续执行其他操作, 因此系统无需保存上一层函数的上下文.

第3章 数据结构

3.1 数据结构分类

3.1.1 逻辑结构: 线性与非线性

线性数据结构指: 元素之间是一对一的顺序关系.

非线性数据结构指:

  • 元素之间可以是一对多的关系, 树形结构
  • 元素之间可以是多对多的关系, 网状结构

3.1.2 物理结构: 连续与分散

物理结构反映了数据在计算机内存中的存储方式, 可以分为:

  • 连续空间存储 (array)
  • 分散空间存储 (list)

所有数据结构都是基于 array 和 list 或者两者的组合来实现的. 比如:

  • 基于数组可实现: 栈, 队列, 哈希表, 树, 堆, 图, 矩阵, 张量等
  • 基于链表可实现: 栈, 队列, 哈希表, 树, 堆, 图等

链表在初始化后, 仍可以在程序运行过程中对其长度进行调整, 因此也称为 “动态数据结构”.

数组在初始化后长度不可变, 因此也称 “静态数据结构”.

3.3.1 原码, 反码和补码

数字都是以补码的形式存储在计算机中的.

  • 补码: 正数的补码与其原码相同, 负数的补码是在其反码的基础上加 1

3.4 字符编码

Unicode 字符集

Unicode 是一个庞大的字符集, 目的是收容世界上所有的字符. 其为每一个字符分配一个码点 (一串二进制序列), 但并没有规定如何在计算机中存储. 通常来说会将文件中所有字符存储为相同的编码, 也就是说, 如果一个文件中的字符是 2 字节和 1 字节的混合, 最终会全部按照 2 字节存储, 这回比较浪费空间.

UTF-8 编码

UTF-8 是一种可变长度的 Unicode 的编码方式, 使用 1 到 4 字节来表示一个字符, 可以根据字符的复杂性而变, 比如 ASCII 字符只需要 1 字节, 拉丁字母和希腊字母需要 2 字节, 常用的中文字符需要 3 字节, 其他一些生僻字需要 4 字节.

编码规则如下:

  • 对于长度为 1 字节的字符, 将最高位设置为 0, 其余 7 位设置为 Unicode 码点. ASCII 字符在 Unicode 字符集中占据了前 128 个码点, 因此 UTF-8 编码可以向下兼容 ASCII 码
  • 对于长度为 n 字节的字符 (其中 n > 1), 将首个字节的高 n 位都设置为 1, 第 n+1 位设置为 0; 从第二个字节开始, 将每个字节的高 2 位都设置为 10; 其余所有位用于填充字符的 Unicode 码点

系统可以通过读取最高位 1 的个数来解析出字符的长度.

其余字节最高位为 10 是用于校验. 选用 10 是因为不可能有字符的最高两位是 10. 可以反证为: 假设一个字符的最高位是 10, 说明该字符的长度为 1, 对应 ASCII 码. 而 ASCII 码的最高位应该是 0, 这与假设矛盾.

第4章 数组与链表

4.1 数组

数组 (array) 是一种线性数据结构, 其将相同类型的元素存储在连续的内存空间中.

由于数组的长度是固定的, 因此插入一个元素 (即后续元素往后移动一位) 必定会导致尾部元素丢失. 等价的, 删除数组中间的元素, 让后续元素往前移动一位, 会让数组末尾变得 “无意义”.

数组的优点在于:

  • 空间效率高, 数组为数据分配了连续的内存块, 无需额外的结构开销
  • 支持随机访问, 数组允许在 $O(1)$ 时间内访问任何元素

缺点在于:

  • 插入和删除效率低
  • 长度不可变, 数组的大小在初始化时固定, 若要扩容, 需要将原始数组的全部内容复制到新数组内, 开销大
  • 空间浪费, 如果数组分配的大小超过实际所需要的, 那么多余的空间就被浪费了

数组可以用于实现栈, 队列, 哈希表, 堆, 图等数据结构. 比如, 图的邻接矩阵表就是一个二维数组.

4.2 链表

链表 (linked list) 是一种线性数据结构, 其中的每个元素都是一个节点对象, 各个节点通过 “引用” 相互连接. 引用记录了下一个节点的内存地址, 通过它可以从当前节点访问到下一个节点. 因此链表可以让各个节点分散到内存中, 不需要内存地址连续:

链表的访问和查找的时间复杂度是 $O(n)$, 插入和删除是 $O(1)$.

常见链表类型

常见的链表类型有 3 种:

  • 单向链表: 即前面介绍的普通链表. 单向链表的节点包含值和指向下一节点的引用两项数据. 通常将第一个节点称为首节点, 最后一个节点称为尾节点, 尾节点指向空 None
  • 环形链表: 如果让单向链表的尾节点指向头节点 (首尾相接), 则得到一个环形链表. 环形链表中, 任意节点都可以视作头节点
  • 双向链表: 与单向链表相比, 双向链表中的每个节点都记录了两个方向的引用, 分别指向后继节点 (下一个节点) 和前驱节点 (上一个节点). 相较于单向链表, 双向链表更具灵活性, 可以朝两个方向遍历链表, 但相应地需要占用更多的内存空间

链表的典型应用

单向链表通常用于实现栈, 队列, 哈希表和图等数据结构.

双向链表常用于需要快速查找前一个和后一个元素的场景.

环形链表常用于需要周期性操作的场景, 比如操作系统的资源调度.

4.3 列表

列表 (list) 是一个抽象的数据结构概念, 它表示元素的有序集合, 支持元素访问, 修改, 添加, 删除和遍历等操作, 无须使用者考虑容量限制的问题. 列表可以基于链表或数组实现:

  • 链表天然可以看作一个列表, 其支持元素的增删改查操作, 并且可以动态灵活扩容
  • 数组也支持元素增删改查, 但由于其长度不可变, 因此只能看作一个具有长度限制的列表 (通常用动态数组实现, 可以在程序运行中进行动态扩容)

第5章 栈与队列

5.1 栈

栈 (Stack) 是一种遵循 “先入后出” 逻辑的线性数据结构.

通常将顶部的元素称为 “栈顶”, 底部元素称为 “栈底”.

常用操作

操作 描述 时间复杂度
push() 元素入栈 (添加到栈顶) O(1)
pop() 栈顶元素出栈 O(1)
peek() 访问栈顶元素 O(1)

Haskell 实现

Haskell 中的 List 是一种天然的栈结构:

  • 列表的头部 (Head), 对应栈顶 (Top)
  • 列表的尾部 (Tail), 对应栈剩余部分

5.2 队列

队列 (queue) 是一种遵循 “先入先出” 逻辑的线性数据结构. 其模拟了排队现象, 即新来的人不断加入队列尾部, 而位于头部的人逐个离开.

通常把队列头部称为 “队首”, 尾部称为 “队尾”, 把元素加入队尾的操作称为 “入队”, 删除队首元素的操作称为 “出队”.

常用操作

不同编程语言可能不同, 这里采用与栈相同的命名方式:

操作 描述 时间复杂度
push() 元素入队 (添加元素到队尾) O(1)
pop() 队首元素出队 O(1)
peek() 访问队首元素 O(1)

常用操作:

第6章 哈希表

哈希表 (Hash table), 又称散列表, 其通过建立键 key 与值 value 之间的映射, 实现高效的元素查询. 具体来说, 向哈希表输入一个键 key, 可以在 O(1) 时间内获取对应的值 value.

哈希表的操作效率与数组和链表的对比:

Operation 数组 链表 哈希表
查找元素 O(n) O(n) O(1)
添加元素 O(1) O(1) O(1)
删除元素 O(n) O(1) O(1)

6.1 哈希表简单实现

可以用一个数组来实现哈希表.

在哈希表中, 将数组的每个空位称为 “桶 (bucket)”, 每个桶可以存储一个键值对. 查询操作就是找到 key 对应的桶, 并在桶中获取 value.

基于 key 定位到桶, 需要用到哈希函数 (Hash function). 哈希函数的作用是将一个较大的输入空间映射到一个较小的输出空间.

在哈希表中, 所有的 key 是输入空间, 输出空间是所有桶索引 (数组实现中对应索引). 输入一个 key, 可以用哈希函数得到对应的桶索引, 也就是数组中的索引, 因此可以取出对应值.

可以这样应用哈希函数:

  1. 通过某州哈希算法 hash() 得到哈希值
  2. 将哈希值对桶数量 (数组长度) capacity 取模, 从而获取该 key 对应的桶索引 (数组索引)

从本质上来看, 哈希函数的作用是将所有 key 构成的输入空间映射到数组所有索引构成的输出空间, 而输入空间往往远大于输出空间. 因此, 理论上一定存在 “多个输入对应相同输出” 的情况.

比如:

这种多个输入对应一个输出的情况称为哈希冲突 (hash collision).

当哈希表容量 n 越大时, 多个 key 被分配到同一个桶中的概率就越低, 冲突就越少. 因此, 可以通过扩容哈希表来减少哈希冲突.

但这种类似数组的扩容方法效率极低. 一些编程语言会预留较大的哈希表容量, 防止频繁扩容.

6.2 哈希冲突

通过改良哈希表结构, 使得哈希表可以在出现哈希冲突时也能正常工作, 且仅在必要时进行扩容.

常见的改良方法有 “链式地址” 和 “开放寻址”.

链式地址

原始哈希表中, 每个桶仅能存储一个键值对. 链式地址 (separate chaining) 将单个元素转换为链表, 将 “键值对” 作为链表节点值, 将所有发生冲突的键值对都存储在同一链表中:

此时哈希表的操作方法有些变化:

  • 查询元素: 输入 key, 经过哈希函数得到桶索引, 此时能够访问到链表头节点, 然后遍历链表并对比 key 以查找目标键值对
  • 添加元素: 首先通过哈希函数访问链表头节点, 然后将节点 (键值对) 添加到链表中
  • 删除元素: 根据哈希函数的结果访问链表头节点, 然后遍历链表以查找目标节点并将其删除

局限性在于:

  • 占用空间大, 链表需存储指针
  • 查询效率低, 因为需要线性遍历链表

开放寻址

开放寻址 (open addressing) 不引入额外的数据结构, 而是通过 “多次探测” 来处理哈希冲突, 探测方式主要包括线性探测, 平方探测和多次哈希等.

线性探查

线性探查采用固定步长的线性搜索来探查, 其操作也与普通哈希表不同:

  • 插入元素: 通过哈希函数计算桶的索引, 若发现桶内已有元素, 则从冲突位置向后线性遍历 (步长通常设置为 1), 直到找到空桶, 然后将元素插入其中
  • 查找元素: 若发现哈希冲突 (桶内元素的 key 与查找的 key 不同), 则使用相同步长向后进行线性遍历, 直到找到对应元素, 返回 value 即可; 若遇到空桶, 说明目标元素不在哈希表中, 返回 None

需要注意, 在开放寻址哈希表中不能直接删除元素. 这是因为删除元素会在数组内产生一个空桶 None, 而当查询元素时, 线性探测到该空桶就会直接返回, 因此后续元素都无法访问到了, 程序会误判这些元素不存在:

通常可以用 “懒删除 (lazy deletion)” 机制, 其不直接从哈希表移除元素, 而是利用一个常量 TOMBSTONE 来标记这个桶. 在该机制下, None 和 TOMESTOME 都代表空桶, 都可以放置键值对. 但不同的是, 线性探测到 TOMESTONE 时应该继续往后遍历.

平方探测

平方探测与线性探测类似, 不同点在于: 当发生冲突时, 不是跳过一个固定的步长, 而是跳过 “探测次数的平方” 个步数, 即 1, 4, 9, ... 步.

平方探测主要优势是:

  • 平方探测通过跳过探测次数平方的距离, 试图缓解线性探测的聚集效应
  • 平方探测会跳过更大的距离来寻找空位, 有助于数据分布更加均匀

但平方探测并不是完美的:

  • 仍然存在聚集现象
  • 由于平方探测, 可能有空桶也没有访问到

多次哈希

多次哈希, 通过使用多个哈希函数 $f_1(x)$, $f_2(x)$, $f_3(x)$ 来进行探测.

  • 插入元素: 若哈希函数 $f_1(x)$ 出现冲突, 则尝试 $f_2(x)$, 以此类推, 知道找到空位后插入元素
  • 查找元素: 挨个调用哈希函数进行查找, 直到找到目标元素时返回; 若遇到空位或已尝试所有哈希函数, 说明哈希表中不存在该元素, 返回 None

多次哈希不易产生聚集, 但会带来额外的计算量.

6.3 哈希算法

无论是链式地址还是开放寻址, 都只能保证哈希表在发生冲突时能正常工作, 而无法减少哈希冲突的发生.

如果哈希冲突过于频繁, 哈希表的性能会急剧劣化. 比如对于链式地址哈希表, 理想情况下键值对均匀分布在各个桶中, 达到最佳查询效率; 最差情况下所有键值对都存储在同一个桶中, 时间复杂度退化至 $O(n)$:

而键值对的分布情况有哈希函数决定:

为了降低哈希冲突发生的概率, 需要设计更好的哈希函数.

好的哈希函数应该有下面特征:

  • 确定性: 对于相同的输入, 哈希算法应该始终产生相同的输出. 这样才能保证哈希表是可靠的
  • 效率高: 计算哈希值的过程应该足够快. 计算开销越小, 哈希表的实用性越高
  • 均匀分布: 哈希算法应该让键值对均匀分布在哈希表中, 分布越均匀, 哈希冲突的概率越低

如果哈希函数还用于安全领域, 还需要有下面安全特征:

  • 单向性: 无法通过哈希值反推出关于输入数据的任何信息
  • 抗碰撞性: 应当极难找到两个不同的输入, 使得它们的哈希值相同
  • 雪崩效应: 输入的微小变化应当导致输出的显著且不可预测的变化

哈希算法设计

对于一些要求不高的场景, 可以设计一些简单的哈希算法:

  • 加法哈希: 对输入的每个字符的 ASCII 码进行相加, 将得到的总和作为哈希值
  • 乘法哈希: 利用乘法的不相关性, 每轮乘以一个常数, 将各个字符的 ASCII 码累积到哈希值中
  • 异或哈希: 将输入数据的每个元素通过异或操作累积到一个哈希值中
  • 旋转哈希: 将每个字符的 ASCII 码累积到一个哈希值中, 每次累积之前都会对哈希值进行旋转操作

第7章 树

7.1 二叉树

二叉树 (Binary tree) 是一种非线性数据结构, 代表 “祖先” 与 “后代” 之间的派生关系, 体现了 “一分为二” 的分治逻辑. 与链表类似, 二叉树的基本单元是节点, 每个节点包含值, 左子节点引用和右子节点引用.

常见术语

二叉树中的常见术语有:

  • Root node (根节点), 二叉树顶层的节点, 没有父节点
  • Leaf node (叶节点), 没有子节点的节点, 两个指针都指向 Empty
  • Edge (边), 连接两个节点的线段, 即节点间的指针
  • Level (层), 节点所在的位置, 如果根节点为 level 1, 那么其子节点是 level 2, 以此向下递增
  • Degree (度), 节点的子节点数量. 在二叉树中, degree 的取值是 0,1,2
  • Height (高度), 节点到最远叶节点所经过的边的数量
  • Depth (深度), 根节点到该节点所经过的边的数量

常见二叉树类型

Perfect Binary Tree, 所有层的节点全部填满:

  • 叶节点的 degree 为 0, 其他所有节点为 2
  • 若树的高度为 h, 则节点总数是 $2^{h+1}-1$

Complete Binary Tree, 完全二叉树:

  • 只有最底层节点没完全填满
  • 最底层节点从左到右依次填充

Full Binary Tree, 满二叉树:

  • 除了叶节点外, 其余所有节点都有两个子节点

Balanced Binary Tree, 平衡二叉树:

  • 任意节点的左右子树高度差的绝对值不超过 1

8.1 堆

堆 (Heap) 是一种满足特定条件的二叉树, 主要分两种类型:

  • 小顶堆 (min heap): 任意节点的值 $\le$ 其子节点的值
  • 大顶堆 (max heap): 任意节点的值 $\ge$ 其子节点的值

在 Haskell 中, 通常用 Leftist Tree 来实现 Heap, 而不是普通的 complete binary tree, 原因是:

  • Complete binary tree 通常实现为用 array 存储, 而 leftist tree 通常用指针/递归数据结构
  • Complete binary tree 需要 $O(n)$ 来 merge 两个 heap, 而 leftist tree 只需要 $O(log n)$

在 Leftist Heap 中, 每个节点需要满足: 左子树的 rank 大于右子树的 rank, 从而能推导出一个性质: 根节点右子树的 rank 最多是 $log_2 (n+1)$ (n 是节点数), 从而能够在 $O(logn)$ 时间内合并两个 heap (后面再推导).

有:

  • 叶节点的 rank 为 0
  • 当前节点的 rank 是右子树 rank 再加上 1 (毕竟左子树的 rank 比右子树大, 而 rank 要取较短的路径长度)

Leftist Property, 这个是基本性质, 不需要证明, 即: .

$$
\displaylines
{
\begin{aligned}
rank(L(x)) >= rank(R(x))
\end{aligned}
}
$$

该性质允许下面 unbalanced tree:

这是能接受的, 因为 merge 的时候只会遍历右子树, 其长度为 1. 该性质的目的是尽可能多地将元素放在左侧, 从而让右侧更短.

Lemma 1.1, 在一个有 n 个元素的 leftist heap 中, root node 的 rank 最大是 $log_2(n+1)$

证明 Lemma 1.1 之前, 先证明一个 Claim: 如果一个 heap 的 rank 为 r, 那么其至少包含 $2^r - 1$ 个 entries.

这里用 n(r) 表示 rank 为 r 的 leftist heap 包含的最少 nodes 数量, 很明显 n(r) 是 monotone function (单调函数), 假如有 $r’ \ge r$, 那么会有 $n(r’) \ge n(r)$. 可以由此得到一个递推关系: 考虑一个 heap 的 root node x 有 rank r, 那么其 right child 的 rank 是 r - 1. 又由于 leftist property, left child 的 rank 大于等于 right child 的 rank: $rank(L(x)) \ge rank(R(x)) = r - 1$. 以 x 为 root 的树的 nodes 数量可以表示为: $1 + |L(x)| + |R(x)|$ (1 是 node x 自己, $|L(x)|$ 是左子树的 node 数, $|R(x)|$ 是右子树的 node 数), 可以用 n(r) 表示为:

$$
\displaylines
{
\begin{aligned}
n(r) = 1 + n(rank(L(x))) + n(rank(R(x))) & \newline~ \newline
& \ge 1 + n(r-1) + n(r-1) = 1 + 2 \cdot n(r-1)
\end{aligned}
}
$$

解这个递推关系, 就能得到 $n(r) \ge 2^r - 1$:
$$
\displaylines
{
\begin{aligned}
n(r) &\ge 1 + 2 n(r - 1) \newline~ \newline
&\ge 1 + 2 (1 + 2 n(r - 2)) \newline~ \newline
&\ge 1 + 2 + 4 n(r - 2) \newline~ \newline
&\ge 1 + 2 + 4 (1 + n(r - 3)) \newline~ \newline
&\ge 1 + 2^1 + 2^2 + 2^2 n(r - 3) \newline~ \newline
… \newline~ \newline
&\ge 1 + 2^1 + 2^2 + … + 2^{r-1} = 2^r - 1
\end{aligned}
}
$$

之后直接用上述关系证明 lemma 1.1:

$$
\displaylines
{
\begin{aligned}
n \ge n(r) \ge 2^r - 1 \rightarrow 2^r \le n + 1 \rightarrow r \le log_2(n+1)
\end{aligned}
}
$$

9.1 图

图 (graph) 是一种非线性数据结构, 由顶点 (vertex) 和边 (edge) 组成, 通常用下面符号表示:

$$
\displaylines
{
\begin{aligned}
V = \{1, 2, 3, 4, 5\} \newline~ \newline
E = \{(1,2), (1,3), (1,5), (2,3), (2,4), (2,5), (4,5)\}
\end{aligned}
}
$$

图的常见类型

根据 edge 是否有方向, 可以分为无向图 (undirected graph) 和有向图 (directed graph):

  • Undirected graph 中, 边表示两顶点之间的 “双向” 连接关系, 比如微信和 QQ 中的 “好友关系”
  • Directed graph 中, 边具有方向性, 即 $A \rightarrow B$ 和 $A \leftarrow B$ 两个方向的边都是相互独立的, 例如 bilibili 上的 “关注” 与 “被关注” 关系

根据所有 vertex 是否连通, 可分为连通图 (connected graph) 和非连通图 (disconnected graph):

  • Connected graph 指, 从某个顶点出发, 可以到达其余任意顶点
  • Disconnected graph 指, 从某个顶点出发, 至少有一个顶点无法到达

根据 edge 是否有 weight, 可以分为 weighted graph 和 unweighted graph:

图的常用术语

  • Adgacency (邻接), 当两顶点之间存在边相连时, 称这两顶点 “邻接”
  • Path (路径), 从顶点 A 到顶点 B 经过的边构成的序列被称为从 A 到 B 的 “路径”
  • Degree (度), 一个顶点拥有的边数. 对于有向图, 入度 (in-degree) 表示有多少条边指向该顶点, 出度 (out-degree) 表示有多少条边从该顶点指出

图的表示

图通常用两种方式表示:

  • Adjacency matrix (邻接矩阵)
  • Adjacency list (邻接表)

邻接矩阵

如果图的顶点数量为 n, 那么邻接矩阵 (adjacency matrix) 会使用一个 n x n 大小的矩阵来表示图, 每一行 (列) 代表一个顶点, 矩阵元素代表边, 用 1 或 0 表示两个顶点之间是否存在边.

设邻接矩阵为 M, 顶点列表为 V, 那么矩阵元素 $M[i, j] = 1$ 表示顶点 $V[i]$ 到顶点 $V[j]$ 之间存在边, 反之 $M[i,j] = 0$ 表示两顶点之间无边.

邻接矩阵有下面特性:

  • 在简单图中, 顶点不能与自身相连, 此时邻接矩阵主对角线元素没有意义
  • 对于无向图, 两个方向的边等价, 此时邻接矩阵关于主对角线对称

10.5 重识搜索算法

搜索算法 (search algorithm) 用于在数据结构中 (例如数组, 链表, 树或图) 中搜索一个或一组满足特定条件的元素.

搜索算法可根据实现思路分为以下两类:

  • 通过遍历数据结构来定位目标元素, 例如数组, 链表, 树或图的遍历等
  • 利用数据组织结构或数据包含的先验信息, 实现高效元素查找, 例如二分查找, 哈希查找和二叉搜索树查找等

暴力搜索

暴力搜索通过遍历数据结构的每个元素来定位目标元素:

  • “线性搜索” 适用于数组和链表等线性数据结构, 其从数据结构的一端开始, 逐个访问元素, 直到找到目标元素或到达另一端仍没有找到目标元素为止
  • “广度优先搜索” 和 “深度优先搜索” 是图和树的两种遍历策略. 广度优先搜索从初始节点开始逐层搜索, 由近及远地访问各个节点. 深度优先搜索从初始节点开始, 沿着一条路径走到头, 再回溯并尝试其他路径, 直到遍历完整数据结构

暴力搜索的优点是简单且通用性好, 无需对数据做预处理和借助额外的数据结构. 但是此类算法的时间复杂度为 $O(n)$, 其中 n 为元素数量, 因此在数据量较大的情况下性能较差.

自适应搜索

自适应搜索利用数据的特有属性 (例如有序性) 来优化搜索过程, 从而更高效地定位目标元素:

  • “二分查找” 利用数据的有序性实现高效查找, 仅适用于数组
  • “哈希查找” 利用哈希表将搜索数据和目标数据建立为键值对映射, 从而实现查询操作
  • “树查找” 在特定的树结构 (例如二叉搜索树) 中, 基于比较节点值来快速排除节点, 从而定位目标元素

此类算法的优点是效率高, 时间复杂度可达到 $O(logn)$ 甚至 $O(1)$.

然而使用这类算法往往需要对数据进行预处理, 例如二分查找需要预先对数组进行排序, 此时要考虑排序的时间.

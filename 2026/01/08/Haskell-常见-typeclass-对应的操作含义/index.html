Haskell-常见-typeclass-对应的操作含义

Functor

Functor typeclass 描述了 “可以映射的结构”, 可以在某种上下文中应用函数, 并且保持上下文的结构不变.

Haskell 定义

  • f a 是某种上下文中的值 a
  • a -> b 是一个普通的函数
  • fmap 将普通函数 lift 到上下文中进行应用

如果我们只给 fmap 提供一个函数 a -> b, 相当于返回了另一个 f a -> f b 的函数. 相当于 fmap 把函数从 “普通世界” lift 到了 “上下文世界”.

也可以另外理解为, f a 是一个盒子, fmap 的作用就是:

  1. 打开盒子
  2. 取出里面的值 a
  3. 对值应用函数 (a -> b)
  4. 把结果 b 放回同一个盒子里

在范畴论 (category theory) 中, functor 是两个 category 之间的映射, 满足:

  1. 映射对象: 将一个 category 的对象映射到另一个 category
  2. 映射 morphism: 将 morphism 映射到 morphism
  3. 保持结构: 保持复合和单位

Functor 定律

任何合法的 Functor instance 必须满足两个定律:

  • 恒等率: fmap id = id. 对任何 Functor 应用 id 函数, 结果不变

  • 复合率: fmap (g . f) = fmap g . fmap f. 先复合函数再 fmap, 等价于分别 fmap 再复合.

Applicative

Applicative 描述了 “可以应用函数的上下文”, 函数是在上下文中执行的.

  • pure 将普通值 lift 到上下文中
  • <*> 将上下文中的函数应用到上下文中的值上, 相当于在 f 上下文内, 把 a -> b 函数应用到了 a 上

Applicative 定律

任何合法的 Applicative instance 都必须满足下面的定律:

  1. 恒等律, 在上下文中应用 id 函数, 值不变
  1. 同态律, 两个 pure 值的结合等价于直接结合后用 pure 包裹
  1. 交换律, 上下文中的函数和纯值可以交换顺序
  1. 复合律, 函数的组合在上下文中保持
  1. 同构律, fmap 可以通过 pure 和 <*> 实现

State Monad

State Monad 描述 “组合带状态传递的计算”.

因为 Haskell 中 “纯函数没有状态”:

状态可以理解为一种记录, 函数通过当前状态可以得到一个当前值

传统的方法是显示传递状态:

  • 这里通过 0 (初始状态), 能得到值 result1. state1 是下一个状态
  • 这里通过 state1, 能得到值 result2. state2 是下一个状态
  • 这里通过 state2, 能得到值 result3. state3 是下一个状态

这样手动传递状态会比较繁琐. 上述过程的本质就是, 有一个函数, 接受一个状态, 返回这个状态的值以及下一个状态.

这个过程可以封装在 State 类型中, 核心思想是让它封装一个函数, 描述状态切换的计算过程:

  • a 是想要传递给外界的信息, s 才是我们想要保留的东西.

对应的 Monad instance 定义如下:

  • h s 是执行一个计算
  • f a 是把 a 放入另一个计算中.
  • g newState 是运行整个计算
  • 这里的目的其实就是简化 newState 这个东西的传递
  • f 声明为 a -> s -> (b, s) 主要是方便结果的传递, 上一个 state 的结果需要传递进下一个 state 中
  • f 需要描述如何把 a 加入到组合的计算中

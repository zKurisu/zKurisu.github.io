Recursive and Inductive definition

所有的 Inductive definition 都是递归的, 但并非所有 recursive definition 都是归纳的.

Recursive definition 定义为: 一个函数在定义中直接或间接地调用自身, 目的是将大问题分解为小问题, 但不保证终止, 比如:

  • 这种方式递归不一定终止, 比如 f x = f x, 也不一定要与数据结构对齐

而 Inductive definition 有时也称 “结构递归 (structurally recursive)”, 定义为: 递归严格按照代数数据类型的构造方式进行:

  • 对于列表, 由 : 和 [] 构造得到:
    • 基础情况: 处理 []
    • 归纳: 处理 x:xs, 递归调用只作用于 xs (直接子结构)
  • 对于自然数, 由 Zero 和 Succ 构造得到:
    • 基础情况: 处理 f Zero = ...
    • 归纳: f (Succ n) = ... f n ...

特征是函数的参数是输入的直接子结构, 且只要数据有限就能保证终止. 这与数学数学归纳法的思想是一样的:

  • 基础情况 (base case): []
  • 归纳步骤 (inductive step): x:xs, 假设对 xs 成立, 则能证明对 x:xs 成立

Haskell 的惰性求值

Haskell 的惰性求值结合了下面两个策略:

  • Call-by-need: 一个表达式只有在它的值被实际需要时才会被计算
  • Outermost Reduction: Haskell 会优先从整个表达式的最外层结构开始进行规约, 而不是先计算里面的子表达式

可以从下面例子看 call-by-need:

  • 如果是在命令式语言中, 调用 f infinity 就需要对 infinity 进行求值, 即构建这个无穷列表, 就会导致程序崩溃
  • 而在 Haskell 中, f 的函数体中没有用到参数 x, 也就是传入的无穷列表, 因此这个表达式永远不会被求值, 程序会顺利打印出 “Hello”

规约的意思是应用一个函数或运算符到一个或多个参数上, 将其替换为一个更简单的形式或最终值, 如 2+3 可以被规约为 5, 下面是一个 Outermost reduction 的示例:

考虑表达式 (2 + 3) * 4, 这个表达式有两层结构:

  • 内层: (2 + 3)
  • 外层 (...) * 4

如果采用最内层规约, 步骤为:

  1. 先找到最内层可规约表达式 2 + 3
  2. 规约它: 2 + 3 -> 5
  3. 得到新表达式: 5 * 4
  4. 再规约: 5 * 4 -> 20

而对于 outermost reduction, 比如这里的 (2 + 3) * 4, 它会先应用 * 到两个参数上, 因此需要知道两个参数的值, Haskell 会选择最左边的参数先求值 (看起来还是先规约 2 + 3, 但实际流程不一样, 多了一步).

对于表达式 area (Circle 5), 求值步骤为:

  1. 最外层结构是 area arg, 求值器查看 area 定义
  2. 尝试匹配模式: 将实参 (Circle 5) 与 area 的第一个模式 (Circle r) 进行匹配:
    a. 这是一个结构匹配, 它看到 Circle, 然后尝试把 5 绑定到 r
    b. 这个过程不要求 5 被求值为一个数字
  3. 匹配成功: 此时将整个 area (Circle 5) 表达式规约到右边的 pi * r * r, 其中 r 被替换为 5
  4. 继续规约: 对表达式 pi * 5 * 5, 最外层的 * 操作需要具体的数值, 所以求值器开始计算 5*5, 然后是 pi * 25

如果要对这个输出做 head 操作, 即取出第一个元素. Haskell 会从最外层进行规约:

  1. 最外层是 (...) ++ c, ++ 运算符会先对左侧 (...) 进行求值
  2. (...) 中是 (...) ++ b, ++ 运算符同样先对左侧 (...) 进行求值
  3. (...) 中是 [] ++ a, ++ 左侧是 [], 再对右侧求值后完成表达式求值
  4. 进行下一个表达式求值, 要计算 a ++ b, 需要遍历 a 中所有元素, 然后挨个插入到 b 的头部
  5. 此时已经能找到第一个元素了, 但整个过程需要先遍历整个调用链, 从最右侧的元素往里钻

而相同的表达式如果用 foldr 计算:

同样从最外层开始规约:

  1. 最外层是 a ++ (...), ++ 先对左边元素进行求值, 然后求值右边, 因为过程会遍历 a 中所有元素追加到 (...) 中, 那么 a 中的元素就是最终结果的前几个元素, 因此如果只需要找到第一个元素, 就不用对后面进行求值了
  2. 直接就找到第一个元素了, 不需要经过整个调用链
表达式 获取 head 需要化简的深度
foldr (++) [] [a,b,c,d] = a ++ (b ++ (c ++ d)) 1 层：直接看 a
foldl (++) [] [a,b,c,d] = (((a ++ b) ++ c) ++ d) 3 层：必须逐层展开才能看到 a

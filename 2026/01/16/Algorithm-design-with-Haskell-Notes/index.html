2 Timing

2.2 Estimating running times

Haskell 对表达式求值时会将其 reducing to normal form, 因此可以用 reduction steps 来衡量 running times.

因为并不是所有 reduction steps 都消耗相同的时间, 比如复杂表达式中找到下一个 reducible expression 并规约可能会花更多时间, 因此用 reduction steps 来衡量不是那么 faithful.

另一个可选方法是用程序开始到结束之间 elapsed time 来衡量.

因为 lazy evaluation 的 reduction steps 不会超过 eager evaluation, 所以通常程序在 eager evaluation 的运行时间 upper bound 也可以看作是 lazy evaluation 的上界.

估计 recursive function 的 reduction steps 时, 需要找到 recurrence relation (递推关系): 其描述了一个递归函数的运行时间是如何依赖于更小规模子问题的运行时间的. 解 recurrence relation 的意思是找到其 closed form, 比如:

  • Recurrence form: $T(n) = T(n-1) + 1, T(0) = 1$, 此时需要展开才能知道 $T(100)$
  • Closed form: $T(n) = n + 1$, 此时直接代入 $n = 100$ 就能得到 $T(100) = 101$

求解 recurrence relation 的常见方法有:

  • Iteration, 迭代展开, 反复代入递推式以达到基础情况
  • Master Theorem (主定理), 通常用于分治算法比如 $T(n) = aT(n/b) + f(n)$
  • 代入法 (Substitution), 猜一个解, 然后用数学归纳法证明

比如用迭代法解下面 recurrence relation:

$$
\displaylines
{
\begin{aligned}
T(n) = nT(n-1) + \Theta (n)
\end{aligned}
}
$$

推导解:
$$
\displaylines
{
\begin{aligned}
T(n) &= cn + n T (n-1) \newline~ \newline
&= cn + n(c(n-1) + (n-1) T(n-2))
&= cn + cn(n-1) + cn(n-1)(n-2) + …
\end{aligned}
}
$$
这里把 $\Theta (n)$ 替换为 $cn$, 用具体的函数替换渐进符号来简化求解过程. 因为 $\Theta(n)$ 是一个集合, 表示 “所有与 n 同阶的函数”, 比如 $3n + 2, 0.5n, 100n - logn$, 需要具体的函数才能进行代数计算.

有:
$$
\displaylines
{
\begin{aligned}
T(n) = c \sum_{k=1}^n \frac{n!}{(n-k)!} = \Theta (n!)
\end{aligned}
}
$$

2.4 Amortised running times

对于某些数据结构而言, 操作序列的总时间复杂度可能会远低于 “单次最坏情况 x 操作次数”, 下面是两个例子.

build 函数用于过滤输入列表, 其定义如下:

  • build 会挨个处理输入列表并添加到输出列表中, 每次插入时, 会删除输出列表中符合 p x y 的元素 (x 是当前要插入的元素, y 是遍历的输出列表中的元素)
  • 单次最坏时间就是 insert 的成本, 由于 dropWhile 可能会遍历整个列表, 因此是 $O(n)$. 此时总的时间复杂度计算是: $B(n+1) = B(n) + O(n)$, 得到 $B(n) = O(n^2)$
  • 但实际上, dropWhile 最多删除 n 个元素, 因为dropWhile 进行遍历时, p x y 为 True 的会被删除, 为 False 时停止遍历, 因此最多只会遍历完输入列表, 长度为 n
  • 因此 insert 操作的平均成本是 $O(1)$, 此时时间复杂度是: $O(n)$, 远小于之前的 $O(n^2)$

又比如 bits 函数, 用于生成一个二进制整数的位序列:

  • inc 函数用于递增一个二进制整数, 每次递增操作的成本在最坏情况下是 $O(k)$, 其中 $k$ 是二进制数的长度. 比如 1110 (例子中左边是低位, 向右边递增), inc 一次的结果是 0001, 遍历了每一位. 此时总的时间复杂度是 $O(n^2)$
  • 但实际上, 每次递增平均下来只会改变一个比特位, 比如:
    • 初始: []
    • inc 1次: [1], 改变1位
    • inc 2次: [0,1], 改变2位
    • inc 3次: [1,1], 改变1位
    • inc 4次: [0,0,1], 改变3位
    • inc 5次: [1,0,1], 改变1位
    • inc 6次: [0,1,1], 改变2位
    • inc 7次: [1,1,1], 改变1位
    • inc 8次: [0,0,0,1], 改变4位
    • inc 9次: [1,0,0,1], 改变1位
    • inc 10次: [0,1,0,1], 改变2位
    • inc 11次: [1,1,0,1], 改变1位
    • inc 12次: [0,0,1,1], 改变3位
    • inc 13次: [1,0,1,1], 改变1位
    • inc 14次: [0,1,1,1], 改变2位
  • 可以看到规律:
    • 每隔2次, 就会只改变 1 位
    • 每隔4次, 就会只改变 2 位
    • 每隔8次, 就会只改变 3 位
    • 总的更新次数假设是 n, 那么会有 $\frac{n}{2}$ 改变 1 位, 有 $\frac{n}{4}$ 改变 2 位, 以此类推
  • 总的时间复杂度是: $ \frac{n}{2} + \frac{2n}{4} + \frac{3n}{8} +… = O(n)$, 也是低于之前计算的 $O(n^2)$

Amortised cost 就是计算每次操作的成本, 比如 insert 和 inc 都是 $O(1)$ (最坏都是 $O(n)$).

这里引入 potential method 来计算, 假设数据结构在第 i 次操作前的状态为 $x_i$, 操作后的状态为 $x_{i+1}$, 有:

  • Actual cost: $C(x_i)$, 表示执行该操作的真实时间开销
  • Potential function: $S(x) \ge 0$, 表示状态 $x$ 所 “存储” 的能量 (或预付成本, 感觉像累积成本)
  • Amortised cost 定义为: $A(x_i) = C(x_i) + S(x_{i+1} - S(x_i)$

可以这样理解:

  • 如果操作之后有 $S(x_{i+1}) > S(x_i)$, 即势能增加, 两者相减得到一个正数, 因此 $A(x_i) > C(x_i)$
  • 如果操作之后有 $S(x_{i+1}) < S(x_i)$, 即势能减小, 两者相减得到一个负数, 因此 $A(x_i) < C(x_i)$

这样就能判断 Amortised cost 与每次操作的实际 cost 的关系. 对上述式子进行移项有: $C(x_i) = A(x_i) + S(x_i) - S(x_{i+1})$, 可以求 n 次操作之和:

$$
\displaylines
{
\begin{aligned}
\sum_{i=0}^{n-1} C(x_i) &= \sum_{i=0}^{n-1}[A(x_i) + S(x_i) - S(x_{i+1})] \newline~ \newline
&= \sum_{i=0}^{n-1} A(x_i) + (S(x_0) - S(x_1) + S(x_1) - S(x_2) + … + s_(x_{n-1}) - S(x_n)) \newline~ \newline
&= \sum_{i=0}^{n-1} A(x_i) + S(x_0) - S(x_n)
\end{aligned}
}
$$

如果能设计出势能函数 S 满足:

  • 初始状态无势能: $S(x_0) = 0$
  • 势能永不为负: $S(x_n) \ge 0$

那么就能得到:
$$
\displaylines
{
\begin{aligned}
\sum C(x_i) \le \sum A(x_i)
\end{aligned}
}
$$

就可以用总的 amortised cost 来表示整个操作序列的时间成本了.

比如对于之前的 build 函数, 可以定义势能函数为: S(xs) = length xs.

对于每次 insert, 其 actual cost 等于 p 的调用次数加上 cons 操作数:

  • 假设 dropWhile 删除了 d 个元素, 则 p 会被调用 d+1 次
  • 由于是 x : ..., 因此 cons 只用操作一个元素
  • 总的代价是: $C = (d+1) + 1 = d + 2$

势能的变化可以计算为: $S(newxs) - S(xs) = [1 + (len xs - d)] - len xs = 1 - d$ (这里的 1 + (len xs -d) 中, 1 是新插入的元素 x, len xs - d 是删除了 d 个元素之后的长度).

因此有:
$$
\displaylines
{
\begin{aligned}
C = d + 2 = 3 + (d - 1) = A + S(x_i) - S(x_{i+1}) \newline~ \newline
A = 3 = O(1)
\end{aligned}
}
$$

总代价: $\le 3n = O(n)$.

另一个例子, 函数 prune:

从右边开始剪裁列表, 直到列表满足条件 p.

假设 p xs 是 p 应用到长度为 k 的列表上, 所花的 steps 是 $O(k)$, 那么 cut x xs 需要花 $O(k^2)$ steps, 因为最坏情况下, until 会调用 k 次 done 函数.

再加上 foldr 会遍历整个列表, 因此总的时间复杂度会是 $O(n^3)$, n 是列表长度.

但实际上, prune 的时间复杂度是 O(n^2). cut 操作的 amortised running time 是 $O(n)$.

假设 cut 作用到长度为 $k_1$ 的列表, 输出列表长度为 $k_2$, 有 $0 \le k_2 \le k_1 + 1$. 假设 done 和 init 作用在长度为 k 的列表上一次都是花费 k steps. 可以知道 init 总共调用 $k_1 + 1 - k_2$ 次 (每调用一次会让列表长度减一, 一共减小了 $k_1 + 1 - k_2$), total cost 为:

$$
\displaylines
{
\begin{aligned}
(k_1 + 1) + k_1 + (k_1 - 1) + … + (k_2 + 1) = \frac{(k_1 + 1)(k_1 + 2)}{2} - \frac{k_2(k_2 + 1)}{2}
\end{aligned}
}
$$

done 调用次数会比 init 多一次 (最后 done 为 True 的时候), 因此 total cost 为:

$$
\displaylines
{
\begin{aligned}
\frac{(k_1 + 1)(k_1 + 2)}{2} - \frac{k_2(k_2 - 1)}{2}
\end{aligned}
}
$$

两者相加再加上 x:xs 的一次 cons 操作, 有:

$$
\displaylines
{
\begin{aligned}
(k_1 + 1) (k_1 + 2) - k_2^2 + 1 = k_1^2 - k_2^2 + 3 (k_1 + 1)
\end{aligned}
}
$$

构造: $S(xs_i) = (length\ xs_i)^2$

有:
$$
\displaylines
{
\begin{aligned}
C = A + S_i - S_{i+1} \newline~ \newline
k_1^2 - k_2^2 + 3(k_1 + 1) = A + k_1^2 - k_2^2 \newline~ \newline
A = 3 (k_1 + 1) = 3 x (length\ xs_i + 1)
\end{aligned}
}
$$
($k_1$ 是操作之前的长度, 因此可以用来计算 $S_i$, 而 $k_2$ 是操作之后的长度, 正好用来计算 $S_{i+1}$)

因此 $A(xs_i) = O(n)$ 而不是 $O(n^2)$.

二进制编码设计的好处在于, 把列表长度按二进制分解, 每一位对应一棵完美二叉树 (One t 表示 “这一位有树”, Zero 表示 “这一位无树”), 这样的设计解决了单一 Tree 的所有问题:

  3. 保证随机访问为 $O(logn)$ 的时间复杂度

Divide and Conquer

接下来的几个章节都属于 Divide and Conquer (也可以翻译成 divide and rule), 其核心思想是, 给定一个问题, 如果其足够小, 则直接对其求解; 否则将其拆为子问题并分别求解, 最后合并子问题的解得到原始问题的解.

这一般要求, 子问题和原问题需要用相同的 strategy 来求解, 子问题仅仅是处理原问题的更小规模的输入. 这样的特征也天然适合递归求解.

Functional algorithm 都非常依赖 recursion, 这些都可以被看作是 divide-and-conquer algorithm. 比如把一个输入 size 为 n > 0 的问题分解为 size 为 n - 1 和另一个 size 为 1 的子问题: foldr 就是这种处理思路, 逐个处理 List 里的元素, 并将结果合并.

Devide-and-conquer algorithm 通常含有两个特征:

  1. subproblem 的输入规模与原问题的输入规模呈比例, 比如是 $\frac{n}{4}$ (子问题的输入规模是原问题的四分之一), 又比如 $\frac{n}{2}$ (子问题的输入规模是原问题的二分之一). 通常分解出的多个 subproblems 的输入规模是一样的, 比如一个 size 为 n 的问题分解为两个 subproblems, 那么各输入规模都是 $\frac{n}{2}$ (当然也存在 subproblems 之间输入规模不同的)
  2. subproblems 之间的求解是相互独立的, 不存在重叠 (如果有重叠那就是 dynamic programming 问题了). 正因为相互独立, divide and conquer 也多用于 parallelism 场景中

4 Binary Search

Binary Search 是最简单的一类 divide and conquer 算法. 其将一个 search problem 拆为两个 subproblems, 每个的输入规模是原始问题的一半.

本章会把 binary search 封装为 binary search tree 这一数据结构.

4.1 A one-dimensional search problem

第一个问题描述如下:

  • 给定一个严格递增的函数 f: N -> N, 即对于任意自然数 $x < y$, 都满足 $f(x) < f(y)$. 该函数还满足 $f (x+1) > x$
  • 目标是找到一个自然数 x, 使得 $f(x) = t$ (t 是给定的目标值)

假设存在 $f(x) = t$, 那么可以用 $f(x) > x-1$ 得到:

$$
\displaylines
{
\begin{aligned}
t > x - 1 \newline~ \newline
t + 1 > x
\end{aligned}
}
$$

由于 x, t 都是自然数, 因此可以写成 $t \ge x$. 比如 t = 5:

  • 对于 $5 + 1 > x$, x 最大取 5
  • 对于 $5 \ge x$, x 最大也取 5

由此可以得到 x 的取值范围是 $0 \le x \le t$, Haskell 实现如下:

这种每次查找一个元素的方法称为 linear search, 时间复杂度与 t 呈正比.

可以重新实现为下面二分形式:

  • a > b, a 是下限, b 是上限, 只会有 $a \le b$, 因此这种情况找不到解
  • t < f m, 可以由 m < f (m+1), m-1 < f m, 得到 $t \le m-1$, 再结合下限 a, 有 $a \le t \le m - 1$
  • t == f m 就是目标情况
  • otherwise 表明 t 可能在剩下的区间内, 即 $m+1 \le t \le b$
  • 其时间复杂度是 $O(logn)$

这个实现的问题在于 (0,t) 这个范围太大了, 对于一些函数, 比如 $f = 2^n$, 目标 t = 1024 时, m = 512, 则需要计算 $2^{512}$, 这是个很大的数, 会导致溢出.

可以一开始就找到这个范围, 满足: $f(a) < t \le f(b)$, 即 $f(a) < f(x) \le f(b)$, 此时有 $a < x \le b$. 可以这样实现, 这里的思路是翻倍找区间, 注意这里找到的区间是左开右闭的:

  • t <= f 0, 说明 t 是负数, 该情况需要特殊处理, 即返回 (-1,0)
  • 注意 util done (x2) 1 这部分, 上限按 2 的倍数增长, 直到找到一个比 t 大的值
  • 返回的区间是 (b/2, b), 且这一定是左开右闭的: 如果左边是闭合, 说明 t <= f (b/2) 是满足的, 如果是这样的话, done b = t <= f b 在 b/2 时就会返回 True, until 就会停止了, 没有停止说明此时 t > f (b/2)

假设经过 p+1 次计算找到了范围, 则有 $f(2^{p-1}) < t \le f(2^p)$, 即 $2^{p-1} < x \le 2^{p}$, 找到满足 $t \le f(x)$ 的最小值, 如果 t == smallest, 则说明存在, 否则不存在, 可以这样实现:

  • 调用 smallest 函数有一个假设 $f(a) < t \le f(b)$, 也就是说这个区间内是一定有解的
  • a + 1 == b 表明此时缩减到了最小的区间, 因为这里的递归条件是左开右闭的 (前面说的假设), a 这个端点值无法使用, 此时只能返回 b (因为一定有解, 这肯定是满足条件的)
  • 注意, smallest 每次调用都会执行 $log(b-a)$ 次来锁定一个解, 不会提前终止
  • 如果范围内没有解, 那么返回值会是最接近的且比解大的值, 除非所有值都小于解. 因为解总是在 a~b 之间, 而最终会取上界返回

这种二分的含义时, 每次减少一半输入, 最终一定会只剩下一个输入, 判定该输入是否是合法解就行: 如果合法, 就是要找的解; 如果不合法, 则无解.

优化常规解法

下面是一个稍微改进的版本, 时间复杂度还是 $\Theta (t^2)$, 只不过会先得到满足条件的解, 在 head, take 等函数取值时, 能利用惰性求值减少一些遍历过程, 查找从左上角开始, 而非左下角:

  • 这个主要是借助了单调性, 从左上角开始查找, y 方向上的值是最大的, 比如在 y==10 的位置找到了解, 那么同一列中 y <- [0..9] 部分肯定不会有解了

一个优化方法是跳过部分一定不会有解的搜索空间, 可以定义一个辅助函数 searchln (ln 是 linear 的缩写), 有 search f t = searchln (0,t), 表示从 (0,t) 这个点开始进行线性查找, 假设 (a,b) 是左上角的点, 因此可以这样定义:

  • 对于左上角的点而言, a 是横坐标最小值, 而 b 是纵坐标最大值

完整的定义为:

  • 由于从左上角查找, 一开始 y 轴就是最大的, 当 z < t 时, 需要增大 z, 因此 x+1 (因为 y 不能继续增大了, 这里相当于 skip 了一列)
  • 当 z > t 时, 需要减小 z, 此时不考虑 x-1, 不然的话就回到了原来的位置, 因此考虑 y-1 (相当于 skip 了一行)
  • 当 z == t 时, 先将此时的 (x,y) 追加到结果中, 由于 f 单调:
    • 如果保持 x 不变, 减小 y, 那么 f 会变小, 不再等于 t
    • 如果保持 y 不变, 增大 x, 那么 f 会变大, 也不再等于 t
    • 因此下一个可能是等于的位置是 (x+1, y-1) (不能是 (x-1,y+1) 不然就是往回走, 是访问过或者排除掉的)

该方法也称为 saddleback search. 能保证每个点只访问一次, 且不会回头, 时间复杂度是 $\Theta(t)$. 比较巧妙的点就是从左上角开始, 这样能保证, 左上角到右下角都是单调的.

考虑原矩阵为 m x n 大小 (注意和之前的坐标 m, n 有所不同, 前面横坐标是 m, 可以理解为 m 列, 但这里 m x n, m 描述的是行数, n 描述的才是列数), 用 $T(m, n)$ 表示最坏情况下找到目标时调用 f 的次数:

  • 假如 m=0, n=0, 由于没有元素可以查找, 因此 $T(m,n) = 0$
  • 假如 m=1 或者 n=1, 此时退化为 one-dimension 的 binary search, 有:
    • $T(1, n)$, 表示 1 行, n 列
    • $T(m, 1)$, 表示 m 行, 1 列

$$
\displaylines
{
\begin{aligned}
T(1, n) = 1 + T(1, n/2) \newline~ \newline
T(m, 1) = 1 + T(m/2, 1)
\end{aligned}
}
$$

当 $m \ge 2, n \ge 2$ 时, 基于之前的找中点策略, 至少能够丢掉一个 $m/2 x n/2$ 的矩形, 假如此时做了 horizontal cut, 得到的两个矩形是:

  • 上方的 $m/2 x n$
  • 下方的 $m/2 x n/2$
  • 此时递推关系为:

$$
\displaylines
{
\begin{aligned}
T(m,n) = 1 + T(m/2,n) + T(m/2, n/2)
\end{aligned}
}
$$

1 是找到中点之后进行判断 f(x,y) 与 t 的大小关系时, 调用了一次 f. $T(m/2, n)$ 表示上方的矩形调用 f 的 steps, $T(m/2, n/2)$ 是下方矩形调用 f 的 steps.

同理, 如果做 vertical cut, 得到的两个矩形是:

  • 左侧的 $m x n/2$
  • 右侧的 $m/2 x n/2$
  • 此时的递推关系是:

$$
\displaylines
{
\begin{aligned}
T(m,n) = 1 + T(m,n/2) + T(m/2,n/2)
\end{aligned}
}
$$

通常来说, 分割策略为:

  • 如果 $m \le n$, 则进行 horizontal cut
  • 如果 $m > n$, 则进行 vertical cut

解上述递推关系时, 需要一些技巧:

  • 假设 m 和 n 都是 2 的幂. 即 $m = 2^i, n = 2^j$
  • 定义 $U(i, j) = T(2^i, 2^j)$, 这里我们用 $i \le j$, 即做 horizontal cut 为例子, 得到

$$
\displaylines
{
\begin{aligned}
U(0,j) = j
U(i+1, j+1) = 1 + U(i,j) + U(i,j+1)
\end{aligned}
}
$$

具体求解比较繁琐 (书上以来就是假设解是多少), 这里直接给出, 可以求的 $T(m,n)$ 为 $O(n log (2m/\sqrt{n}))$. 当 m, n 中其中一个远小于另一个时, 运行速度会比 saddleback search 的 $O(m+n)$ 要快.

这个就是 h 的下限值与节点数 n 的关系.

当 binary tree 的任意一个节点的左子树和右子树的高度相差不超过 1 时, 则称其为 balanced.

对于一个 balanced binary tree, 其 height 往往很小, 满足下面式子:

$$
\displaylines
{
\begin{aligned}
h \le 1.4404 log(n+1) + \Theta (1)
\end{aligned}
}
$$

定义树的 balance 性质, 就是为了描述一种 height 很小的树.

证明上述公式的思路很巧妙, 用数学归纳法推导出的. 用反向定义进行逻辑转换:

  • 直接问题: 给定节点数 n, 求最大高度 $H(n)$
  • 反向问题: 给定高度 h, 求最少节点数 $S(h)$
  • 逻辑转换的思路:
    • 如果节点数量为 n 的树最大高度为 $H(n)$, 将这个高度代入 S(..), 得 $S(H(n))$, 表示到达高度 $H(n)$ 所需的最小节点数. 根据这个最小就有: $S(H(n)) \le n$

得到这个不等式的好处在于, 只要知道了 $S(h)$ 的表达式, 就可以用 $S(h)$ 的反函数 $S^{-1}(h)$ 来推出 $H(n)$ 的不等式了: $H(n) \le S^{-1}(n)$.

为了让树的 height 最大, 并且满足 AVL 平衡的特性, 则结构必须为:

  • 一个子树高度为 h - 1 (h 是父节点的 height)
  • 另一个子树高度为 h - 2

这就形成了一个递推公式:

$$
\displaylines
{
\begin{aligned}
S(h) = S(h - 1) + S(h - 2) + 1
\end{aligned}
}
$$

  • 求高度为 h 的父树所需的最小节点数, 等价于求高度分别为 h - 1 和 h - 2 的子树之和, 再加上一个父节点

这个公式与 Fibonacci 非常相似, 可以推出下面式子:

$$
\displaylines

\begin{aligned}
S(h) = fib(h + 2) - 1
\end{aligned}
}
$$

用反证的思路:
$$
\displaylines
{
\begin{aligned}
a < (a + b)\ div\ 2 < b \newline~ \newline
利用 div 的定义得到: \newline~ \newline
a < \lfloor (a + b) / 2 \rfloor < b \newline~ \newline
根据算术性质的: \newline~ \newline
a + 1 \le \lfloor (a + b) / 2 \rfloor < b \newline~ \newline
利用 floor\ rule 有: \newline~ \newline
a + 1 \le (a + b) / 2 < b \newline~ \newline
a + 1 < b
\end{aligned}
}
$$

之后用 rule of celings 证明, 对于整数 n 和 h, 有 $n < 2^h$, 那么就有:

$$
\displaylines
{
\begin{aligned}
\lceil log (n + 1) \rceil \le h
\end{aligned}
}
$$

这个用来说明二叉树节点数量和 height 的关系, 有 $n \le 2^h - 1$, 即 $n < 2^h$, 能推出 $h$ 的下限.

    • 如果 x 更小, 那么直接构建 Tree, x 作为 Node value
    • 如果 y 更小, 那么需要交换 x 和 y 的位置. 虽然 Node y u v 是满足 Heap 特性的, 但是 Node x u v 不一定满足, 因为 x 比 y 大, 此时 u 和 v 中可能有更小的, 因此需要继续 siftDown x u v 处理

siftDown 的调用次数与 height 有关, 毕竟 siftDown x u v, 相当于每次处理一层. 由于递归作用于每个子树, 假设一开始树高为 h, 那么子树高度为 h-1 (最坏情况下两个都是这么高), 下一层子树高度为 h-2, 可以得到:
$$
\displaylines
{
\begin{aligned}
h + 2(h-1) + 4(h-2) + … + 2^(h-1) = \sum_{k=1}^{h} 2^{h-k} k = 2^h \sum_{k=1}^h k/2^k < 2^{h+1}
\end{aligned}
}
$$
由于一棵 size-balanced tree 的高度最高为 $\Theta(logn)$, 因此 $2^{h+1} = 2^{logn + 1}$, 相当于就是 $\Theta(n)$.

  • mktree 和 heapify 的时间复杂度是 $\Theta(n)$
  • flatten 的时间复杂度是 $\Theta(nlogn)$
  • hsort 的时间复杂度是 $\Theta(nlogn)$

5.5 Sorting sums

Sorting sums 问题是这样的, 假如 A 是一个可比较的元素类型, + 是二元运算函数, 那么有:

  • sortsums 为 xs, ys 中元素的所有组合的 + 运算结果进行排序
  • 假设 xs 和 ys 的长度都为 n, 那么 sort 处理的列表长度为 $n^2$ (xs 和 ys 的元素的所有组合有这么多种), 总的排序时间复杂度就是 $n^2 log (n^2) = 2n^2log(n) = \Omega(n^2 logn)$. 这里还没考虑 + 的运算复杂度

之后书中引入了 Abelian group 这个概念, 因此这里我查了一些内容补充到这里.

Group (群), 是一个代数结构. 其描述了一个集合及其上的一种运算, 这个运算需要满足四个核心性质 (通常称为 “群的公理”).

Group 严格定义为: 一个非空集合 G 和一个定义在 G 上的二元运算 $\cdot$, 如果满足下面四条公理, 则称 $(G, \cdot)$ 构成一个 group:

  1. 封闭性: 对于 G 中的任意两个元素 a 和 b, 运算结果 $a \cdot b$ 也一定属于 G
  2. 结合性: 对于 G 中任意三个元素 a,b,c, 有 $(a \cdot b) \cdot c = a \cdot (b \cdot c)$. 意味着运算的次序不影响结果
  3. 单位元存在性: G 中存在一个唯一的特殊元素 e, 使得对于 G 中的任意元素 a, 都有 $a \cdot e = e \cdot a = a$. 元素 e 称为单位元
  4. 逆元存在性: 对于 G 中任意元素 a, 都存在 G 中一个元素 b, 使得 $a \cdot b = b \cdot a = e$. 元素 b 称为 a 的逆元, 通常记为 $a^{-1}$

Abelian group 是指, Group 还额外满足交换律, 即 $a, b \in G$, 有 $a \cdot b = b \cdot a$.

其他几个可能遇到的术语:

  • Goup 的 rank, 指 Group 中元素的个数. 如果有限, 则称为有限群; 如果无限, 则称为无限群
  • 子群, 如果群 G 的一个子集 H 在相同的运算下也构成一个群, 则 H 称为 G 的子群

回到书中的例子, 假设 $(A, +)$ 构成一个 Abelian group, 则有下面性质:

  • + 满足 associative 和 commutative, 有 identity element 写作 0, 有获取逆元的操作 negate 满足 x + negate x = 0

  • 是否允许 duplicates

6.2 Selection from one set

本节要讨论的问题是: 如何在乱序的数组中, 以最快的速度找到 “第 k 小的元素”, 即 select 问题.

对于有 n 个元素的 set 而言, 如果一个元素是 kth smallest element 意味着有 $k - 1$ 个元素比它更小 (毕竟排到了第 k 小, 那前面肯定有第 $1, 2, 3, …, k-1$ 小的元素), 整体来说, 集合中 1st smallest 的元素就是最小的元素, 而 nth smallest 的元素就是最大的元素.

这里引入 median (中位数), 其本质是将集合对半分开, 但根据集合大小 n 是奇数还是偶数, 情况会有所不同.

如果 n 为 odd, 这个集合的 median m (中位数) 能够将剩下元素分为大小相同的两部分, 即这个 median m 需要: 比集合中 $\lfloor n/2 \rfloor$ 个元素大, 比 $\lfloor n/2 \rfloor$ 个元素小. 比如集合有 5 个数, $\lfloor 5/2 \rfloor = 2$, 中位数左边 2 个, 右边两个.

如果 n 为 even, 没有一个元素能作为 “正中间”, 按理说是中间两个元素都可以, 这里选择中间两个元素中较小的一个, 因此是 lower median m, 满足: 比 $\lfloor n/2 \rfloor - 1$ 个元素大, 比 $\lfloor n/2 \rfloor$ 个元素小. 比如集合有 4 个元素, $\lfloor 4/2 \rfloor = 2$, lower median 左边有 1 个点, 右边有 2 个点.

无论 n 是 odd 还是 even, 都可以用一个通用公式表示 median 满足的性质:

  • 比它小的元素个数为: $\lfloor (n+1) / 2 \rfloor - 1$
  • 比它大的元素个数为: $\lceil (n+1) / 2 \rceil - 1$
    (证明怎么得来的不清楚, 但是符合之前的 cases, 可以证明正确性)

对于更快的解法, 无论是什么, 都要至少把整个数组看一遍, 才能确定谁是第 k 小, 因此, 这个问题的运行时间下界是 $\Omega(n)$.

这里的疑问就是, 有没有一个算法能在 $O(n)$ 时间内找出谁是第 k 小, 而不需要进行完整的排序.

有一个算法引入 divide and conquer 思想达到了这个效果.

需要先将之前的 sort 替换为下面的 qsort 实现:

  • pivot 从列表中找到合适的 pivot 值
  • partition3 将一个列表拆为 3 份, 小于 pivot 的, 等于 pivot 的, 大于 pivot 的. 这种实现便于处理有重复元素的情况

用 divide-and-conquer 思路加快的原理如下 (只是描述过程):

  • 如果只考虑取元素, 这里还没有引入排序, k 比 xs 的长度小, 那么只需要从 xs 中取就行, 不需要管 ys

加入 qsort 的 partition 特性, 可以结合上述到 select 中如下:

  • partition3 分成的 3 部分, 虽然没有进行细致排序, 但是这 3 部分是有顺序关系的, 剩下的就是锁定在那一个部分中
  • 之所以对 vs 直接取 !! 而不是 select, 是因为 vs 中的元素都一样 (与 pivot 相等的元素列表), 因此随便取都行实际上
  • 这里借鉴的就是 qsort 思想, 先分区, 然后判断是哪个区中, 再进行定位

这里分析下时间复杂度, 假设 $T(n)$ 是 select 作用到长度为 n 的列表上的时间. partition3 将列表分为 3 部分, 第一部分和最后部分的长度最大为 $(n-1)/2$ (中间部分的长度至少是 1, 也就相当于是 median 中位数), 因此有:

$$
\displaylines
{
\begin{aligned}
T(n) \le T((n-1)/2) + \Theta(n)
\end{aligned}
}
$$

解出来有 $T(n) = O(n)$.

如果 partition3 返回的列表非常不均衡, 最坏情况会变为:

$$
\displaylines
{
\begin{aligned}
T(n) = T(n - 1) + \Theta(n)
\end{aligned}
}
$$

解出来有 $T(n) = \Theta (n^2)$.

如果 partition 用 list 的 median 作为 pivot, 就能保证 equal split 且运行时间是线性时间.

用一个函数 group 将 list 拆为固定大小的几个子 list:

  • 比如 group 5 [1..12] = [[1..5], [6..10], [11,1]], 一共有 $\lceil n/5 \rceil$ 个 groups, 运行时间为 $O(n)$

之后找到每个 group 的 median 值, 具体来说就是先排序, 然后取中间元素. medians 会得到一组 median 值:

  • 比如 medians [1..12] = [3,8,11]
  • 为每个 group 排序消耗常数时间, 由于有 $\lceil n/5 \rceil$ 个 group, 因此时间复杂度总体是 $O(n)$ ($C \cdot \lceil n/5 \rceil$ 也是线性时间).

之后取这一组 medians 中的 median 值作为 pivot:

  • 从 medians 中找到 median 的时间是 $T(\lceil n/5 \rceil) + \Theta(n)$, 毕竟 median 处理的列表长度是 $\lceil n/5 \rceil$, $\Theta(n)$ 是运行 select 的时间

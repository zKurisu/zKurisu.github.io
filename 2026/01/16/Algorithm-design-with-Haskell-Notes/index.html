2 Timing

2.2 Estimating running times

Haskell 对表达式求值时会将其 reducing to normal form, 因此可以用 reduction steps 来衡量 running times.

因为并不是所有 reduction steps 都消耗相同的时间, 比如复杂表达式中找到下一个 reducible expression 并规约可能会花更多时间, 因此用 reduction steps 来衡量不是那么 faithful.

另一个可选方法是用程序开始到结束之间 elapsed time 来衡量.

因为 lazy evaluation 的 reduction steps 不会超过 eager evaluation, 所以通常程序在 eager evaluation 的运行时间 upper bound 也可以看作是 lazy evaluation 的上界.

估计 recursive function 的 reduction steps 时, 需要找到 recurrence relation (递推关系): 其描述了一个递归函数的运行时间是如何依赖于更小规模子问题的运行时间的. 解 recurrence relation 的意思是找到其 closed form, 比如:

  • Recurrence form: $T(n) = T(n-1) + 1, T(0) = 1$, 此时需要展开才能知道 $T(100)$
  • Closed form: $T(n) = n + 1$, 此时直接代入 $n = 100$ 就能得到 $T(100) = 101$

求解 recurrence relation 的常见方法有:

  • Iteration, 迭代展开, 反复代入递推式以达到基础情况
  • Master Theorem (主定理), 通常用于分治算法比如 $T(n) = aT(n/b) + f(n)$
  • 代入法 (Substitution), 猜一个解, 然后用数学归纳法证明

比如用迭代法解下面 recurrence relation:

$$
\displaylines
{
\begin{aligned}
T(n) = nT(n-1) + \Theta (n)
\end{aligned}
}
$$

推导解:
$$
\displaylines
{
\begin{aligned}
T(n) &= cn + n T (n-1) \newline~ \newline
&= cn + n(c(n-1) + (n-1) T(n-2))
&= cn + cn(n-1) + cn(n-1)(n-2) + …
\end{aligned}
}
$$
这里把 $\Theta (n)$ 替换为 $cn$, 用具体的函数替换渐进符号来简化求解过程. 因为 $\Theta(n)$ 是一个集合, 表示 “所有与 n 同阶的函数”, 比如 $3n + 2, 0.5n, 100n - logn$, 需要具体的函数才能进行代数计算.

有:
$$
\displaylines
{
\begin{aligned}
T(n) = c \sum_{k=1}^n \frac{n!}{(n-k)!} = \Theta (n!)
\end{aligned}
}
$$

2.4 Amortised running times

对于某些数据结构而言, 操作序列的总时间复杂度可能会远低于 “单次最坏情况 x 操作次数”, 下面是两个例子.

build 函数用于过滤输入列表, 其定义如下:

  • build 会挨个处理输入列表并添加到输出列表中, 每次插入时, 会删除输出列表中符合 p x y 的元素 (x 是当前要插入的元素, y 是遍历的输出列表中的元素)
  • 单次最坏时间就是 insert 的成本, 由于 dropWhile 可能会遍历整个列表, 因此是 $O(n)$. 此时总的时间复杂度计算是: $B(n+1) = B(n) + O(n)$, 得到 $B(n) = O(n^2)$
  • 但实际上, dropWhile 最多删除 n 个元素, 因为dropWhile 进行遍历时, p x y 为 True 的会被删除, 为 False 时停止遍历, 因此最多只会遍历完输入列表, 长度为 n
  • 因此 insert 操作的平均成本是 $O(1)$, 此时时间复杂度是: $O(n)$, 远小于之前的 $O(n^2)$

又比如 bits 函数, 用于生成一个二进制整数的位序列:

  • inc 函数用于递增一个二进制整数, 每次递增操作的成本在最坏情况下是 $O(k)$, 其中 $k$ 是二进制数的长度. 比如 1110 (例子中左边是低位, 向右边递增), inc 一次的结果是 0001, 遍历了每一位. 此时总的时间复杂度是 $O(n^2)$
  • 但实际上, 每次递增平均下来只会改变一个比特位, 比如:
    • 初始: []
    • inc 1次: [1], 改变1位
    • inc 2次: [0,1], 改变2位
    • inc 3次: [1,1], 改变1位
    • inc 4次: [0,0,1], 改变3位
    • inc 5次: [1,0,1], 改变1位
    • inc 6次: [0,1,1], 改变2位
    • inc 7次: [1,1,1], 改变1位
    • inc 8次: [0,0,0,1], 改变4位
    • inc 9次: [1,0,0,1], 改变1位
    • inc 10次: [0,1,0,1], 改变2位
    • inc 11次: [1,1,0,1], 改变1位
    • inc 12次: [0,0,1,1], 改变3位
    • inc 13次: [1,0,1,1], 改变1位
    • inc 14次: [0,1,1,1], 改变2位
  • 可以看到规律:
    • 每隔2次, 就会只改变 1 位
    • 每隔4次, 就会只改变 2 位
    • 每隔8次, 就会只改变 3 位
    • 总的更新次数假设是 n, 那么会有 $\frac{n}{2}$ 改变 1 位, 有 $\frac{n}{4}$ 改变 2 位, 以此类推
  • 总的时间复杂度是: $ \frac{n}{2} + \frac{2n}{4} + \frac{3n}{8} +… = O(n)$, 也是低于之前计算的 $O(n^2)$

Amortised cost 就是计算每次操作的成本, 比如 insert 和 inc 都是 $O(1)$ (最坏都是 $O(n)$).

这里引入 potential method 来计算, 假设数据结构在第 i 次操作前的状态为 $x_i$, 操作后的状态为 $x_{i+1}$, 有:

  • Actual cost: $C(x_i)$, 表示执行该操作的真实时间开销
  • Potential function: $S(x) \ge 0$, 表示状态 $x$ 所 “存储” 的能量 (或预付成本, 感觉像累积成本)
  • Amortised cost 定义为: $A(x_i) = C(x_i) + S(x_{i+1} - S(x_i)$

可以这样理解:

  • 如果操作之后有 $S(x_{i+1}) > S(x_i)$, 即势能增加, 两者相减得到一个正数, 因此 $A(x_i) > C(x_i)$
  • 如果操作之后有 $S(x_{i+1}) < S(x_i)$, 即势能减小, 两者相减得到一个负数, 因此 $A(x_i) < C(x_i)$

这样就能判断 Amortised cost 与每次操作的实际 cost 的关系. 对上述式子进行移项有: $C(x_i) = A(x_i) + S(x_i) - S(x_{i+1})$, 可以求 n 次操作之和:

$$
\displaylines
{
\begin{aligned}
\sum_{i=0}^{n-1} C(x_i) &= \sum_{i=0}^{n-1}[A(x_i) + S(x_i) - S(x_{i+1})] \newline~ \newline
&= \sum_{i=0}^{n-1} A(x_i) + (S(x_0) - S(x_1) + S(x_1) - S(x_2) + … + s_(x_{n-1}) - S(x_n)) \newline~ \newline
&= \sum_{i=0}^{n-1} A(x_i) + S(x_0) - S(x_n)
\end{aligned}
}
$$

如果能设计出势能函数 S 满足:

  • 初始状态无势能: $S(x_0) = 0$
  • 势能永不为负: $S(x_n) \ge 0$

那么就能得到:
$$
\displaylines
{
\begin{aligned}
\sum C(x_i) \le \sum A(x_i)
\end{aligned}
}
$$

就可以用总的 amortised cost 来表示整个操作序列的时间成本了.

比如对于之前的 build 函数, 可以定义势能函数为: S(xs) = length xs.

对于每次 insert, 其 actual cost 等于 p 的调用次数加上 cons 操作数:

  • 假设 dropWhile 删除了 d 个元素, 则 p 会被调用 d+1 次
  • 由于是 x : ..., 因此 cons 只用操作一个元素
  • 总的代价是: $C = (d+1) + 1 = d + 2$

势能的变化可以计算为: $S(newxs) - S(xs) = [1 + (len xs - d)] - len xs = 1 - d$ (这里的 1 + (len xs -d) 中, 1 是新插入的元素 x, len xs - d 是删除了 d 个元素之后的长度).

因此有:
$$
\displaylines
{
\begin{aligned}
C = d + 2 = 3 + (d - 1) = A + S(x_i) - S(x_{i+1}) \newline~ \newline
A = 3 = O(1)
\end{aligned}
}
$$

总代价: $\le 3n = O(n)$.

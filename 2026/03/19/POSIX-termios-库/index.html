POSIX-termios-库

介绍

termios 是 POSIX 标准里定义的一套终端 I/O 接口, 在类 Unix 系统的头文件 <termios.h> 中实现. 其可以用来读取和修改终端设备的属性, 从而控制终端的输入输出行为.

termios 结构体

索引 名称 默认值 说明

VEOL2 EOL2 未定义 第二行结束符

VREPRINT REPRINT Ctrl+R 重打印
VDISCARD DISCARD Ctrl+O 丢弃输出
VWERASE WERASE Ctrl+W 擦除单词
VLNEXT LNEXT Ctrl+V 逐字输入

更改 terminal settings

获取 terminal settings

获取与 fd 关联的 terminal 的参数, 保存在 termios_p 指向的地址.

这里的 tc 前缀是 terminal control 的缩写, 是一种命名约定.

设置 terminal seettings

设置与 fd 关联的的 terminal 的参数, 具体参数在 termios_p 中指定, optional_actions 指定更改什么时候生效, 可以有下面几个值:

  • TCSANOW (now), 立即生效
  • TCSADRAIN (drain, 立即排空), 在所有输出都写入 fd 之后生效. 如果改变的 terminal 参数会影响 output 时用这个
  • TCSAFLUSH (flush, 冲洗输入输出), 先将所有输出都写入 fd, 然后丢弃还没有读取的 input, 之后生效更改

TCSA 是 Terminal Control Set Attribute 的缩写

Canonical and noncanonical mode

Canonical mode (规范模式), 意思是 Unix 终端的默认模式, 该模式符合大多数用户对终端行为的预期 (输入一行, 按回车, 程序处理).

c_lflag 的 ICANON flag 用来设置 terminal 工作在哪一种 mode. 如果设置, 则在 canonical mode (默认); 如果不设置, 则在 noncanonical mode.

在 canonical mode 下的行为有:

  • 输入一行一行读取 (行缓冲). 终端接收到数据后, 不会立即交给程序, 而是缓存一整行, 直到遇到 delimiters 才会传输
  • 可以进行 Line editing, 比如退格 (ERASE, 删掉前一个字符), 删掉整行 (KILL). 如果开启了 IEXTEN 标志, 还能删一个单词 (WERASE), 重新打印当前行 (REPRINT)
  • 一行最长 4096 个字符 (包含最后的换行符). 当输入字符超过 4095 个字符以后, 终端仍会继续处理信号, 但遇到换行符之前的所有字符都会被丢弃

在 noncanonical mode 下:

  • 终端接收到的输入会立即交给程序, 不会等待遇到 line-delimiter
  • 不能进行 line editing
  • 一行最多 4095 个字符 (如果切换到了 canonical mode, 就能够预留一个位置给 newline char)

具体行为可以通过 c_cc[VMIN] (后面简称 MIN) 和 c_cc[VTIME] (后面简称 TIME) 两个参数的组合来设置, 有 4 种组合方式:

  • MIN == 0, TIME == 0 (polling read), 不阻塞输入, 有数据立即读取, 没有数据时返回 0 (轮询)
  • MIN > 0, TIME == 0 (blocking read), 阻塞, 直到接收到 MIN 个字节才返回
  • MIN == 0, TIME > 0 (read with timeout), 等待 TIME x 0.1s 之后, 有数据就立即读, 超时没数据就返回 0
  • MIN > 0, TIME > 0 (read with interbyte timeout), 收到第一个 byte 后开始计时, 直到读取了 MIN 个字节或者超时

Raw Mode

cfmakeraw 将 termios_p 设置成类似 old version 7 terminal driver 的行为, 也称 raw mode:

  • 输出立即读取
  • Echoing is disabled
  • 所有对 input 和 output 中特殊字符的处理也 disabled

发送 zero-valued bits

如果 terminal 设置为 asynchronous serial data transmission, tcsendbreak 函数会向 fd 一直发送 zero-valued bits, 时长为 duration 设置, 如果 duration = 0, 则发送时间为 $0.25s \le t \le 0.5$. 如果 terminal 不是 asynchronous serial data transmission, 则直接返回不进行任何操作.

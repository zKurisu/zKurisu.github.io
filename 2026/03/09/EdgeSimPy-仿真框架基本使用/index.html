EdgeSimPy Tutorial
EdgeSimPy Github Repo

介绍

EdgeSimPy 是一个基于 Python 的 Edge Computing simulator, 其抽象出 edge servers, network devices, applications 等组件来便于使用, 还提供有内置的 user mobility, application composition, power consumption 等仿真模型.

Monitoring

EdgeSimPy 的 monitoring mechanism 能收集大量关于 simulated entities 的信息, 在每个 time step 都有记录.

EdgeSimPy 的 monitoring data 用 MessagePack 格式 (一种 serialization format, binary file) 来保存. 可以将其再转换为 Python dictionaries 和 Pandas data frames 等.

EdgeSimPy 默认会间隔一个 interval 写入日志到磁盘中, 而不是每个 time step 都会写入 (都会在内存中进行记录, 只是还没持久化写入), 这个 interval 可以用 Simulator 类的 dump_interval 属性来设置, 比如 dump_interval=10, 表示每 10 个 time steps 写入一次. 如果不想持久化, 可以设置为 dump_interval=float("inf").

EdgeSimPy 会自动创建名为 logs 的 dictionary 来保存日志信息.

  • dump_interval = 5, 定义保存 log 的间隙是 5 个 time steps
  • tick_duration = 1, 一个时间步的长度是 1 份单位时间
  • tick_unit = "seconds, 时间步的单位是秒
  • stopping_criterion, 指定一个 Boolean function, 返回 True 时结束模拟, 示例程序如下:
  • run_model 方法表示开始 simulation 过程

Custom mobility model

EdgeSimPy 仿真中, 与 datasets 关联的最基础的组件是 map model, 其封装了 coordinates system 便于其他 entities (比如 base stations, edge servers, users) 使用. Map model 通常不会影响其他的 system model 比如 power consumption model, 但对于 custom mobility models 就很重要.

EdgeSimPy 的 map model 基于 Aral et al. 的理论, 其会将一个 map 拆分为一个个 haxagonal cells. 在 EdgeSimPy 中可以用 hexagonal_grid 函数来创建一组 hexagonal cell, 对应一组自动生成的坐标. 比如:

  • x_size 和 y_size 指定了 map 的大小, map 上每一个位置就是一个 hexagonal cell
  • map_coordinates 是一个 list, 其元素是包含两个坐标值的 list

有了 coordinates set 之后, 就可以开始创建 base stations. EdgeSimPy 默认的 hexagonal map 假设每个坐标上都有一个 base station, base station 之上可以连接 edge server, network switch 等 (一个 base station 可以与多个 entities 相连).

每个 hexagonal cell 中至少要有一个 base station 和一个 network switch. 所有 base stations 之间是连接的, 形成一个 wired network infrastructure.

  • 2 个用户, 各自使用一个 application:

  • cls 参数是自动传入的

Edge server 中有两个 attributes 用来支持 power consumption model:

  • static_power_percentage, 当 device 静息运行时, 其消耗的功率是 max_power_consumption 的多少比值

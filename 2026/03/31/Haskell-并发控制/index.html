Haskell-并发控制

这里说明几个 Haskell 里的并发同步机制:

  • MVar, Mutable Variable

MVar

详细可以看 hackage MVar 页面: https://hackage-content.haskell.org/package/base-4.22.0.0/docs/Control-Concurrent-MVar.html.

data MVar a 定义在 Control.Concurrent.MVar 模块中. MVar t 表示一个 mutable location, 可以包含一个具体值 t 或者为空.

对于 MVar t 可以进行两种基本操作:

  • putMVar, 如果 mutable location 为 empty, 则填充值; 否则 block 直到为空
  • takeMVar, 如果 mutable location 有值, 则将其取出 (取出后就为空了); 反之 block 直到有值

像 swapMVar, withMVar, modifyMVar 这些都是对 putMVar 和 takeMVar 的组合.

MVar 的常见用法有:

  • 异步的 mutable variable
  • 利用 putMVar 和 takeMVar, 当作 channel 使用
  • 将 MVar () 作为不携带值的信号使用

Chan

Control.Concurrent.Chan 中定义的 data Chan a 是 unbounded channels, 意思是无限容量, 队列不会被填满, 因此:

  • 写端永远不会因为 full 而被阻塞
  • 读端只有在空的时候阻塞

Chan 满足 FIFO 的性质, 且内部是 MVars 实现的, 因此 MVars 的 races, deadlocks 等问题 Chan 也会有.

BoundedChan

Control.Concurrent.BoundedChan 与 Chan 类似, 区别在于前者限制写入的数量不超过某一个阈值. 当写的速度快于读的速度时通常用这个. 比如写入了 10 个, 此时才读 1 个, 那么 Chan 的长度就会快速增长, 写入 100 个时, 才读出 10 个, 此时 Chan 大小是 90 了. 如果是 BoundedChan, 限制写 10 个, 那么不管写得有多快, Chan 的大小都不会超过 10.

Control.Concurrent.BoundedChan 支持 Chan 中除了 dupChan 和 unGetChan 以外的其他所有函数. 这里就不举例子了, 直接给出一个列表:

  • newBoundedChan :: Int -> IO (BoundedChan a)
  • writeChan :: BoundedChan a -> a -> IO ()
  • tryWriteChan :: BoundedChan a -> a -> IO Bool
  • readChan :: BoundedChan a -> IO a
  • tryReadChan :: BoundedChan a -> IO (Maybe a)
  • isEmptyChan :: BoundedChan a -> IO Bool
  • getChanContents :: BoundedChan a -> IO [a]
  • writeList2Chan :: BoundedChan a -> [a] -> IO ()

  • 两个维度中某一个是 1

Attention

需要记住 attention 计算中的几个维度大小:

  • 输入的维度是 (nbatches, seq_len, d_model), 每个 token 的 features 个数是 d_model 个
  • 经过 attention 处理之后的输出也是 (nbatches, seq_len, d_model), attention 操作不会改变输入的维度, 其只是根据序列之间的关联更新了各个 feature 而已
  • $W_Q, W_K, W_V$ 参数矩阵通常维度都是 (d_model, d_model), 比如 Q = X W_Q, 因此第一个维度是 d_model 就很好理解, 输出维度在这里通常不做升维或降维操作, 因此也保持 d_model
  • 从上述来看, $Q,K,V$ 矩阵通常维度也是 (nbatches, seq_len, d_model), 和输入的维度保持一致
  • 对于 attention score 而言, 维度始终是 (nbatches, seq_len, seq_len), 这很容易理解, 每一个 token 对整个序列都会有相似度打分, 因此一个 token 对应的 score 为 (seq_len,), 而总共有 seq_len 个 token, 因此一个序列的 score 是 (seq_len, seq_len)
  • mask 是作用在 score 上的, 因此维度要与 score 一致, 因此也是 (seq_len, seq_len)

Mask 用在 score 上的作用比如是因果, 目的是让一个 token 不会看到未来的 token, 但 $q_i \times K^T$ 的过程中, q_i 就会为未来的 token 打相似度分数, 这会产生影响吗?

答案是不会, 因为每个相似度分数的计算都是独立的, 比如 $q_i$ 与 token 1 计算相似度, 那么是 $q_i \times k_1$; 如果与 token 2 计算相似度, 那么是 $q_i \times k_2$; 相互之间不会影响. 因此在计算了 score 之后再进行 mask 是合理的.

假如 mask 原本的维度是 (batch, seq_len_q, seq_len_k), unsqueeze 表示指定在某个地方添加一维, 这里变成 (batch, 1, seq_len_q, seq_len_k).

按理说 mask 的维度为 (seq_len, seq_len) 就行了, 这样就能为 score 中每一个位置都对应一个 mask. 加上 batch 这个维度的原因是, 每个样本 (序列) 的 padding 长度是不一样的, 对于 padding 的位置, 通常要将其 mask 掉, 因此不能让所有的 batch 共享一个 mask.

下面转换很重要:

  • 这里用 query 做例子, 其维度原本是 (nbatches, seq_len, d_model), 由于 multi-head 的因素, 将 d_model 拆为 h 份, 有 d_model = h x d_k, 这本质是对 query 矩阵的最后一维做操作, 将这一层拆成两维因此有 view(nbatches, -1, self.h, self.d_k): 最后一层是 d_k, 倒数第二层大小是 h, 合起来可以想象成 h 个 d_k 层堆叠起来. -1 表示自动计算大小, 这里算出是 seq_len
  • transpose(1,2) 将 h 层和 seq_len 进行交换, 方便后面对 seq_len 进行并行计算 attention (attention 的计算通常就是取 -1 层作为 features 和 -2 层作为 sequence)
  • zip(self.linears, (query, key, value)) 将 q, k, v 矩阵分别绑定到一个 linear 层上

最终拼接回 d_model:

  • transpose(1, 2) 将 h 和 seq_len 两个维度换回来
  • contiguous() 会将 tensor 在内存中重新整理为连续布局, 否则后面 view 会报错, 这样才能安全进行 reshape
  • view(nbatch, -1, self.h * self.d_k) 将维度重新变为 (nbatch, seq_len, d_model)

Position-wise Feed-Forward Networks

Embedding

lut 是 Lookup table 的缩写, embedding 本质上就是一个表, 一个 token 对应其中的一行.

乘以 $\sqrt{d_{model}}$ 的原因: 主要为了对抗位置编码的影响以及稳定深度模型的数值方差.

在 Transformer 中, embedding 会直接加上位置编码:

  • 问题: 位置编码的值通常在 [-1, 1] 之间, 且方差是固定的
  • 由于 Embedding 层初始化的权值通常是按照 $\frac{1}{\sqrt{d_{model}}}$ 缩放后的正态分布, 随着 $d_{model}$ 的增大, 其数值会变得很小. 如果不放大 Embedding, 位置信息就会在数值上占主导地位, 淹没掉词本身的语义信息.

因此乘以 $\sqrt{d_{model}}$ 的本质是提高词向量的 “能量”, 确保在和位置编码相加时, 语义信号不会被位置信号覆盖掉.

在深度学习中, 我们希望每一层输出的方差尽可能是 1, 而在 PyTorch 中, nn.Embedding 默认初始化通常会让每个维度的方差约为 $\frac{1}{d_{model}}$, 而我们乘以 $\sqrt{d_{model}}$ 之后, 根据方差的性质 $Var(a X) = a^2 Var (X)$, 输出的方差就变成了:

$$
\displaylines
{
\begin{aligned}
(\sqrt{d_{model}})^2 \cdot \frac{1}{d_{model}} = 1
\end{aligned}
}
$$

这能让 Embedding 的输出处于一个稳定的量级.

Positional Encoding

对于一个特征维度为 d_model 的 token 而言, 对应的 positional encoding 维度也应该是 d_model, 这样就能够直接相加.

  • 对于 0~max_len 中的每一个位置, 都生成一个 d_model 长度的 encoding, 这比较符合直觉
  • 维度是 (max_len, 1), 每一行对应一个位置, 便于广播到 (max_len, d_model) 然后用于相加

对于论文中的 positional encoding 的公式, 每一个 feature 都会对应一个 sinusoid.

理解下面计算:

原始公式可以这样变换, 先取对数再 exp 等于没变, 但是计算会更快:

$$
\displaylines
{
\begin{aligned}
\omega_k = 10000^{-2i/d_{model}} \newline~ \newline
log(\omega_k) = -\frac{2k}{d_{model}} log(10000) \newline~ \newline
omega_k = exp(-\frac{2k}{d_{model}} log(10000))
\end{aligned}
}
$$

性质 self.pe = pe (普通属性) self.register_buffer('pe', pe)

对于 Positional Encoding (PE), 它有几个特殊属性:

  • 它是确定的, 不需要更新: PE 是通过正余弦函数算出来的确定值, 不需要梯度下降来优化, 因此不需要作为 nn.Parameter
  • 它必须和输入数据在同一个设备上: 如果模型在 GPU 上跑, 那么 PE 矩阵也必须在 GPU 上, 否则 x + self.pe 会报错. 如果用 self.pe = pe, 当执行 model.to('cuda') 时, pe 还是留在内存 (CPU) 里
  • 它是模型的一部分, 当你把训练好的模型发给别人时, 你希望这个 PE 矩阵也包含在模型文件里, 而不是让别人再手动计算一遍
  • 首先先了解一点, x 的 seq_len 不一定是 max_len. x.size(1) 取出来的就是 seq_len, self.pe[:, :x.size(1)] 表示取 pe 的前 seq_len 行, 而不是 max_len 行

在 PyTorch 0.3 及以前:

  • Tensor 不会自动记录计算图
  • 必须用 Variable 包一层

在 PyTorch 0.4 以后, Tensor 自动支持 requires_grad, 因此 Variable 被弃用

Optimizer

解析下这里用到的公式:
$$
\displaylines
{
\begin{aligned}
lrate = d_{model}^{-0.5} \cdot min(step\_num^{-0.5}, step\_num \cdot warmup\_steps^{-1.5})
\end{aligned}
}
$$

这是 Transformer 原始论文中的 Noam Scheduler 学习率预热和衰减的方案, 其本质是为了平衡 deep learning 训练中的两个矛盾: 早期的稳定性 (防止梯度爆炸) 和后期的收敛性 (便于精准调整参数):
- $d_{model}^{-0.5}$ 是基础的缩放项, $d_{model}$ 是模型的隐藏层维度, 如果模型的维度越大, 那么初始的学习率越低, 比如: $512^{-0.5} \approx 4.42e-2, 1024^{-0.5} \approx 3.13e-2$>. 毕竟维度越高, 神经元激活值的方差可能越大, 导致梯度波动剧烈
- min(..) 部分是由两个随 step 变化的函数组成, 由于取的是最小值, 因此学习率会跟随 “更低” 的曲线走
- 当 step 较小时 (线性预热阶段), min(..) 由 $step\_num \cdot warmup\_steps^{-1.5}$ 主导, 且由于 step 在分子上, 这是一个正比例函数, 能够从 0 开始线性缓慢增长, 给模型一个 “热身” 过程, 使其逐渐适应数据
- 当 step 大于 warmup_steps 后 (衰减阶段), min(..) 会由 $step\_num^{-0.5}$ 主导, 这是一个反比例函数, 当模型进入稳定期后, 随着步数增加, 学习率逐渐降低

利用 warmup, 在训练初期使用较小的 learning rate, 防止模型参数剧烈震荡.

  • self._step 用来记录训练经过的 step 数量
  • self._rate 表示当前的 learning rate
  • 每调用一次 step, 让 self._step 自增 1
  • 调用 self.rate() 函数来得到新的学习率

Label Smoothing

Label Smoothing (标签平滑), 能够减轻过拟合, 以及模型过于 “自信” 的问题.

比如普通 CrossEntropy, 标签为:

模型被逼着:

  • 对正确类输出概率 $\rightarrow 1$
  • 对其他类 $\rightarrow 0$

这容易过拟合, 且对噪声标签不鲁棒, 模型过于 “自信” 等.

而 Label smoothing 的思想是, 不再用 one-hot, 而是:

  • 正确类: $1 - \epsilon$, 从 1 变小
  • 其余类: $(epsilon) / (K-1)$, 从 0 增大, 且平分概率

比如 $\epsilon = 0.1$ (平滑因子), $K = 5$ (标签数), 有:

KL divergence (Kullback-Leibler Divergence), 用来衡量两个概率分布之间的差异 (从 divergence “分散” 这个词也可以看出). 也可以说明用一个概率分布 Q 去近似另一个分布 P 时, 损失了多少信息.

其数学定义如下:

$$
\displaylines
{
\begin{aligned}
D_{KL} (P || Q) = \sum_x P(x) log \frac{P(x)}{Q(x)}
\end{aligned}
}
$$

  • 当 $P$ 和 $Q$ 完全一样时, $KL = 0$
  • 如果 $P(x)$ 很大时 $Q(x)$ 很小, 此时 $KL$ 会很大
  • 如果 $P(x)$ 很小, 此时 $Q(x)$ 的大小无所谓, 最终 $KL$ 都会很小

需要注意 KL 散度不是对称的:
$$
\displaylines
{
\begin{aligned}
D_{KL} (P || Q) \ne D_{KL} (Q || P)
\end{aligned}
}
$$

如果模型的输入是一个分布, 输出是另一个分布, 那么通过最小化 KL 散度能让信息损失最小.

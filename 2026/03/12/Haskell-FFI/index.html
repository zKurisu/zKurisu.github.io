Haskell-FFI

介绍

FFI, Foreign Function Interface, 是一种编程语言各自实现的机制, 用于让一种语言调用另一种语言编写的函数或库. 它不是由一个单一组织制定的协议 (比如 ISO 或 IEEE 标准), 而是一个通用的概念和技术.

Haskell 的 FFI 相关内容都在 Foreign 以及其子模块中, 后面会说明一些常用的部分.

Storable

Storable typeclass 定义在 Foreign.Storable 中, 用来提供基本的 marshalling 功能.

完整的定义如下:

  • peekElemOff 和 peekByteOff 的区别在于, 前者的 offset 单位是 sizeOf a, 而后者是 byte (前者指定 index offset, 后者指定 byte offset)
  • sizeOf, alignment 必须实现, 而 peek, peekElemOff, peekByteOff 只需要实现其中一个, 剩下的能够自动实现; 同理 poke, pokeElemOff, pokeByteOff 也只需要实现其中一个

通常手动实现 peek 和 poke, 而这些函数的实现需要借助 Foreign.Ptr 模块中的函数.

Ptr

Ptr (Pointer) 类型定义在 Foreign.Ptr 模块中, 该模块还提供了一系列操作指针的函数. 除了 Ptr 类型外, 还定义有 FuncPtr, IntPtr 等.

空指针

nullPtr 常量是一个特殊值, 表示一个不指向任何有效内存地址的指针.

指针类型转换

castPtr 函数将一个指向 a 类型的指针 cast 为指向 b 类型的指针.

这里的目标类型通常是 Haskell 自己推理出的, 不需要手动标注.

向前移动指针

plusPtr 将指针 Ptr a 向前移动指定 offset.

指针对齐

给定一个 alignment constraint, alignPtr 会将指定指针对齐到更高的地址以满足 alignment constraint (此时新地址能够被 alignment 整除).

指针之差

minusPtr 用于计算两个指针之间的 offset.

FunPtr

data FunPtr a 就是字面含义, 指向函数的指针, 只不过这个函数是用 foreign language 定义的. a 通常是 foreign type.

  • Argument types 需要是 marshallable foreign types
  • Return type 需要是 either a marshallable foreign type or has the form IO t, 其中 t 是 marshallable foreign type 或 ()

ForeignPtr

Foreign.ForeignPtr 模块定义了 data ForeignPtr a 类型以及相关操作.

ForeignPtr 表示一个指向特定对象, 且内存是由 foreign language 维护的指针.

ForeignPtr 和 Ptr 的区别在于, 前者需要和 finalizer 绑定在一起. finalizer 指一个函数, 在 Haskell storage manager 发现没有 references 指向该 ForeignPtr 时自动调用. 通常 finalizer 是 foreign language 中的函数, 用来释放对应的资源.

Marshal

Foreign.Marshal 主要包含下面几个子模块:

  • Foreign.Marshal.Alloc
  • Foreign.Marshal.Array
  • Foreign.Marshal.Error
  • Foreign.Marshal.Pool
  • Foreign.Marshal.Utils

下面分别说明几个常用的地方.

Alloc 子模块

Foreign.Marshal.Alloc 子模块中提供了一些用于 allocate 和 deallocate blocks of raw memory 的工具. 这些 memory 通常用来传递 foreign function 中用到的 compound data structures, 或用来存储这些 foreign function 的结果.

任何 allocation functions 运行失败, 都会抛出 exception.

如果 free 或 reallocBytes 函数作用在 alloca 或 allocaBytes 分配的区域, 其结果是未定义的.

如果使用了传递给 free, realloc, reallocBytes 后的 pointer, 结果也是未定义的 (经过 free 处理的指针不再指向有效的内存区域, 同样, realloc 会重新分配一片内存, 因此 pointer 原始指向的区域也是不再有效).

在分配内存时, 需要考虑对应数据结构的 alignment.

下面的函数都是封装自 stdlib.h 中的 malloc, realloc 和 free.

Local allocation

Local allocation 是指, 分配的内存空间会在作用域结束后自动释放.

  • alloca 接受一个函数作为参数, 然后会执行对应的 IO b , 其会为 Ptr 指向的结构 a 提供足够的内存. 这相当于是创建了一个执行环境, 让分配的内存在 IO b 执行结束时自动释放
  • allocaBytes 与 alloca 的区别在于, 分配的空间由传入的 Int 参数指定
  • allocaBytesAligned size align f, 与 allocaBytes 的区别在于会将分配的内存对齐指定的 align 大小

Dynamic allocation

Dynamic allocation 指, 内存从堆上分配, 不会因为作用域结束而自动释放, 需要手动管理.

  • malloc 会分配一片足够容纳 type a 的内存, 具体大小用 sizeOf 得到, 具体的类型由 Haskell 自动推导. 这部分内存需要用 free 或 finalizerFree 来手动释放
  • mallocBytes 与 malloc 的区别在于, 分配的空间的大小由参数 Int 指定
  • 与 malloc 类似, 只不过会先用 0 填充内存区域
  • 与 mallocBytes 类似, 只不过会先用 0 填充内存区域
  • 将 malloc 或 mallocBytes 分配的区域重新调整为 type b 的大小, 返回的指针通常指向与原指针不同的地址
  • 将 malloc 或 mallocBytes 分配的区域重新调整为指定大小
  • 当指定参数为 0 时, 其行为类似 free
  • free 用于释放一片由 malloc, mallocBytes, realloc, reallocBytes, new, newX 等函数分配的内存
  • 表示一个指向 Finalizer 的指针
